# GuandanKitty 编码规范

> 版本：v1.0 | 日期：2026-07-08

---

## 一、基本原则

### 1.1 大括号风格

所有方法体必须使用**大括号 `{}`**，禁止使用 `=>` 表达式体。

```csharp
// ✅ 正确
public void OnInit()
{
    InitializeDecks();
    ResetAgentStates();
    _fsm.TransitionTo<BattleStartState>();
}

// ❌ 错误
public override void OnEnter() => Battle.OnInit();
```

### 1.2 单一职责

一个方法只实现一种目的。禁止在一个方法内堆砌多个不同层次的逻辑。

```csharp
// ✅ 正确
public void OnInit()
{
    InitializePlayerData();
    InitializeEnemyData();
    PrepareDecks();
}

// ❌ 错误
public void OnInit()
{
    foreach (var agent in Agents)
    {
        agent.Deck?.Initialize();
        agent.HasPassed = false;
        if (agent.IsPlayer)
            agent.RemainingCallCards = agent.MaxCallCards;
    }
    _fsm.TransitionTo<BattleStartState>();
}
```

### 1.3 注释规范

每个方法必须写注释说明它的职责。注释使用 `/// <summary>` 格式。

```csharp
/// <summary>
/// 初始化玩家数据：重置牌堆、手牌、叫牌次数。
/// </summary>
private void InitializePlayerData()
{
    var player = PlayerAgent;
    if (player?.Deck != null)
    {
        player.Deck.Initialize();
        player.RemainingCallCards = player.MaxCallCards;
    }
}
```

注释需要回答"这个方法干什么"，而不是逐行翻译代码。

### 1.4 命名规范

| 类型 | 规范 | 示例 |
|------|------|------|
| 类/方法 | PascalCase | `OnInit`, `EndAgentTurn` |
| 私有字段 | _camelCase | `_enemyTimer`, `_skipCount` |
| 局部变量 | camelCase | `agent`, `remaining` |
| 参数 | camelCase | `damage`, `cards` |
| 常量 | PascalCase | `MaxCallCards` |

### 1.5 可访问性

- `public`：对外暴露的 API（UI 调用、状态机回调）
- `private`：内部实现细节
- 无需 `internal` 或 `protected` 除非必要

## 二、架构分层规范

### 2.1 三层职责

```
BattleUI（显示层）
  ↓ 调用 Battle.PlayerPlay() / PlayerPass() / PlayerCallCards()
  ↑ 实现 Battle 调用的 On* 方法
  
Battle（逻辑层）
  ↓ 调用 _fsm.TransitionTo<T>()
  ↑ 实现状态机调用的 On* 回调
  
BattleFSM（流转层）
  ↓ 调用 Battle.On*()
  ↑ 状态继承 BattleState
```

### 2.2 BattleFSM 状态

状态类**禁止包含游戏逻辑**，只做回调分发：

```csharp
// ✅ 正确
class InitState : BattleState
{
    public override void OnEnter()
    {
        Battle.OnInit();
    }
}
```

### 2.3 Battle 逻辑分层

Battle 中的方法按以下模式组织：

```
OnXxx()             ← 状态机回调入口，只做流程编排
├── 调用私有封装方法  ← 每个封装方法只做一件事
└── _fsm.TransitionTo<T>()  ← 决定下一状态

私有方法             ← 具体的游戏逻辑实现
```

## 三、文件组织

### 3.1 目录结构

```
Core/
  Battle/
    Battle.cs          ← 战斗逻辑（主文件）
    BattleFSM.cs       ← 状态机 + 状态类
Scenes/
  BattleUI.cs          ← 战斗 UI 显示
```

### 3.2 方法顺序

每个文件内的方法按以下顺序排列：

1. `public` 属性 / 字段
2. `private` 字段
3. 生命周期方法（`_Ready`）
4. `public` 方法（UI 调用入口）
5. 状态机回调（`OnXxx`）
6. `private` 封装方法
7. UI 通知工具方法
