# 玩法原型计划书 v1.0

> 版本：v1.0 | 创建：2026-07-06
> 关联：`核心玩法文档`、`内线玩法GDD.md`、`伤害公式.md`、`怪物设计.md`
> 目标：第一期——可玩的裸规则战斗原型 + 完整 run 循环 + 开火车验证

---

## 一、范围定义

### 包含

- ✅ 完整战斗机制（牌型判定、压制、接龙、伤害结算、回合流程）
- ✅ 8~10 种怪物（含牌池、战败效果、等级缩放）
- ✅ 1 次完整 run（地图选路 → 怪物遭遇 → 开火车 → 战斗 → 循环直到失败/通关）
- ✅ 牌桌 TRPG 美术方向的基础 UI

### 不包含

- ❌ 卡牌贴纸、遗物、金币经济（构筑层）
- ❌ 商店、事件、休息点（外线系统）
- ❌ 玩家成长/升级
- ❌ 完整的视觉美术资源（先用占位素材）

## 二、系统模块拆分

### 2.1 扑克引擎

| 模块 | 职责 | 关键逻辑 |
|------|------|------|
| **StandardDeck** | 52 张标准扑克（4花色×13数字），不含大小王 | 创建、洗牌、抽牌 |
| **CardPattern** | 牌型检测 | 识别单张/一对/三张/顺子/联队/炸弹，返回牌型和对比数值 |
| **SuppressionJudge** | 压制判定 | 同牌型 + 更大数值才能压制（炸弹可跨牌型）；返回能否压制 |

### 2.2 战斗核心

| 模块 | 职责 | 关键逻辑 |
|------|------|------|
| **BattleManager** | 回合流程总控 | 初始化→回合开始→出牌循环→回合结算→回合结束→胜负检查 |
| **HandManager** | 手牌管理 | 抽牌、打出、叫牌(+6)、手牌区查询 |
| **ChainTracker** | 接龙追踪 | 记录玩家出牌次数（深度）、上家出牌信息、自压支持 |
| **CardRiver** | 牌河（战斗网格） | 存放本回合所有打出的牌，回合结束洗回 |
| **DamageCalculator** | 伤害计算 | `牌面值 × 出牌张数 × 2^(深度−1)` + 赢回合×2 / 清空×10 |
| **DefeatEffectExecutor** | 战败效果执行 | 根据怪物定义移出玩家手牌 + 补抽 |

### 2.3 怪物系统

| 模块 | 职责 | 关键逻辑 |
|------|------|------|
| **Monster** | 怪物数据定义 | HP基础值、牌池(≥5张含重复)、抽牌速率、战败效果、等级 |
| **MonsterDatabase** | 怪物注册表 | 8~10 种怪物，含高低难变体 |
| **EnemyAgent** | 敌方AI | 回合结束时从所有怪物牌池抽牌；出牌时用最小合法压制 |
| **MonsterCardPool** | 怪物骰子模拟 | 从牌池中等概率抽1张复制品（不消耗牌池） |

### 2.4 Run 循环

| 模块 | 职责 | 关键逻辑 |
|------|------|------|
| **RunManager** | Run 状态管理 | 当前关卡、玩家状态、总分 |
| **MapGenerator** | 地图生成 | 生成房间序列，每个房间含高/低难怪物选项 |
| **TrainPullUI** | 开火车界面 | 显示后续房间怪物、选择拉几波（1~5）、确认开战 |
| **EncounterResolver** | 遭遇结算 | 战斗→胜负判定→奖励占位→下一房间或Run结束 |

## 三、怪物清单（8~10 种）

### 设计约束

- 每个怪物的牌池 5 张，可重复
- 高低难变体 = 同一怪物不同等级（HP 差异）或不同抽牌速率
- 战败效果覆盖不同维度：随机弃牌、定向弃牌、按牌面、按花色…

| # | 怪物 | Lv1 HP | 抽牌 | 牌池倾向 | 战败效果 |
|:--:|------|:--:|:--:|------|------|
| 1 | 哥布林 | 400 | 1/回合 | [2,3,3,4,5] | 随机弃 3 张手牌，补抽 3 张 |
| 2 | 哥布林首领 | 600 | 1/回合 | [2,3,4,4,5] | 随机弃 4 张手牌，补抽 4 张 |

## 四、Run 流程

```
┌─────────────────────────────────────────────────┐
│                  Run 开始                        │
│  玩家获得 52 张标准牌组                          │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│              地图房间选择                         │
│  显示当前房间：高难 / 低难 怪物选择               │
│  显示后续 4 个房间预览（仅怪物名称+HP）           │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│                 开火车决策                        │
│  玩家选择：                                      │
│    □ 仅当前房间（1波）                           │
│    □ 拉 2 波                                    │
│    □ …                                          │
│    □ 拉满 5 波                                  │
│                                                  │
│  预览总 HP、总怪物数                             │
│  确认 → 进入战斗                                 │
└────────────────────┬────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────┐
│                    战斗                          │
│  详见 核心玩法文档 & 内线玩法GDD                  │
└────────────────────┬────────────────────────────┘
                     ↓
              ┌──────┴──────┐
              ↓              ↓
         🎉 胜利          💀 失败
         奖励占位         Run 结束
              ↓
         下一房间 ──→ 循环
         或通关
```

### 通关条件（暂定）

- 连续完成 N 个房间的战斗（N 待定，原型阶段可设为 8~12）
- 或在最后房间击败 Boss 级怪物
