## 一、游戏概述

GuandanKitty 是一款**类斗地主规则的 Roguelike 纸牌游戏**。内线玩法（战斗层）采用纯进攻制——玩家与敌人用扑克牌型互相压制，以清空敌人 HP 为胜利目标，以牌堆耗尽为失败条件。

### 1.1 设计目标

| 黄金规则 | 目标 | 实现方式 |
|------|------|------|
| **正反馈优先** | 正确行为→巨大正反馈，错误→小正反馈 | 四层伤害倍率 + 赢家通吃 + 清空奖励 |
| **高快乐密度** | 快乐总量 / 进程最大化 | 纯进攻、无防御、无拖延、敌方 AI 瞬判 |
| **快决策节奏** | 裸规则不设精细决策，为构筑层留白 | 能出什么出什么，构筑层再叠深度 |

### 1.2 术语

| 术语 | 定义 |
|------|------|
| **牌河** | 本回合双方打出的牌临时存放区，结算后清空洗回牌堆 |
| **压制** | 以同牌型、更大数值的牌盖过对手的前一手 |
| **接龙** | 双方轮流出牌对拼的过程 |
| **牌型** | 一次出牌的组合方式（单张/一对/顺子等） |
| **终结技** | 本回合最后一手牌（对手无法压制的牌），其牌型决定倍率 |
| **战败效果** | 敌人赢得回合时触发的效果，由敌人类型定义 |

## 二、战斗模型

### 2.1 核心规则

```
┌─────────────────────────────────────────┐
│  纯进攻制                                 │
│                                          │
│  胜利条件：敌人 HP ≤ 0                    │
│  失败条件：牌堆 + 手牌 = 0                │
│  无护甲 / 无治疗 / 无防御                 │
└─────────────────────────────────────────┘
```

### 2.2 敌人 HP

- 敌人共享**一个总血条**，无基础 HP
- 总 HP = 各敌人 HP 贡献之和
- 每个敌人拥有独立的等级-血量函数

> 例：Lv3 哥布林(50×3=150) + Lv2 巨魔(80×2=160) = 总 HP 310

## 三、扑克牌体系

### 3.1 牌组

- **52 张标准扑克**（4 花色 × 13 数字），不含大小王
- 数字从小到大：2 < 3 < 4 < … < 10 < J(11) < Q(12) < K(13) < A(14)

### 3.2 牌型

| 牌型 | 构成 | 伤害张数因子 |
|------|------|:--:|
| 单张 | 1 张 | ×1 |
| 一对 | 2 张同数字 | ×2 |
| 三张 | 3 张同数字 | ×3 |
| 顺子 | ≥5 张连续 | ×5~×13 |
| 联队 | ≥3 对连续 | ×6~×26 |
| 炸弹 | ≥4 张同数字 | ×4~×13 |

> 伤害因子 = 出牌张数。出几张就×几，无需查表。炸弹张数虽可能不及长顺子，但可跨牌型压制——伤害让利，战术通吃。

### 3.3 压制规则

- 必须以**同牌型、更大数值**压制**上一手牌**（可为敌人打的，也可为自己上一手——自压）
- 敌人 Pass → 控制权交还玩家，回合不结束
- **玩家 Pass → 回合结束**（仅有玩家有权终结回合）
- 炸弹可以压制任何牌型（详见 `炸弹问题.md`）

## 四、战斗流程

```
战斗初始化
    ↓
┌─── 回合循环 ───────────────────────────┐
│                                          │
│  回合开始（触发"回合开始"效果）           │
│    ↓                                     │
│  【出牌阶段】 玩家先手                     │
│    玩家出牌 → 敌方出牌 → 玩家出牌 → …     │
│    （循环至一方无法压制）                   │
│    ↓                                     │
│  回合结算（判胜负 + 计伤害 + 结算牌河效果） │
│    ↓                                     │
│  回合结束（触发效果 + 牌河清空洗回）       │
│    ↓                                     │
│  检查胜负条件                              │
└──────────────────────────────────────────┘
    ↓
战斗结束
```

### 4.1 战斗初始化

- 触发"进入战斗时"效果
- 从 52 张牌组中抽取 **8 张**手牌（手牌无上限）
- 根据敌人阵容为敌方抽牌
- **敌方手牌对玩家完全可见**

### 4.2 回合开始

- 触发"回合开始"效果
- 总是玩家先出牌

### 4.3 出牌阶段（玩家行动）

| 行动 | 效果 | 限制 |
|------|------|:--:|
| **出牌** | 打出牌型压制上一手牌（可自压） | — |
| **叫牌** | 从牌堆补 6 张手牌 | 每场 3 次 |
| **跳过** | 主动结束回合（**仅玩家有权终结**） | 不鼓励，UI 低调 |
| **清空手牌** | 打出最后一张牌→立即获胜→伤害×10→补抽至 8 张 | 终极操作 |

> **自压**：敌人 Pass 后控制权交还玩家，玩家可继续用同牌型更大数值压制自己。确保持有两付稀有牌型时不会浪费接龙深度。

**敌方出牌规则**：敌人总是用最小的合法出牌压制上一手牌，即使拆散潜在牌型。无法压制则 Pass，控制权交还玩家。意图完全可预知，无计算延迟。

### 4.4 牌河

- 本回合双方打出的牌暂存牌河
- 回合结算时统一处理
- 回合结束后**全部洗回抽牌堆**（无弃牌堆，资源永循环）

### 4.5 回合结算

**胜负判定**：
- 回合仅在**玩家主动 Pass** 时结束
- 玩家先 Pass → 敌人获胜；敌人先 Pass → 玩家后续 Pass → 玩家获胜
- 清空手牌 → 玩家立即获胜（不等 Pass）

**执行效果**：

| 结果 | 效果 |
|------|------|
| **玩家获胜** | ① 伤害 = 牌河总值 × 牌型倍率 × 接龙倍率（清空时再 ×10） ② 牌河全部结算效果归玩家（赢家通吃） ③ 若敌人 HP ≤ 0 → 战斗胜利 |
| **敌人获胜** | ① 触发战败效果（弃牌+补抽，牌堆变薄） ② 牌河结算效果不触发 ③ 若牌堆+手牌=0 → 玩家失败 |

### 4.6 回合结束

- 触发"回合结束"效果
- 牌河清空，所有牌洗回抽牌堆
- 敌方通常抽 1 张牌（成长机制）

## 五、伤害公式

### 5.1 基础公式

```
每手即时伤害 = 牌面值之和 × 出牌张数 × 接龙深度倍率
```

| 因子 | 说明 |
|------|------|
| 牌面值之和 | 该手牌所有牌的牌面数字累加 |
| 出牌张数 | 出了几张就×几（单张×1、顺子5张×5、顺子9张×9…） |
| 接龙深度 | 当前回合内玩家第几次出牌，每手累乘2 |

> 详细公式和计算示例见 [`伤害公式.md`](./伤害公式.md)。

### 5.2 接龙深度倍率

| 玩家出牌次序 | 倍率 |
|:--:|:--:|
| 第 1 手 | ×1 |
| 第 2 手 | ×2 |
| 第 3 手 | ×4 |
| 第 4 手 | ×8 |
| 第 N 手 | ×2^(N−1) |

### 5.3 额外倍率

| 条件 | 倍率 |
|------|:--:|
| 赢得回合的一手 | ×2 |
| 清空手牌的一手 | ×10（立即获胜 + 补抽至8张） |

## 六、牌堆消耗（玩家生命值）

牌堆是玩家的"生命值"。正常情况下牌循环使用，唯一消耗途径：

```
敌人赢回合 → 战败效果 → 手牌中若干张永久移出 → 补抽等量
```

### 6.1 负反馈循环

```
连输回合 → 牌堆变薄 → 抽牌一致性↑ → 强度回升 → 回到均势
（若始终追不回 → 牌堆耗尽 → 失败）
```

### 6.2 设计要点

- 补抽 = 移出，手牌权能不变（杜绝失败陷阱）
- 牌堆薄 → 一致性高（短期利好）但 → 消耗生命（长期威胁）
- 玩家可主动 Pass 利用战败效果打薄牌堆（战术维度）

## 九、设计原理

### 9.1 正反馈结构

```
第一层：打牌 → 牌河膨胀（即时，每张）
第二层：高难度牌型终结 → 牌型倍率（每回合）
第三层：多轮压制 → 接龙倍率滚雪球（每轮）
第四层：清空手牌 → ×10 终极爆发（偶发极致）
```

额外正反馈源：
- **赢家通吃**：敌人出大牌帮你堆伤害
- **战败效果**：策略B——错误→小正反馈（牌堆变薄提高一致性）
- **敌人手牌可见**：阅读→规划→执行的智力自洽

### 9.2 进程保障

- 纯进攻：所有行为推进战斗
- 无防御/护甲/治疗：不存在"只拖不打"
- 敌方 AI 瞬判：无计算等待
- 清空鼓励最大出牌量：最快消耗手牌 = 最快进程

### 9.3 决策节奏

裸规则保持快速决策（"能出什么出什么"），为构筑层预留决策深度空间。构筑层（贴纸/遗物/金币）将在此之上叠加精细化决策，而不是在裸规则中过早引入。
