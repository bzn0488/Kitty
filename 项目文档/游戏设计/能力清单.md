## 分类说明

| 分类 | 用途 | 类比 |
|------|------|------|
| **Agents**（智能角色） | 定义可调用的 AI 专业角色，封装完整工作流。用户可直接 `@角色名` 召唤，自动执行整套设计流程 | 雇佣一个“专家”——他知道整个流程怎么走 |
| **Skills**（技能包） | 领域知识和方法论沉淀，包含设计原则、自检清单、已犯错误记录。被 Agent 和 Prompts 按需调用 | 专家的“工具箱”——里面是专业知识和经验总结 |
| **Prompts**（提示指令） | 面向 AI 的详细操作指令，规定某类任务的执行步骤、输出格式、约束条件 | 专家的“操作手册”——告诉他具体每一步怎么做 |
| **Workflows**（工作流） | 可执行的工作流定义（JSON）和引擎（Python），定义步骤依赖链，强制顺序执行、防止跳步 | 专家的“流程清单”——确保不遗漏任何步骤 |
| **Scripts**（自动化脚本） | 独立的自动化工具，用于索引生成、备份、自检、同步等运维任务 | 专家的“自动化工具”——一键完成重复性工作 |
| **Documents**（文档） | 工作区使用说明、架构设计、定制指南等供人阅读的文档 | 专家的“说明书”——告诉你这个工作区怎么用 |

### 协作关系

```
Agent（角色）
 ├─ Skill（领域知识）
 ├─ Prompt（执行指令）—— 调用设计引擎推进步骤
 │   └─ Workflow / Engine（步骤定义 + 强制检查）
 └─ Script（运维工具）
```

## Workflows（工作流）

| 文件 | 描述 |
|------|------|
| `GDD分析.json` | 用知识库理论体系解析 GDD → 反向推导设计意图 → 冲突检测 → 体验路径分析 → 知识映射 → 多维度分析 → 生成报告 → 讨论决策 → 更新知识库 |
| `分析.json` | 对某个主题进行多维度分析，先理解再输出观点 |
| `反思.json` | 对已有方案进行反思和改进，不制造新知识 |
| `对弈自检.json` | 对抗式自我检查：Agent 自己出题、自己答题、自己裁判、自己改进，通过多轮迭代暴露盲区。基于 Ctx2Skill 论文思想适配到工作流引擎。 |
| `技能精修.json` | 对技能文件进行全面审查、联网补充新知、知识入库、优化迭代 |
| `日报.json` | 汇总当日工作，生成结构化报告 |
| `框架设计.json` | 按照框架设计 Skill 的方法论，从需求到输出完整框架方案的设计流程。含时间压力适配、四原则审核、相对最优解空间分析 |
| `框架转GDD.json` | 基于框架设计方案，扩展生成完整的游戏设计文档（GDD）。含联网搜索补充章节、知识库验证、双层自检（文档质量+设计深度）。详见 技能/专业/框架转GDD.skill.md |
| `生成柒星报告.json` | Agent 根据后端收集的上下文数据，撰写 8 段式柒星报告。 |
| `知识入库.json` | 新增/修改知识内容时，自动打标签、更新索引、校验一致性 |
| `知识回流.json` | 从验证报告、同事文档、失败经验、决策日志等来源中提取可回流的知识，分类后执行到 skill/工作流/prompt/知识库中。 |
| `知识精修.json` | 通过结构化问答深入讨论一个知识点，从中提炼洞察，更新/扩充知识库文档。适用于现有知识文档的深化、修正、扩展 |
| `知识维护.json` | 批量更新知识标签、修复 frontmatter、清理冗余、重建索引 |
| `知识调研.json` | 联网搜索相关理论→评分筛选→冲突检测→用户确认→整合入库→后处理 |
| `自问自答精修.json` | AI 自行对知识文档进行2-5轮自适应深度自问自答——先阅读文档提出最关键问题，搜索知识库解答，逐轮聚焦收敛，最终输出改进方案。用户仅在综合报告阶段做整体方向确认 |
