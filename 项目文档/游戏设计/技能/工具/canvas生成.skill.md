# Canvas 流程图生成

> 按照 `_共享资源/技能/canvas生成规范.md` 中的规范生成 .canvas 文件。
> .canvas 文件是 VS Code 原生支持的无限画布格式，遵循 JSON Canvas 规范 1.0。

## 执行指引

### 步骤 1：确定布局类型

根据内容选择合适的布局：

| 布局类型     | 适用场景               | 排列方式                     |
| ------------ | ---------------------- | ---------------------------- |
| **垂直流程** | 游戏流程、进程、时间线 | 节点垂直居中排列，x对齐      |
| **水平分支** | 系统拆解、决策树、分类 | 根节点居中，子节点向两侧展开 |
| **星形辐射** | 核心系统关联、模块依赖 | 核心居中，关联节点环绕       |
| **自由布局** | 头脑风暴、早期设计     | 根据逻辑关系自由放置         |

### 步骤 2：定义节点

每个节点需确定以下属性：

```
- id:     16位十六进制随机字符串
- type:   text（目前只使用文本节点）
- x, y:   坐标位置（像素）
- width:  节点宽度（px）
- height: 节点高度（px）
- text:   内容（Markdown格式）
- color:  可选颜色代码 "1"~"6"
```

**默认尺寸参考**：

| 内容量            | width   | height  | 示例                 |
| ----------------- | ------- | ------- | -------------------- |
| 短标题（≤10字）   | 250     | 60      | 战斗、系统、流程     |
| 简要说明（≤30字） | 250     | 80-100  | 带描述的概念节点     |
| 详细说明（≤80字） | 340-380 | 140-200 | 含多行描述的设计说明 |
| 长文（≤200字）    | 560-680 | 200-560 | 详细设计文档节点     |

**颜色语义**：

| 颜色 | 代码  | 语义      | 使用场景           |
| ---- | ----- | --------- | ------------------ |
| 红   | `"1"` | 核心/关键 | 核心系统、关键节点 |
| 橙   | `"2"` | 行动/过程 | 流程步骤、动作节点 |
| 黄   | `"3"` | 标记/注意 | 需要注意的元素     |
| 绿   | `"4"` | 目标/完成 | 目标、成果、奖励   |
| 蓝   | `"5"` | 信息      | 辅助信息、参考     |
| 紫   | `"6"` | 分类      | 次要分类、可选内容 |

**文本格式**：使用 Markdown，标题 `##`/`###`，列表 `- `，加粗 `**`。

### 步骤 3：定义连线

每条连线需确定：

```
- id:        16位十六进制随机字符串
- fromNode:  起始节点ID
- fromSide:  方向 top|right|bottom|left
- toNode:    目标节点ID
- toSide:    方向 top|right|bottom|left
- label:     可选标签（2-4字，动词+名词）
- color:     可选颜色代码
```

**方向约定**：

| 布局     | 方向          | fromSide | toSide |
| -------- | ------------- | -------- | ------ |
| 垂直向下 | 上→下         | `bottom` | `top`  |
| 水平向右 | 左→右         | `right`  | `left` |
| 反向     | 下→上 / 右→左 | 相反     |

**箭头规则**：`toEnd` 默认 `arrow` 可省略，`fromEnd` 默认 `none` 可省略。

### 步骤 4：计算布局坐标

**垂直流程布局算法**：
```
首节点 y = 起始Y
后续节点 y = 前节点.y + 前节点.height + 间距
垂直间距 = 120-160px
所有节点 x 统一
```

**水平分支布局算法**：
```
层级0: x = 0
层级n: x = 层级0.x + n × (节点宽度 + 水平间距)
水平间距 = 200-300px
同一层级 y 对齐
```

**间距速查**：

| 关系             | 间距            |
| ---------------- | --------------- |
| 父子连接（垂直） | 120-160px       |
| 兄弟对齐（水平） | 200-300px       |
| 无关联模块       | 200-300px (x/y) |
| 区域分隔         | 400-600px       |

## 输出格式

生成 `.canvas` 文件，JSON 结构如下：

```json
{
  "nodes": [
    {
      "id": "84b2fca91f976a8d",
      "type": "text",
      "x": -280,
      "y": -180,
      "width": 250,
      "height": 100,
      "text": "### 标题\n内容",
      "color": "2"
    }
  ],
  "edges": [
    {
      "id": "e30a07ed8f3c8723",
      "fromNode": "84b2fca91f976a8d",
      "fromSide": "bottom",
      "toNode": "095bb2755894f544",
      "toSide": "top",
      "label": "产出"
    }
  ]
}
```

## 自检清单

```
[  ] 1. 布局类型选择正确
[  ] 2. 节点尺寸匹配内容量
[  ] 3. 颜色语义正确（红=核心，橙=行动，绿=目标，等）
[  ] 4. 连线方向正确（bottom→top 向下，right→left 向右）
[  ] 5. 间距符合规范（垂直120-160px，水平200-300px）
[  ] 6. 所有 ID 唯一（16位十六进制）
[  ] 7. 文本格式规范（标题##，列表-，加粗**）
[  ] 8. 标签简洁（2-4字动词+名词）
```
