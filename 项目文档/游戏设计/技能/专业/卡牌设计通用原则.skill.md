## 卡牌设计上下文

先确定你设计的是哪种卡——上下文决定原则如何落地。

### 上下文 A：独立卡牌

卡牌本身就是完整的最小游戏单位（MTG、杀戮尖塔）。

```
强度 = 卡牌效果本身
自闭环检测：卡牌自身能否独立运作？
诅咒卡检验："三选一里选它吗？"
费用价值：N费=效果N
```

### 上下文 B：组件卡牌

卡牌是更大系统（武器、角色、装备）的组件（荒野与曙光武器卡、自走棋单位卡）。

```
卡牌强度 = 卡牌效果 + 载体特性 + 卡牌x载体协同
自闭环检测：脱离载体后是否还有基本价值？
诅咒卡检验："配合当前载体好用吗？"
费用价值：N费=效果N + 载体特性触发次数
```

**组件卡设计要点**：
- 组件卡应至少有**部分价值独立于载体**——脱离武器后仍有基本功能
- 卡牌与载体的协同应**依赖玩家决策**而非自动触发
- 检查"无脑强"组合——某张卡配合特定载体时强度远超预期
- 载体层的设计由对应 skill 处理（如 `荒野与曙光武器设计.skill.md`），本 skill 只处理卡牌层

## 核心原则

### 原则 1：费用是卡牌的核心平衡维度

费用决定了卡牌在游戏中的节奏位置和强度上限。选择费用模型是设计的第一步。

#### 三种费用模型

|     模型     | 核心约束            | 玩家权衡               | 设计注意                       |
| :----------: | ------------------- | ---------------------- | ------------------------------ |
| **能量消耗** | 每回合固定能量上限  | "我这回合能打出哪些牌" | 低费牌效率不能碾压高费         |
| **冷却/CD**  | 卡牌使用后需等待    | "我什么时机打出这张牌" | 单卡独立冷却，不共享资源       |
| **时间代价** | 推进时间轴=敌方向前 | "我愿意为这效果等多久" | 0费=立即结算，N费=高风险高回报 |

> 费用即时间代价模型中，费用的本质是**风险定价**而非能量定价。高费卡=高风险高回报的决策。
>
> **0费卡的特殊地位**：0费卡在所有费用模型中都有特殊定位，但正反馈和风险因模型而异——
>
> | 模型 | 定位 | 正反馈（好处） | 风险（坏处） |
> |------|------|---------------|-------------|
> | **能量消耗** | 免费操作 | 保障基础操作空间，避免空过挫败感；连击/爆发组合技的基石；维持手牌流动性润滑剂 | 若效果碾压有费同类，会破坏"费用=价值"的核心定价原则 |
> | **时间代价** | 即时结算 | 提供额外决策弹性——本回合多一次操作而不增加风险敞口，让玩家感到"赚到了" | 不推进时间轴=零风险代价，数量和强度需严格控制，否则瓦解"风险定价"核心机制 |
>
> 设计时应同时利用正反馈（让0费卡成为策略选择的延伸）和约束风险（避免0费卡成为唯一最优解）。详见 `82-费用即时间代价.md`。

#### 费用曲线原则（理想分布）

```
低费(1-2): ~25% — 节奏铺垫、过牌、小幅场面影响
中费(3-5): ~50% — 核心战术执行、关键转折
高费(6+):  ~25% — 终结比赛、大场面翻盘
```

**自检**：卡牌的费用分布是否合理？低费卡不会让中费卡完全无用？高费卡是否足够强大到配得上代价？

### 原则 2：每张卡必须有独特定位

**重复检测**：这张卡的功能是否已被另一张卡覆盖？如果两张卡能做到同样的事，删一张。

**自闭环检测**（来自跨项目经验）：

| 检测                       |          判定          |
| -------------------------- | :--------------------: |
| 移除所有队友后仍能满额运作 | ❌ 自闭环——缺乏协作维度 |
| 需要外部状态/队友才能生效  |       ✅ 需要协作       |
| 一次性结算后不持续         |    ✅ 结算型≠自闭环     |

> 例外：经济卡不受自闭环约束（独立于战斗体系运作）。

**诅咒卡检验**：
> 设计完成后自问：*"如果三选一里只有这一张，我会选它吗？"*
> 答案是否 → 这张卡不够吸引人，要么 buff 要么重设计。

**低稀有度底线**：1级/普通卡必须有即时生效的行动，纯条件触发不能作为唯一效果。提供兜底主动（如"基础每6秒1点伤害"→有高星时替换为更强效果）。

> **组件卡说明**：以上自闭环检测、诅咒卡检验、低稀有度底线在组件卡上下文中需叠加载体维度解读。详见上方「卡牌设计上下文」的组件卡说明。

### 原则 3：协同依赖玩家决策

协同（Synergy）是构筑游戏的核心乐趣，但设计不当会导致"说明书式协同"——照着套路抄作业。

| 好的协同                    | 差的协同                      |
| --------------------------- | ----------------------------- |
| 依赖玩家**主动选择**触发    | 自动触发，玩家无需思考        |
| 基于共享中间层（中毒/护甲） | 指名道姓联动（"与卡牌X配合"） |
| 有取舍感（选A就不能选B）    | 无脑堆叠收益                  |
| 卡本身独立可用              | 单卡废物，只靠协同存活        |

**七种基础协同模式**：
1. **同名堆叠**：多张同卡叠加效果（如升星）
2. **异名互补**：不同类型卡配合覆盖弱点
3. **资源转换**：一类资源→另一类（金→卡/卡→金）
4. **条件触发**：满足条件获得额外收益
5. **连锁响应**：A触发B，B触发C
6. **构筑奖励**：牌组构成满足条件获得全局收益
7. **节奏配合**：低费卡铺场→高费卡收割

**自检**：协同是否依赖玩家的主动决策？移除协同后单卡是否还有基本功能？

### 原则 4：效果设计可量化、可验证

每张卡的效果必须满足：

- **成本收益明确**：打出这张卡，玩家付出什么，得到什么
- **数值可比较**：不写"大量伤害"，写"造成 5 点伤害"
- **无歧义**：效果描述应唯一解读，不依赖设计师解释
- **可验证**：测试时能明确判断"效果是否生效"

**术语规范**：
- **充能** ≠ **冷却缩减**：充能使当前CD前进N秒；冷却缩减永久减少技能CD属性
- **叠层** ≠ **计数**：叠层达到阈值触发质变；计数是线性累加

**自检**：效果描述去掉后，测试人员能否准确理解卡牌功能？

## 卡牌类型框架

|     类型      | 核心作用     | 设计要点           | 费用倾向 |
| :-----------: | ------------ | ------------------ | :------: |
| **攻击/伤害** | 直接输出     | 伤害数字+触发条件  | 全费用段 |
| **防御/生存** | 保护单位     | 护盾/治疗/减伤     |  中低费  |
| **辅助/功能** | 资源流转     | 过牌/减费/充能     |   低费   |
| **协同/增益** | 团队增幅     | 光环/间接加成      |  中高费  |
| **经济/资源** | 获取额外资源 | 独立于战斗体系     |   低费   |
| **终结/高费** | 翻盘/终结    | 效果显著配得上费用 |   高费   |

## 设计流程（五步法）

```
第1步：定定位
  → 这张卡在什么场景下使用？核心战术/功能补充/节奏铺垫？
  → 独特定位——与已有卡不重复

第2步：定费用
  → 选择费用模型（能量/CD/时间代价）
  → 对照费用曲线，确定费用区间

第3步：定效果
  → 选择效果类型（伤害/增益/减益/资源/召唤）
  → 满足可量化、可验证标准
  → 检查自闭环

第4步：定数值
  → 参照同类卡的数值基准
  → 对照费用曲线——N费卡的效果≈N
  → 稀有度系数：普通1x，稀有1.5x，史诗2x

第5步：检核
  → 诅咒卡检验？自闭环？独特定位？
  → 效果描述无歧义？数值可比较？
  → 是否满足协同原则？
```

## 快速自检清单

**独立卡牌上下文**：
```
[ ] 费用模型明确（能量/CD/时间代价）
[ ] 费用曲线合理（低中高分布符合设计意图）
[ ] 独特定位——不与已有卡功能重叠
[ ] 自闭环检测通过（或明确标注为经济卡例外）
[ ] 诅咒卡检验通过——三选一里会选它
[ ] 低稀有度有兜底——1级卡不纯条件触发
[ ] 协同依赖玩家决策——非说明书式联动
[ ] 移除协同后单卡仍有基本功能
[ ] 协同模式多样性——使用的协同模式不单一（至少涉及2种不同模式）
[ ] 效果可量化、可验证——无歧义描述
[ ] 成本收益明确——玩家知道付出什么得到什么
[ ] 术语规范——充能≠冷却缩减
[ ] 稀有度与强度正相关
```

**组件卡牌上下文（额外）**：
```
[ ] 卡牌有部分价值独立于载体——脱离后仍有基本功能
[ ] 卡牌与载体的协同依赖玩家决策——非自动触发
[ ] 无"无脑强"组合——不存在某卡配合特定载体时强度远超预期
[ ] 载体层由对应 skill 独立验证
```

## §四 行为边界

### 不适用场景

- **非卡牌游戏**（回合制RPG、动作游戏）——不需要卡牌费用模型和构筑设计
- **纯收集/展示**（如集换式卡牌的收藏属性）——设计重心在美术/叙事而非机制
- **已有成熟体系覆盖**（如 MTG 的色组/费用体系）——不应与成熟体系冲突

### 常见反模式

| 反模式            | 表现                           | 修复                   |
| ----------------- | ------------------------------ | ---------------------- |
| **自闭环**        | 单卡完全自给自足，不需要队友   | 增加外部条件依赖       |
| **诅咒卡**        | 三选一里绝对不选               | 提数值或增加即时效果   |
| **说明书协同**    | "与卡牌X配合时+2"              | 改为共享中间层         |
| **费用倒挂**      | 低费卡效率碾压高费             | 拉平费用曲线           |
| **效果歧义**      | 描述需要设计师解释             | 重写为可量化、唯一解读 |
| **经济/战斗混用** | 经济卡参与战斗或战斗卡产生经济 | 明确分路设计           |

## §五 设计方法论

> 正向引导——面对具体设计场景时如何选择方案。

### 5.1 费用模型选择决策树

| 游戏特征                                | 推荐模型          | 理由                            |
| --------------------------------------- | ----------------- | ------------------------------- |
| 每回合有固定资源（能量/魔力）           | **能量消耗**      | 资源管理是核心策略维度          |
| 卡牌使用后有独立冷却间隔                | **冷却/CD**       | 时机选择 > 资源分配             |
| 行动直接推进游戏进程（时间轴/回合消耗） | **时间代价**      | 风险定价为核心机制              |
| 混合系统                                | **主导模型+辅助** | 如：能量为主+少数CD牌做节奏调控 |

### 5.2 卡牌类型定位启发式

| 设计目标      | 推荐类型              | 费用倾向 | 不推荐                   |
| ------------- | --------------------- | :------: | ------------------------ |
| 直接改变战局  | 攻击/伤害             | 全费用段 | —                        |
| 保护关键单位  | 防御/生存             |  中低费  | 高费纯防御（节奏损失大） |
| 手牌/资源流转 | 辅助/功能 + 经济/资源 |   低费   | 高费经济（回本太慢）     |
| 构建协同体系  | 协同/增益             |  中高费  | 低费协同（效果不显著）   |
| 终结比赛      | 终结/高费             |   高费   | 低费终结（破坏费用曲线） |

### 5.3 协同模式选择决策

- **如果目标**是纵向强化单一战术 → 同名堆叠 / 构筑奖励
- **如果目标**是横向覆盖弱点盲区 → 异名互补
- **如果目标**是增加资源操作深度 → 资源转换
- **如果目标**是提升单局操作上限 → 条件触发 + 连锁响应
- **如果目标**是控制整体游戏节奏 → 节奏配合

### 5.4 自闭环与协作判断

| 卡牌角色    |             自闭环要求             | 典型示例          |
| ----------- | :--------------------------------: | ----------------- |
| 泛用工具卡  | 允许自闭环（但需检查强度是否超模） | 杀戮尖塔·耸肩无视 |
| 体系组件卡  |   必须依赖协作，自闭环=设计失败    | 杀戮尖塔·暴走     |
| 经济/资源卡 |           不受自闭环约束           | 炉石·工程师学徒   |

### 5.5 数值估算基准（能量模型参考）

```
1费 ≈ 3-4单体伤害 / 1-2 AOE伤害
每增1费 ≈ +2-3伤害
附带功能性效果 → 基准降1-2伤害
冷却(秒) = 秒伤期望 × 效果价值系数
时间代价(N) = 风险 = N倍基础值
```
> 以上为通用参考，实际数值需结合具体游戏框架调整。参见 `理论知识/16-卡牌设计规范.md`。
