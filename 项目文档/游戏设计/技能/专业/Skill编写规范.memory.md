## 2026-06-09 初始创建

- **场景**：用户提出"创建一个怎么写 skill 的 skill"，通过知识调研收集了现有 7 个技能的模式、模板、schema、注册流程，提炼为本技能

## 2026-06-09 首次实战：审查荒野与曙光武器设计.skill.md

- **场景**：用户要求用本技能审查现有技能文件
- **使用方式**：加载自检清单（10 项）逐条检查，发现 5 个问题（其中 2 个修复后被用户纠正——description 加触发暗示不合理、删除使用后回流是误判）
- **教训**：
  - 📎 description 加"当你...时自动加载"不适用于项目级 skill（triggers 精准匹配即可）
  - 📎 "使用后回流"是技能的自维护指令，不应归为"操作手册"而删除——技能维护说明属于技能内容
  - 🚀 **发现了技能编写规范自身的未覆盖边界**：自检清单第 8 条要求配套 `.memory.md`，但**使用本技能后我自己也没有写 `.memory.md`**——说明技能规范缺少"使用技能后必须写 memory"的自指约束
- **本文件是补写的**：使用后未即时记录，违背了规范本身的要求。建议在本技能正文中新增一条"使用后立即写 `.memory.md`"的提醒

## 2026-06-09 "Skill 的读者是 Agent" 论文调研 + 第二轮对弈自检修复

- **触发**：用户指出"skill 是给 agent 看的不是给人看的"，要求在 Cornell/arXiv 调研
- **调研发现**：
  - COLLEAGUE.SKILL (2605.31264) 的 progressive disclosure — agent 先读 metadata，需要时才加载详细内容
  - Memento-Skills (2603.18743) — skill 作为 agent 的结构化 markdown 记忆，通过 Read-Write Reflective Learning 演化
  - 现有指南已覆盖大部分碎片（triggers/description/鸟瞰/§四§五），但缺 agent 阅读路径的显式描述
- **对弈自检**（第二轮）：5 探针仅 1 通过，暴露 6 项失败模式：
  (1) 无 agent 阅读路径图
  (2) 🪶 "可选但推荐"低估重要性
  (3) description/prompt 分不清
  (4) 自检清单无 agent 可读性检查
  (5) 指南自身无决策启发式
  (6) MUSE-Autoskill 引用无正文体现
- **修复**：
  - 新增原则 0「Skill 的读者是 Agent」含 agent 阅读路径图
  - 🪶 从"可选但推荐"→"必须除非总 token<500"+ 格式约束 + 省略条件
  - 注册流程区分 description（给人）和 prompt（给 agent）
  - 自检清单 +3 条 agent 可读性检查
  - 内容组织技巧新增第 5 条"内容格式决策表"（6 种信息类型×推荐格式）
  - MUSE-Autoskill 标注具体贡献（§5.5 + 注册流程）
- **自指检查**：原则 0 要求"表格/列表优先于段落"——本次记录本身是段落格式，违背了原则 0 第 2 条。原因：.memory.md 是自然语言记录，不是给 agent 读的 skill 正文，所以不适用。

- **发现的问题**：
