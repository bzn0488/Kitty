## 核心层——驱动力三大模型

### 模型一：心理力学——行为改变倾向

回答"玩家会不会改变当前行为"。这是驱动力分析的**核心入口**。

```
行为改变倾向 = 过程吸引度 + 结果吸引度 
             - 过程排斥度 - 结果排斥度 
             - 当前行为惯性 - 改变成本
```

| 因素       | 检查问题             | 修复方向                  |
| ---------- | -------------------- | ------------------------- |
| 过程吸引度 | 新行为过程中有趣吗？ | 改善体验、加心流设计      |
| 结果吸引度 | 新行为的结果诱人吗？ | 加强奖励预告、双轨收益    |
| 过程排斥度 | 新行为有痛苦环节吗？ | 减冗余操作、降门槛        |
| 结果排斥度 | 失败惩罚重吗？       | 调策略 B（失败→小正反馈） |
| 行为惯性   | 旧习惯有多强？       | 逐步引导而非强制切换      |
| 改变成本   | 切换成本高吗？       | 降低学习成本、给过渡期    |

**用法**：逐项分析。6 项中负数 > 正数 → 玩家不会改变行为。

**自检**：你希望玩家做的行为，6 项中正数多于负数吗？

### 模型二：过度理由效应——驱动力排序机制

行为阈值像一个水桶，驱动力按固定顺序倒入：

```
外驱先倒入 → 内驱再倒入 → 桶满了 → 行为发生
                            ↕
                     溢出的内驱流失（未起作用）
```

**关键规则**：
1. **外驱优先**：人总是先考虑看得见的奖励（外驱），再用内驱补差额
2. **溢出即流失**：如果外驱已灌满桶，内驱连倒的机会都没有 → 长期萎缩
3. **移除外驱后**：只剩萎缩的内驱，不足以驱动行为

| 状态     | 外驱  |     内驱     | 结果                             |
| -------- | :---: | :----------: | -------------------------------- |
| 健康     |  中   |      中      | 两者共同达到阈值，内驱持续被使用 |
| 外驱依赖 |  高   | 低（萎缩中） | 外驱一撤，行为停止               |
| 理想     |  低   |      高      | 玩家为乐趣而玩，外驱只是启动引导 |

**设计启示**：
- 外驱"够用"就好——不要远超过阈值，给内驱留空间
- 需要启动时用外驱引导，同时给内驱培育空间
- 外驱可被"包装"成内驱（不揭示奖励全貌）
- 魂系列弱化数值奖励，正是为了让内驱主导

**自检**：移除所有外部奖励后，玩家还会做这个行为吗？如果不会，说明外驱已过度。

### 模型三：多行为竞争——总驱动力比较

当玩家面对多个可选行为时，每个行为都有自己的驱动力总值：

```
驱动力总值 = 内在驱动力 + 外在驱动力 - 排斥力
```

- 超过**行为阈值**的行为进入候选
- 多个行为超过阈值 → **选择驱动力总值最高的**
- 无一超过阈值 → 选择驱动力最高的（但可能不做）

| 力                 | 定义                   | 示例                                 |
| ------------------ | ---------------------- | ------------------------------------ |
| **内在驱动力**     | 行为本身就是奖励       | 享受战斗过程、好奇探索欲、成长叙事感 |
| **外在驱动力**     | 行为是手段，奖励是目的 | 金币、任务进度、排行榜               |
| **排斥力（负数）** | 让玩家不想做的因素     | 难度太高、节奏太慢、惩罚太重         |

**应用**：当玩家不做你期望的行为时，不一定是"驱动力不够"——可能是另一个行为的驱动力更高。

**自检**：玩家当前还有哪些替代行为可选？你期望的行为的总驱动力是否高于所有替代行为？

## 检验层——SDT 三需求

SDT 不直接驱动行为，而是用来**检验**驱动力设计的健康度。

| 需求       | 定义                  | 设计实践                 | 自检                               |
| ---------- | --------------------- | ------------------------ | ---------------------------------- |
| **胜任感** | 有能力克服挑战        | 数值成长 + 技巧验证      | 玩家是否感到"我变强了"？           |
| **自主感** | 行为出于自我意愿      | 权力下放 + 有意义选择    | 玩家是否觉得"我想做"而非"我得做"？ |
| **归属感** | 与他人/世界的情感纽带 | NPC 关系、叙事参与、社交 | 玩家是否感到"被需要"或"属于这里"？ |

**用法**：当驱动力分析发现问题后，用 SDT 检查——是哪个需求长期缺失导致了驱动力不足？

## 补充层——辅助工具

### Fogg 模型：行为 = 动机 x 能力 x 提示

行为不发生时，先问——真是动机问题吗？

| 缺失要素     | 表现                   | 修复方向                          |
| ------------ | ---------------------- | --------------------------------- |
| **动机**不足 | 玩家知道怎么做但不想做 | 回到核心层用心理力学/过度理由分析 |
| **能力**不足 | 玩家想做但做不到       | 降低门槛、改善教程、简化操作      |
| **提示**缺失 | 玩家想做也能做但忘了   | 增加 UI 提示、新手引导、行为预告  |

**自检**：先排 Fogg——排除能力和提示问题后，再回到核心层分析动机。

### 权力下放框架

| 权力           | 实现成本 | 效果上限 |            优先级             |
| -------------- | :------: | :------: | :---------------------------: |
| **节奏权**     |    低    |    高    | 1——优先（暂停/倍速/环境减速） |
| **难度选择权** |    中    |    高    |               2               |
| **身份定义权** |  中-高   |  中-高   |               3               |
| **体验选择权** |    高    |    高    |               4               |

### 玩家类型分类

| 类型         | 核心驱动力 | 设计方向                   |
| ------------ | ---------- | -------------------------- |
| 红桃（社交） | 归属感     | 协作系统、表情互动、被需要 |
| 方块（成就） | 胜任感     | 高难度挑战、排名、收集     |
| 梅花（杀戮） | 过程爽感   | 割草、连击、大量即时反馈   |
| 黑桃（探索） | 好奇心     | 隐藏内容、解谜、未知发现   |

## 诊断流程

```
第1步：用 Fogg 排除能力和提示问题
第2步：用心理力学分析"行为改变倾向"
第3步：用过度理由效应检查"驱动力排序是否健康"
第4步：用多行为竞争比较"可选行为中哪个胜出"
第5步：用 SDT 检验"三需求有无长期缺失"
第6步：用权力下放/玩家类型找具体改进方向
```

---

## 快速自检清单

```
[ ] 心理力学——6 因素中正数 > 负数，行为改变倾向为正
[ ] 过度理由——外驱未过度，移除外驱后行为仍可持续
[ ] 多行为竞争——期望行为的总驱动力 > 所有替代行为
[ ] SDT——三需求均无长期缺失
[ ] Fogg——排除能力和提示问题后才是动机问题
[ ] 权力下放——玩家至少拥有节奏权或选择权之一
[ ] 玩家类型——核心动机至少适配 2 种玩家类型
```

## 四、行为边界

### 不适用场景

- **纯工具系统**（设置界面、物品整理）——需要可用性优化，非驱动力分析
- **一次性引导**（固定教程步骤）——按流程走即可，不需动机设计
- **已有成熟惯例覆盖**——如抽卡 gacha 的动机已被行业参数充分研究

### 常见反模式

| 反模式           | 表现                         | 根因                       | 修复                             |
| ---------------- | ---------------------------- | -------------------------- | -------------------------------- |
| **外驱依赖**     | 只追奖励，奖励一停就不玩     | 外驱远大于阈值，内驱已萎缩 | 降外驱保底、加内驱设计、奖励包装 |
| **心理力学无视** | 玩家就是不改                 | 改变成本+惯性>吸引度       | 降低切换成本，提高过程吸引力     |
| **替代行为更强** | 玩家在做别的事               | 另一个行为总驱动力更高     | 要么提期望行为，要么降替代行为   |
| **三缺一**       | SDT 有一条长期缺失           | 只关注了 1-2 个需求        | 用 SDT 诊断并补足                |
| **Fogg 盲区**    | 以为是动机问题，实为能力问题 | 只考虑了吸引力没考虑门槛   | 先查 Fogg 再进核心层             |

## 五、设计方法论

### 5.1 心理力学修复优先级

```
负数项排序（从最严重开始）：
1. 改变成本 -> 降门槛（最容易修，效果立竿见影）
2. 行为惯性 -> 逐步引导（需要时间）
3. 过程/结果排斥度 -> 减痛苦（可能涉及核心机制）
4. 过程/结果吸引度 -> 加强（可能需要新内容）
```

**最小成本方案**：先降改变成本——往往比提高吸引度更容易，效果一样显著。

### 5.2 驱动力修复优先级

```
Level 1（阻断性）——必须立即修
  -> 心理力学负>正、Fogg 能力/提示缺失、替代行为驱动力更高
Level 2（体验折损）——应尽快修
  -> 外驱依赖、SDT 缺一项
Level 3（精益求精）——有余力时修
  -> 驱动力效率优化、外驱包装成内驱、玩家类型扩展
```
