# 正反馈技法-具体设计

> 将系统层正反馈原则落地到具体设计对象。系统层原则参见 `正反馈技法-系统框架.skill.md`。
>
> 本 skill 回答："我这张卡/这个武器/这个事件的正反馈够不够？哪里有问题？"
>
> 来源：知识库 `03-卡牌设计检核.md`、`10-家族纪元-正反馈循环修正经验.md`、`选择与决策设计通用规则.md`

---

## 🪶 速查

| 设计对象 | 核心检查点 |
|----------|-----------|
| **卡牌** | 即时反馈、协作感知、自闭环检测、诅咒卡检验、CET 信息性 |
| **武器** | 资源可见性、升级感知、空白期预防、特殊触发 |
| **事件** | 选项差异化、避免两害相权、资源转换、后果可见 |
| **敌人/关卡** | 击败归因、难度匹配、失败保底、阶段性奖励 |

## 核心原则

### 原则 1：每次玩家操作后都要有反馈

无论设计对象是什么，**玩家每做一次有意义的操作，都必须得到至少一种正反馈**。

| 操作类型 | 反馈形式 |
|----------|----------|
| 打出卡牌 | 即时数值变化 + 效果动画 + 后续影响预告 |
| 升级武器 | 属性变化数字 + 新效果解锁 + 视觉/文本变化 |
| 完成事件 | 资源获得 + 叙事推进 + 属性/关系变化 |
| 击败敌人 | 掉落展示 + 经验/进度变化 + 成就触发 |

**边界情况——"空过"检测**：如果玩家什么都不做/无事可做，系统是否仍给予反馈？
- 增量自走棋：每回合自动产出 → 挂机也有正反馈（放置感）
- 回合制策略：无事可做时 → 至少显示"跳过"确认 + 暗示下次行动的预告
- ❌ 空过时无任何反馈 → 玩家感到"卡住了"，负反馈

**自检**：选出设计对象，问"玩家做完这件事后，我给了ta什么反馈？"如果答案是"没有"或"不明显"，就是问题。

### 原则 2：反馈归因必须清晰

鸽子实验教训：玩家必须知道"为什么得到了这个奖励"。

**自检清单**：
- [ ] 玩家能否从反馈中推断出自己的哪个行为触发了它？
- [ ] 反馈是否紧跟在行为之后？（延迟 > 2 秒会削弱归因）
- [ ] 如果多个行为同时触发了反馈，每个反馈是否分别归因？
- [ ] 反馈文本是否包含"因 XXX，获得 YYY"的句式？

### 原则 3：协作反馈必须双方都感知到

协作卡/多人系统中的反馈问题：**施放者获得反馈不难，难的是让被施放者也获得正反馈**。

来自家族纪元实战：
- 帮工：施放者获得 +0.5 产出，被施放者至少应有视觉/音效反馈
- 锻造：消耗 8 金给目标 +0.8 → 双方都应看到"+0.8"的浮动数值
- 讲学：双方智力各 +1 → 双方各自看到自己的属性变化

**自检**：
- 单人局中，协作卡的目标是自己——此时反馈是否仍然有效？
- 多人局中，被施放者是否收到了与施放者同等质量的反馈？（不一定是同等数值，但感知质量应匹配）

### 原则 4：反馈应是信息性的，而非控制性的（CET）

认知评价理论（CET）区分两种反馈：
- **信息性反馈**："你变强了"（你 +1 力量）→ 增强胜任感 + 内驱
- **控制性反馈**："系统给了你金币"（+10 金）→ 削弱自主感，外驱挤出内驱

| 设计对象 | ❌ 控制性 | ✅ 信息性 |
|----------|-----------|-----------|
| 卡牌 | "获得 +3 金" | "你的商业谈判让家族 +3 金" |
| 武器升级 | "攻击力 +5" | "你的剑刃变得更加锋利，攻击力 +5" |
| 事件 | "支持秦朝，+20 金" | "你的家族选择了归顺秦朝，获得 20 金赏赐" |

**自检**：反馈文本去掉后，玩家是否仍能感知到自己"变强了"或"做了有意义的决定"？

## 卡牌设计正反馈检查

### 先过自闭环检测

**自闭环**：一张卡移除场上所有其他队友后，仍能满额运作自身所有效果。

为什么这是反馈问题：自闭环卡不需要任何协作，玩家打出后没有"互动感"，正反馈只有"数字变大了"——属于控制性反馈，信息量为零。

**检测方法**：把场上其他所有单位移除后，该单位是否仍能满额运作？

| 类型 | 示例 | 判定 | 反馈问题 |
|------|------|:----:|----------|
| 纯自我增益 | "自己+1攻击力" | ❌ 自闭环 | 反馈单调——只有数字变化 |
| 依赖自身属性 | "以自身生命值30%造成伤害" | ❌ 自闭环 | 同上 |
| 依赖外部状态 | "相邻友方数量×攻击力" | ✅ 通过 | 每次布阵变化都产生新反馈 |
| 一次性结算 | "上场时为同列友方加血" | ✅ 通过 | 结算时有多单位反馈 |

**例外**：经济卡不受自闭环约束（经济单位独立于战斗体系运作）。

### 诅咒卡检验

设计完成后自问："如果三选一里只有这一张，我会选它吗？"

如果答案是"不会" → 这张卡的反馈不足以让玩家觉得"值得出手"——要么 buff 数值，要么增加即时可见的反馈效果。

### 卡牌正反馈七项检查表

| 检查项 | 通过标准 | 常见问题 |
|--------|----------|----------|
| **即时可见** | 卡牌打出后 1 秒内有数值/动画反馈 | 卡牌打出后无声无息 |
| **后续预告** | 如果有后续效果，有明确的视觉标记 | DoT 卡没有任何"已中毒"标记 |
| **协作感知** | 多人局中双方都收到反馈；单人局中目标==自己时仍有效 | 帮工卡只给施放者显示收益 |
| **代价明确** | 自毁/消耗类卡牌的损耗有明确数值显示 | "牺牲"卡不显示具体损失了多少 |
| **归因清晰** | 反馈文本包含"因 XXX，获得 YYY" | "获得 +3 金"→ 不知道哪张卡给的 |
| **边际递减预防** | 同张卡重复使用时有变化（数值/动画/文本） | 每回合一样的"造成 5 点伤害"→ 腻了 |
| **CET 信息性** | 反馈让玩家感知到"我变强了"而非"系统给了奖励" | 纯数字变化→缺乏叙事归属感 |

## 武器设计正反馈检查

| 检查项 | 通过标准 | 常见问题 |
|--------|----------|----------|
| **资源可见** | 武器特有资源（蓄力槽/箭矢/连击数）每个行动轮都有显示 | 玩家不知道当前蓄力槽有几格 |
| **资源变化反馈** | 资源增减时有动画/数字/音效 | 蓄力槽增加时没声音没提示 |
| **升级感知** | 每级升级都有可感知的变化（不只有数值） | 从 Lv1→Lv2 只变了数字 |
| **特殊触发反馈** | 武器专属效果触发时有独特反馈 | "连击×5"触发时跟普通攻击一样 |
| **空白期预防** | 武器进入冷却/空白期时玩家仍有事可做 | 弓箭手箭矢用完→只能干等 |
| **CET 信息性** | 升级反馈强调"你的技巧精进了"而非"武器数值+5" | 纯数值增长→缺乏成长叙事 |

## 事件设计正反馈检查

| 检查项 | 通过标准 | 常见问题 |
|--------|----------|----------|
| **选项差异化** | 每个选项改变**不同**的资源类型 | 三个选项都产出金币→永远选最大的 |
| **避免两害相权** | 每个选项至少有"得"的一面 | 选啥都亏→玩家感到被惩罚 |
| **资源转换** | 多个选项产出不同资源，非单纯的数值比较 | 选项A产金 vs 选项B产卡 → 真正选择 |
| **触发条件合理** | 事件出现时背景逻辑自洽 | 焚书令出现在文盲家族→不合理 |
| **叙事反馈** | 选择后除了数值还有叙事/文本反馈 | 纯数字变化→缺乏沉浸感 |
| **后果可见** | 选择的长期影响有后续通知 | 选了"A"后 5 回合看不到影响→不知选对了没 |
| **霍布森选择检测** | 去掉 log 文本后各选项仍有可感知的差异 | 三条不同 log，实际效果完全一样 |

## 敌人/关卡设计正反馈检查

| 检查项 | 通过标准 | 常见问题 |
|--------|----------|----------|
| **击败奖励归因** | 击败后立即显示"因击败 XXX 获得 Y" | 击败后自动加经验→玩家不知为什么 |
| **难度匹配** | 越难的敌人→越大的正反馈（且幅度差异可感知） | 打 Boss 跟打小怪掉一样的东西→不值 |
| **失败保底** | 连续失败 N 次后自动给予正反馈 | 连败 5 次啥都不给→习得性无助 |
| **阶段性奖励** | 关卡中途有多个小反馈点，不只在关底 | 只在关底给奖励→中途枯燥 |

## 快速自检清单

```
[ ] 玩家每次有意义操作都有至少一种反馈
[ ] 空过时也有反馈（"跳过"确认或自动产出）
[ ] 反馈归因清晰——玩家知道"为什么得到了这个"
[ ] 协作反馈双方都感知到
[ ] CET 信息性——反馈让玩家感到"我变强了"而非"系统给了奖励"
[ ] 自闭环检测——卡牌不依赖外部也能运作时，反馈是否仍丰富
[ ] 诅咒卡检验——如果只有这张卡可选，玩家会选吗？
[ ] 卡牌打出有即时可见反馈
[ ] 武器资源变化有视觉/音效
[ ] 事件选项避免两害相权 + 霍布森选择
[ ] 难度与奖励幅度匹配
[ ] 连续失败时有保底正反馈
[ ] 同行为反复出现时有多样化反馈（防边际递减）
```

## §四 行为边界

### 不适用场景

- **操作无意义**：如果玩家的操作本身没有游戏内后果（如纯叙事选择、纯展示页面），本 skill 的反馈检查不适用
- **已有专用 skill 覆盖**：如果已经有了具体项目的武器/卡牌/事件设计 skill，应优先参考那些 skill 中的专门检查项
- **设计尚未定型**：在概念发散阶段过早检查反馈细节会扼杀创意——建议在设计收敛后再用本 skill
- **经济系统**：经济卡/纯资源系统不受自闭环和诅咒卡检验约束（通过独立渠道获取，玩家已知其经济定位）

### 常见反模式

| 反模式 | 表现 | 修复 |
|--------|------|------|
| **过度反馈** | 每个行为都有大特效+数值+音效，玩家麻木了 | 区分"重要反馈"和"背景反馈"，重要时刻才用大特效 |
| **归因错位** | 反馈内容与玩家预期不符（鸽子实验） | 检查反馈文本是否说明了触发原因 |
| **协作盲区** | 多人卡只有施放者看到反馈 | 给被施放者至少视觉反馈（发光/震动/浮动数字） |
| **空过惩罚** | 玩家无事可做时无任何正反馈 | 加入"跳过"确认 + 下次行动的预告/期待感 |
| **霍布森选择** | 事件选项只有 log 不同，实际效果一样 | 每个选项必须改变不同资源类型——用"去掉 log 还有差异吗"自检 |

## §五 诊断方法论

### 5.1 多症状联合诊断

单个反模式可能由多个根因导致。当以下症状同时出现时：

| 症状组合 | 指向根因 | 优先修复 |
|----------|----------|----------|
| 自闭环 + 诅咒卡检验不过 | 卡牌缺乏协作维度 | 重设计效果：加入外部依赖 |
| 两害相权 + 叙事反馈缺失 | 事件设计过于功利 | 先修"避免损失"，再加叙事层 |
| 空白期 + 资源不可见 | 武器系统反馈链断裂 | 先修资源可见性（成本最低），再解决空白期 |
| 频繁弃游 + 失败保底缺失 | 习得性无助风险 | 加入连续失败后的保底正反馈 |

### 5.2 正反馈修复优先级

```
Level 1（阻断性）——必须立即修
  → 两害相权、空过惩罚、习得性无助风险
Level 2（体验折损）——应尽快修
  → 归因模糊、自闭环、协作盲区
Level 3（精益求精）——有余力时修
  → 边际递减预防、CET 信息性增强
```

### 5.3 快速排查流程

```
1. 玩家做完操作有反馈吗？             → 无 → 阻断性问题
2. 反馈归因清晰吗？                   → 无 → Level 2
3. 反馈是信息性还是控制性？           → 控制性 → Level 3
4. 协作双方都感知到了吗？             → 没有 → Level 2
5. 同操作反复出现时反馈还有变化吗？   → 没有 → Level 3
```
