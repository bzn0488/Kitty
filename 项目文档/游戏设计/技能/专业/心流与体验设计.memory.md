## 2026-06-09 核心理解修正

- **触发**：用户指出对心流确定机制理解不够深
- **修正**：
  - 明确心流位置由「玩家强度（数值+操作）」vs「游戏难度（关卡+敌人+决策复杂度）」决定
  - 理想状态是在通道内**浮动**（偶尔碰边界），而非稳定在正中
  - **最关键的新增**：session 结束时必须从心流中**直接中止**，而非滑落到无聊或焦虑
  - 海明威效应不仅是"积极离场"，更是心流管理的核心手段
- **改动**：原则 1 重写、自检清单修改、心流曲线模板更新

## 2026-06-09 彻底重写 v1.0 → v1.1（围绕"系统性约束"）

- **触发**：用户认为修补式的修改不够，需要围绕核心重新组织
- **深度调研**：
  - 重新精读 `85-可选回合制设计.md`，理解系统性约束的根本原理
  - 搜索 arXiv 心流相关论文（结果有限，KB 内容更丰富）
- **核心认知升级**：
  - **之前**：心流是"被调节"的（给玩家节奏权/DDA）
  - **之后**：心流是"被对齐"的——设计规则让玩家的理性选择自然与心流通道重合
  - DDA vs 系统性约束的根本区别：暗箱调节 vs 透明规则
  - 可选回合制不是"跳过按钮"，而是一套让最优决策 = 心流有利的系统规则
- **结构变化**：
  - 取消之前的"5 条原则"结构
  - 改为：一、核心命题（两种解法对比）→ 二、系统性约束设计 → 三、Session 管理（入口→浮动→中止）→ 四、支撑工具 → 五、设计流程
  - MDA/快乐密度/峰终降级为"支撑工具"而非核心
  - description 更新、triggers 替换（去掉广义触发词，加入"可选回合制""系统性约束""DDA"等精准词）
