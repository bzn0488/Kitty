# 正反馈技法-系统框架

> 设计游戏中的正反馈循环、信号包装与强化体系。核心目标是让玩家"持续有动力玩下去"。
>
> 关联 skill：`框架设计.skill.md`（§4.4 信号与强化验证）、`工作流与prompt设计通用经验.skill.md`（心理力学行为改变公式）

---

## 🪶 正反馈技法速查

| 技法 | 一句话 | 最佳场景 |
|------|--------|----------|
| **正反馈循环** | 行为→奖励→加强行为→继续 | 核心循环（打怪→掉落→更强→打更难怪） |
| **双轨收益** | 固定保底 + 随机惊喜 | 打怪掉落、开箱、任务奖励 |
| **信号包装** | 把奖励包裹成信号，信号本身引发期待 | 潜在关联链路（Boss→奖励预告） |
| **强化程式** | 按规则给奖励，控制行为频率 | 日常奖励、抽卡、进度条 |
| **积极离场** | 在"顺手"时让玩家下线 | 长线游戏、日常循环 |
| **塑造** | 逐步强化越来越接近目标的行为 | 渐进式教程、复杂技巧分解 |

## 核心原则

### 原则 1：正反馈 > 负反馈

| 策略 | 做法 | 效果 |
|------|------|------|
| ❌ 策略A | 正确→大正反馈，错误→负反馈 | 快速建立认知但触发损失厌恶 |
| ✅ 策略B | 正确→巨大正反馈，错误→小正反馈 | 基础满足 + 相对剥夺感 |

**需要调整行为时，用小的正惩罚而非移除奖励**——损失厌恶使玩家对损失的敏感度远高于同等收益。

**自检**：玩家犯错时感受到的是"没拿到最佳奖励"还是"被惩罚了"？

### 原则 2：信号先于奖励

把奖励"包裹"成信号，让信号本身就能引发期待。步骤：

```
设定关联：行为A → 信号S → 奖励R
实施关联：反复让 S 出现在 R 之前
最终效果：S 本身触发期待感（巴甫洛夫条件反射）
```

**应用**：
- Boss 战前的迷雾/音乐/门——暗示"战斗后有奖励"
- 抽卡动画——拉伸期待感，让奖励本身成为高潮
- 升级音效——点击时已有愉悦感

**自检**：玩家在看到信号时是否已经有期待感？如果没有，说明信号未被有效建立。

### 原则 3：双轨保底 + 惊喜

为同一行为设置两套奖励机制：

| 轨道 | 类型 | 作用 |
|------|------|------|
| 轨道1（固定） | 每次行为都有 | 基础正反馈保底 |
| 轨道2（随机） | 不固定出现 | 可变比率强化制造惊喜 |

**为什么有效**：固定收益保证"不空手而归"，随机收益制造惊喜峰值。双轨结合覆盖浅层和深度玩家。

**自检**：玩家的每次行为是否都有"至少不会亏"的保底感？同时是否有"这次可能出好东西"的期待？

## 强化程式分类与选择

| 程式 | 规则 | 行为模式 | 最佳场景 |
|------|------|----------|----------|
| **连续强化** | 每次都奖励 | 学习最快，消退最快 | 教程期、新手引导 |
| **固定比率** | 固定次数后奖励 | 频率高，强化后短暂停顿 | 每日登录、累计签到 |
| **可变比率** | 平均次数后奖励（不固定） | 频率最高、最稳定、最持久 | 抽卡、稀有掉落、Boss loot |
| **固定时距** | 固定间隔后第一次行为奖励 | 临近时激增，刚获时骤降 | 周常任务、限时活动 |
| **可变时距** | 不固定间隔后第一次行为奖励 | 稳定中偏高，消退慢 | 随机事件、巡逻遭遇 |

> 可变比率是行为频率最高、最持久的强化程式，建议作为随机收益轨道的核心机制。

### 鸽子实验教训

给予奖励时，必须**清晰告知玩家奖励为何而来**。否则玩家归因到错误行为 → 预期落空 → 负反馈。

## 快速自检清单

```
[ ] 正反馈为主：玩家犯错时的体验是"没拿到最佳"而非"被惩罚了"
[ ] 信号先于奖励：关键奖励前有明确的信号/预告
[ ] 双轨保底：每次行为有固定收益，同时有随机惊喜空间
[ ] 强化程式与场景匹配：教程期用连续强化，长线用可变比率
[ ] 奖励归因清晰：玩家知道"为什么得到了这个奖励"
[ ] 边际递减预防：同一种奖励形式不反复使用，有多样化机制
[ ] 积极离场设计：玩家可以在"顺手"时下线，而非耗尽精力
[ ] 习得性无助预防：没有让玩家连续失败多次后不给任何正反馈的场景
```

## §四 行为边界

### 不适用场景

- **单次博弈游戏**（如叙事体验、步行模拟器）——反馈系统不是核心驱动力，过多强化设计反而破坏沉浸感
- **已有成熟品类惯例覆盖的场景**（如抽卡 gacha 已有行业标准参数）——不应与品类惯例冲突
- **以惩罚为核心体验的游戏**（如 Darkest Dungeon、Frostpunk）——正反馈并非设计核心，负反馈才是故意制造的情绪

### 常见反模式

| 反模式 | 表现 | 根因 | 修复 |
|--------|------|------|------|
| **奖励归因模糊** | 玩家触发奖励但不知为何 | 鸽子实验——奖励与行为未绑定 | 给明确归因提示（"因XXX获得奖励"） |
| **固定比率过密** | 每次行为都有大奖励 | 将双轨中的固定轨道设得过高 | 降固定收益，把差值移到随机轨道 |
| **信号坍塌** | 信号出现已不再让人期待 | 信号与奖励关联过密无变化 | 加入可变幅度，让信号不100%对应特定奖励 |
| **负反馈溢出** | 玩家持续受挫 | 策略A（错误→负反馈）使用过多 | 切换到策略B（错误→小正反馈） |

## §五 设计方法论

### 5.1 反馈策略选择

| 场景条件 | 推荐策略 |
|----------|----------|
| 玩家在学习新系统 | 连续强化 + 塑造 |
| 需要维持日常活跃 | 固定比率 + 可变比率双轨 |
| 需要制造惊喜时刻 | 可变比率为核心 |
| 需要控制行为节奏 | 固定时距（周常/限时） |
| 需要检测玩家参与度 | 可变时距（巡逻/随机） |

### 5.2 正反馈循环构建

```
1. 选定一个核心行为 → 2. 定义该行为的正反馈 → 3. 正反馈增强行为动机 →
4. 动机驱动下一个行为 → 5. 检查是否形成循环链
```

**示例**：打怪→掉落装备→更强→打更难的怪→更好的装备

**检查**：循环链中是否有断点？某个行为的正反馈是否不足以驱动下一个行为？

### 5.3 双轨收益设计

```
双轨收益 = 固定轨道（连续强化） + 随机轨道（可变比率）

设计步骤：
1. 确定行为频率（玩家多久做一次这个行为）
2. 设定固定轨道：每次都有，量适中
3. 设定随机轨道：概率触发，量显著大于固定
4. 确保固定轨道让玩家"不空手而归"
5. 确保随机轨道让玩家"有惊喜期待"
```

### 5.4 边际递减预防

同一种奖励反复出现且形式单一时，正反馈显著减少。预防方法：

- **多样化奖励类型**：不只用金币，混合经验/道具/成就/叙事解锁
- **变化奖励幅度**：同样杀怪，有时掉 10 金有时掉 15 金
- **加入随机成分**：双轨中的随机轨道天然对抗边际递减
- **阶段性重置**：刷新周期（每日/每周）让奖励感知恢复基线

### 5.5 积极离场设计

"在写得顺手时停下来"——不要让玩家因疲劳被迫下线。设计要点：

- 提供自然的中断点（章节结束、任务完成、资源耗尽）
- 中断时给予明确的回归奖励预告（"明天登录领取额外奖励"）
- 避免"再玩一局"的无限循环导致疲劳
- 海明威效应：积极离场减少回归的心理阻力


### 5.7 运行时诊断

当系统中出现以下症状时，对照诊断：

| 症状 | 可能根因 | 确认方法 | 修复方案 |
|------|----------|----------|----------|
| 玩家"刷腻了"，活跃下降 | 边际递减——奖励形式单一或固定轨道过高 | 统计同一行为的反复次数 vs 奖励形式种类数 | 多样化奖励类型 / 降固定收益提随机收益 |
| 玩家每天签到但不深入玩 | 固定比率过强，可变比率缺失 | 检查核心循环是否有随机惊喜成分 | 在核心行为中加入低概率高价值奖励 |
| 玩家频繁在某个节点弃游 | 负反馈溢出——连续失败无正反馈兜底 | 检查该节点的失败链路——第几次失败后仍有正反馈？ | 加入保底正反馈（策略B：错误→小正反馈）|
| 新系统/新卡上线后无人使用 | 信号未建立——玩家不知道"用了会怎样" | 检查新内容是否有关联信号/教程 | 加入使用前的奖励预告 / 首次使用特别反馈 |
| 玩家上线后不知所措 | 积极离场缺失——上次下线时未留"钩子" | 检查离场时是否有"明天做什么"的提示 | 设计明确的回归目标 / 离线奖励预告 |
| 玩家对抽卡/开箱失去兴趣 | 信号坍塌——抽卡动画已不再引发期待 | 检查信号是否100%对应固定结果 | 加入信号与结果的可变映射 / 重置信号节奏 |
