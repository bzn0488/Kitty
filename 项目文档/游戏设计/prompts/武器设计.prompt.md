根据设计目标设计一套武器卡组（3~4 张卡 + 武器被动效果）。**只产出设计方案文档，不产生代码。**

**必须先读取目标游戏的武器系统数据模型**——不同游戏的武器系统有不同的结构。以荒野与曙光为例，每把武器包含：武器被动效果 + 3~4 张卡牌（每张有费用/效果/标签）。

**经验参考：** 荒野与曙光的 `技能/项目/荒野与曙光/荒野与曙光武器设计.skill.md` 中积累了设计经验（§四），设计时遇到「这个设计对不对？」的疑问，可以先查阅是否有对应经验教训。

**⚡ 引擎约束（不可跳过）：** 每步开始前 `python 工作流/设计引擎.py check <step_id>`，完成后 `python 工作流/设计引擎.py done <step_id> <摘要>`。步骤 ID 见 `工作流/设计路由.json` 的 `武器设计` 工作流。自检时调用 `python 工作流/设计引擎.py audit` 并将结果嵌入预览。

- **数值参考表**（1费4~5伤害等）基于荒野与曙光的数值体系。其他项目需参考目标游戏的已有数值。
- **核心原则 5**（费用=时间价值）是荒野与曙光的核心规则。其他项目如有不同的费用机制，以目标项目的规则为准。

## 创造力标记

设计可以基于已有机制，也可以创造新机制。**区分两者的来源类型即可：**

| 标记 | 含义 |
|------|------|
| ✅ 基于 | 效果引用项目文档中明确存在的机制/数值/事件 |
| 💡 推断 | 效果基于已知机制但做了合理延伸（如阈值假设、组合效果） |
| ✨ 新机制 | 项目文档中不存在，Agent 自主创造的新概念/效果 |

鼓励创造新机制，但必须标注。用户看到预览时可以判断是否接受。

## 核心原则（正向引导）

这些原则指导你做出好的设计，而非限制你的创造力。

1. **武器类型即玩法** — 每种武器的核心机制（蓄力/护盾/连击/箭矢/自动蓄力）从根本上决定了它的战斗风格。设计卡牌时必须围绕该核心机制展开，而非套用通用卡牌模板。
2. **卡组是整体而非拼装** — 一套武器的 3~4 张卡应围绕一个核心循环有机组织，而非将几个独立效果拼在一起。每张卡在循环中有明确的功能角色。
3. **反自闭** — 出牌决策必须依赖战场状态（怪物状态、队友行动、当前资源量），不能存在独立于战场的"最优出牌序列"。
4. **联动基于共享中间层** — 禁止指名卡牌联动。联动通过连击数、标记、护盾值、蓄力槽等共享数值/状态实现。注意：部位破坏是怪物系统的输出，武器只能作为条件引用，不可作为核心设计概念（详见 `_武器设计规范.md` §二）。
5. **费用 = 时间价值** — 费用推进时间轴。0 费效果必须有明确限制，高费效果须确保回报匹配时间成本。
6. **差异化** — 每种武器类型内的每套卡组应有独特的决策倾向，避免同类型武器"换皮"。
7. **联想空间 > 精确规则** — 卡牌效果写完后问自己"玩家第一次看到这张卡时脑中会闪过什么画面？"如果答案是空白，说明缺少联想空间。好的卡牌效果在打出前就产生了正反馈（幻想联动的多巴胺）。
8. **设计接口而非combo** — 提供操作权限（可丢弃/可抽牌/可移动等），让玩家自己去发现和谁的卡配合，而不是把联动写在卡面上。接口型设计比combo型更有生命力。
9. **"无法打出"卡是被动引擎** — 当武器需要频繁生产基础资源时，设一张"无法打出"卡让资源在抽牌/弃牌等日常操作中自然积累，降低主动操作密度。
10. **阶段感 > 最终奖励** — 成长曲线的每一层应有独立的可感知信号（伤害/速度/范围等变化），而非只在终层给一个大奖励。阶段性正反馈维持动力，最终高潮兑现情绪。
11. **每把武器有自己的评判标准** — 风险博弈型武器的评判标准不适用于正反馈循环型武器，反之亦然。设计新武器时先回答"这把武器的核心体验是什么"，用它的核心体验做评判标准，而不是其他武器的标准。

## 步骤

### 0. 读取目标游戏武器系统数据模型  ← engine: model

在开始设计前，先确定本游戏中「武器卡组」包含哪些组成部分。

方法：
- 搜索项目中已有的武器定义（武器被动效果、卡牌数据结构、费用体系）
- 读取目标游戏的战斗系统基础规则（如荒野与曙光的"小队共享血量""费用=时间推进"等）
- 读取目标武器类型的系统规格和设计规范（如有）
- 输出武器数据模型清单

示例输出：
```
## 武器数据模型（荒野与曙光）
| 组成部分 | 必须 | 说明 |
|----------|------|------|
| 武器名称 | ✅ | 体现怪物素材来源 |
| 武器被动效果 | ✅ | 定义核心机制（如"连击达10衍生绞杀"） |
| 卡牌列表 | ✅ | 3~4 张，每张含费用/效果/标签/角色定位 |
| 核心资源 | ✅ | 连击数/蓄力槽/护盾值/箭矢数/自动蓄力值 |
| 联动维度 | ✅ | 1~2 个额外的战场相关维度（标记/破甲/毒素等） |
| 循环逻辑 | ✅ | 一句话描述"积攒→消费"或"蓄力→打出"的节奏 |
```

### 1. 概念发散  ← engine: ideation

**产出：至少 3 个候选概念方向，每方向附一句话核心体验（必须展示）**

围绕武器类型（双刀/大剑/剑盾/弓箭/斧子）发散可能的机制方向，注意与同类型已有设计**差异化**：

```
概念A"方向名称"：核心体验是"一句话体验描述"
概念B"方向名称"：核心体验是"一句话体验描述"
概念C"方向名称"：核心体验是"一句话体验描述"
```

概念发散的检查原则：
- 避免与同武器类型已有设计的核心循环撞车
- 每个方向应导致不同的卡牌角色分工
- 方向之间差异明显（不是同一概念的微调）

#### 1.5 选定方向后的正反馈验证

选定概念方向后，必须执行正反馈验证，确保玩家体验中有明确的、可追溯的正反馈来源。

**第一层：概念层面的正反馈来源**

1. 在 `知识库/` 中搜索正反馈相关理论文档（如 `理论知识/46-正反馈与负反馈循环.md`、`游戏设计理论/归纳文档/04-快乐总量与正反馈技法.md`、`项目经验/10-家族纪元-正反馈循环修正经验.md`）
2. 若找到理论文档：引用对应技法说明该概念如何产生正反馈
3. 若未找到可用理论：由Agent自行阐述该概念产生正反馈的明确理由（必须具体，不可泛泛说"这很有趣"）

> 正反馈验证（第一层）通过后进入设计锚点。卡牌层面的正反馈信号验证（第二层）在 §4.3 执行，通不过的卡牌需要重新设计效果。

### 2. 设计锚点  ← engine: core_anchor

**产出：设计目的 + 武器调性 + 核心循环 + 共享中间层检查结论（必须展示）**

从概念发散中选择一个方向，收敛为设计锚点。**本步骤不涉及武器被动效果和卡牌设计**——它们是后续的独立步骤，与此同级。

> ⚠️ **武器被动效果与卡牌系统是从设计锚点出发的两条同级设计线。** 不要在定义锚点时提前决定被动或卡牌的形式，先回答"这把武器是什么"，再回答"它的被动和卡牌分别怎么做"。

#### 2.1 设计锚点

回答两个问题锁定设计方向：

| 问题 | 说明 | 示例 |
|------|------|------|
| 一句话设计目的 | 这套武器让玩家多一种什么思考方式？ | "让玩家在'集中标记一个部位'和'切换目标重新缠绕'之间权衡" |
| 武器调性 | 一句话描述这把武器的氛围气质 | "锁链双刀——甩出锁链缠绕怪物部位，缠住后拉近距离痛击" |

> ⚠️ 武器调性是氛围描述，不是功能说明。不需要详细描述武器外观——猎人如何使用就够了。

#### 2.2 核心循环

描述这套武器的完整循环：

```
积攒期（怎么积累核心资源）→ 阈值（达到多少触发质变）→ 消费（怎么消耗资源兑现收益）→ 回归（如何回到积攒期）
```

#### 2.3 共享中间层复用检查

在定义新的联动维度之前，先系统性地检查能否复用现有的共享中间层。

**目标效果**：[一句话描述你想要的效果]

**现有共享中间层检查**：

| 中间层 | 是否可表达目标效果 | 如不可，原因 |
|--------|:---:|------------|
| 连击数 | 是/否 | ... |
| 护盾值 | 是/否 | ... |
| 蓄力槽 | 是/否 | ... |
| 箭矢数 | 是/否 | ... |
| 标记 | 是/否 | ... |
| 破甲 | 是/否 | ... |
| 部位位置（头/身/尾等） | 是/否 | ... |
| **部位已破坏**（部位破坏系统输出） | **有条件** | **武器只能作为判断条件引用，不可作为核心设计概念。详见 `_武器设计规范.md` §二** |

**判定**：
- ✅ 能找到复用方案 → 用现有中间层实现，不引入新概念
- ❌ 全部不可行 → 允许引入新概念，但须满足以下条件之一：
  - 新概念能同时服务于 2+ 种武器类型（跨武器可复用）
  - 新概念与核心机制有本质的"质变"关系（不只是数值叠加）

> 共享中间层检查的结果将传递给后续的武器被动效果设计和卡牌系统设计步骤。

### 3. 武器被动效果设计  ← engine: weapon_passive

**产出：武器被动效果定义（须与核心概念对齐，必须展示）**

> ⚠️ **武器被动效果与卡牌系统是同级设计任务。** 两者都从设计锚点推导，不存在谁先决定谁。如果被动效果写成了通用模板，说明这一步没有做到位。

> 💡 **与 §4 的执行顺序**：`weapon_passive` 和 `card_system` 互不依赖，引擎允许任意顺序。你可以先做被动再做卡牌，也可以先定卡牌角色再设计被动来配合——哪种思路更顺就从哪种开始。

> ⚠️ **被动效果是卡牌决策空间的粘合剂，不是体验的中心。** 如果被动效果本身已经决定了整个武器的玩法，说明卡牌角色分工没有做透——回退到 §4.1 重新设计卡牌的独立决策空间。

设计武器被动效果（进入战斗自动生效，不占用手牌位）：

- **必须与核心概念对齐**：被动效果应是核心循环的一部分，而非通用模板。回答"这个被动如何体现设计锚点？"
- 必须定义核心资源的**积攒方式**（如"全队每次攻击+1连击数"；或"每回合获得1层毒液"）
- 必须定义核心资源的**质变消费方式**（如"连击数≥10时，衍生1张终端卡"；或"蓄力完成时自动攻击"）
- **联动维度**：从共享中间层复用检查的结果中选择。如果通过了复用检查引入了新概念，在此定义其规则
- 被动效果的强度控制：被动不直接造成伤害，它定义规则。伤害由卡牌兑现。

**与被动的正确关系**：
- ✅ 正确：被动提供稳定的资源收入，卡牌提供资源转换/爆发的决策
- ❌ 错误：被动是通用模板（蓄力5/20伤），卡牌自成体系——两者没有对话

**自检**：
- [ ] 这个被动效果如果换成通用模板，卡牌系统是否还成立？→ 如果成立，说明被动没有发挥作用
- [ ] 被动效果是否让玩家每回合面对与核心概念相关的决策？→ 如果不是，被动只是填空

### 4. 卡牌系统设计  ← engine: card_system

**产出：卡牌角色分工 + 具体卡牌设计 + 卡牌正反馈验证（必须展示）**

> ⚠️ **卡牌系统与武器被动效果是同级设计任务。** 两者都从设计锚点推导，不存在谁先决定谁。卡牌不应假设被动是通用模板而来设计，反之亦然。

> 💡 **与 §3 的执行顺序**：`card_system` 和 `weapon_passive` 互不依赖，引擎允许任意顺序。如果你先设计卡牌角色分工，再回来设计被动来配合它们，是完全可行的。

#### 4.1 卡牌角色分工

先定卡牌的**独立决策空间**。每张卡的核心价值在于**让玩家多一种思考方式**，而非配合自动攻击。

设计 3~4 张卡，每张回答**它在核心循环中承担什么独特功能**和**玩家用这张卡时在想什么**：

| 角色 | 数量 | 作用 | 独立决策空间（玩家用这张卡时在想什么） | 设计要求 |
|------|------|------|----------------------------------------|----------|
| 启动端 | 1~2 张 | 加速核心资源积累，绑定战场收益 | "应该优先集火哪个部位？" | 多重触发、自动使用、被动触发等 |
| 数值端 | 1~2 张 | 引入额外维度，提供决策空间 | "现在投入资源划算吗？还是再等一轮？" | 与怪物/队友/战场状态互动 |
| 终端 | 0~1 张 | 消费核心资源，提供爆发 | "时机到了吗？现在打出还是等更好的窗口？" | 额外+一次性+虚无，质变效果 |

> 不是所有武器都必须严格按三端分工。复杂度低的武器可以只有 3 张卡，将启动和数值合并。核心判据是：**每张卡不可替代**。

#### 4.2 具体卡牌设计

将概念和角色分工落地为具体的卡牌。每张卡必须包含：

| 字段 | 说明 |
|------|------|
| 费用 | 0=不推进时间轴 / 1~4=推进N回合 / 无法打出 |
| 卡牌名称 | 使用技法/战技概念，不体现怪物素材 |
| 效果 | 精确无歧义，包含数值、条件、持续时间 |
| 标签 | 额外/一次性/虚无/漂浮/多重触发N/无法打出（按需选用） |
| 循环角色 | 启动端/数值端/终端 |
| 来源 | ✅ 基于 / 💡 推断 / ✨ 新机制 |
| 边界行为 | 单人局/极端情况的行为定义 |

> **卡牌级质量参考**：每张卡的设计应满足 `_共享资源/prompts/卡牌设计.prompt.md` 的基本质量标准，特别是设计意图明确（非凑数）、极端情况已考虑、效果描述无歧义。

```
### 武器被动效果
[一句话描述被动规则]

### 卡牌列表

| 费用 | 名称 | 效果 | 标签 | 循环角色 | 来源 | 边界 |
|------|------|------|------|----------|------|------|
| 1 | 卡牌A | 效果描述 | — | 启动端 | ✅ 基于 | 单人局... |
| 3 | 卡牌B | 效果描述 | 多重触发3 | 数值端 | 💡 推断 | ... |
| 0 | 卡牌C | 效果描述 | 额外+一次性+虚无 | 终端 | ✨ 新机制 | ... |
```

**来源列取值：** `✅ 基于` / `💡 推断` / `✨ 新机制`。每张卡的整体来源取所有效果中最低级别。

> ⚠️ **设计完成后的自检提醒**：写完全部卡牌后，回头对照武器被动中的数值——确认每张卡的条件阈值与被动数值自洽（如被动蓄力最大值 vs 条件中的蓄力阈值）。最常见的低级错误是写卡牌时用了和被动不一致的数值。

#### 4.3 卡牌正反馈验证（第二层）

每张卡设计完成后，逐卡验证：

> **信号 ≠ 视觉/音效反馈。信号 = 玩家阅读卡牌时脑中自动形成的因果链：**
> "如果我做A → 就会得到B → B对我有利 → 我该追求A"
>
> 这是完全发生在玩家脑内的认知过程，在打出卡牌之前就已经产生了正反馈的联想。

逐卡检查：
- 玩家读这张卡时，脑中能否形成清晰的因果链（"如果我做X→就会得到Y"）？
- 这个因果链是否让玩家产生了"我判断对了"的智力优越感？
- 这个因果链是否加强了玩家重复该行为的意愿（正反馈循环）？

**验证格式：**
```
卡牌正反馈信号验证：
卡A「卡牌名称」
- 玩家读卡时形成的因果链：如果我[行为] → 就会[结果]
- 是否产生智力优越感："我判断对了"的点在哪里
- 是否形成正反馈循环：这次[行为]获得[结果] → 下次更倾向重复该行为

卡B「卡牌名称」
- ...
```

**示例（收割者卡牌）：**
```
卡牌正反馈信号验证：
卡A「尾棘扫击」
- 因果链：如果我集火破坏一个部位 → 再打它就能从6伤变成14伤
- 智力优越感："我等部位破了再打，果然更赚——我判断对了时机"
- 正反馈循环：这次收割了14伤 → 下次更主动规划优先破坏哪个部位

卡B「棘刺蓄力」
- 因果链：如果我选一个部位 → 自动攻击就会优先打它 → 我可以控制集火方向
- 智力优越感："我把自动攻击引到了我想打的部位——我的路线选择起了作用"
- 正反馈循环：这次成功集火了目标部位 → 下次更注意观察战场决定引导方向
```

> 卡牌层面的正反馈信号验证（第二层）通不过的卡牌需要重新设计效果。

### 5. 行为预检  ← engine: precheck

按 `_共享资源/agents/游戏设计师.agent.md` 中的行为预检框架执行六维度分析。以下是武器设计（区别于单张卡牌设计）的特有提问角度：

**选择情境补充**
- 这套武器的卡牌是在什么场景下进入玩家视野的？（初始自带/素材锻造/怪物掉落/任务奖励）
- 如果是锻造获得，玩家投入资源的预期回报是什么？
- 如果是初始自带，新玩家第一次使用的学习曲线如何？

**内在驱动力补充**
- 玩家在一场战斗中能体验到几次完整的"积攒→消费"循环？
- 循环中的决策点是否清晰（积攒期知道在积攒什么，阈值期有期待感）？
- 多次使用后是否还有新鲜感，还是沦为"机械执行"？
- 武器被动效果是否与核心概念对齐？还是说换一把武器换掉被动后，这套卡牌还能满额运作？

**外在驱动力补充**
- 武器被动效果是否足够明显（战斗中有视觉/反馈提示）？
- 队友能否感知到这套武器的节奏（全队连击/共享护盾等）？

**排斥力补充**
- 是否存在"卡手"风险（关键牌上手时机不对导致循环断裂）？
- 学习成本是否在目标玩家的接受范围内？
- 是否有"操作复杂但收益不明显"的风险？

**武器特有风险模式**
- 🔴 **自闭环**：移除所有队友后，这套武器是否仍能满额运作？→ 是则违反反自闭原则，需引入战场依赖
- 🔴 **被动脱节**：武器被动效果与卡牌核心概念无关联，属于通用模板填空？→ 需回退到 §3 重新设计被动
- 🔴 **节奏真空**：积攒期过长导致玩家没事做 → 需缩短周期或填充积攒期的微决策
- 🔴 **数值碾压**：与同类型武器相比全面占优 → 要么提升其他武器，要么调整
- 🔴 **认知负荷**：循环需要追踪过多状态（核心资源+联动维度+队友状态）→ 简化到 2~3 个追踪要素

> **卡牌级认知风险（单张卡）**：如果某张卡的效果需要大量游戏知识才能判断使用价值，考虑简化或增加牌面提示。但武器整体的劣势面是允许存在的——武器作为最小单位，设计上有弱点反而增加构筑深度，玩家通过副卡组弥补弱点是预期行为。

**产出**：六维度驱动力分析 + 武器特有风险检查（含被动脱节检测）+ 设计目的校对结论。通过预检后进入自检。

### 6. 内部自检（过程不展示，审计结果嵌入预览）  ← engine: selfcheck

自检过程中发现问题时自动修正并重复自检，**此迭代过程不展示给用户**。自检全部通过后调用 `python 工作流/设计引擎.py audit`，将审计结果嵌入步骤 7 的预览。

完成草稿后，逐条检查以下清单。**全部通过后才能输出预览。**

```
## A. 硬约束检查

### 🌐 通用约束（所有游戏适用）
[ ] 禁止指名卡牌联动，联动基于共享中间层

### 🏕️ 荒野与曙光特有限制（其他项目替换为目标游戏规则）
[ ] 小队共享血量，无"最低生命值队友"类效果
[ ] 费用=时间推进，0费有明确限制
[ ] 禁止无视/穿透护甲（除非武器特殊机制允许）
[ ] 不提供 Buff/护盾

## B. 角色定位检查

[ ] 每张卡在核心循环中有明确的不可替代功能
[ ] 每张卡有明确的设计意图（不是"凑数"）
[ ] 不存在"看不出什么时候用"的卡
[ ] 同一武器内无功能重叠的卡

## C. 反自闭检查

[ ] 出牌决策依赖战场状态（怪物/队友/资源）
[ ] 不存在独立于战场的"最优出牌序列"
[ ] 自闭环检测：移除所有队友后卡组无法满额运作

## D. 节奏检查

[ ] 积攒期与消费期交替，有紧有松
[ ] 循环高点有足够回报感
[ ] 循环可在一场战斗中多次触发
[ ] 积攒速度与阈值匹配（不焦虑不无聊）
[ ] 数值自洽性：所有条件阈值与武器被动数值一致（如蓄力最大值 vs 条件阈值、费用 vs 伤害基准等）
[ ] 时序一致性：被动触发时序与卡牌效果不在同一动作中加减同一资源（检查：打出每张卡时，被动是否抵消了卡牌的效果导致净效果为零？）

## E. 可追踪性检查

[ ] 所有触发条件基于当前可见状态（非历史记忆）
[ ] 数值比较使用绝对值阈值
[ ] 条件措辞直接可读（玩家能一眼判断触发条件是否满足）
[ ] 效果描述无歧义（实现者能准确理解）
[ ] 伪条件检测：条件是否因被动效果或基础规则而永远为真？（是→伪条件，需修改）

## F. 来源标记检查

[ ] 每张卡牌的来源类型已标注（✅基于 / 💡推断 / ✨新机制）
[ ] 每项效果有来源追溯（若整卡标记为 ✅ 基于，确认所有效果都有明确的项目文档依据）

## G. 一致性检查

[ ] 被动效果与核心概念对齐：被动不是通用模板，而是核心循环的一部分（检查：把被动换成通用模板后，卡组是否还成立？）
[ ] 卡牌机制是从怪物概念自然推导（遮住怪物名能反推）
[ ] 与同武器类型已有设计的核心循环无撞车
[ ] 效果的"结构"非已有设计的数值换皮

## H. 联想与信号检查（2026-06-09 新增）

[ ] 每张卡阅读时能引发玩家"和谁配合？"的联想（不是一眼算完数值就完）
[ ] 卡牌提供了"操作权限"（可丢/可抽/可移等）而非写死联动
[ ] 如果使用"无法打出"卡，它的触发路径清晰且自然（玩家不需要专门操作就能触发）
[ ] 成长曲线有多个可感知的阶段里程碑（不只是最终奖励）
[ ] 跨角色交互设计优先考虑"操作对方资源"（弃队友牌等），引发build联想而非纯数值叠加

## I. 新资源放行检查

[ ] 引入的新联动维度是否经过了共享中间层复用检查（步骤 2.3）？
[ ] 是否确认没有现成的共享中间层可以表达目标效果？
[ ] 新概念是否能被 2+ 种武器类型复用（或与核心机制有质变关系）？
```

如果发现违规 → **自动修正设计** → 重复自检，直到全部通过。
