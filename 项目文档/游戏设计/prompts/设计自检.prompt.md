设计质量内部自检。**由其他设计 prompt（卡牌设计、事件设计）在内部调用，不直接输出给用户。**

当检测到违规时，应自动修正设计并重复自检，直到全部通过。

**自检前必须先调用 `python 工作流/设计引擎.py audit` 获取工作流审计结果。审计中如有 ❌ 步骤，必须先完成所有缺失步骤，再重新自检。**

## 自检清单

### 方法论完整性检查项（通用）

```
[ ] 步骤 0 已执行（目标游戏数据模型已读取）
[ ] 情景分析已完成（有明确的玩家状态描述）
[ ] 设计目的已确立（有一句话设计目标）
[ ] 概念发散已完成（至少 3 个候选概念）
[ ] 行为预检已执行（六维度分析完整，设计目的校对一致）
[ ] 每步推论的引用来源已标注
[ ] 本次改动涉及架构/流程/配置？如是，决策日志已记录
```

### 卡牌设计检查项

```
[ ] 每张卡牌有明确的设计意图（不是凑数）
[ ] 卡牌间有差异化定位（无功能重叠）
[ ] 协作卡定义了单人局/无目标时的行为
[ ] 新机制有参考依据或理论支撑
[ ] 极端情况已考虑（零资源/满员/单人）
[ ] 卡牌数值在同类中有参照系（不过高也不过低）
[ ] 效果描述无歧义（实现者能准确理解）
[ ] 获得条件已定义且与强度匹配
[ ] 行为预检中识别的风险已在设计中处理
```

### 事件设计检查项

```
[ ] 每个选项有实质性差异（去掉文本后仍可区分）
[ ] 无"两害相权"式负反馈选项
[ ] 选项间是资源转换而非数值大小比较
[ ] 事件触发条件合理
[ ] 将所有选项收益换算为同一种资源时，无"永远最优"
[ ] 极端情况已考虑（资源不足时选项的行为）
[ ] 行为预检中识别的风险已在设计中处理
```

### 通用检查项

```
[ ] 设计方案文档格式完整（表+意图+边界）
[ ] 新引入的概念/术语有定义
[ ] 与已有系统的交互关系已说明
[ ] 设计约束被遵守（技术/资源/风格）
```

## 使用方式

调用此 prompt 时，附上当前设计草稿。自检完成后：

1. **全部通过** → 返回"✅ 自检通过"，继续输出预览
2. **有违规** → 列出违规项，自动修正设计，再次自检
3. **反复违规无法解决** → 在预览时标注"⚠️ 未解决项"供用户判断
