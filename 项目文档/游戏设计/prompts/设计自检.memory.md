## 2026-06-04 三国卡牌自检

- 检查清单未覆盖：**跨时代卡牌池一致性**——新加时代后需验证其卡牌 handler 已全部注册（本次差点漏了采药的延迟效果处理器）
- 检查清单未覆盖：**时代特有经济参数**——战争时代税率50%需确认职业产出数值与之匹配（本次未深入验证，建议将来检查）
- 协作卡 fallback 是常见遗漏点，建议在清单中加一项："协作卡在无目标时自动降级或自选"

## 2026-06-08 自检机制重构

- 自检从独立步骤改为内部工具，由设计 prompt 在预览前自动调用
- 检查清单根据设计类型分类（卡牌项/事件项/通用项），不再混用
- 核心原则从"禁令"改为"正向引导"，质量把控交给自检清单而非 prompt 约束
- 自检失败时自动修正设计并重复，用户看到的永远是通过自检的版本
- 未来方向：积累常见违规模式，将自动化修正逻辑逐步沉淀到 prompt 中
