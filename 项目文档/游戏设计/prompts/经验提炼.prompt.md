将本次设计过程中发现的通用原则提炼到知识库和技能文件中。

## 步骤

### 1. 回顾对话
- 本次设计遇到了哪些问题？
- 解决了哪些之前未遇到的情况？
- 有哪些模式可能适用于其他项目？
- 当前流程/prompt 是否有需要改进的地方？

### 1½. 边界检测（不可跳过）

在写入之前，先扫描已有相关记录，判断本次经验的类型。

**扫描范围**：
- 本次使用过的 prompt 对应的 `.memory.md`
- 相关 skill 文件中的已犯错误/经验总结
- 相关知识库文档

**判断标准**：

| 类型 | 标记 | 触发条件 | 处理方式 |
|------|------|---------|---------|
| **全新经验** | 🚀 新边界 | 与已有内容重叠 <30% | 完整写入，标注为新发现 |
| **增量补充** | 📎 增量 | 重叠 30%-70% 但有新角度 | 写入并注明"补充：已有记录 X 未覆盖的情况" |
| **重复/冗余** | 🔄 已存在 | 重叠 >70% | 不写入，可在已有记录下追加一条 "+1 验证" |
| **冲突** | ⚡ 冲突 | 与已有内容明显矛盾 | 写入并注明"取代/修正 已有记录 X"，同步更新旧记录 |

**产出**：本次经验的类型判定结果（在后续步骤中体现对应标记）

### 2. 更新 prompt 的 .memory.md
- 找到本次使用过的 prompt 对应的 `.memory.md`
- 追加本次的经验笔记（日期 + 标记 + 内容）
- 内容应包括：遇到的情况、处理方法、边界类型判定结果
- 标记按第 1½ 步的判定结果：🚀 / 📎 / 🔄 / ⚡

### 2½. 更新 skill 的 .memory.md（不可跳过）

本次设计过程中，Agent 可能自动加载了多个技能文件（通过 triggers 匹配）。需要将这些技能的 .memory.md 一并更新。

**检测哪些 skill 被使用**：
- 先读取 `_共享资源/技能/_registry.json`，获取所有 skill 的 `triggers` 列表（防止凭记忆比对遗漏）
- 回顾本次对话（用户消息 + Agent 推理），提取出现的关键词
- 将关键词与 `_registry.json` 中每个 skill 的 `triggers` 列表逐一比对
- 若某个 skill 的任一 trigger 出现在对话中 → 该 skill 本次被触发过
- 特别规则：`工作流与prompt设计通用经验.skill.md` 的 triggers（设计工作流/修改工作流/创建prompt/修改prompt/设计技能/修改技能/技能设计/prompt设计/步骤设计/元经验）需要根据设计路由的主步骤来判断——如果本次流程涉及工作流/prompt/skill 设计变更，即使对话中未出现明确关键词也应视为已触发

**写入格式**：
- 对每个被触发的 skill，打开其对应的 `.memory.md`（位于 `_共享资源/技能/` 下同名文件）
- 追加笔记，格式与 prompt 的 `.memory.md` 一致：

```markdown
## YYYY-MM-DD 简要标题
- **场景**：本次设计任务简述
- **心得**：本次使用该技能时发现的边界案例、未覆盖情况或改进建议
- **边界类型**：🚀 / 📎 / 🔄 / ⚡（来自第 1½ 步判定）
```

- 如果某个 skill 虽然被触发但没有实质性经验（纯粹验证了已有知识正确），也记录一条简短的"+1 验证"以积累使用频率

### 3. 更新知识库
- 如果发现 🚀 新边界或 ⚡ 冲突，优先考虑更新知识库文档
- 如果是跨项目通用的新原则：
  - 更新 `_共享资源/跨项目知识/` 中对应文档
  - 或创建新文档
- 如果是项目特有的经验：
  - 更新 `项目经验/` 中对应文档
- 📎 增量类经验可仅在 .memory.md 中记录，不需更新知识库
- 🔄 重复类不在知识库中新建任何内容

### 4. 输出 Prompt 改进建议
- 如果发现 prompt 的指引不够覆盖本次情况，输出修改建议
- 包括：哪些内容需要补充、修正或移除
- 格式：
  ```
  ## Prompt 改进建议
  - 卡牌设计.prompt.md：新增"极端场景检查"步骤
  - 数值平衡.prompt.md：示例中的税率数据应更新
  ```

## 产出

- 边界检测判定结果（🚀 / 📎 / 🔄 / ⚡）
- 已更新的 `.memory.md` 文件列表（prompt + skill）
  - 每个文件含更新摘要（写入了什么、对应标记）
- 被触发但无新增经验的 skill 列表（+1 验证记录）
- 知识库更新建议（仅 🚀 和 ⚡ 类）
- Prompt 改进建议（如需）
