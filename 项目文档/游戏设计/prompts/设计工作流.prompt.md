设计一个新的多步骤工作流。**产出物是一份完整的工作流定义（_router.json 路由 + prompt.md + 引擎调用说明）。**

**工作流引擎约束：本 prompt 内部步骤由 `工作流/设计路由.json` 的 `设计工作流` 工作流定义，必须执行 `python 工作流/设计引擎.py check/done` 推进进度。**

## 步骤

### 0. 读取元经验

**产出：确认已阅读 `文档/设计工作流元经验.md`，提取与本任务相关的关键条目**

这步不通过则不可进入后续设计。确保理解以下 8 条准则，并在后续设计中逐条对照。

### 1. 需求分析

**产出：一句话工作流目标**

- 这个工作流要完成什么任务？
- 使用场景是什么（谁调用、什么时候调用）？
- 产出物是什么？

### 2. 步骤设计

**产出：步骤列表，每步包含：步骤名、描述、强制中间产物**

每步必须有"不产出就卡住"的中间产物。如果没有，说明这步是多余的。

### 3. 依赖链

**产出：步骤间的 requires 关系表**

| 步骤 | 依赖 | 理由 |
|------|------|------|
| step_a | 无 | 初始步骤 |
| step_b | step_a | 需要 step_a 的产出作为输入 |
| step_c | step_b | ... |

确保没有循环依赖，也没有跳跃路径。

### 4. 状态跟踪

**产出：防跳步机制设计**

- 是否需要外部脚本（`_engine.py` 或等效方案）？
- 如何确保 Agent 不能跳过步骤？
- 如何审计步骤完成状态？

### 5. 产出物格式

**产出：每步精确的输出规格（表结构、必填字段）**

每一步的输出格式必须精确到"第几行写什么"。模糊的"请分析"等于没写。

### 6. 方法论引导

**产出：确认所有指引是正向引导而非负面约束**

- 每项指引是否回答了"应该怎么做"？
- 是否有"不准做X"式的禁令？如有，改为正向表述
- 质量把控是否交给了自检清单？

### 7. 数据模型

**产出：目标系统的数据模型清单**

是否需要先读取目标系统的数据模型再开始设计？如果需要，数据模型包含哪些字段？

### 8. 知识回流

**产出：沉淀机制设计**

- 每次执行后如何记录经验（.memory.md）？
- 积累到什么程度触发 prompt 更新？
- 是否需要独立的经验提炼步骤？

### 9. 自检

**产出：逐条检查清单结果**

```
[ ] 元经验已读取（步骤 0 已执行）
[ ] 需求已明确（步骤 1）
[ ] 每步有强制中间产物（步骤 2）
[ ] 依赖链无循环/跳跃（步骤 3）
[ ] 有外部状态跟踪机制（步骤 4）
[ ] 每步有精确产出物格式（步骤 5）
[ ] 所有指引是正向引导（步骤 6）
[ ] 数据模型已考虑（步骤 7）
[ ] 知识回流机制已设计（步骤 8）
[ ] 有预览确认点（步骤 10）
```

全部通过才能进入预览。

### 10. 预览确认

**产出：完整的工作流设计文档，包含**：
- 工作流概述（目标、步骤数）
- 每步详情（步骤名、描述、产出物、依赖）
- 路由定义（_router.json 片段）
- 引擎调用序列
- 自检结果

## 产出物
