## 2026-06-09 实战经验：共享中间层复用检查

- **场景**：设计斧子 05 棘刺鬣蜥时，想要"集火同一部位越打越痛"的效果，发现没有现成的"部位级伤害加深"中间层，于是创造了"裂伤"概念
- **根因**：概念发散（③）到核心机制设计（④）之间，没有强制检查"能否复用现有共享中间层"的环节。Agent 和设计者都会下意识造新概念
- **修复**：在步骤 3 新增「3.3 共享中间层复用检查」，要求系统性检查连击数/护盾/蓄力/箭矢/标记/破甲/部位状态 7 种中间层
- **教训**：创造新概念不一定是错的，但必须经过系统性的复用检查——不是"感觉没有"而是"逐一排查后确认没有"

## 2026-06-09 实战经验：被动效果粘合剂原则

- **场景**：连续 3 次武器设计（07 噬花 → 06 锤尾蝎 → 05 棘刺鬣蜥）都是先定自动攻击效果再写卡牌，自动攻击占掉了所有机制带宽
- **根因**：3.2 卡牌角色分工的表述太弱——只定义了功能标签，没有强制回答每张卡的独立决策空间是什么。设计者（人或 Agent）写完分工表后脑子里仍然装着"被动效果"
- **修复**：3.2 的表新增"独立决策空间"列（玩家用这张卡时在想什么），3.4 开头增加粘合剂约束（被动效果不是体验中心）
- **教训**：设计顺序应该是"先定卡牌决策空间，再定被动效果如何配合"——被动效果是卡牌之间的粘合剂，不是体验的起点
