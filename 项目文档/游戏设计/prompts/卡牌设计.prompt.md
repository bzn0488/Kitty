根据设计目标设计游戏卡牌。**只产出设计方案文档，不产生代码。**

**必须先读取目标游戏的卡牌数据模型，确定一张"完整的卡牌设计"需要包含哪些字段**（不同游戏的卡牌有不同的属性结构，例如杀戮尖塔需要费用/稀有度，家族纪元需要获得条件/单人局行为等）。

**⚡ 引擎约束（不可跳过）：** 每步开始前 `python 工作流/设计引擎.py check <step_id>`，完成后 `python 工作流/设计引擎.py done <step_id> <摘要>`。步骤 ID 见 `工作流/设计路由.json` 的 `卡牌设计` 工作流。自检时调用 `python 工作流/设计引擎.py audit` 并将结果嵌入预览。

## 创造力标记

设计可以基于已有机制，也可以创造新机制。**区分两者的来源类型即可：**

| 标记 | 含义 |
|------|------|
| ✅ 基于 | 效果引用项目文档中明确存在的机制/数值/事件 |
| 💡 推断 | 效果基于已知机制但做了合理延伸（如阈值假设、组合效果） |
| ✨ 新机制 | 项目文档中不存在，Agent 自主创造的新概念/效果 |

鼓励创造新机制，但必须标注。用户看到预览时可以判断是否接受。

## 核心原则（正向引导）

这些原则指导你做出好的设计，而非限制你的创造力。

1. **差异化定位** — 每张卡牌应有独特的机制和定位，避免功能重叠
2. **协作卡需定义单人局行为** — 设计时明确 target==caster 或单人时的效果
3. **卡牌效果应有可理解的成本-收益关系** — 玩家能直观判断使用时机
4. **规模效应设计** — 涉及"按数量缩放"的效果应非线性（防止线性最优解）
5. **获得条件影响强度** — 卡牌的获得门槛与其强度正相关，越难获得的效果应越强

## 步骤

### 0. 读取目标游戏卡牌数据模型

在开始设计前，先确定本游戏中的"卡牌"包含哪些字段。

方法：
- 搜索项目中已有的卡牌定义（如 `CAREER_CARD_POOLS`、`Card` 类定义）
- 分析游戏机制对卡牌的特殊需求（是否需要获得条件？是否需要联动事件？）
- 输出字段清单

示例输出：
```
## 卡牌数据模型（家族纪元）
| 字段 | 必须 | 说明 |
|------|------|------|
| 名称 | ✅ | 简短有辨识度 |
| 效果描述 | ✅ | 精确无歧义，包含数值 |
| 类型 | ✅ | 成长/协作/全局/延迟/自毁/转化/环境/经济 |
| 获得条件 | ✅ | 何时/如何进入玩家手牌 |
| 单人局行为 | ✅ | target 不可用时的降级方案 |
| 设计意图 | ✅ | 为何设计这张卡 |
| 联动效应 | □ | 打出后是否触发事件/为事件加选项 |
```

### 1. 情景分析

**产出：玩家状态描述 + 玩家需求推断（一段话，必须展示）**

推断玩家此时所处的局面：
- 游戏阶段（初期/中期/后期？）
- 资源状态（金库、人口、手牌数量和质量）
- 核心压力（生存？发展？应对危机？）
- 玩家此时的心理预期（希望稳步发展？寻求突破？被迫应对？）

```
示例：
"玩家处于游戏开局阶段，家族仅1-2人，手牌3张基础成长卡，金库40金。
此时核心压力是人口增长和经济基础——单人产出有限，需要尽快成家立业。
玩家的心理预期是'安全起步'，不希望开局就遭遇毁灭性打击。"
```

### 2. 设计目的确立

**产出：一句话设计目标（必须展示）**

明确回答：**这组卡牌要带给玩家什么样的体验？**

```
示例：
"让初代玩家体验'白手起家'的规划感——有限的资源下做出关键决策。"
```

### 3. 概念发散

**产出：3-5 个候选概念，每个附一句话核心体验 + 参考来源（必须展示）**

围绕设计目的发散可能的主题/机制，**不要过早陷入具体数值**。

```
引用来源：跨项目知识/卡牌设计通用规则.md / 选择与决策设计通用规则.md

概念A"启动资金"：核心体验是"第一桶金时的选择兴奋"
概念B"夫妻档"：核心体验是"成家立业的小确幸"
概念C"代际传承"：核心体验是"牺牲与希望的沉重抉择"
概念D"抱团取暖"：核心体验是"人少时的团结力量"
概念E"安家立业"：核心体验是"有了家才有干劲"
```

### 4. 行为预检

按 `_共享资源/agents/游戏设计师.agent.md` 中的行为预检框架执行六维度分析。以下是卡牌设计的特有提问角度：

**选择情境补充**
- 这张卡是在什么场景下进入玩家视野的？（选牌/自动获得/商店/产牌/事件奖励）
- 如果是在选牌情境（如3选1），替代选项的竞争力如何？
- 如果是在自动获得情境，玩家拿到这张卡的第一反应是什么？

**内在驱动力补充**
- 新手玩家能看懂这张牌的用法吗？是否需要试错才能理解？
- 这张卡第一次使用时有"惊喜感"吗？
- 多次使用后是否仍然有趣，还是沦为"机械执行"？

**外在驱动力补充**
- 打出后的效果是否足够明显（数值变化、视觉效果）？
- 如果效果是隐性的（如概率提升、build组件），玩家能否感知到？

**排斥力补充**
- 这张卡是否有"卡手"风险（时机不对时打出就是浪费）？
- 上手后是否有明确的使用指引，还是会让玩家困惑"什么时候用"？

**卡牌特有风险模式**

> ⚠️ 以下风险上下文相关：前两条（未来build卡/诅咒卡）取决于游戏设计哲学，非绝对约束。

- 🔴 **未来build卡当前无作用** ⚠️ 上下文相关 — 适用于牌库随进程累积的 roguelike/DBG。在副卡组可自由替换的游戏中不构成问题，甚至希望玩家思考如何补强弱项
- 🔴 **诅咒卡/纯负面** ⚠️ 上下文相关 — 如果设计单位是单张卡（如家族纪元），应避免。如果设计单位是武器/卡组（如荒野与曙光），劣势面可增加构筑深度，允许存在
- 🔴 **数值碾压**：与同类卡相比全面占优 → 要么提升其他卡，要么削弱
- 🔴 **认知门槛过高**：需要大量游戏知识才能判断价值 → 考虑简化或加教程引导

**产出：六维度驱动力分析 + 设计目的校对结论。** 通过预检后进入具体设计。

### 5. 具体设计

**产出：按步骤 0 确定的卡牌数据模型，输出完整设计表（必须展示）**

将概念落地为具体的卡牌：

- **获得条件**：何时/如何加入玩家手牌（与其他卡牌的区别点）
- **效果与数值**：参考已有卡牌的数值量级
- **卡牌类型**：选择最匹配的类型
- **单人局/边界行为**：明确定义
- **联动效应**（如有）：打出后是否触发事件、是否为事件加选项

```
引用来源：家族纪元/原型/engine/core.py / CAREER_CARD_POOLS / 数值量级参考

| 卡牌 | 类型 | 效果 | 获得条件 | 来源 | 单人局 | 设计意图 |
|------|------|------|----------|------|--------|----------|
| 筚路蓝缕 | 环境 | 人数≤3时全族+0.5 | 开局自带 | ✅ 基于规则 | ✅ | 抱团取暖 |
| 养生之道 | 环境 | 全族体质+1，5年 | 郎产出中 | ✅ 基于职业设定 | ✅ | 健康投资 |
| ... | ... | ... | ... | ... | ... | ... |
| 铁官令 | 协作 | 持有政令卡→全体+0.3 | 铁匠产出 | ✨ 新协同机制 | ✅ | 跨职业协同 |
```

**来源列取值：** `✅ 基于` / `💡 推断` / `✨ 新机制`。每张卡的整体来源取所有效果中最低级别。

### 6. 内部自检（过程不展示，审计结果嵌入预览）

自检过程中发现问题时自动修正并重复自检，**此迭代过程不展示给用户**。自检全部通过后调用 `python 工作流/设计引擎.py audit`，将审计结果嵌入步骤 7 的预览。

完成草稿后，逐条检查以下清单。**全部通过后才能输出预览。**

```
[ ] 卡牌数据模型已读取（步骤 0 已执行）
[ ] 情景分析已完成（有明确的玩家状态描述）
[ ] 设计目的已确立（有一句话设计目标）
[ ] 概念发散已完成（至少 3 个候选概念）
[ ] 行为预检已执行（六维度分析完整）
[ ] 每张卡牌有明确的设计意图（不是"凑数"）
[ ] 卡牌间有差异化定位（无功能重叠）
[ ] 获得条件已定义（与其他卡牌可区分）
[ ] 协作卡定义了单人局/无目标时的行为
[ ] 极端情况已考虑（零资源/满员/单人）
[ ] 效果描述无歧义（实现者能准确理解）
[ ] 每张卡牌的来源类型已标注（✅基于 / 💡推断 / ✨新机制）
[ ] 每项效果有来源追溯（若整卡标记为 ✅ 基于，确认所有效果都有明确的项目文档依据）
[ ] 强化程式多样性——是否混合了多种奖励节奏（固定比率兜底 + 可变比率惊喜），而非依赖单一程式
```

如果发现违规 → **自动修正设计** → 重复自检，直到全部通过。

### 7. 输出设计方案预览（展示给用户）

通过自检后，按以下格式输出完整预览。**每步的推论和引用来源必须保留。**

```
## 卡牌设计方案

### 使用的引用
- 卡牌数据模型：家族纪元 CAREER_CARD_POOLS / Card 类
- 数值参考：家族纪元 农夫/工匠 职业卡牌数值量级
- 设计原则：跨项目知识/卡牌设计通用规则.md

### ① 情景分析
[玩家状态描述]

### ② 设计目的
[一句话设计目标]

### ③ 概念发散
[候选概念列表]

### ④ 行为预检
[六维度分析结论]

### ⑤ 具体设计

| 卡牌 | 类型 | 效果 | 获得条件 | 来源 | 单人局 | 设计意图 |
|------|------|------|----------|------|--------|----------|
| ... | ... | ... | ... | ... | ... | ... |

#### 边界说明
- [卡牌名] 单人局：[行为描述]
- [卡牌名] 极端情况：[行为描述]

#### 联动效应（如有）
- [卡牌名] 打出后：[触发的事件/新选项]
```

预览后等待用户确认。用户可要求修改，修改后重复 5→6。

### 8. 输出最终设计规格书（确认后）

用户确认后，输出一份完整的、可供实现者直接使用的设计规格书。
