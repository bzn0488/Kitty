# HUD 信息架构与视觉层次

> HUD（Heads-Up Display）是玩家在游戏过程中始终可见的信息层。好的 HUD 设计在"提供足够信息"和"不干扰沉浸感"之间找到平衡。

---

## 一、信息优先级分层

### 1.1 三层优先级模型

游戏中的信息可按**对当前决策的重要性**划分为三层：

```
┌─────────────────────────────────────────┐
│          核心层（Core）                    │
│  必须始终可见，影响即时生存与操作           │
│  ├─ 血量/护盾/能量                       │
│  ├─ 当前武器/能力                        │
│  ├─ 弹药/资源计数                        │
│  └─ 关键状态效果（中毒/燃烧/昏迷）         │
├─────────────────────────────────────────┤
│          次要层（Secondary）               │
│  需要时可快速获取，不影响即时反应           │
│  ├─ 小地图/方位                          │
│  ├─ 任务目标/导航点                       │
│  ├─ 队友/敌人状态概览                     │
│  └─ 连杀/积分/计时                        │
├─────────────────────────────────────────┤
│          背景层（Background）              │
│  仅在玩家主动查看时呈现                    │
│  ├─ 成就/进度                            │
│  ├─ 详细属性面板                          │
│  ├─ 聊天/社交信息                         │
│  └─ 系统设置/帮助                         │
└─────────────────────────────────────────┘
```

**关键原则**：核心层占用不超过 15% 的屏幕面积，且应集中在**视野中心区域**。

### 1.2 注意力竞争模型

玩家在游戏中的注意力是有限资源。HUD 的设计应避免**多个重要信息同时竞争注意力**：

- **协同呈现**：将相关信息整合（如血量+能量条在同一个视觉区域）
- **时序错开**：重要通知避免同时弹出
- **状态压缩**：常态下简略显示，异常/临界时显著显示（如血量低于 20% 时闪动）

> **与知识库关联**：心流通道理论指出——当玩家同时需要处理的信息超出其认知带宽时，心流会被打断。——`理论知识/36-心流通道.md`、`游戏设计理论/04-游戏节奏与心流设计.md`

## 二、屏幕空间区域化

### 2.1 经典屏幕区域分配

```
┌──────────────────────────────────────┐
│   左上          顶部          右上     │
│  血量/能量    资源/金币    小地图/数据 │
│                                      │
│              中心区域                  │
│  左         游戏视口          右      │
│  侧         （操作空间）     侧       │
│  边                             边   │
│                                      │
│   左下          底部          右下     │
│  聊天/日志     技能栏      状态/Buff  │
└──────────────────────────────────────┘
```

### 2.2 区域功能分工

| 区域         | 适合放置              | 原因                              |
| :----------- | :-------------------- | :-------------------------------- |
| **左上**     | 玩家状态（血量/能量） | 阅读起点（LTR 文化中左上优先）    |
| **右上**     | 地图/数据/分数        | 非即时但重要信息                  |
| **底部中央** | 技能/物品栏           | 操作区，Fitts 定律下离摇杆/鼠标近 |
| **左下**     | 聊天/日志             | 次要文字信息                      |
| **右下**     | Buff/Debuff           | 辅助监控，非核心操作区            |
| **屏幕中央** | 关键通知/QTE          | 仅临时出现，结束后消失            |

### 2.3 屏幕覆盖率控制

不同类型游戏对 HUD 面积占用的容忍度不同：

| 游戏类型   | HUD 理想占比 | 策略                               |
| :--------- | :----------- | :--------------------------------- |
| 动作/射击  | < 10%        | 极简 HUD，信息靠视觉提示集成到场景 |
| RPG/策略   | 10-20%       | 丰富但可折叠，非战斗时展开         |
| MMO/竞技   | 15-30%       | 信息密度高，但可自定义布局         |
| 沉浸式叙事 | < 5%         | 尽可能无 HUD，靠场景/角色反馈      |

> **与知识库关联**：「减法设计」理念在 HUD 设计中的体现——减少非必要信息以增强沉浸感。——`理论知识/12-减法设计.md`

## 三、动态 HUD 策略

### 3.1 情境感知显示

HUD 元素应根据**当前游戏情境**自动调整：

- **战斗中 vs 非战斗**：战斗中显示战斗相关状态（技能冷却、敌人血量），非战斗时简化
- **探索中 vs 对话中**：探索时显示导航/地图，对话时淡化/HUD
- **正常 vs 危险**：血量正常时细条显示，低血量时放大+闪动

### 3.2 渐进式披露

所有信息不应在一开始全部开放。应随着玩家学习进度逐步揭示：

- **教学期**：仅有核心信息，额外提示辅助
- **成长期**：逐渐引入次要信息，玩家熟悉后再展现
- **精通期**：全量信息可用，允许玩家自定义 HUD

> **与知识库关联**：「学习与遗忘曲线」理论支持了渐进式披露的合理性——信息过载发生在玩家尚未建立心智模型时。——`理论知识/39-学习与遗忘曲线.md`

## 四、视觉层次设计

### 4.1 视觉重量金字塔

```
            ⬆ 重要度
        🔴 动态变化（闪烁/动画）
      🟠 颜色高亮（红/橙警告）
    🟡 尺寸/形状突出
  🟢 位置/区域固定
🔵 静态背景信息
```

### 4.2 视觉层次手法

| 手法           | 说明                       | 示例                                   |
| :------------- | :------------------------- | :------------------------------------- |
| **大小对比**   | 重要元素更大               | 主目标图标 > 次要目标图标              |
| **色彩对比**   | 重要元素饱和度/亮度更高    | 任务目标黄色标记 vs 普通兴趣点白色标记 |
| **动效导引**   | 新出现的元素用动效吸引注意 | 新任务弹入、Buff 图标发光出现          |
| **景深分离**   | 利用模糊/阴影创造前后层次  | HUD 元素带轻微阴影浮于游戏画面之上     |
| **透明度层级** | 低优先级元素半透明         | 非战斗时技能栏变半透明                 |

### 4.3 信息密度节奏

HUD 的信息密度应与游戏节奏匹配：

| 游戏节奏  | HUD 密度 | 策略                             |
| :-------- | :------- | :------------------------------- |
| 紧张战斗  | 低 → 中  | 仅保留核心战斗信息，次要信息隐藏 |
| 探索/解谜 | 低       | 仅导航提示，无战斗 UI            |
| 营地/城镇 | 高       | 展开全部面板（背包/商店/技能树） |
| 过场/对话 | 极低     | 仅任务提示，其余淡化             |

> **与知识库关联**：游戏节奏设计中的"紧张-松弛"交替模式应反映在 HUD 的信息密度变化上。——`理论知识/43-游戏节奏设计.md`

## 五、HUD 设计检查清单

- [ ] 核心信息（血量/资源）是否在固定位置始终可见？
- [ ] 不同优先级的信息是否在视觉上有明确区分？
- [ ] 战斗中和非战斗中 HUD 是否自动适配？
- [ ] 屏幕覆盖率是否在对应类型的合理范围内？
- [ ] 新元素出现时是否有动效/高亮引导注意力？
- [ ] 玩家是否可以自定义 HUD 布局？
- [ ] 信息密度是否匹配当前游戏节奏？
- [ ] 是否避免了多个重要通知同时弹出？
