# 游戏UI设计原则

> 游戏 UI（User Interface）设计关系到玩家如何理解游戏状态、做出决策、执行操作。好的 UI 降低认知摩擦，让玩家专注于"玩游戏"本身，而非"学习怎么操作这个界面"。

---

## 一、核心设计原则

### 1.1 一致性（Consistency）

同一款游戏内部，语义相同的元素在所有场景中应有**相同的视觉样式、交互行为和位置**。

| 维度           | 说明                       | 反例                                       |
| :------------- | :------------------------- | :----------------------------------------- |
| **视觉一致性** | 同类按钮同色同形同大小     | 背包里的"确认"是蓝色，商店里的"确认"是绿色 |
| **交互一致性** | 同种操作用同一方式触发     | 物品栏左键使用，装备栏也应是左键使用       |
| **布局一致性** | 同一类型信息出现在固定区域 | 血量条有时在左上、有时在中间               |
| **术语一致性** | 同一概念用同一名称         | 有的地方叫"法力"，有的叫"能量"             |

> **与知识库关联**：「格式塔法则中的相似律」解释了为什么一致性对认知效率至关重要——视觉相似的元素被视为同一类，降低理解成本。——`游戏设计理论/02-玩家心理学.md §格式塔法则`

### 1.2 可见性（Visibility）

玩家所需的信息和可执行的操作应在需要时**清晰可见**，而非隐藏于深层菜单或模糊的提示中。

- **核心信息**：血量、资源、当前位置、任务目标 → 始终可见或一键可查
- **情境信息**：仅在相关时刻出现（如交互提示、伤害数字）
- **隐藏信息**：需要玩家探索或达到条件才解锁（如成就详情）

**可见性的黄金法则**：在玩家思考"接下来该做什么"之前，界面已经给了答案。

> **与知识库关联**：Fogg 行为模型中的"能力"维度指出——如果玩家看不到可执行的选项，再强的动机也无法转化为行动。——`游戏设计理论/02-玩家心理学.md §Fogg B=MAP`

### 1.3 反馈（Feedback）

**系统对每一次玩家操作都应给予即时、明确、可辨别的反馈。**

| 反馈类型 | 延迟    | 例子                               |
| :------- | :------ | :--------------------------------- |
| 即时反馈 | < 100ms | 按钮按下立即变色/缩放              |
| 操作确认 | < 200ms | 点击后播放选中声音                 |
| 结果反馈 | 视情况  | 造成伤害飘字、拾取物品提示         |
| 状态反馈 | 持续    | 读条、冷却倒计时、buff/debuff 图标 |

> **与知识库关联**：「信号与强化体系」中强化程式的前提是信号必须被玩家感知到——反馈就是信号。——`理论知识/37-信号与强化体系.md`
> 另见「Game Feel」中"操作→反馈"链路紧密度的讨论。——`游戏设计理论/03-游戏机制设计技法.md §1.10`

### 1.4 容错性（Forgiveness）

玩家犯错时，界面应**减少损失的严重性**并提供**恢复路径**。

| 原则 | 实现方式                                               |
| :--- | :----------------------------------------------------- |
| 预防 | 危险操作前弹出确认（如"确定要分解这件装备？"）         |
| 撤销 | 提供"撤回/重做"功能                                    |
| 宽容 | 误触不造成严重后果（如点击菜单外区域关闭而非执行操作） |
| 恢复 | 操作后一段时间内可以撤销                               |

### 1.5 效率（Efficiency）

**高频操作的路径应最短，低频操作的路径可以较长。**

抽象而言：设 $P$ 为操作频率，$D$ 为操作路径长度（点击次数/移动距离），应满足：

$$P \propto \frac{1}{D}$$

即使用最频繁的操作，其路径应最短。

## 二、交互定律在游戏UI中的应用

### 2.1 Fitts 定律

$$T = a + b \cdot \log_2 \left(1 + \frac{D}{W}\right)$$

其中 $T$ 为到达目标的时间，$D$ 为起始点到目标的距离，$W$ 为目标宽度。

**游戏UI中的推论**：
- **常用按钮要大**：攻击/技能按钮、拾取按钮应显著大于次要按钮
- **边缘与角落是高效区域**：屏幕四角（位置无限大）适合放置需要快速点击的 HUD 元素
- **相邻操作保持距离**：避免将"确认"和"取消"、"购买"和"出售"放得太近

### 2.2 Hick 定律

$$RT = a + b \cdot \log_2(n)$$

其中 $RT$ 为反应时间，$n$ 为选择数量。

**游戏UI中的推论**：
- **减少单屏选择数**：背包/商店界面一次展示太多选项会延长决策时间
- **分组降低认知**：将物品按类别分组比平铺所有选项更快定位（反应时间从 $\log_2(n)$ 降为 $\log_2(k) + \log_2(m)$）
- **推荐标注**：标记"推荐"或"最近使用"可缩小有效选择集

### 2.3 格式塔法则在游戏UI中的应用

> 详见 `游戏设计理论/02-玩家心理学.md §格式塔法则`，以下为游戏UI专用展开：

| 法则          | UI 应用                                                |
| :------------ | :----------------------------------------------------- |
| **接近律**    | 同一功能的控件在空间上靠近（如血条+能量条放在一起）    |
| **相似律**    | 同类型物品同色同形（药水都是圆形、装备都是方形）       |
| **闭合律**    | 通过部分边框暗示完整区域（如半透明背景暗示可交互区域） |
| **连续律**    | 技能树、任务链用箭头/连线引导视觉流向                  |
| **图形-背景** | 对话框中背景变暗突出前景内容                           |

## 三、专项设计检查清单

### 3.1 界面布局检查

- [ ] 核心信息是否在屏幕中心/次中心视野区域？
- [ ] 高频操作是否在快捷位置（Fitts 定律）？
- [ ] 危险操作和普通操作是否有足够间距？
- [ ] 同类信息的位置、颜色、交互方式是否一致？
- [ ] 低优先级信息是否会干扰高优先级信息？

### 3.2 交互流程检查

- [ ] 每个操作是否有明确的视觉/音效反馈？
- [ ] 误触是否可撤销或影响可控？
- [ ] 单屏选择数是否在 7±2 以内（Hick 定律）？
- [ ] 深层操作（3 次点击以上）是否有进度提示？

### 3.3 文字与图标检查

- [ ] 术语是否全游戏统一？
- [ ] 图标是否能通过形状（而非颜色）区分？（色盲玩家可识别）
- [ ] 文字大小是否符合阅读距离和屏幕尺寸？
