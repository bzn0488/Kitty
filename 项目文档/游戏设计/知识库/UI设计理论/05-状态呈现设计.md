# 状态呈现设计

> 玩家状态可视化（血量/能量/资源）和敌我态势感知是游戏 UI 的核心功能。好的状态呈现让玩家**零思考**地理解当前局势。

---

## 一、玩家状态可视化

### 1.1 常见状态元素

| 状态             | 可视化方式             | 注意事项                         |
| :--------------- | :--------------------- | :------------------------------- |
| **生命值（HP）** | 进度条、数值、分段方块 | 临界值（<20%）应有特殊反馈       |
| **能量/法力**    | 进度条、圆点、分段条   | 满时应有提示（满法力提示放技能） |
| **体力/耐力**    | 圆环、进度条           | 不应干扰视线，常置于侧边         |
| **经验值**       | 进度条、数字           | 可显示当前/下一级所需值          |
| **Buff/Debuff**  | 图标+倒计时            | 图标应通过形状可区分（色盲友好） |
| **资源**         | 数字+图标              | 应显示当前/最大值                |

### 1.2 生命值条设计

生命条是游戏中最常用、最重要的 UI 元素。

**设计考量**：

| 维度     | 选项                | 适用场景               |
| :------- | :------------------ | :--------------------- |
| **形状** | 长条形              | 传统 RPG、动作游戏     |
|          | 圆形/环形           | 第三人称射击、竞技游戏 |
|          | 分段式（心形/菱形） | 休闲游戏、简化 UI      |
| **位置** | 左上角              | 最传统，阅读起点       |
|          | 底部中央            | 第三人称，靠近角色     |
|          | 角色周围（3D HUD）  | 沉浸式、无界面风格     |
| **分段** | 连续条              | 精确数值重要时         |
|          | 分段（10格/条）     | 减少认知负荷           |
|          | 仅临界警示          | 极简 UI                |

### 1.3 状态异常的呈现

当玩家状态异常时，UI 应主动提示：

| 异常类型                  | 呈现方式                          |
| :------------------------ | :-------------------------------- |
| **低血量**                | 屏幕边缘泛红 + 心跳声 + 血条闪烁  |
| **中毒/燃烧**             | 角色变绿/红 + 持续伤害飘字 + 图标 |
| **控制效果（眩晕/冰冻）** | 屏幕边缘色效 + 图标 + 剩余时间    |
| **资源不足**              | 技能按钮灰显 + 提示文字           |

> **与知识库关联**：「信号与强化体系」中，状态变化是信号的重要来源——清晰的状态呈现让玩家能做出正确的下一步决策。——`理论知识/37-信号与强化体系.md`

## 二、敌我态势感知

### 2.1 敌人状态呈现

| 呈现方式          | 适用场景 | 优点         | 缺点           |
| :---------------- | :------- | :----------- | :------------- |
| **敌人头顶血条**  | 所有类型 | 直观定位     | 画面杂乱       |
| **锁定目标UI**    | 动作/RPG | 聚焦单一目标 | 丢失周围视野   |
| **伤害数字**      | 所有类型 | 即时反馈     | 不反映剩余血量 |
| **敌人标记/图标** | 策略/RPG | 区分敌人类别 | 信息量有限     |

### 2.2 Boss 战状态呈现

Boss 战对状态呈现有特殊要求：

| 需求          | 呈现方式                    | 对应项目经验                   |
| :------------ | :-------------------------- | :----------------------------- |
| **Boss 血量** | 屏幕顶部大型血条 + 阶段标记 | `Boss行为设计.md` — 阶段转换   |
| **部位状态**  | 每个部位独立血条 + 破坏标记 | `部位破坏设计.md` — 部位可见性 |
| **技能预告**  | Boss技能前摇 + 区域预警     | `Boss行为设计.md` — 行为前摇   |
| **狂暴计时**  | 进度条/倒计时               | 压力机制可视化                 |

### 2.3 队友/团队状态

| 信息     | 呈现方式                    | 时机       |
| :------- | :-------------------------- | :--------- |
| 队友血量 | 侧边栏列表 + 屏幕边缘标记   | 持续可见   |
| 队友状态 | 小图标（存活/倒地/远离）    | 状态变化时 |
| 队友位置 | 小地图标记 + 屏幕外方向指示 | 需要时     |
| 团队资源 | 团队进度条/计数             | 持续可见   |

> **与知识库关联**：「荒野与曙光」项目中的 Boss 行为设计和部位破坏设计直接依赖清晰的状态呈现。——`项目经验/01-荒野与曙光-Boss行为设计.md`、`项目经验/02-荒野与曙光-部位破坏设计.md`

## 三、伤害数字与飘字

### 3.1 伤害数字设计

| 设计点   | 建议                      | 原因             |
| :------- | :------------------------ | :--------------- |
| **颜色** | 白=普通、黄=暴击、红=破防 | 颜色区分伤害类型 |
| **动画** | 向上飘动 + 淡出           | 自然的视觉方向   |
| **字体** | 粗体、大号、带描边        | 在任何背景下可读 |
| **分组** | 多次伤害合并显示          | 避免数字刷屏     |

### 3.2 信息飘字类型

| 类型 | 颜色      | 持续时长 | 示例         |
| :--- | :-------- | :------- | :----------- |
| 伤害 | 红色/白色 | 1.5s     | "-50"        |
| 暴击 | 黄色/橙色 | 2s       | "暴击! -120" |
| 治疗 | 绿色      | 1.5s     | "+30"        |
| 经验 | 金色      | 2s       | "+150 EXP"   |
| 拾取 | 蓝色/白色 | 2s       | "+5 金币"    |
| 状态 | 紫色/粉色 | 2.5s     | "中毒!"      |

## 四、特殊场景状态呈现

### 4.1 隐身/潜行状态

- 屏幕边缘逐渐变暗/模糊
- 自身半透明效果
- 可见度指示器（眼睛图标：睁开/半闭/闭合）
- 声音反馈（脚步声大小指示）

### 4.2 战斗疲劳/耐力

- 耐力条变红 + 闪烁
- 操作限制提示（"体力不足"）
- 耐力恢复时变色变化（红→黄→绿）

### 4.3 部位破坏状态

- 部位血条独立显示
- 破坏后部位标记（断裂/燃烧/消失）
- 各部位破坏进度同时可见
- Boss 身上部位的物理状态变化作为视觉反馈

## 五、设计检查清单

- [ ] 玩家核心状态（血量/能量）是否在显眼固定位置？
- [ ] 生命值低于 20% 时是否有特殊视觉反馈？
- [ ] Buff/Debuff 是否可通过形状（非颜色）区分类型？
- [ ] 敌人血量是否可见且位置合理？
- [ ] Boss 阶段是否通过 UI 明确标记？
- [ ] 伤害数字是否可读且不遮挡关键视野？
- [ ] 潜行/隐匿状态是否有清晰的状态指示？
- [ ] 团队状态下是否有队友位置和状态概览？
