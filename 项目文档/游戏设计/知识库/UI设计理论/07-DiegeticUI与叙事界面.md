# Diegetic UI 与叙事界面设计

> 游戏界面是一种独特的媒介挑战——它既需要传递信息，又不应破坏玩家在游戏世界中的沉浸感。Diegetic UI 试图将界面元素"融入"游戏世界内部。

---

## 一、UI 的四分类框架

根据界面元素是否存在于游戏世界内部，可分为四类：

### 1.1 四象限分类

```
                   存在于游戏世界中
                   ───────┬───────
                      是  │  否
              ┌──────────┼──────────┐
          是  │ Diegetic │ Meta     │
              │  叙事界面  │  元信息  │
              │          │          │
非游戏世界    ─┼──────────┼──────────┤
  元素        │ Spatial  │ Non-     │
          否  │ 空间界面  │ diegetic │
              │          │ 非叙事   │
              └──────────┴──────────┘
```

| 类型                       | 定义                               | 示例                             |
| :------------------------- | :--------------------------------- | :------------------------------- |
| **Diegetic（叙事界面）**   | 界面存在于游戏世界中，角色也能看到 | 角色手腕上的全息投影、车内仪表盘 |
| **Spatial（空间界面）**    | 界面浮在游戏空间中，但角色看不到   | 敌人头顶的血条、3D 导航箭头      |
| **Meta（元信息）**         | 角色看不到，但作为UI信息展示       | 屏幕边缘的伤害数字、连杀计数     |
| **Non-diegetic（非叙事）** | 传统 2D 覆盖层 UI                  | 左上角血条、底部技能栏           |

### 1.2 各类型优缺点

| 类型             | 优点               | 缺点                       |
| :--------------- | :----------------- | :------------------------- |
| **Diegetic**     | 沉浸感最强         | 信息清晰度受限、实现成本高 |
| **Spatial**      | 位置直观、融入场景 | 可读性受场景影响           |
| **Meta**         | 信息明确           | 破坏沉浸感                 |
| **Non-diegetic** | 最清晰、实现简单   | 最破坏沉浸感               |

> **与知识库关联**：「玩家投影与游戏肌理」中讨论了玩家在游戏世界中的存在感——Diegetic UI 的本质是强化玩家的"我在这个世界里"的感受。——`理论知识/31-玩家投影与游戏肌理.md`

## 二、Diegetic UI 设计

### 2.1 常见实现方式

| 方式            | 说明                     | 示例游戏                                   |
| :-------------- | :----------------------- | :----------------------------------------- |
| **角色装备 UI** | 角色身上的设备显示信息   | 《死亡空间》主角身后的生命条、《地铁》腕表 |
| **场景嵌入 UI** | 信息出现在场景物体上     | 《战争机器》全息战术地图                   |
| **道具信息 UI** | 使用游戏内道具查看信息   | 《逃出生天》角色拿出照片查看               |
| **角色动作 UI** | 通过角色动作触发信息显示 | 《荒野大镖客》角色掏怀表看时间             |

### 2.2 Diegetic UI 的设计原则

| 原则       | 说明                                               |
| :--------- | :------------------------------------------------- |
| **合理性** | 界面存在于游戏世界需有合理解释（科技/魔法/世界观） |
| **可读性** | 融入世界的同时保持信息清晰可辨                     |
| **操作性** | 玩家与界面交互的方式应符合世界规则                 |
| **一致性** | 所有 Diegetic UI 应遵循同一套"世界内"规则          |

### 2.3 案例分析：《死亡空间》的 Diegetic UI

《死亡空间》是 Diegetic UI 的经典案例：

- **生命值**：主角 Isaac 的脊椎上的蓝色灯光条，在背后可见
- **弹药**：武器上的全息数字显示剩余弹药
- **导航**：角色的导航线（通过主角的装备投射到地面）
- **菜单**：全息投影菜单浮在空中，角色会手持查看

> **启示**：所有 UI 元素都存在于游戏世界中——没有任何传统 HUD 覆盖层。这不仅增强了沉浸感，还增加了紧张感（查看血量需要转身看背后）。

## 三、Spatial UI 设计

### 3.1 空间界面的应用

| 应用         | 说明                | 示例                    |
| :----------- | :------------------ | :---------------------- |
| **敌人标记** | 敌人头顶/身旁的标记 | 血量条、等级、状态      |
| **导航点**   | 3D 空间中的目标指示 | 任务标记、路径指引      |
| **交互提示** | 可交互对象的提示    | "按 E 拾取"浮在物体上方 |
| **范围指示** | 技能范围/危险区域   | 地面红圈/AoE 指示       |

### 3.2 设计要点

| 要点         | 说明                                       |
| :----------- | :----------------------------------------- |
| **深度优先** | 距离远的元素应更小/更透明（透视一致）      |
| **遮挡处理** | 被物体遮挡时应显示轮廓或切换为屏幕边缘指示 |
| **密度控制** | 同一画面不超过 5-7 个空间标记              |
| **颜色区分** | 友方=绿色/蓝色、敌方=红色、中立=白色/灰色  |

## 四、混合策略

### 4.1 分层混合 UI

大多数现代游戏采用混合策略：核心信息用 Non-diegetic（清晰），情境信息用 Diegetic/Spatial（沉浸）：

```
沉浸感 ←─────────────── 清晰度
    极简Non-diegetic ← ─ → Diegetic/Spatial
    （血条/技能条）       （交互提示/导航点）
                                ↓
                        Meta 信息（伤害数字）
```

### 4.2 UI 切换策略

| 场景          | 推荐策略                    | 理由           |
| :------------ | :-------------------------- | :------------- |
| 过场动画/对话 | 隐藏 Non-diegetic UI        | 最大化沉浸感   |
| 紧张战斗      | 突出 Spatial + Non-diegetic | 信息优先于沉浸 |
| 探索/解谜     | 隐藏大部分 UI               | 沉浸体验       |
| 菜单/背包     | Non-diegetic 全展开         | 信息密度优先   |

> **与知识库关联**：「叙事中的父系结构设计」——叙事界面的选择本身就是一种设计立场：父系游戏倾向于清晰的 Non-diegetic UI，母系游戏倾向于融合的 Diegetic UI。——`理论知识/38-叙事中的父系结构设计.md`

## 五、设计检查清单

- [ ] 核心玩法态下是否选择了合适的 UI 类型（Non-diegetic vs Diegetic）？
- [ ] 沉浸时刻（过场/对话）是否隐藏了 Non-diegetic UI？
- [ ] Diegetic UI 是否有世界观内的合理解释？
- [ ] Spatial UI 是否遵循深度优先原则？
- [ ] 同一画面的空间标记是否过多（> 7 个）？
- [ ] 混合策略下各层 UI 之间是否有明显视觉区分？
- [ ] 信息清晰度与沉浸感之间是否取得了平衡？
