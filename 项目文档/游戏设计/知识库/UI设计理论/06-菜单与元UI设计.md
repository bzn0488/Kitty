# 菜单与元 UI 设计

> 背包、商店、技能树、设置面板等"元 UI"是玩家脱离核心玩法循环后频繁交互的界面。其组织效率直接影响玩家的整体体验。

---

## 一、元 UI 分类

### 1.1 分类框架

```
元 UI 分类
├── 管理类（背包、装备、仓库）
│   ├── 高频：物品整理、装备切换
│   └── 低频：丢弃、出售
├── 交易类（商店、铁匠、交易）
│   ├── 买卖操作
│   └── 价格/价值比较
├── 成长类（技能树、升级、天赋）
│   ├── 选择路径
│   └── 结果预览
├── 信息类（图鉴、日志、统计）
│   ├── 查阅型浏览
│   └── 目标追踪
└── 系统类（设置、保存、退出）
    └── 低频但关键
```

### 1.2 使用频率与路径深度的关系

对于所有元 UI 界面，应遵循：

$$\text{路径深度} \propto \frac{1}{\text{使用频率}}$$

| 使用频率                   | 路径深度   | 界面位置       |
| :------------------------- | :--------- | :------------- |
| 极高（更换装备、使用物品） | 1 次点击   | 主菜单直接可见 |
| 高（查看任务、技能）       | 1-2 次点击 | 主菜单快捷键   |
| 中（商店、图鉴）           | 2 次点击   | 子菜单         |
| 低（设置、退出）           | 2-3 次点击 | 系统菜单深层   |

## 二、背包/物品栏设计

### 2.1 信息组织方式

| 方式          | 适用场景     | 优点                   |
| :------------ | :----------- | :--------------------- |
| **网格**      | 物品类型多样 | 视觉直观，适合触控     |
| **列表**      | 物品数量多   | 信息密度高，适合鼠标   |
| **分类标签**  | 物品类型明确 | 快速过滤，降低认知     |
| **搜索/过滤** | 物品数量极多 | 精准定位，适合资深玩家 |

### 2.2 物品卡片设计

每个物品展示的信息应包含：

```
┌─────────────────────┐
│    [物品图标]         │  ← 辨识度优先
│                      │
│    **钢剑**          │  ← 名称
│    品质: ★★★★       │  ← 稀有度
│    攻击力: +15       │  ← 核心属性
│    速度: 普通       │  ← 次要属性
│                      │
│    [装备] [出售]     │  ← 操作按钮
└─────────────────────┘
```

### 2.3 信号标注

物品列表中应通过视觉信号帮助玩家快速定位：

| 信号           | 含义               | 示例           |
| :------------- | :----------------- | :------------- |
| **新物品标记** | 上次查看后新增     | 黄色"New"标签  |
| **升级提示**   | 该物品优于当前装备 | 绿色向上箭头   |
| **套装标记**   | 该物品属于某套装   | 套装名称+图标  |
| **任务关联**   | 该物品与任务相关   | 任务图标角标   |
| **数量标注**   | 可堆叠物品数量     | 图标右下角数字 |

> **与知识库关联**：Build 信号原理同样适用于物品标注——好的标注让玩家"一看就知道该不该拿"。——`理论知识/16-卡牌设计规范.md`、`项目经验/03-增量自走棋_荒野与曙光-卡牌设计检核.md`

## 三、技能树/成长界面设计

### 3.1 信息层级

```
技能树
├── 全局概览（当前点数、总进度）
├── 分支/类别（按功能/风格划分）
│   ├── 技能节点（名称、图标、说明）
│   │   ├── 已解锁 ✓
│   │   ├── 可解锁 ⭐
│   │   └── 未解锁 🔒
│   └── 连接线（前置依赖关系）
└── 详细面板（选中技能的详细信息）
    ├── 效果描述
    ├── 数值变化
    └── 前置/后续节点
```

### 3.2 预览与后果可视化

玩家在做出选择前应能预览后果：

| 预览类型     | 实现方式                             |
| :----------- | :----------------------------------- |
| **数值预览** | "+15 攻击力" 的数值变化动画          |
| **技能预览** | 悬停/选中时播放技能演示动画          |
| **路径预览** | 高亮学习该节点后可解锁的后续节点     |
| **重置预览** | 显示重置后返还点数和可重新分配的选项 |

### 3.3 选择焦虑控制

当技能树过大时，玩家可能陷入"选择焦虑"（Hick 定律风险）：

| 策略           | 做法                             |
| :------------- | :------------------------------- |
| **分组**       | 按功能/风格将技能分为 3-5 个类别 |
| **推荐标记**   | 标记当前 Build 最适配的技能节点  |
| **预设方案**   | 提供预设加点方案供参考           |
| **渐进披露**   | 前 10 级仅开放核心分支，逐步解锁 |
| **低风险重置** | 允许低成本重置加点以鼓励尝试     |

> **与知识库关联**：Hick 定律指出单屏选择数增加会导致决策时间对数增长。——`01-游戏UI设计原则.md §2.2`

## 四、商店/交易界面设计

### 4.1 交易界面架构

```
┌──────────────────────────────────────────┐
│  商店                            [退出]   │
├──────────────────────────────────────────┤
│  分类: [武器] [防具] [消耗品] [特殊]      │
├──────────────────────────────────────────┤
│  物品网格 / 列表                          │
│  ┌────┐ ┌────┐ ┌────┐                    │
│  │ A  │ │ B  │ │ C  │    ← 物品卡片      │
│  └────┘ └────┘ └────┘                    │
│                                          │
│  选中物品详情面板                          │
│  名称 | 属性 | 价格 | 拥有数量             │
│                                          │
│  [购买] [比较] [收藏]                     │
├──────────────────────────────────────────┤
│  玩家信息: 金币: 500    背包: 12/20       │
└──────────────────────────────────────────┘
```

### 4.2 价格感知优化

| 手法           | 效果                     | 说明         |
| :------------- | :----------------------- | :----------- |
| **参考锚定**   | 原价划掉显示折扣价       | 增强"划算感" |
| **价格对比**   | 显示当前拥有的金币和价格 | 降低决策摩擦 |
| **价值标识**   | 标记"性价比高""超值"     | 引导新手选择 |
| **稀缺性标识** | "限时""仅剩 3 件"        | 刺激购买动机 |

## 五、设计检查清单

### 5.1 通用元 UI

- [ ] 高频操作路径是否最短（1 次点击可达）？
- [ ] 所有界面是否可通过 ESC/返回键逐级退出？
- [ ] 同类信息在不同界面中的布局是否一致？
- [ ] 是否有搜索/过滤功能（物品 > 20 种时）？

### 5.2 背包/物品栏

- [ ] 新物品是否有明显标记？
- [ ] 装备对比是否直观（当前 vs 待选）？
- [ ] 批量操作（出售/分解/整理）是否支持？
- [ ] 物品图标是否在缩略图尺寸下也能辨识？

### 5.3 技能树/成长

- [ ] 前置依赖是否清晰（连线/箭头）？
- [ ] 选择后果是否有预览？
- [ ] 是否提供低风险重置路径？
- [ ] 当前可用点数是否始终可见？

### 5.4 商店

- [ ] 价格信息是否一目了然？
- [ ] 购买前是否有确认步骤？
- [ ] 玩家当前资源是否始终可见？
- [ ] 已拥有的物品是否有标识？
