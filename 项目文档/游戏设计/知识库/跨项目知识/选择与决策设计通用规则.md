# 选择与决策设计通用规则

> 标签：`decision-design`, `player-psychology`, `event-design`, `choice-design`
> 来源项目：家族纪元（原型迭代验证）
> 状态：draft

---

## 一、选项必须有实质性差异

玩家面对多个选项时，如果各选项只有叙事文本不同（log）而实际游戏效果相同，则不是真正的选择。

**检测方法**：去掉所有选项的 log 文本后，各选项是否仍有可感知的差异？
- 否 → ❌ 霍布森选择（伪选择），需要重设计

**正确做法**：每个选项至少改变一种不同的游戏资源（金钱/属性/成员/卡牌等），让玩家根据当前局面做取舍。

```
❌ 大泽乡起义 → 三条不同log，实际效果：无
✅ 大泽乡起义 → 支持(-20金+体魄) / 效忠(+20金) / 中立(-5金)
```

## 二、避免负反馈（两害相权→正面投资）

当玩家被要求在"两种损失之间选择"时，无论选哪个都会产生被迫感和挫败感。

**检测方法**：将每个选项的收益/损失分别列出。如果所有选项都有净损失，则触发该问题。

**修复方向**：将"损失 vs 损失"重构为"投资方向 A vs 投资方向 B"，让玩家感知到自己在做**积极的战略选择**而非承受惩罚。

```
❌ 焚书令 → 上交(+10金🧠-1) / 隐藏(-10金🧠+1)  → 选啥都亏
✅ 焚书令 → 上交(+10金) / 隐藏(-10金，后代智力+1)  → 各有收益感
```

## 三、资源转换 > 数值比较

当多个选项产出同一种资源时，玩家永远选数值最大的那个——这不是设计选择，是设计答案。

**检测方法**：将所有选项的收益换算为同一种资源。如果某个选项在所有局面下都优于其他选项，则触发该问题。

**修复方向**：让选项在不同资源类型之间转换。不同时空条件下，玩家的资源稀缺度不同，自然产生不同的最优解。

```
❌ 商队 → 赚+3 / 赚+10 / 赚0
✅ 商队 → 手牌→金 / 金→手牌 / 金→扩建折扣
```

## 四、核心机制不应纯随机

当一项对玩家至关重要的进程（如人口增长/角色获取/资源生产）完全依赖随机判定时，玩家会感到无力且缺乏策略感。

**解决方案**：提供一种**确定性手段**作为"安全网"，让玩家在愿意付出额外成本时绕过随机判定。

```
❌ 自然生育：每年15%概率 → 看脸，连续4年不生概率52%
✅ 添丁卡：已婚+年龄<40+有住房 → 确保怀孕
   （消耗手牌位作为成本，换取确定性）
```

**设计要点**：
- 确定性手段的成本应高于随机手段的期望成本（否则随机手段被淘汰）
- 确定性与随机并存 → 玩家自行权衡"资源 vs 概率"
- 确定性手段不应完全取代随机机制

## 五、合成/融合系统的非线性数学陷阱

当玩家可以将多个低阶物品合成为高阶物品时，如果效果是线性缩放（2×低阶 = 1×高阶），则合成无收益——玩家不如分开使用。

```
线性（❌）：
  1★:  +0.2/次
  2★:  +0.4/次（= 2×1★分别打）
  → 融合无意义

非线性（✅）：
  1★:  +0.2/次
  2★:  +0.6/次（3×1★，优于2×1★分别打）
  → 只有融合才能达到这个倍率
```

**通用规则**：合成后物品的价值应是合成前物品价值之和的 **1.5x~2x**，玩家才有合成的动力。具体倍率取决于：
- 合成的不可逆性（永久消耗低阶物品）
- 获取低阶物品的难度
- 高阶物品是否有额外解锁效果

## 六、晋升/里程碑事件是庆祝而非选择

当玩家达到某个里程碑时，弹出"是否接受晋升"的对话框会让玩家困惑——晋升本身就是值得庆祝的事，不应该让玩家来选择"要不要"。

**通用规则**：晋升/升级/里程碑事件应**自动执行**并伴随正反馈（特效/奖励/动画），而非抛出一个需要玩家确认的对话框。确认对话框隐含"拒绝"选项，削弱了成就感知。

---

## 七、内容/事件应有触发条件

避免不合逻辑的内容触发——没有相关背景或能力的玩家不应遇到某些事件。

**通用规则**：每个事件/内容块应定义其触发的前置条件，防止"缺乏文化底蕴的家族被焚书令困扰"这类逻辑断裂。

常见条件类型：
- 属性门槛（平均智力 ≥ 4）
- 职业要求（需要特定职业成员）
- 资源门槛（需要特定数量的某资源）
- 进度锁（已解锁特定区域/阶段）
