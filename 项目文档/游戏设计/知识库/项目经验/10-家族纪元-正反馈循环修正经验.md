# 家族纪元·正反馈循环修正经验

> 来源项目：家族纪元（家族模拟×卡牌策略×穿越中国历史）

## 一、诊断方法

### 1.1 理论对照法

将原型现状与知识库理论逐条对比，定位根因：

| 理论        | 原问题               | 根因                                                 |
| ----------- | -------------------- | ---------------------------------------------------- |
| CET认知评价 | 控制性反馈（得金币） | 反馈告诉玩家「你得到了什么」而非「你变强了」         |
| 正反馈循环  | 行为之间无因果链     | 打牌→产出+0.3→**断掉了**，没有形成下一个行为的驱动力 |
| 驱动力模型  | 只有外驱（金币）     | 内驱≈0，职业无身份感，打牌无策略感                   |
| 双轨收益    | 只有单轨             | 所有收益固定，没有惊喜轨道                           |
| 强制循环    | 无权力下放           | 玩家没有「这回合选A还是B」的决策空间                 |
| 心理力学    | 行为惯性锁死         | 所有策略都导向金币，改变成本高但收益不明确           |

### 1.2 日志分析法

跑69步实机操作数据，统计卡牌使用频率、金库波动、人口变化、职业变迁，用数据验证诊断。

## 二、核心修改

### 2.1 卡牌效果：从临时buff到永久成长

**旧**：打出「丰收」→ 农业+25%持续10年 → 10年后消失
**新**：打出「深耕」→ 💪+1，💰+0.3 → 永久持有

**理论依据**：
- CET理论：信息性反馈（「我变强了」）增强内驱，控制性反馈（「系统给我金币」）削弱内驱
- 永久成长让玩家每次打牌都有积累感，正反馈从「临时工具感」变为「成长叙事」
- 角色寿命70±5岁，时效限制逻辑上多余——人都会死，卡牌时效无意义

### 2.2 跨角色协作卡：7种不同机制

每个职业的第三张卡替换为完全不同的交互机制：

| 职业  | 卡名 | 机制                   | 设计意图             |
| ----- | ---- | ---------------------- | -------------------- |
| 🌾农夫 | 帮工 | 双方产出各+0.5         | 双赢，鼓励培养搭档   |
| 🔨工匠 | 锻造 | 消耗8金，目标+0.8      | 资源转换，花钱买增长 |
| 💰商人 | 投资 | 消耗10金，目标+1.2     | 高投入高回报         |
| ⚔️军人 | 练兵 | 双方体力各+1（需年轻） | 年龄条件，老带新     |
| 📜官吏 | 举荐 | 无职获得职业+产出      | 痛点解决             |
| 📜士   | 讲学 | 双方智力各+1           | 双向成长             |
| 🗡️游侠 | 援护 | 条件产出+0.8/交换      | 条件触发+交换机制    |

**理论依据**：
- 心理力学模型：行为改变需要打破当前行为惯性。所有卡都「选人→加数值」会形成路径依赖
- 边际递减：同一种正反馈反复出现且形式单一时，效果显著减少

### 2.3 非目标型卡牌（5种新增类型）

在成长卡+协作卡+经济卡的基础上，引入完全不同的卡牌类型：

| 类型  | 例子              | 决策维度             |
| ----- | ----------------- | -------------------- |
| 全局🌍 | 全族产出+0.3      | 不选人，追求整体效益 |
| 延迟⏳ | 2年后到账15金     | 等待vs即时满足       |
| 转化🔄 | 消耗5金换2张卡    | 资源形态转换         |
| 自毁💥 | +1.0产出但年龄+3  | 短期收益vs长期代价   |
| 环境🌤 | 5年内容产牌率翻倍 | 改变概率而非数值     |

### 2.4 晋升自动化

从「弹窗询问是否晋升」改为「达标自动晋升+庆祝动画」。

**理论依据**：
- 晋升是正反馈（变强），弹窗让玩家选择「要不要变强」是多余步骤
- 权力下放的误用——玩家不需要对自己明显想要的结果做确认
- 避免「放弃晋升」这种反直觉选项

### 2.5 双重消耗机制

| 成员状态  | 年消耗     | 设计意图                   |
| --------- | ---------- | -------------------------- |
| 儿童(<16) | -1金       | 生娃有养育成本             |
| 无职      | -1金       | 养闲人有代价，驱动安排职业 |
| 老人(≥55) | -1金       | 退休有社会成本             |
| 在职      | 产出正收入 | 工作创造价值               |

### 2.6 年收支可视化

顶栏从「💰 40」改为「💰 40 (+2.3/年)」，让经济数据成为决策信号。

## 三、未采纳方案及原因

| 方案             | 原因                                       |
| ---------------- | ------------------------------------------ |
| 角色不能自给自足 | 惩罚式设计。CET理论：控制性反馈削弱内驱    |
| 家族里程碑系统   | 核心循环尚未跑顺，叠加上层系统混淆问题来源 |
| 深化心态系统     | 需要一整套独立机制支撑，当前阶段不划算     |
| 纯数值平衡调整   | 治标不治本，根本问题是行为链条断裂         |

## 四、经验总结

### 4.1 判断反馈性质的方法

```
问：这个反馈告诉玩家什么？
  → 「你得到了X金币」→ 控制性（外驱）
  → 「你的角色变强了」→ 信息性（内驱）
```

### 4.2 正反馈循环的检验标准

```
行为 → 反馈 → 目标进展 → 驱动力强化 → 继续行为
                                  ↑
                            链条不能断
```

如果某个行为的结果不能增加下一个行为的驱动力，这个行为就是孤立的，需要重新设计。

### 4.3 卡牌多样性原则

同一套卡牌体系中，两张卡的交互模式不应相同。判断标准：

```
玩家执行这两个操作时的「心智动作」是否一样？
  是 → 需要差异化（改条件/改效果/改目标方式）
  否 → OK
```

### 4.4 晋升设计的要点

- 晋升应该是「庆祝」而非「选择」——玩家不会拒绝变强
- 晋升必须有实质奖励（稀有卡），否则只是数字+1
- 晋升门槛必须可见（进度条），利用目标梯度效应
