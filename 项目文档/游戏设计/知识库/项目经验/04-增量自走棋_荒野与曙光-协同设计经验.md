# 增量自走棋 & 荒野与曙光 — 协同设计经验

> 项目经验
> 来源：增量自走棋、荒野与曙光
> 关联知识：`理论知识/35-协同效应.md`, `理论知识/04-叠层设计与延迟满足.md`

## 核心原则

- 协同效应应基于**共享中间层**（如标签、属性类型），而非指名卡牌联动
- 协同达成应依赖玩家决策，而非纯随机
- 协同强度应有上限，避免单一构筑垄断

## 协同的两种底层方向

构筑游戏中有两种底层Build方向，设计时应有意识分化：

| 方向         | 机制                             | 玩家心智模型           | 示例                               |
| ------------ | -------------------------------- | ---------------------- | ---------------------------------- |
| **同名堆叠** | 多个同名单位叠层/叠星→越叠越强   | "我要找更多同一个东西" | 小兵的"学习"复制、同名堆叠暴击光环 |
| **异名混搭** | 不同类型/标签的单位互补→协同效应 | "我要找不同东西来互补" | 小丑马戏团的"不同Tag→舞台增益"     |

**核心规则**：一种卡牌/机制设计出来后，应能明确回答"它指向哪个方向"。

## 示例

| 项目       | 协同模式                        |
| ---------- | ------------------------------- |
| 增量自走棋 | Tag标签体系——同标签单位互相增益 |
| 荒野与曙光 | 武器卡牌组合——职业内卡库协同    |

## 与自走棋通用协同机制的对比

### Tag体系 vs 职业体系

增量自走棋的 Tag 标签体系与经典自走棋的协同系统有相通之处，但维度更单一：

| 维度           | 增量自走棋（Tag体系） | 经典自走棋（职业+种族） |
| :------------- | :-------------------- | :---------------------- |
| **维度数**     | 单维度                | 双维度/多维度           |
| **触发方式**   | 同Tag数量达标         | 同职业/种族数量达标     |
| **强度曲线**   | 连续型                | 阶梯型（2/4/6/8层）     |
| **构筑灵活度** | 高——可混合            | 中——需双维度配合        |
| **复杂度**     | 低                    | 中-高                   |

**参考**：`理论知识/53-自走棋协同机制设计.md` §三 — Tag体系与职业体系的详细对比。

### 协同强度曲线

增量自走棋的协同是**连续型增长**（多一个同Tag单位就多一层增益），而经典自走棋常用**阶梯型触发**（达到2/4/6个才触发质变）：

```
连续型（增量自走棋）：
    3个同Tag → 30%增益
    4个同Tag → 40%增益
    → 每个单位都有贡献，但缺少"质变感"

阶梯型（经典自走棋）：
    2个同种族 → 基础增益（引导）
    4个同种族 → 高额增益（成型）
    6个同种族 → 质变（终极）
    → 有明确的里程碑感
```

**设计启示**：可以在连续型协同中引入"阈值拐点"——在多Tag体系中设置关键数量节点，在保持灵活性的同时提供"质变"时刻。

### 涌现式协同的应用

增量自走棋的协同设计应遵循**涌现式协同**（`理论知识/35-协同效应.md` §涌现式协同）：

```
✅ 涌现式： 
   "攻击时触发" → 找高攻速/多段攻击单位
   "暴击时触发" → 找暴击率/暴击伤害单位

❌ 说明书式：
   "与XX单位的Tag相同" → 堆Tag就行
```

逐步将部分协同从"名字匹配"转向"行为匹配"，提升玩家发现的乐趣。

## 设计自检

- [ ] 协同是否基于共享中间层？
- [ ] 玩家是否有决策空间？
- [ ] 是否存在通解/最优解？
- [ ] 协同强度的上限是否可控？
- [ ] 是否有"质变时刻"（而非纯连续增长）？
- [ ] 协同方向是否分化清晰（同名堆叠 vs 异名混搭）？
