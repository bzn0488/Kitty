# 增量自走棋 — 升星机制设计

> 项目经验
> 来源：增量自走棋
> 关联知识：`理论知识/04-叠层设计与延迟满足.md`, `理论知识/35-协同效应.md`, `理论知识/16-卡牌设计规范.md`

## 设计背景

在卡牌构筑游戏中，升星是一个非常普遍的机制。大多数游戏的升星就是"同卡合并→数值×N"，实质上只解决了"抽到重复卡怎么办"的问题，没有产生新的乐趣密度。增量自走棋的升星机制试图打破这一局限——让升星本身成为构筑决策的一部分。

## 核心设计

### 战斗单位升星

| 规则     | 说明                                         |
| :------- | :------------------------------------------- |
| 触发条件 | 2 张同卡面同星级的卡牌同时出现在玩家控制区域 |
| 操作方式 | 弹出合成界面，玩家手动确认                   |
| 基底选择 | 玩家手动选择保留哪张卡作为基底               |
| 技能保留 | 玩家手动选择保留哪些技能到基底上             |
| 技能超限 | 超出槽位上限时，玩家手动卸下多余技能         |

### 三档升星效果

升星不是纯数值提升，而是**改变决策逻辑**。分为三个档位：

| 档位           | 效果                             | 设计意图                 |
| :------------- | :------------------------------- | :----------------------- |
| **数值提升**   | 基础属性等额提升                 | 基础强度增长（保留）     |
| **条件放宽**   | 固有技能的条件阈值降低或范围扩大 | 让卡牌"更好用"而非"更强" |
| **中间层加速** | 写入中间层的效率提升             | 构建涌现式协同的基础     |

**条件放宽示例**：
- 1 星：条件为"相邻有友方" → 2 星：条件放宽为"相邻+斜角有友方"
- 1 星：条件为"暴击时" → 2 星：条件为"暴击或格挡时"

**中间层加速示例**：
- 1 星：每次攻击写入 1 层"战意" → 2 星：每次攻击写入 2 层"战意"
- 1 星：每秒产出 2 金币 → 2 星：每秒产出 3 金币

### 经济单位升星（独立机制）

经济单位的升星不同于战斗单位——**不通过合成实现**，而是每个经济单位自带内置升星任务：

- 任务类型：累计型（如"累计产出 50 金币"、"累计触发 10 次结算"）
- 任务进度：战斗中实时累积，进度条始终可见
- 升级操作：任务完成 → 手动点击 → 星级+1
- 循环机制：升级后自动生成下一级升星任务（条件更高）

**设计意图**：使用行为驱动升星，形成"用得越多→升得越快→产出更多→用得更多"的正反馈闭环。

## 设计经验总结

### 升星应该改变决策逻辑，而非纯数值

这是升星设计最核心的原则。如果升星仅提升数值，玩家不会对升星产生"期待"——只是"应该做的日常操作"。只有当升星改变卡牌的使用方式（条件放宽）或协同潜力（中间层加速），玩家才会主动思考"怎么让这张卡升星"。

### 手动保留技能 = 权力下放 + 避免损失厌恶

合成时玩家手动选择保留哪些技能，避免自动合成导致玩家损失已投入的技能。这给了玩家控制权，也让技能投资成为一个有意义的决策。

### 三档升星效果的结构化

升星的三个档位对应不同的乐趣来源：
- **数值** → "我变强了"（胜任感）
- **条件放宽** → "这张卡有更多用法了"（探索欲）
- **中间层加速** → "我的构筑涌现了新的协同"（智力优越感）

三个档位叠加，让升星不仅是"变强"，更是"拓展构筑空间"。
