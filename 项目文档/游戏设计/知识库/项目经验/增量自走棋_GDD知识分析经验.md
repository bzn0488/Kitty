# 增量自走棋 GDD 知识分析经验

> 使用 GDD 知识分析工作流 v2.0 对《增量自走棋》GDD 进行系统性分析的成果总结。

## 一、设计意图推导方法论

### 1.1 三类意图的区分

在反向推导设计意图时，可以按以下三类区分：

|      类型      | 标记  | 定义                   | 示例                           |
| :------------: | :---: | ---------------------- | ------------------------------ |
| **功能性意图** |   🛠️   | 解决具体的游戏机制问题 | "防止无限僵持"、"降低选择门槛" |
| **体验性意图** |   🎭   | 创造特定的玩家体验     | "让玩家感觉变强了"、"戏剧张力" |
| **心理性意图** |   🧠   | 利用/服务于玩家心理    | "利用损失厌恶"、"蔡格尼克效应" |

### 1.2 推导方法

1. **如果 GDD 已写设计目的** → 评估其完整性、合理性，补充缺失维度
2. **如果 GDD 未写设计目的** → 根据核心规则反推：这个系统让玩家做什么？解决了什么痛点？创造了什么体验？
3. **判定置信度**：高（从规则直接推断）/ 中（推测但有规则支持）/ 低（纯推测）

### 1.3 经验总结

- **不要被理论"框住"思维**：先理解设计者自己想解决什么问题，再用理论去验证
- **商业性意图判断**：独立游戏通常商业意图较少，但需要留意付费点设计的隐性意图
- **意图的"冰山模型"**：GDD 明确写的设计目的是"水上部分"，补充推导的往往是"水下部分"

## 二、GDD 分析的典型发现

### 2.1 常见设计模式

以下模式在《增量自走棋》中反复出现，可能是自走棋/构筑类游戏的通用设计模式：

| 模式              | 在增量自走棋中的体现           | 理论依据                |
| :---------------- | ------------------------------ | ----------------------- |
| **两阶段决策**    | 选包→选卡（两次期待+两次自主） | 信号系统、SDT自主感     |
| **三段式奖励**    | 10%/30%/60% 分段发放           | 固定比率强化、快乐总量  |
| **三维构筑**      | 位置关系×中间层×Tag            | 涌现式设计、协同效应    |
| **三级强化**      | 低级/中级/高级                 | 投篮效应、目标梯度      |
| **三货币闭环**    | 金币/代币/刷新币               | 避免单一通胀            |
| **自动+手动混合** | 自动战斗+手动构筑              | 减少无意义操作+权力下放 |

### 2.2 冲突检测重点

| 冲突维度 | 检查要点                               |
| -------- | -------------------------------------- |
| 意图层面 | 两个系统的设计方向是否矛盾？           |
| 规则层面 | 规则执行时是否产生矛盾？               |
| 资源层面 | 资源分配是"有意义取舍"还是"隐性冲突"？ |
| 心流层面 | 系统A的节奏是否破坏系统B的体验？       |

### 2.3 体验路径分析四阶段

|   阶段   |  时间   | 关注重点                       |
| :------: | :-----: | ------------------------------ |
| 首次接触 | 前5分钟 | 第一次正反馈速度、第一个困惑点 |
| 早期探索 |  前3局  | 核心循环理解速度、经济系统认知 |
| 中期熟练 | 3-10局  | 外线过渡平滑度、系统开放节奏   |
|  精通期  |  10+局  | 构筑深度、数值稳定性、长期目标 |
