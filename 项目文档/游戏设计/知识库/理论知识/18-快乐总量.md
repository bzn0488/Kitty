# 快乐总量

> 标签：`happiness-total`
> 关联概念：`46-正反馈与负反馈循环.md`, `33-无效进程与三层乐趣体系.md`, `17-快车道设计.md`
> 知识整理参考：`01-核心概念与思维模型.md`（快乐总量定义+单位进程快乐总量）、`03-游戏机制设计技法.md`（正反馈技法完整展开）

## 定义

游戏中正反馈（正值）与负反馈（负值）的总和。

- **默认讨论范围**：所有玩家的快乐总量
- **核心推论**：快乐总量越高，总体验越好

### 示例：非指向性技能
《英雄联盟》非指向性技能：
- 命中 → 攻击者获得大正反馈 + 防守者小负反馈 → **快乐总量提高**
- 未命中 → 攻击者小负反馈 + 防守者大正反馈 → **快乐总量提高**

## 单位进程快乐总量

```
单位进程快乐总量 = 快乐总量 / 游戏进程
```

**手段**：
- 增加快乐总量
- 减少进程（非缩减体量，而是减少无效时间/压缩低正反馈进程）

## 提供正反馈的技法列表

| 技法 | 核心 | 适用场景 |
|------|------|----------|
| 协同效应 | 1+1>2 | 构筑类游戏 |
| 叠层 | 累积到阈值触发大奖 | 技能/攻击循环 |
| 投篮效应 | 低成本+高惊喜概率 | 任意可重复行为 |
| 精准操作 | 操作技巧换奖励 | 动作/射击游戏 |
| 浪费规避 | 减少浪费+补救途径 | 资源管理类 |
| 他者确认 | 多人交互认可 | 多人/社交游戏 |
| 额外挑战&及格线 | 分层目标 | 关卡/挑战设计 |
| 创造性表达 | 规则内自由选择 | 母系游戏/沙盒 |
| 双轨收益 | 固定+随机奖励 | 通用 |
