# 相对最优解与三层变化模型——Roguelike决策设计的底层逻辑

> 标签：`decision-design`, `optimal-solution`, `roguelike`, `emergence`, `player-psychology`, `game-analysis`

## 一、核心抽象：游戏 = 问答

游戏可以抽象为最底层的结构：

```
设计师提出问题 → 玩家从选项中作答 → 系统给予反馈
```

这个抽象适用于所有游戏。不同的游戏类型，本质上是"问题如何提出、选项如何生成、反馈如何给予"的差异。

从这一抽象出发，可以推导出游戏设计中一个根本性的困境和它的解法。

---

## 二、霍布森选择困境

### 2.1 两种霍布森选择

当玩家面临选择时，有两种情况会导致选择无意义：

| 类型 | 本质 | 例子 | 后果 |
|:-----|:------|:------|:------|
| **选项不平衡** | 一个选项明显优于其他 | "捡100块还是捡50块"（无后续影响） | 玩家实际没有选择，只能选最优 |
| **重复体验** | 同一问题反复出现 | 再次面对已经知道答案的谜题 | "这题我做过了"——审美疲劳 |

霍布森选择 = 看似有选择，实则没有。

### 2.2 游戏类型的本质差异

| 类型 | 定义 | 特征 | 代表 |
|:-----|:------|:------|:------|
| **绝对最优解** | 问题有唯一正确答案 | 找到"设计师预期的答案"→ 通过 | 传统解谜、传统塔防、线性关卡 |
| **相对最优解** | 答案取决于当前上下文 | 固定答案不存在，只有"在当前情境下比较好" | Roguelike、CRPG、策略游戏 |

## 三、相对最优解的三层变化模型

相对最优解要运作，需要**三个层次同时变化**。只有一层或两层变化，相对最优解空间仍然有限。

```mermaid
flowchart TB
    subgraph 三层变化
        L1[L1(问题变化)<br>玩家面对什么]
        L2[L2(解答方式变化)<br>玩家拥有什么]
        L3[L3(规则变化)<br>游戏怎么运作]
    end
    
    L1 -->|新鲜感| S[策略思考]
    L2 -->|构筑差异| S
    L3 -->|打破经验| S
    
    S -->|智力优越感| F[强正反馈]
    
    L3 -.->|外因改变| L1
    L3 -.->|外因改变| L2
```

### 3.1 三层定义

| 层级 | 变化什么 | 防止的路径依赖 | 实现方式 |
|:-----|:---------|:--------------|:---------|
| **L1(问题变化)** | 敌人/地图/事件 | "这关我知道怎么打"——解法固定化 | 随机地图、随机敌人组合、程序化生成 |
| **L2(解答方式变化)** | 卡牌/角色/Build | "我就用这套最优构筑"——构筑固定化 | 随机卡牌掉落、每局重新构筑、不同角色 |
| **L3(规则变化)** | 游戏底层规则 | "这个游戏我玩透了"——体验固定化 | 规则变异、外因改变、版本改动 |

### 3.2 三层各自的效率

```
防止路径依赖的效率：
  L3(规则变化)  >>>  L2(解答方式变化)  >>  L1(问题变化)
```

| 场景 | 为什么效率低 | 玩家心理 |
|:-----|:------------|:---------|
| 只变 L1(问题变化)（不变 L2/L3） | 用同一套打法应对不同敌人 | "换个皮而已" |
| L1+L2 变（不变 L3） | 换构筑但底层逻辑不变 | 能玩很久，但终有天花板 |
| 三层全变 | 需要重新理解游戏逻辑 | "每次都是新游戏"——天花板被大幅推后 |

### 3.3 相对最优解空间公式

```
相对最优解空间 = L1(问题变化)变化量 × L2(解答方式变化)变化量 × L3(规则变化)变化量
```

| 变量 | 含义 | 决定因素 |
|:-----|:------|:---------|
| 变量 | 含义 | 决定因素 |
|:-----|:------|:---------|
| L1(问题变化)变化量 | 敌人/地图/事件的种类数量 | 内容制作量 |
| L2(解答方式变化)变化量 | 有效构筑路径的数量 | 机制轴设计、协同效应 |
| L3(规则变化)变化量 | 规则变异/外因改变的组合数 | 规则变异系统的组合爆炸 |

**关键推论**：当任何一层为 0 时，整个空间归零。这正是传统 Roguelike 有天花板的根本原因——L3(规则变化)规则层不变，玩家终将"玩透"。

### 3.4 三层变化与体验图

```
体验宽度
  ↑
  |              L1+L2+L3（三层全变→空间最大）
  |           ╱╲
  |         ╱   ╲
  |       ╱      ╲        L1+L2（两层→宽但不深）
  |     ╱         ╲     ╱╲
  |   ╱            ╲ ╱   ╲
  | ╱               ╲      ╲        L1 only（一层→很快就窄了）
  |╱                  ╲     ╲    ╱╲
  └──────────────────────────────────────→ 游戏进程
```

三层全变时，体验图不仅是"宽"，而且是**持续宽**——玩家在不同规则下走不同的构筑路径面对不同的问题，体验宽度不会在中后期收窄。

参考：`理论知识/29-体验图分析框架.md`

## 四、Roguelike 的解法验证

### 4.1 Roguelike 的随机化本质

从三层模型来看，Roguelike 的真正创新不是"随机"本身，而是通过**双重随机**撑开相对最优解空间：

```
① 随机化问题（L1(问题变化)）——这次遇到什么敌人/事件？
② 随机化答案（L2(解答方式变化)）——这次给你什么牌/遗物？

→ 即使玩家保留了经验（"这个Boss我知道怎么打"）
→ 但当前构筑不同、可用资源不同→ 最优解不同
→ 所以每次决策都是新鲜的
```

### 4.2 判断 Roguelike 质量的标准

> 一款 Roguelike 游戏，如果你玩了半小时，决策时有重复感，这半小时内体验没有差别——那这款游戏的设计师可能并没有真的理解 Roguelike 的底层逻辑。

从三层模型来看，半小时内出现决策重复感 = 三层变化空间不够大。通常是 L2(解答方式变化)（构筑多样性不足）或 L3(规则变化)（规则层毫无变化）的问题。

### 4.3 规则变异作为 L3(规则变化) 解法

L3(规则变化)是最强但也最容易被忽视的层次。`77-规则变异设计.md` 中提出的规则变异系统正是 L3(规则变化)的具体实现：

- 每次游戏规则改变 → 玩家需要重新评估最优解
- 规则变异的组合爆炸 → L3(规则变化)变化量接近无穷
- 规则变异影响 L1(问题变化)（敌人规则变化）和 L2(解答方式变化)（构筑规则变化）→ 三层联动

## 五、协同效应的双刃剑——相对最优解的自我矛盾

> 这是三层模型最深刻的内部矛盾：创造正反馈的机制（协同效应），同时也是破坏正反馈的源头。

### 5.1 矛盾的本质

```
协同效应 → 当构筑成型时 → 乘法正反馈 → "我太聪明了" ✅
协同效应 → 当组件凑不齐时 → 选项无效化 → "游戏在搞我" ❌
```

相对最优解依赖**协同效应**来撑开 L2(解答方式变化)空间——不同卡牌/能力组合产生乘法效果，让玩家获得智力优越感。但协同效应本身有一个隐藏代价：

| 协同的正面 | 协同的负面 |
|:-----------|:-----------|
| 凑齐组件时快感强烈 | 凑不齐组件时挫败感强烈 |
| 创造"发现组合"的惊喜 | 同一组合反复出现后变成路径依赖 |
| 鼓励深度探索 | 排除非协同选项 → 玩家实际选择空间缩小 |
| 玩家归因于自己的智慧 | 随机性阻断协同 → 归因转向外部（"游戏设计有问题"） |

### 5.2 归因漂移——从内驱到外驱的转折点

这是最关键的心理学机制——当协同被随机性阻断时，玩家的归因会从**内部**漂移到**外部**：

```
玩家拿到符合构筑方向的选择：
  "我选的" → 内部归因 → "我很聪明" → 胜任感 ↑ ✅

玩家拿到不符合构筑方向的选择（且无法改变）：
  "RNG没给我" → 外部归因 → "游戏不公平" → 胜任感 ↓ ❌
```

**转折点**在于玩家是否感受到**自己仍然有选择的余地**。如果玩家说"这一局我只能玩火伤Build，因为游戏只给了我火伤牌"——这就是**自主感崩塌**的征兆。玩家不是在"相对最优解"中做选择，而是在"被规则强迫的唯一路径"中做确认。

### 5.3 "策划教我玩游戏"的本质

当以下条件同时满足时，就会出现"策划教玩家玩游戏"的体验：

```
① L3(规则变化)暗示了一个强势方向（如"敌人弱火"）
② L2(解答方式变化)的空间被该方向垄断（火伤卡牌远强于其他）
③ 玩家的自主选择权被压缩（不选火伤 = 这一局很难玩）

→ 玩家感觉：这局游戏已经替我决定了"最优解"
→ 我做的不是"选择"，只是"确认"
→ 霍布森选择以新的形式回来了
```

这与模型的根本目标——创造相对最优解——形成了矛盾。协同效应本是撑开 L2(解答方式变化)空间的手段，但当协同变得太强时，它反而**坍缩了选择空间**。

### 5.4 核心公式：类Rogue 正反馈 = 最终强度 - 初始强度

在深入解法之前，先理解一个关键公式：

```
一局的正反馈 ≈ 最终强度 - 初始强度

类Rogue 的独特优势：
  初始强度 = 极低（每局从零开始）
  最终强度 = 极高（因永久死亡，无需顾虑后续心流）
  → 差值极大 → 正反馈极强
```

这解释了为什么类Rogue框架受欢迎——差距大不是问题，**走不完这个差距**才是问题。协同效应放大的不仅是快乐总量，更是**这个差距的完成速度**：

```
达成协同时：
  快乐总量 ↑↑（协同爆发）+ 进程 ↓↓（敌人死得快）
  → 快乐密度 ↑↑↑

未达成协同时：
  快乐总量 ↓（挫败感）+ 进程 ↑（打不动敌人）
  → 快乐密度 ↓↓↓
```

因此解法方向不是"缩小上下界"（那会牺牲快乐总量），而是**给玩家更多方式去走通这个差距**。

### 5.5 解法方向

#### 5.5a 情境协同（中间层设计）

最深层的解法——协同不绑定在卡牌 ID 上，而是绑定在**共享中间层**上：

```
固定协同："如果你有【燃烧】和【火焰风暴】，造成双倍伤害"
  → 玩家必须同时拿到两张特定牌 → 凑不齐 = 挫败

情境协同（机制轴）：
  "如果敌人身上有【燃烧】，火焰伤害翻倍"
  → 谁都能给敌人上燃烧
  → 不同卡牌/能力都能触发
  → 路径更多，挫败感更低
```

这就是 `理论知识/11-机制轴设计方法.md` 的核心价值——**选择影响中间层属性，而非孤立的两两配对**。卡牌增加"力量"属性，力量属性影响所有力量相关的后续选择——选择不是孤立的，而是对"角色成长方向"的连续投票。

#### 5.5b 随机性暴露（透明度设计）

玩家不反对随机，但反对**无法理解的随机**：

```
模糊的随机 → "RNG在搞我" → 外部归因 → 习得性无助
透明的随机 → "我知道为什么" → 理解 → 掌控感

暴露分为三个级别：
```

| 级别 | 做什么 | 玩家感知 |
|:-----|:-------|:---------|
| **Lv1: 结果暴露** | 掉落后说明"为什么是这个" | "哦，原来如此" |
| **Lv2: 过程暴露** | 选择前提示"你会看到什么类型的选项" | "我能预期到" |
| **Lv3: 操纵暴露** | 让玩家知道如何影响随机结果 | "我能控制它" |

**Lv1 落地方式**——选牌界面旁显示概率分布：

```
当前构筑：3 张火系卡牌
本局变异：【烈焰深渊】火系卡牌 +25%
本包预览——火系：40% | 冰系：20% | 通用：40%
```

**Lv3 落地方式**——主动调节"概率旋钮"：

```
"花费 1 变异点数 → 火系卡牌出现概率 +20%（本层有效）"
→ 玩家主动选择影响随机性
→ 归因回到玩家："是我选择花点数去定向的"
```

#### 5.5c 负反馈调节的公开化——"Boss 式翻盘"

这是最巧妙的设计模式。与传统 DDA（动态难度调节）对比：

| 维度 | 隐藏式 DDA | 公开式负反馈调节 |
|:-----|:-----------|:----------------|
| **透明度** | 隐藏，玩家不知道 | 公开，玩家清楚发生了什么 |
| **玩家归因** | "系统让我赢了"（成就感注水） | "我选择了翻盘路径"（内部归因） |
| **张力** | 无 | 强制 1 血 = 高张力 |
| **可重复性** | 每秒都在调 | 仅触发一次（首死），独特体验 |
| **本质** | 维持体验曲线 | 创造戏剧性转折点 |

**设计模板**（以《The Bazaar》为参考）：

```
① 触发条件：玩家首次死亡（公开明确）
② 强制状态：变为 1 血（制造紧张）
③ 方向选择：与当前阶段强相关（前期/中期/后期强化方向）
④ 具体加成：在选定方向内做强力加成选择
⑤ 效果：强度骤然提升 → 玩家继续爽一段时间 → 直到难度匹配

为什么比 DDA 好：
  · 透明的——玩家知道发生了什么
  · 玩家做了选择——归因于自己
  · 有戏剧张力——1 血 vs 高额奖励的紧张感
  · 代价真实——"我死过一次了"不是系统白给
```

**负反馈公开化三要素**：
```
① 触发条件明确（首次死亡/血量低于阈值/等）
② 补偿是一次选择而非自动发放
③ 补偿本身创造新的风险（1血 = 随时再死）
```

#### 5.5d 代价性刷新的替代方案

传统"花金币刷新"的问题：负反馈被延迟而非消除——玩家为系统的随机性买单。

```
没有刷新时：看到烂选项 → "这局没了"
有刷新时：纠结 → 花钱 → 可能还是烂 → "我早该知道"
```

更好的方向——**免费+有限+定向**：

```
免费刷新（每商店/每层 1 次免费）
→ 有限次数（不花钱，但有次数限制）
→ 定向逻辑（"筛掉当前与你最不匹配的 1 张"而非完全重随）
→ 玩家感知："系统在帮我调整"而非"我又花钱了"
```

### 5.6 设计自检——协同健康度

| 问题 | 健康状态 | 危险信号 |
|:-----|:---------|:---------|
| 当前 L3(规则变化) 下有几条有效路径？ | ≥ 3 条 | 只有 1 条 |
| 非当前路径的卡牌是否完全无用？ | 有基础价值 | 完全零价值 |
| 玩家能否从中途转到其他路径？ | 可以，有代价但可行 | 不能，锁死了 |
| 玩家归因于自己的智慧还是 RNG？ | "我选的" | "RNG给的" |

## 六、时间压力——与三层变化正交的设计轴

时间压力是独立于三层变化模型的一个设计轴。它不改变相对最优解的存在，但决定了玩家能感知到多少相对最优解空间。人是有极限的——我们不应当让一个限时1秒钟的决策变得极其复杂。

### 6.1 不同时间压力下的 ROS 空间特征

| 类型 | 时间压力 | ROS 空间特征 | 设计重心 | 代表 |
|:-----|:---------|:-------------|:---------|:------|
| **无时间压力** | 无限制 | L1/L2/L3 三层都可充分展开 | L2(解答方式变化) 做深 | 杀戮尖塔、构筑游戏 |
| **有时间压力但不大** | 秒级 | L1 频率高，L2 折叠到战前/预读 | L1 × 战前 L2 配置 | 动作 RPG、黑魂 |
| **高时间压力** | 帧级 | L1 高频，L2 执行深度，L3 低频 | L1 频率 × 执行精度 | 格斗游戏、音游 |

### 6.2 L2(解答方式变化) 在高时间压力下的折叠

低时间压力下 L2 是横向展开的（"很多选项可选"）。高时间压力下 L2 的形态发生了变化：

**折叠形式一：预读层**
格斗游戏中，在对手出招前预判他的行动。这不是"宽选择"（你不能临时换选项），而是"深判断"（读心 + 条件反射训练）。L2 被折叠到了"决策树的根"——选择不是在响应时做，而是在响应前就决定了。

**折叠形式二：战前配置层**
角色选择、装备搭配、天赋配置——这些是正常的 L2 空间（"我选择用什么方式战斗"），但一旦选定，战斗中就不会变了。L2 被折叠到了游戏外/局间。

### 6.3 设计指导

```
根据目标游戏的时间压力水平选择 ROS 落地方式：

无时间压力（策略/构筑）
  → L2(解答方式变化) 做深，让玩家尽情探索组合
  → L1(问题变化) 和 L3(规则变化) 辅助丰富体验

有时间压力但不大（动作 RPG）
  → L1(问题变化) 做丰富（多种敌人/情境组合）
  → L2 放在战前准备阶段（角色/装备配置）
  → 战斗中 L2 缩小为即时决策（闪避/格挡/攻击）

高时间压力（格斗/音游）
  → L1 变化频率要高（每 1-3 秒换一个新情境）
  → 每次 L1 变化的幅度要肉眼可辨
  → L2 在战前/角色选择中展开，战斗中 L2 体现在执行精度
```

## 七、对游戏设计的指导意义

### 7.1 设计自检

在设计一款策略/构筑/类Rogue游戏时，用三层模型自问：

| 层次 | 自检问题 | 不通过的表现 |
|:-----|:---------|:------------|
| L1(问题变化) | 玩家每次面对的敌人/地图/事件是否不同？ | 玩家能背板 |
| L2(解答方式变化) | 玩家每次的构筑方向是否有多条有效路径？ | 最优构筑被速通社区锁定 |
| L3(规则变化) | 游戏的底层规则是否会变化？ | 玩家说"这游戏我玩透了" |

### 7.2 设计优先级

```
1. 先确保 L2(解答方式变化)空间够大（这是核心深度的来源）
   → 通过机制轴设计（11-机制轴设计方法.md）撑开构筑多样性
   → 确保选择影响中间层属性，而非孤立的两两配对

2. 再通过 L1(问题变化)的变化来展示 L2(解答方式变化)的不同侧面
   → 不同敌人类型让不同构筑有各自的"高光时刻"

3. 最后用 L3(规则变化)打破天花板
   → 规则变异/外因改变在玩家快要玩透时改变游戏逻辑
```

### 7.3 正反馈公式的应用

```
一局的正反馈 ≈ 最终强度 - 初始强度
类Rogue 的优势在于初始强度极低、最终强度极高——差距大是特征。

设计者应关注的不应是"缩小差距"，而是：
  · 确保玩家每局都能走通"从低到高"的完整曲线
  · 当协同被阻断时，给玩家其他方式去走通（中间层属性/公开式补偿）
  · 不要让任何单一的 L3(规则变化)配置只支持一条路径
```

### 7.4 DDA vs 公开式负反馈调节

在需要"帮助落后玩家"时，优先选择**公开式**而非**隐藏式**：

```
优先选择公开式负反馈调节：
  · 触发条件明确
  · 补偿是一次选择（玩家自主）
  · 补偿伴随新风险（维持张力）
  
避免使用隐藏式 DDA：
  · 玩家归因于"系统让我赢了"
  · 成就感注水
  · 一旦被察觉，信任崩塌
```

### 7.5 三层模型的适用范围

| 品类 | L1(问题变化)适用性 | L2(解答方式变化)适用性 | L3(规则变化)适用性 |
|:-----|:-------------------|:----------------------|:--------------------|
| Roguelike/Roguelite | ✅ 核心 | ✅ 核心 | ✅ 可选（进阶） |
| CRPG | ✅ 核心（不同敌人） | ✅ 核心（不同Build） | 🟡 受限（不能频繁改规则） |
| 策略游戏 | ✅ 核心 | ✅ 核心 | 🟡 每局可改 |
| 动作游戏 | ✅ 核心（不同关卡） | ❌ 弱 | ❌ 弱 |
| 解谜游戏 | ❌ 固定 | ❌ 固定 | ❌ 固定 |
