# 叠层设计与延迟满足

> 标签：`stacking`, `happiness-total`
> 关联概念：`35-协同效应.md`, `37-信号与强化体系.md`

## 定义

将较小的即时奖励累积起来，到达某个阈值时触发一个**显著更大**的奖励或质变效果。

## 核心机制

- **累积**：小奖励逐步叠加
- **阈值**：到达某个点触发质变
- **预期**：玩家对阈值的强烈期待

## 示例

- 技能每次施放永久提升攻击力 → 强化该行为
- 攻击3次后下一次释放强力攻击 → 从攻击行为获得更大正反馈

## 视觉强化

叠层效应可通过视觉/听觉强化（如叠满时手部特效、打击顿帧、火花特效等）进一步放大正反馈。

## 叠层不消耗原则（项目经验）

当卡牌设计中有"成长→爆发"的节奏时，遵循以下原则：

### 核心原则

> 如果一个单位的设计承诺是"越久越强"（叠层/资源成长），则**效果不应消耗叠层**——除非消耗本身就是成长循环的一环。

### 错误示范

> "每2秒获得1层摇摆。消耗所有摇摆层数造成伤害。"
> → 玩家看到"消耗"→叠加的成长瞬间清零→"越久越强"的承诺被打破→负反馈

### 正确做法

> "每2秒获得1层摇摆。根据当前层数造成伤害，不消耗层数。"
> → 叠层只增不减→时间越长冲撞越痛→"越久越强"的承诺兑现

### 什么时候可以消耗叠层？

|         场景         | 例子                 |               原因               |
| :------------------: | -------------------- | :------------------------------: |
| 消耗是正反馈的一部分 | 击杀→重置CD→再杀     |   消耗的是CD冷却，不是成长资源   |
|  消耗后获得更大收益  | 消耗5层→造成10倍伤害 |   消耗换来爆发，玩家有心理预期   |
| 消耗后有快速恢复途径 | 子弹消耗→有单位供弹  | 消耗可被外部补充，不破坏成长预期 |

### 自检问题

> **"叠层被打回原形时，玩家感受到的是'哇，我爆了！'还是'哎，又归零了？'"**

- 前者 → 消耗合理（正反馈爆发）
- 后者 → 不应消耗（成长承诺破裂）

### 成长条件的绑定原则

成长条件必须绑定**玩家可主动干预的行为**，而非纯挂机等待：

|                ❌ 不可接受                |                  ✅ 可接受                   |
| :--------------------------------------: | :-----------------------------------------: |
| "累计完成X场战斗"→玩家无法干预，挂机就行 | "连续存活X回合"→玩家必须主动保护，断链惩罚  |
|          "累计伤害X点"→自动累积          | "每上场X场次→永久成长"→玩家必须主动派它上阵 |

### 项目实践

参考增量自走棋经验（`项目经验/07-增量自走棋-升星机制设计.md`），升星不仅提升数值，更应改变条件阈值和中间层写入效率，让"叠层→质变"的承诺通过多种方式兑现。
