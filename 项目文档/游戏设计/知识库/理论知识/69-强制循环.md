# 强制循环（Compulsion Loop）

> 标签：`player-psychology`, `reinforcement`, `core-loop`, `game-mechanics`
> 关联概念：`37-信号与强化体系.md`, `46-正反馈与负反馈循环.md`, `21-驱动力模型.md`, `08-父系与母系设计哲学.md`, `22-权力下放设计.md`
> 来源：John Hopson, Gamasutra（2001）
> 完整展开见：`游戏设计理论/03-游戏机制设计技法.md §1.11`

## 定义

强制循环（Compulsion Loop）是游戏核心循环（Core Loop）的一种特殊形式——通过精心设计的正反馈节奏，让玩家**不由自主地**持续进行某一行为模式。

### 核心循环 vs 强制循环

| 维度         | 核心循环（Core Loop） | 强制循环（Compulsion Loop） |
| :----------- | :-------------------- | :-------------------------- |
| **目的**     | 提供完整的游戏体验    | 维持玩家的行为频率          |
| **时间跨度** | 贯穿整个游戏          | 每次登录/每局游戏的循环     |
| **驱动力**   | 内驱+外驱混合         | 主要依赖外驱（强化程式）    |
| **玩家感知** | "我想玩"              | "我想再玩一次"              |

## 结构

强制循环由三个环节构成：

```
行为 → 奖励 → 强化 → 行为 → ...
```

### 关键要素

| 要素             | 说明                       | 示例                 |
| :--------------- | :------------------------- | :------------------- |
| **低门槛行为**   | 行为本身简单，无需大量精力 | 点击、滑动、回合结束 |
| **可变比率奖励** | 奖励时间/数量不可预测      | 抽卡、掉落、随机事件 |
| **即时反馈**     | 行为后立即获得反馈         | 动画、音效、数值变化 |
| **渐进目标**     | 始终有一个"下一步"         | 进度条、等级、任务   |

## 与强化程式的配合

强制循环高度依赖于可变比率强化（Variable Ratio Schedule）：
- 行为不一定每次都有奖励
- 但平均而言，投入越多回报越大
- 这与老虎机的机制一致——也是最容易导致"成瘾"行为的方式

## 强制循环 vs 父系思维

强制循环本质上是**父系设计哲学的典型表现**：

> 设计师设计了一条"行为→奖励→行为"的路径，玩家走在上面。

| 维度         | 父系特征                   | 强制循环的体现                             |
| :----------- | :------------------------- | :----------------------------------------- |
| **控制权**   | 设计师掌握节奏控制权       | 循环的频率由设计者预设（冷却/CD/刷新时间） |
| **路径设计** | 单人线性体验，路径不可变更 | 行为→奖励→行为的闭环是固定的               |
| **玩家感知** | 被引导、被安排             | "系统在让我做这个"                         |
| **适用场景** | 线性叙事、引导式教学       | 每日任务、新手引导期、付费转化             |

### 父系强制循环的风险

当强制循环的父系特征过强时：

1. **自主感丧失**（SDT）——玩家感觉被系统操控而非自主选择
2. **过度理由效应**——玩家把行为归因为外驱，一旦外驱消失则停止行为
3. **习得性疲劳**——循环缺乏变化导致边际递减

## 权力下放——对父系强制循环的修正

权力下放是解决强制循环"过度父系"问题的关键手段：

| 权力类型       | 在强制循环中的实践                     | 效果                       |
| :------------- | :------------------------------------- | :------------------------- |
| **游戏节奏权** | 允许玩家自己决定何时进入循环、何时退出 | 消除被强迫感，保护自主感   |
| **体验选择权** | 提供多种循环路径供玩家选择             | 同一循环不同玩法，减少单调 |
| **目标定义权** | 让玩家自定义小目标（自选每日任务）     | 行为归因从外驱转向内驱     |

### 设计实践对比

```
父系强制循环（不推荐）：
    登录→强制每日任务→固定奖励→引导付费→...

母系强制循环（推荐）：
    登录→可选任务列表→自由选择完成→奖励可自选→自然退出
```

**核心原则**：即使使用强制循环来驱动行为，也应通过权力下放让玩家**感觉不到"强制"**。

## 健康设计指南

### 该做的

| 策略               | 说明                               | 对应的设计哲学       |
| :----------------- | :--------------------------------- | :------------------- |
| **有意义的决策**   | 循环中的行为应该需要玩家思考和选择 | 母系——玩家自主决策   |
| **提供主动退出点** | 玩家应该能在达成某个目标后自然停止 | 权力下放——节奏权让渡 |
| **多样化的奖励**   | 避免同一奖励导致边际递减           | 母系——扩展体验宽度   |
| **搭配海明威效应** | 在"还想再玩"时给予积极的停止信号   | 权力下放——积极离场   |

### 不该做的

| 陷阱               | 原因                            | 违背的设计哲学                   |
| :----------------- | :------------------------------ | :------------------------------- |
| 纯随机无策略       | 变成纯赌博机制                  | 父系过度——剥夺玩家的决策意义     |
| 强制循环无终点     | 导致玩家疲劳和习得性无助        | 父系过度——完全剥夺节奏权         |
| 用惩罚驱动行为     | 触发损失厌恶，适得其反          | 父系过度——用强制手段替代内在动机 |
| 消除所有退出的机会 | 违背自主感（SDT），最终引发反感 | 违背所有权力下放原则             |
