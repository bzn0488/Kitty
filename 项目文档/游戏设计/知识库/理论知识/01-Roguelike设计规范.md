# Roguelike设计规范

> 综合 Roguelikeness Factors（Slash, 2014）、Berlin Reloaded（Z, 2026）及现代 Roguelike/Roguelite 分化共识。

## 一、Roguelike 与 Roguelite 的核心分界

现代游戏中，"Roguelike 特征"已分化为两个流派：

| 维度         | Roguelike             | Roguelite                         |
| :----------- | :-------------------- | :-------------------------------- |
| **永久死亡** | ✅ 角色死亡 → 完全重来 | ✅ 角色死亡 → 重来，但保留局外成长 |
| **局外成长** | ❌ 无（每次重新开始）  | ✅ 有元进程（Metaprogression）     |
| **单局时长** | 可长可短，一次性体验  | 通常较短，多次累积                |
| **代表**     | NetHack, DCSS, Brogue | 杀戮尖塔, Hades, 死亡细胞         |
| **类Rogue**  | 属于类Rogue子集       | 也属于类Rogue子集                 |

> **"类Rogue"是统称**：指代包含 Roguelike 特征的**所有游戏**（含 Roguelike 和 Roguelite），不是某一方的专属术语。

常见的混用场景：
- 你说"类Rogue游戏"→ 可能指包含随机+永久死亡机制的广义品类
- 你说"Roguelike" → 应特指传统格式（回合制+网格+无局外成长）
- 你说"Roguelite" → 指轻量化变体（有局外成长）

## 二、Roguelike 为什么让人上瘾——期待感驱动

> 补充来源：千水《游戏创作心灵鸡汤》#4、#6

### 核心发现：多巴胺来源于期待，而非获得

舒尔茨经典实验：给猴子喂糖水前播放提示音 → 猴子习惯后，一听到提示音就分泌多巴胺。**是期待奖励带来多巴胺，而非获得奖励本身。**

这对 Roguelike 设计的意义：

| 机制          | 产生的期待           | 效果                  |
| :------------ | :------------------- | :-------------------- |
| 随机掉落      | "这次会掉什么？"     | 每次开箱/击杀都有期待 |
| 三选一        | "哪张牌能让我变强？" | 每次选择都有兴奋感    |
| 新怪物/新地图 | "下一层有什么？"     | 探索驱动              |
| 稀有事件      | "终于遇到了！"       | 间歇强化，长期粘性    |

> **设计启示**：Roguelike 的随机性首要目的不是"提供多样体验"，而是**持续制造期待感**。多样体验是附带结果，期待感才是核心驱动力。

### Roguelite 解法——缓解"必然的负反馈"

Roguelike 的原罪：**玩家一定会输，一定会有负反馈。**

千水提出的"体验图"模型：
```
玩家强度 ↗
          │ 玩家成长曲线（缓慢上升）
          │
          │ ═══════════════ 游戏难度（恒定直线）
          │
          └──────────────────→ 时间
```
- 初次游玩：游戏难度远高于玩家强度 → 挫败
- 多次游玩后：玩家强度慢慢超过游戏难度 → 才能通关
- **人和人有差异**：有人始终无法通关

**Roguelite 的解决**：加入局外成长（元进程），每次死亡积累资源：
```
玩家强度 ↗
          │ 玩家强度（每次+局外成长）
          │         ↗
          │       ↗    游戏难度（被局外成长抵消，相对下降）
          │     ↗   ═══════════
          │   ↗
          │ ↗
          └──────────────────→ 时间
```
即使学习较慢的玩家，也能通过多次积累最终通关。

## 三、Roguelike 三大高价值因子（Roguelikeness Factors）

> 来源：Slash, Temple of the Roguelike (2014)，已取代 2008 年 Berlin Interpretation

| 因子                       | 说明                                   | 设计意义                             |
| :------------------------- | :------------------------------------- | :----------------------------------- |
| **随机环境生成**           | 地图/物品/怪物位置随机化，提升重玩性   | 固定内容（预设迷宫/剧情）降低重玩性  |
| **永久失败（含永久死亡）** | 角色必须为错误和选择付出代价，甚至死亡 | 存档不可无限读取（可存档但载入即删） |
| **回合制交互**             | 所有实体在循环队列中轮流行动           | 玩家有充分时间决策，不依赖反应速度   |

### 中价值因子

| 因子           | 说明                                                 |
| :------------- | :--------------------------------------------------- |
| **单一命令集** | 任何时候都能使用所有游戏命令，不分战斗/非战斗模式    |
| **自由形式**   | 非线性的推进路径，玩家自己决定做什么、怎么做、何时做 |
| **发现机制**   | 必须研究/探索物品的性质和用途，非直接告知            |

### 低价值因子（可作为参考指标）

| 因子           | 说明                               |
| :------------- | :--------------------------------- |
| 单角色         | 控制单个角色                       |
| 丰富内容       | 足够多的怪物种类和物品类型         |
| 复杂非平凡交互 | 物品有非显而易见的用途             |
| 高陡峭难度     | 上手极难，需要大量经验才能通关     |
| 怪物类似玩家   | 怪物也有装备/属性/AI，遵守相同规则 |
| 字符显示       | 用 ASCII 字符显示                  |
| 砍杀           | 以击杀怪物获取资源为核心循环       |

## 三、Berlin Reloaded 四维度（现代视角）

> 来源：Z, RogueBasin (2026)，作为对 Berlin Interpretation 的修正和补充。**不视为定义，而是作为设计自检框架。**

### 🅰 自由感（Freedom）

| 层面     | 设计要点                                           |
| :------- | :------------------------------------------------- |
| 创造自由 | 玩家与开发者之间无隔阂，可 mod / 改进游戏          |
| 模拟自由 | 不限制玩家能做的事情——改地图、建造事物、做奇怪的事 |
| 自定规则 | 提供挑战/休闲两条路径，不设门槛（Wizard模式）      |
| 无守门   | 免费/低硬件要求/无需游戏技巧                       |

### 🅱 沉浸角色扮演（Immersive Roleplaying）

| 层面                | 设计要点                              |
| :------------------ | :------------------------------------ |
| 想象力驱动          | 符号化呈现 > 写实画面，给想象力留空间 |
| 未知探索            | 你不知道一切——需要学习                |
| 有意义决策          | 每一个游戏内决策都可能是生死的差别    |
| 有意义损失          | 即使输了，你的行为会被记住            |
| 你是Rogue，不是英雄 | 需要用技巧生存                        |

### 🅲 每次新体验（A New Experience Every Time）

| 层面       | 设计要点                         |
| :--------- | :------------------------------- |
| 程序化生成 | 复杂算法创造新体验               |
| 玩家选择   | 玩家自主选择玩法路径             |
| 可通关性   | 不应出现无法通关或过于简单的局面 |

### 🅳 决策密度（Decision Density）

| 层面             | 设计要点                                     |
| :--------------- | :------------------------------------------- |
| 无停机           | 思考即行动——你知道怎么解决就立即执行         |
| 不浪费在假决策上 | 不强迫玩家做无意义或琐碎的、仍耗费时间的选择 |
| 不浪费在呈现上   | 呈现要清晰，动画/效果不拖慢节奏              |
