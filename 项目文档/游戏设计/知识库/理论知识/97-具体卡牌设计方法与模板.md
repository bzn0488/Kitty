# 具体卡牌设计方法与模板

> 调研目标：提供可直接套用的卡牌设计方法、效果模板、数值基准、设计流程，
> 让设计师拿到需求后能快速产出一张"可用"的卡牌。

---

## 一、一张卡牌的构成要素

### 1.1 卡牌的完整结构

```
┌─────────────────────────────────┐
│  [费用] [类型] [稀有度]         │
│                                 │
│        卡牌名称                  │
│         ╱ 插画 ╲               │
│                                 │
│  [关键词]：[数值]               │
│  [效果描述文字]                  │
│  "背景/风味文字"                │
│                                 │
│  [标签/Tag] [所属角色/阵营]    │
│  卡牌编号 #001                  │
└─────────────────────────────────┘
```

### 1.2 每个字段的设计规范

| 字段         | 用途                 | 设计规范                                                                            |
| ------------ | -------------------- | ----------------------------------------------------------------------------------- |
| **费用**     | 打出这张卡需要的资源 | 确定费用模型（能量/行动点/时间）。参考 `93-构筑多样性与平衡性的冲突分析` 的费用基准 |
| **类型**     | 卡牌的功能分类       | 攻击/技能/能力/诅咒/状态。每类有自己的规则约束                                      |
| **稀有度**   | 出现频率和复杂度     | 普通=1行效果，稀有=1-2行，史诗=2行+独特机制，传说=3行+改变玩法                      |
| **卡名**     | 卡牌的标识           | 4-7字为佳。能传达效果意图。避免生僻字                                               |
| **关键词**   | 标准化效果缩写       | 每游戏建议5-10个关键词。关键词=减少文字量+增加一致性                                |
| **效果描述** | 卡牌的具体效果       | 使用标准句式："造成X点伤害""获得X点护甲""抽X张牌"                                   |
| **数值**     | 效果的量化部分       | 参考第§三节的数值基准表                                                             |
| **风味文字** | 叙事/背景点缀        | 1行，斜体。不参与游戏逻辑。纯氛围                                                   |
| **标签**     | 用于协同/检索        | 给卡牌打上分类标签（"火""冰""斩击""魔法"）                                          |
| **归属**     | 属于哪个角色/阵营    | 多角色游戏中标注所属角色                                                            |

## 二、效果设计的标准句式

### 2.1 标准效果动词库

| 动词     | 含义               | 用例                   | 变体                 |
| -------- | ------------------ | ---------------------- | -------------------- |
| **造成** | 对目标产生伤害     | "造成6点伤害"          | 造成X点伤害→目标     |
| **获得** | 给自己添加增益     | "获得5点护甲"          | 获得X点→某种资源     |
| **施加** | 给敌人添加减益     | "施加2层虚弱"          | 施加X层→某种状态     |
| **抽取** | 从牌库拿牌到手牌   | "抽2张牌"              | 抽取X张→来源         |
| **回复** | 恢复生命/资源      | "回复3点生命"          | 回复X点→资源类型     |
| **减少** | 降低费用/数值      | "本回合下一张牌费用-1" | 减少X→目标           |
| **复制** | 生成卡牌的副本     | "复制一张手牌"         | 复制→目标            |
| **消耗** | 使用后从游戏中移除 | "消耗。"               | 永久移除，不进弃牌堆 |
| **升级** | 永久强化一张卡     | "升级一张手牌"         | 升级→目标            |

### 2.2 标准效果句式模板

```
句式1：即时效果
  "造成X点[伤害类型]伤害。"
  → "造成8点物理伤害。"
  → "造成12点火焰伤害。"

句式2：条件效果
  "如果[条件]，[效果]。"
  → "如果目标有中毒状态，伤害翻倍。"
  → "如果你的护甲≥10，造成额外5点伤害。"

句式3：复合效果
  "[效果A]。然后[效果B]。"
  → "造成6点伤害。然后抽1张牌。"
  → "获得4点护甲。如果本回合没有受到伤害，再获得4点护甲。"

句式4：持续效果/能力
  "每回合[触发条件]，[效果]。"
  → "每回合第一次受到伤害时，获得等量护甲。"
  → "每回合结束时，回复2点生命。"

句式5：选择效果
  "选择一项：[选项A]或[选项B]。"
  → "选择一项：造成8点伤害；或回复6点生命。"
  → "选择一项：所有敌人受到5点伤害；或所有友方获得3点护甲。"
```

### 2.3 效果描述的清晰度检查

```
✅ 好的效果描述：
  "造成6点伤害。每有1点力量，额外造成1点伤害。"
  → 清晰：玩家知道基础值 + 倍率

❌ 差的效果描述：
  "根据你的力量造成伤害。"
  → 模糊：到底造成多少？玩家算不出来

✅ 好的效果描述：
  "获得等于你手牌数量的护甲。"
  → 清晰：玩家可以算出来

❌ 差的效果描述：
  "获得一些护甲。"
  → 模糊："一些"是多少？
```

## 三、数值基准表

### 3.1 通用数值基准（适用于大多数卡牌游戏）

以下基准假设"1费 ≈ 造成6点伤害"作为基准线：

```
伤害基准：
  1费：单体6-9伤害 / 群体3-5伤害
  2费：单体12-16伤害 / 群体6-9伤害
  3费：单体18-25伤害 / 群体10-14伤害
  0费：2-4伤害（功能性为主）

护甲/格挡基准：
  1费：获得5-8护甲
  2费：获得10-15护甲
  3费：获得16-22护甲

治疗基准（≈伤害的70%-80%）：
  1费：回复4-6生命
  2费：回复8-12生命
  3费：回复14-18生命

抽牌基准：
  1费：抽1-2张牌（抽2通常有额外限制）
  2费：抽2-3张牌
  0费：抽1张牌（通常附带条件或负面效果）

减费基准：
  1费的价值 ≈ 让下一张卡费用-1 或 -2
  为强力卡减费的效果应该更昂贵
```

### 3.2 时间轴战斗的数值调整（针对荒野与曙光）

在时间轴模型中，时间成本 = 费用 × 怪物预期伤害。需要调整基准：

```
时间轴模型的伤害基准（假设怪物预期伤害=5）：
  0费：2-3伤害（纯白嫖，必须限制）
  1费：8-10伤害（需要超过时间成本5 + 至少2点净收益）
  2费：14-18伤害（2次怪物行动成本10 + 净收益）
  3费：22-28伤害（3次怪物行动成本15 + 净收益 + 补偿机制）

关键变化：时间轴模型中的"净收益"比传统模型低，
          因为费用本身也"允许怪物行动"。
```

### 3.3 模拟经营卡牌的数值基准（针对家族纪元）

模拟经营卡牌的效果不是"伤害"，而是资源调整。基准不同：

```
金钱调整：
  1费：+10-20金钱（或 -5-10敌方金钱）
  2费：+25-40金钱（或 -15-25敌方金钱）
  3费：+50-80金钱（全局性调整）

人口调整：
  1费：+1-2人口（或生育率+20%持续3回合）
  2费：+3-4人口（或解锁特定职业人口上限）
  3费：全局性人口调整（移民潮/战争损耗）

效率调整（百分比）：
  1费：+10-15%某资源产出（持续3-5回合）
  2费：+20-30%某资源产出（持续3-5回合）
  3费：全局性效率调整（+50%但附带负面效果）

注意：模拟经营的"一回合"概念不同于战斗——
      资源调整的持续时间以"年/朝代阶段"为单位
```

## 四、类杀戮尖塔游戏的卡牌设计准则

### 4.1 StS 卡牌类型学：四种功能定位

杀戮尖塔的卡牌按功能可归为四种类型，每种类型有自己的设计规则：

| 类型                 | 占比 | 功能                | 价值判断标准                       |
| -------------------- | ---- | ------------------- | ---------------------------------- |
| **攻击卡（Attack）** | ~35% | 造成伤害            | 伤害/费用比、多段收益、条件增伤    |
| **技能卡（Skill）**  | ~45% | 防御/抽牌/控制/复合 | 资源转化效率、条件触发收益         |
| **能力卡（Power）**  | ~15% | 整场战斗的持续效果  | 能否在打出回合产生价值？启动速度？ |
| **诅咒卡（Curse）**  | ~5%  | 负面效果（占牌位）  | 是否有趣味性或叙事价值？           |

**每张卡必须且只能属于一个类型**，类型决定了它的基础规则：
- 攻击卡：受力量/脆弱等状态影响，触发"打出一张攻击卡"类遗物
- 技能卡：不受力量影响，触发"打出一张技能卡"类遗物
- 能力卡：打出后进入"能力栏"，永久生效，不可收回

**对荒野与家族的参考**：同样需要明确卡牌类型分类——每张卡只能属于一类，类型决定其交互规则。

### 4.2 StS 卡牌设计的十条隐形规则

```
规则1：攻击卡的基础伤害基准
  1费单体攻击卡的标准伤害 = 6（"打击"）
  升级后 1费 = 9（+50%，这个增幅比例是 StS 的升级基准）
  设计原则：让玩家一眼看出"这张卡比打击强"即可

规则2：AOE 伤害约为单体的 60-70%
  1费单体 = 6 → 1费全体 = 4-5
  2费单体 = 12 → 2费全体 = 8-10
  规则：AOE 的效率低于单体，但处理多目标时总收益更高

规则3：护甲 ≈ 伤害 × 0.85
  1费伤害 6 → 1费护甲 5
  2费伤害 12 → 2费护甲 10
  原因：护甲不会过量（不会超出需求），伤害会
  例外：有叠甲协同的卡可以打破此基准

规则4：抽牌本身有价值但不直接增强
  1费抽2张 = 标准过牌（不直接产生场面效果）
  抽牌的价值体现在"让你拿到更好的牌"，而不是牌本身变强
  设计原则：纯抽牌卡的费用 ≈ 抽牌数 × 0.5（1费抽2，2费抽3-4）

规则5：能力卡必须在打出回合有"感觉"
  能力卡的问题是：打出花了一回合，但效果要后续回合才体现
  解决方案：能力卡打出时附带一个即时效果（"造成3点伤害并能力：每回合..."）
  杀戮尖塔的"偏差认知"就是这个设计的反面教材——打出回合无即时效果

规则6：高费卡（2-3费）需要"即时价值"
  在 StS 的 3 费系统中，打出一张 3 费卡意味着当回合只能做这一件事
  因此 3 费卡必须产生足以改变战局的效果
  杀戮尖塔的"回收"（3费费用型）很弱就是因为不满足这条规则

规则7：条件触发的难度决定倍率
  容易触发的条件（如有护甲）：1.2-1.5 倍效果
  中等条件（如生命<50%）：1.5-2 倍效果
  困难条件（如手牌为0）：2-3 倍效果
  极难条件（如场上无其他单位）：3 倍以上效果

规则8：每张卡在"三选一"中应该有自己的读者
  核心测试：这张卡在什么构筑路线中最有价值？
  如果一张卡在所有构筑路线中价值相同 → 它是"通用强卡"（危险）
  如果一张卡只在一两条路线中价值极高 → 它是"Build定义卡"（优秀）
  如果一张卡在任何路线中价值都不高 → 它是"诅咒卡"（不合格）

规则9：删牌的"反面设计"——初始卡
  初始卡（打击/防御）的设计目的是被删掉
  它们的强度故意设为基础值（1费6伤/1费5甲）
  所有非初始卡都应该比初始卡强（或提供初始卡没有的功能）
  设计测试：如果一张卡不如初始卡→它没有存在价值

规则10：协同强度与触发条件成正比
  强协同（组合技）需要满足 2+ 条件
  中协同（条件触发）需要满足 1 个条件
  弱协同（标签聚合）无条件，但效果有限
  设计原则：不要让玩家"无意中"触发强协同
```

### 4.3 DBG 构筑中的卡牌角色分类

在 DBG（牌库构筑游戏）中，每张卡在构筑中扮演特定角色：

| 角色       | 定义               | 数量占比 | 设计要点                                   |
| ---------- | ------------------ | -------- | ------------------------------------------ |
| **核心卡** | 构筑路线的主引擎   | 3-5 张   | 效果独特，不可替代；Build 信号最强         |
| **支撑卡** | 配合核心卡发挥     | 5-8 张   | 与核心卡有明确协同；可被同类替代           |
| **过渡卡** | 前期填充，后期删掉 | 0-5 张   | 强度一般，但前期需要；有明确的"被替换窗口" |
| **功能卡** | 解决特定问题       | 1-3 张   | 针对性强（如应对特定Boss）；非必需         |
| **必拿卡** | 任何构筑都值得拿   | —        | ⚠️ 危险信号——如果存在必拿卡说明平衡有问题   |

**设计含义**：一张卡应该能对应到上面的某个角色。如果一张卡"哪个角色都不是"——它没有设计目标。

### 4.4 稀有度与卡牌复杂度的匹配

StS 的稀有度系统本质上是**复杂度管理工具**而非强度标尺：

```
普通（Common）：
  - 效果：一行文字搞定
  - 复杂度：低
  - 作用：填充费用曲线，提供基础功能
  - 例子："造成8点伤害"
  - 设计规则：不需要玩家"学习"就能正确使用

稀有（Uncommon）：
  - 效果：一行半到两行
  - 复杂度：中
  - 作用：构筑路线的核心组件
  - 例子："造成10点伤害。如果敌人有中毒，额外造成5点伤害。"
  - 设计规则：需要玩家理解触发条件

史诗/稀有（Rare）：
  - 效果：两行以上或改变玩法
  - 复杂度：高
  - 作用：Build定义器或终结手段
  - 例子："能力 - 每回合第一次打出一张攻击卡时，复制它。"
  - 设计规则：玩家需要围绕这张卡重新规划构筑
```

**核心约束**：普通卡的数量应占 50-60%，稀有 25-30%，史诗 10-15%。如果高稀有度卡比例过高，玩家在构筑中难以形成"找到核心卡"的期待感。

### 4.5 卡牌设计中的"费用分布"策略

StS 四个角色的卡牌费用分布：

```
铁甲战士：偏向 1-2 费（简单直接）
静默猎手：偏向 0-1 费（高速过牌）
故障机器人：偏向 1-3 费（高费强力卡多）
观者：极端 1 费（灵活切换姿态）

原则：每个角色的费用分布应该不同。
      如果所有角色的费用分布一样 → 角色失去差异化。
```

**给荒野的参考**：如果你的游戏有多个角色/职业，他们的卡牌费用分布应该不同。一个角色的特色可以通过"她擅长什么费用段"来定义。

### 4.6 "协同卡"的设计准则

协同卡是最容易出问题的地方。设计协同卡的四个准则：

```
准则1：协同应该是"奖励"而非"必需"
  ✅ "造成6点伤害。如果敌人有中毒，额外造成6点伤害。" 
     → 不触发也有基础值，触发是奖励
  ❌ "如果敌人有中毒才能造成伤害"
     → 没有中毒就是废卡，玩家不敢拿

准则2：协同链不应超过2层
  ✅ 卡A→卡B（一层协同）
  ❌ 卡A→卡B→卡C→卡D（四层协同，条件太多）

准则3：协同的触发条件应"自然发生"
  ✅ "每回合第一次受伤时获得 3 点护甲"（自然会受伤）
  ❌ "每回合恰好打 3 张牌时获得 5 点护甲"（需要精确数牌）

准则4：不要让玩家为了触发协同而做"反直觉操作"
  ✅ 玩毒的角色自然会堆毒 → 毒相关协同是自然的
  ❌ 一个攻击型角色需要"不打攻击卡"来触发协同 → 反直觉
```

### 4.7 类 StS 卡牌的"数字语言"

StS 形成了一套玩家熟悉的数字语言——某些数值有特定的含义：

```
数字 6 = 基准（1费单体攻击的标准值）
数字 8 = ⭐ 升级后的标准值（6→8，不是9，实际 StS 是 6→9）
数字 5 = 基准护甲（1费）
数字 10 = 某种门槛/里程碑
数字 2 = 最常见的时间/次数（持续2回合、造成2次）
数字 3 = 三选一的标准、每回合的基础能量
数字 1 = 抽牌的最小单位

设计启示：使用这些"通用数字"可以降低玩家的认知成本。
          玩家看到"造成6点伤害"就知道"这是一张标准攻击卡"。
          看到"造成10点伤害"就知道"这张卡比标准强"。
```

### 4.8 卡牌设计的"一句话测试"

```
每张卡设计完成后，问自己：
"玩家在一句话内能说清楚这张卡的价值吗？"

✅ 通过的例子：
  "这张卡是标准输出" → 1费6伤攻击卡
  "这张卡是毒体系的触发器" → 叠毒相关
  "这张卡是保命用的" → 护甲/回复
  "这张卡是组合技的核心" → 需要配合

❌ 不通过的例子：
  "这张卡...呃...在特定情况下...如果...可能..." 
  → 玩家说不清它的价值 → 说明设计目标不清晰
```

### 4.1 从概念到成卡

```
步骤1：确定卡牌定位
  □ 这张卡属于哪个角色/阵营？
  □ 这张卡支持哪种构筑路线？
  □ 这张卡的"Build信号"是什么？

步骤2：选择费用和效果类型
  □ 确定费用段（0/1/2/3费）
  □ 选择效果类型（伤害/防御/抽牌/控制/复合）
  □ 选择是否有条件/协同

步骤3：填入数值
  □ 参考数值基准表填入初步数值
  □ 检查：这个数值是否值得这个费用？
  □ 检查：这个数值在同类卡中是否合理？

步骤4：写效果描述
  □ 使用标准句式模板
  □ 确保描述清晰无歧义
  □ 写风味文字（可选）

步骤5：自检
  □ Build信号自检："三选一里看到这张卡，玩家知道往什么方向构筑吗？"
  □ 费用自检："这张卡的效果是否值得它的费用+时间成本？"
  □ 自闭环自检："移除其他所有卡后，这张卡还能工作吗？"
  □ 诅咒卡自检："如果三选一里只有这张，我会选它吗？"
  □ 功能重叠自检："已经有卡能更好地完成同样功能吗？"
```

### 5.2 设计中的常见决策

```
Q: 一张卡同时做两件事（伤害+抽牌），费用怎么定？
A: 复合效果 = 主效果费用 + 副效果费用 × 0.6-0.8（打折）
   "造成6点伤害 + 抽1张牌" ≈ 1费（伤害6=1费 + 抽牌≈0.8费 × 0.7=0.56 → 取整1费）

Q: 条件触发卡比无条件卡强多少？
A: 条件卡的触发难度决定强度倍率：
   容易触发（如"如果有护甲"）：1.2-1.5倍
   中等触发（如"如果生命<50%"）：1.5-2倍
   困难触发（如"如果手牌为0"）：2-3倍

Q: 0费卡应该怎么设计？
A: 0费卡的四个安全设计原则：
   1. 效果值低（伤害≤3）
   2. 附带条件（"如果XXX，则XXX"）
   3. 功能性为主（过牌/过滤/转换）
   4. 有隐性成本（占用抽牌位/手牌位）
```

## 六、卡牌效果模板库

### 5.1 攻击类模板

```
#1 标准攻击
  [费用]：造成X点伤害。
  例 - 1费：造成8点伤害。

#2 多段攻击
  [费用]：造成X次Y点伤害。
  例 - 2费：造成3次4点伤害。
  设计要点：多段=享受多次力量加成，但被护甲多次抵消

#3 条件增伤
  [费用]：造成X点伤害。如果[条件]，额外造成Y点伤害。
  例 - 1费：造成5点伤害。如果目标有中毒，额外造成5点伤害。

#4 连击（本回合使用多张卡后触发）
  [费用]：造成X点伤害。本回合每打出过一张其他卡，额外造成Y点伤害。
  例 - 2费：造成8点伤害。本回合每打出过一张其他卡，额外造成2点伤害。

#5 穿透（无视护甲）
  [费用]：造成X点穿透伤害。
  例 - 2费：造成10点穿透伤害。
```

### 6.2 防御类模板

```
#6 标准护甲
  [费用]：获得X点护甲。
  例 - 1费：获得6点护甲。

#7 条件护甲
  [费用]：获得X点护甲。如果[条件]，额外获得Y点护甲。
  例 - 2费：获得8点护甲。如果处于防御姿态，额外获得6点护甲。

#8 伤害转化
  [费用]：本回合受到的下X次伤害减Y。
  例 - 1费：本回合受到的下2次伤害减少3点。
```

### 6.3 抽牌/过牌类模板

```
#9 标准抽牌
  [费用]：抽X张牌。
  例 - 1费：抽2张牌。

#10 选择性抽牌
  [费用]：从牌堆顶X张牌中选择Y张加入手牌，其余弃掉。
  例 - 1费：从牌堆顶3张牌中选择1张加入手牌，其余弃掉。

#11 弃抽（弃牌换抽牌）
  [费用]：弃掉X张牌，然后抽Y张牌。
  例 - 1费：弃掉1张牌，然后抽2张牌。

#12 回收（从弃牌堆回到手牌）
  [费用]：从弃牌堆中选择X张[类型]牌回到手牌。
  例 - 1费：从弃牌堆中选择1张攻击牌回到手牌。
```

### 6.4 控制/减益类模板

```
#13 减攻/减防
  [费用]：施加X层[减益]，目标[效果]。
  例 - 1费：施加2层虚弱，目标造成的伤害减少50%。

#14 延迟
  [费用]：目标的下一次行动延迟X回合（时间轴战斗中）。
  例 - 2费：目标怪物下次行动延迟1回合。

#15 沉默/打断
  [费用]：取消目标的[指定行动]。
  例 - 1费：取消怪物的蓄力攻击。
```

### 6.5 持续效果类模板

```
#16 能力（整场战斗持续）
  [费用]：能力 - 每回合[触发条件]，[效果]。
  例 - 2费：能力 - 每回合第一次打出攻击牌时，抽1张牌。

#17 临时buff（持续X回合）
  [费用]：在X回合内，[效果]。
  例 - 1费：在2回合内，每次攻击额外造成2点伤害。

#18 光环（全场持续）
  [费用]：所有友方/敌方[效果]。
  例 - 2费：所有友方每回合回复2点生命。
```

## 七、卡牌设计实战案例

### 7.1 案例1：为荒野与曙光设计一张卡

```
需求：需要一张2费攻击牌，适合"速攻型"构筑路线

步骤1：定位
  - 归属：输出型角色
  - 构筑路线：速攻型（低费多段）
  - Build信号："这张卡鼓励多段攻击"

步骤2：费用+效果
  - 2费（时间成本=10，需要产生>10价值）
  - 多段攻击（2次 × 7伤害 = 14伤害 > 10 → ✅）
  - 附加条件触发（与速攻型构筑的"多段"主题一致）

步骤3：数值填入
  - 2费多段：造成2次7点伤害 = 14总伤害
  - 时间成本10 → 净收益4（合理）

步骤4：效果描述
  "造成2次7点伤害。如果本回合已经打出过2张其他卡，额外造成1次7点伤害。"

步骤5：自检
  - Build信号：✅ "速攻型玩家看到后脑补：打很多卡→多段触发"
  - 费用自检：✅ 14-18伤害在2费基准内。条件触发后21伤害
  - 自闭环：✅ 无条件也可工作（2×7=14），条件触发更赚
  - 诅咒卡自检：✅ 就算不触发条件，14伤害也是合格的2费卡
  - 功能重叠：✅ 独特的多段+条件连击定位
```

### 7.2 案例2：为家族纪元设计一张卡

```
需求：需要一张1费的事件卡，体现"摊丁入亩"历史事件

步骤1：定位
  - 归属：清朝时期事件卡
  - 效果定位：经济调整（税收改革）
  - 叙事目标：让玩家通过机制感受到"摊丁入亩"的历史意义

步骤2：费用+效果
  - 1费（家族纪元的费用模型：1费=轻度经营调整）
  - 资源调整型（金钱+人口的双重影响）

步骤3：数值
  - 金钱：+25（正面效果）
  - 人口：生育率-20%持续5回合（负面效果，反映改革阵痛）

步骤4：效果描述
  "摊丁入亩：金钱+25。生育率-20%，持续5回合。"
  风味文字："将丁银并入田赋，无田者不再纳丁税。"

步骤5：自检
  - Build信号：✅ "经济型玩家看到→这是金钱路线"
  - 费用自检：✅ +25金钱≈1费价值，-20%生育率是合理代价
  - 叙事一致：✅ 机制反映历史——增加税收但人口增长放缓
```

## 十六、卡牌设计进阶技巧

### 16.1 数字的选择心理

| 数字      | 心理效果        | 设计建议                             |
| --------- | --------------- | ------------------------------------ |
| **3**     | 小、安全、基础  | 用于低费卡、触发次数                 |
| **5**     | 中等、标准      | 最安全的"基础数值"                   |
| **6**     | 比5好一点但不多 | 常见的基础伤害值                     |
| **7**     | 幸运感          | 适合暴击/额外效果                    |
| **8**     | 扎实            | 中等费用的标准值                     |
| **10**    | 里程碑感        | 高费卡的基准值、整十产生"很强"的感觉 |
| **12/13** | 超过10的突破感  | 2费卡的强效区间                      |

### 16.2 卡牌设计的简化原则

```
每张卡最多做"两件事"：
  1件事（推荐）："造成8点伤害。" → 简单直接
  2件事（可接受）："造成6点伤害。抽1张牌。" → 复合但不复杂
  3件事（避免）："造成6点伤害。抽1张牌。如果XXX再XXX。" → 太复杂

每张卡的效果描述不超过2行：
  超过2行 → 玩家需要在战斗中阅读 → 打断节奏
  解决方案：使用关键词压缩文字量
```

### 16.3 关键词系统的设计

关键词是压缩效果文字量的工具。每游戏建议5-10个关键词：

```
关键词设计原则：
  1. 关键词应该在游戏中出现5次以上（否则加关键词不如直接写文字）
  2. 关键词应该有统一的视觉标记（粗体/颜色/图标）
  3. 玩家应该能凭直觉理解关键词的含义

关键词示例（适用于荒野与曙光）：
  连击：本回合每打出过一张其他卡，额外触发一次效果
  穿透：无视护甲造成伤害
  延时：目标的行动在时间轴上延后X格
  拼刀：本卡参与拼刀判定时，拼刀值+X
  充能：使用后进入冷却，冷却结束后可再次使用
```

## 十一、卡牌升级设计

### 11.1 StS 升级系统的设计哲学

杀戮尖塔的升级系统（篝火 Smith）是 DBG 卡牌设计的精髓之一。核心哲学：

```
升级 ≠ 加强，升级 = 改变选择

杀戮尖塔的升级原则：
  一张卡升级后，它在构造中的"角色"应该改变或强化。
  升级不是简单地"数值+50%"，而是让玩家做出新的选择。
```

### 11.2 升级的七种模式

| #     | 模式             | 原理                               | StS 例子              | 适用场景         |
| ----- | ---------------- | ---------------------------------- | --------------------- | ---------------- |
| **1** | **数值提升**     | 伤害/护甲/治疗量按比例增加         | "造成6→9伤害"（+50%） | 攻击卡、防御卡   |
| **2** | **费用降低**     | 升级后费用-1                       | 完美打击 2→1费        | 高费卡的核心升级 |
| **3** | **条件移除**     | 升级后移除触发条件                 | "如果有中毒"→无条件   | 条件卡的升级奖励 |
| **4** | **效果增加**     | 升级后增加额外效果                 | 抽X→抽X+1张           | 过牌/功能性卡    |
| **5** | **目标扩展**     | 单体→全体                          | 单一目标→所有敌人     | AOE 的升级路径   |
| **6** | **持续时间延长** | 持续X→X+1回合                      | 缓冲 2→3回合          | buff/debuff 卡   |
| **7** | **固有**         | 升级后获得"固有"（战斗开始在手中） | 多种能力卡            | 能力卡/核心卡    |

### 11.3 升级的数值基准（StS 标准）

```
StS 标准升级幅度：
  攻击卡：伤害 +50%（如6→9，8→12）
  护甲卡：护甲 +33%~+50%（如5→8，10→14）
  抽牌卡：抽牌数 +1（如抽2→抽3）
  能力卡：获得"固有" 或 费用-1
  持续效果：持续回合 +1

设计原则：
  升级应该让一张"可用"的卡变成"好用"的卡
  不应该让一张"废卡"通过升级变成"好卡"（废卡该删掉）
  升级后不应该改变卡牌的核心功能（升级不是重做）
```

### 11.4 升级自检

- [ ] 升级前后的差异是否直观可见？
- [ ] 升级奖励是否与卡牌的核心功能一致？
- [ ] 升级后的卡是否会让玩家重新考虑构筑选择？
- [ ] 升级补偿是否与费用相符？（2费→1费是强升级，应该对应弱效果）
- [ ] 是否存在"不升级就不能用"的卡？（如果有，说明设计本身有问题）

## 十二、状态/减益系统设计

### 12.1 常见的状态效果类型

| 类型           | 例子                              | 效果                 | 叠加方式       | 持续时间   |
| -------------- | --------------------------------- | -------------------- | -------------- | ---------- |
| **属性调整**   | 力量/敏捷                         | 永久改变基础属性     | 加法叠加       | 整场战斗   |
| **百分比调整** | 虚弱（-50%伤害）/易伤（+50%受伤） | 百分比乘区           | 不叠加（刷新） | 有限回合   |
| **层数累积**   | 中毒/毒                           | 每回合按层数造成伤害 | 加法叠加→消耗  | 直到被净化 |
| **状态标记**   | 标记/锁定                         | 作为其他效果的触发器 | 不叠加         | 有限回合   |
| **控制**       | 眩晕/冰冻                         | 跳过行动             | 不叠加         | 有限回合   |

### 12.2 状态效果的强度基准

```
伤害类状态（毒/灼烧等）：
  1费标准：施加3-4层
  2费标准：施加6-8层
  3费标准：施加10-14层
  层数伤害 ≈ 每层每回合造成 1-2 点伤害

百分比减益（虚弱/易伤）：
  标准数值：±50%
  设计原则：50% 是一个关键的"感受"阈值——玩家能明显感受到差异
  如果低于 25%，玩家感觉不到 → 效果太弱
  如果高于 75%，过于强力 → 变成"不得不带"

叠加规则：
  同来源不叠加（刷新持续时间）
  不同来源可叠加（上限通常 2-3 层）
  百分比乘区之间加法叠加（×50% + ×50% = ×100%）
```

### 12.3 状态效果的设计准则

```
准则1：每种状态应该有明确的"对策"
  ✅ 中毒 → 有解毒手段/净化卡
  ✅ 虚弱 → 可以等它过期/有消除手段
  ❌ 一种状态只能硬扛 → 玩家感觉"不公平"

准则2：百分比效果不要超过 75%
  超过 75% 的减伤 ≈ 免疫，让玩家/敌人无敌
  超过 75% 的易伤 ≈ 一碰就碎

准则3：层数累积状态需要"质变节点"
  毒层数到某个值时"毒爆"触发额外效果
  让层数累积不只是线性收益，还有阶段性奖励

准则4：负面状态在玩家手上和敌人手上应该不对称
  玩家对敌人使用的效果：可以强一些（需要消耗卡牌和费用）
  敌人对玩家使用的效果：应该弱一些（或者有应对空间）
```

## 十三、卡牌池的整体规划

### 13.1 卡牌池的规模规划

```
卡牌池规模建议：
  小型游戏（如小型独立游戏）：30-50 张（够用）
  中型游戏（如大多数 StS-like）：75-150 张（丰富）
  大型游戏（如杀戮尖塔）：每个角色 75+ 张，总计 300+

卡牌池的构成比例：
  攻击卡：30-35%
  技能卡（防御/抽牌/控制）：45-50%
  能力卡（持续效果）：10-15%
  诅咒卡/特殊卡：5-10%
```

### 13.2 费用分布的宏观规划

```
以 100 张卡的中型卡牌池为例：

0费卡：8-12 张（10%）
  - 功能性为主，伤害类不超过 3 张
  - 用于填充低费曲线，创造操作感

1费卡：35-40 张（40%）
  - 主体战力，每局游戏的主力
  - 覆盖攻击/防御/抽牌各类功能

2费卡：30-35 张（35%）
  - 核心战术卡，决定构筑方向
  - 效果比 1 费卡更强但不只是数值翻倍

3费卡：10-15 张（15%）
  - 终结技/强力组合卡
  - 必须有补偿机制或独特效果

稀有度分布（与费用交叉）：
  普通：50-60 张（费用覆盖 0-2，主要是 1 费）
  稀有：25-35 张（费用覆盖 1-3，主要是 2 费）
  史诗：10-15 张（费用覆盖 2-3，主要是高费）
```

### 13.3 构筑路径的数量规划

```
构筑路径数 = 卡牌池中可识别的"Build方向"数

建议：
  小型（30-50张）：2-3 条构筑路径
  中型（75-150张）：3-5 条构筑路径
  大型（300+张）：5-8 条构筑路径

每条构筑路径至少需要的支撑卡牌：
  核心卡（定义路线）：2-3 张
  支撑卡（配合核心）：5-8 张
  过渡卡（前期填充）：3-5 张
  合计每条路线：10-15 张支撑

卡牌池 = 各路线支撑卡 × 路线数 + 通用卡 + 填充卡
  例：3 条路线 × 12 张/路线 = 36 张支撑卡
      + 20 张通用卡 + 10 张填充卡 = 66 张（小型池）
```

### 13.4 卡牌池规划自检

- [ ] 每条构筑路径至少有 10 张支撑卡？
- [ ] 各费用段的卡牌比例是否合理？（0费<15%，1费~40%，2费~35%，3费~15%）
- [ ] 各稀有度的卡牌比例是否合理？（普通~55%，稀有~30%，史诗~15%）
- [ ] 是否存在某个费用段"没有可选卡"？
- [ ] 各构筑路径之间是否有重叠卡？（允许玩家中途换路线）
- [ ] 是否有足够的功能卡应对特殊场景（Boss/AOE/续航）？

## 十四、费用模型的变体设计

### 14.1 五种费用模型的对比

| 模型           | 代表游戏   | 核心机制           | 策略特征                | 适用场景     |
| -------------- | ---------- | ------------------ | ----------------------- | ------------ |
| **固定能量**   | 杀戮尖塔   | 每回合固定 3 能量  | 每回合决策"怎么花 3 费" | 标准卡牌战斗 |
| **增长水晶**   | 炉石传说   | 每回合 +1，上限 10 | 节奏规划（前期/后期）   | 天梯对战     |
| **时间轴推进** | 荒野与曙光 | 费用=推进回合数    | 高费卡=让怪物行动       | 时间轴战斗   |
| **行动点**     | 欺诈之地   | 每回合 3-4 行动点  | 任意分配，无卡牌费用    | 灵活组合     |
| **空间格子**   | 背包英雄   | 背包位置 = 费用    | 空间管理 = 资源管理     | 物品管理     |

### 14.2 时间轴费用模型（荒野）的卡牌设计特例

时间轴模型最大的特殊性：**费用每张卡单独支付，不共享资源池**。

这对卡牌设计的含义：

```
传统模型：每回合 3 费 → 打 1 张 3 费卡或 3 张 1 费卡
时间轴模型：打 1 张 3 费卡（怪物动 3 次）或 3 张 1 费卡（怪物动 3 次）
→ 总时间成本相同，但节奏不同

时间轴模型的节奏差异：
  3 张 1 费卡：3 次小效果 + 怪物行动分散在 3 段等待期（可控）
  1 张 3 费卡：1 次大效果 + 怪物集中行动 3 回合（风险集中）

设计影响：
  - 高费卡必须比"3 张 1 费卡"打出更好的效果（否则没人用）
  - 但具体好多少？≈ 高出 20-30%，因为高费卡有"风险集中"的隐形成本
  - 0 费卡的价值被系统性放大（见 93 号文档）
```

### 14.3 混合模型的设计思路

如果你的游戏有多种玩法模式（战斗/探索/经营），可以考虑混合模型：

```
荒野的可能方案：
  战斗内：时间轴费用模型（费用=推进时间）
  战斗外（探索/领地）：传统能量模型（每回合/每行动固定资源）

家族纪元的可能方案：
  家族经营：模拟经营型费用（每朝有固定"行动额度"）
  事件处理：卡牌消耗型费用（打事件卡消耗"影响力"资源）

混合模型的设计原则：
  1. 不同模式下使用不同的费用逻辑
  2. 模式之间通过资源转换连接（战斗外获取资源→战斗内消耗）
  3. 不要让玩家同时管理两套完全独立的费用系统
```

## 十五、卡牌设计进阶话题

### 15.1 卡牌与外部系统的交互

卡牌不是孤立的——它们需要与游戏的战斗系统、角色系统、资源系统交互。

```
卡牌与外部系统交互的类型：

1. 卡牌 ↔ 角色技能
   角色技能影响卡牌效果（如"本角色打出的攻击卡伤害+2"）
   设计：技能的效果应该可被玩家感知到（+2 伤害能改变斩杀线）

2. 卡牌 ↔ 遗物/物品
   遗物触发卡牌打出时的额外效果
   设计：遗物的触发条件应清晰（"每回合第一次打出技能卡时"）

3. 卡牌 ↔ 怪物状态
   卡牌对特定状态的怪物产生额外效果
   设计：状态的持续时间应该匹配卡牌的使用窗口

4. 卡牌 ↔ 地形/位置
   卡牌效果受角色/怪物位置影响
   设计：位置变化的手段应该与卡牌效果一起提供给玩家
```

### 15.2 卡牌设计中的"反直觉但有效"的模式

```
模式1："看起来很弱的卡反而是核心"（StS 的"耸肩无视"）
  看起来"获得 8 护甲"很普通，但它是铁甲战士最强的防御卡之一
  原因：低费 + 高效率 + 无条件的组合在实战中非常可靠
  启示：不要只设计"华丽"的卡，务实的卡同样重要

模式2："限制条件反而让卡更有趣"（StS 的"金刚臂"）
  "造成 12 点伤害。如果敌人意图攻击，额外造成 6 点"
  条件限制让玩家有"满足条件"的成就感和策略规划
  启示：无条件的强卡是危险的，有条件的强卡是策略的

模式3："一次性消耗品创造稀缺感"（StS 的药水）
  药水是一次性的，用完就没了
  稀缺性让玩家在使用时更谨慎、更有成就感
  启示：不是所有卡都应该反复使用

模式4："弃牌堆的卡比手牌更有价值"（StS 的"黑暗之拥"）
  "每回合第一次弃牌时抽 1 张"——让弃牌从负面变成正面
  设计：创造一个机制让"废弃物"变得有价值
  启示：改变玩家对"负面操作"的认知是高阶设计
```
