# 规则变异设计——Roguelite局外成长的替代方案

> 标签：`roguelite`, `metaprogression`, `player-psychology`, `reinforcement`, `design-pattern`
> 关联概念：`01-Roguelike设计规范.md`, `21-驱动力模型.md`, `09-过度理由效应.md`, `18-快乐总量.md`, `37-信号与强化体系.md`
> 来源：框架设计 Skill 工作流实践推导

---

## 一、问题：传统局外成长（内容解锁）的三大困境

Roguelite 品类中，局外成长系统（Metaprogression）通常采用**内容解锁**模式——玩家通过多次游玩积累资源，解锁新角色、新卡牌、新遗物。但这种模式存在三个结构性问题：

### 困境一：线性有穷

```
内容解锁是一条直线：
  新角色1 → 新角色2 → 新角色3 → …… → 全部解锁 → 不再有期待
```

内容总量是有限的，玩家终将"全解锁"。此后局外成长系统失去作用，体验图底部不再拓宽。

### 困境二：制作成本高

每解锁一个新角色，意味着：
- 全新的数值体系设计
- 全新的卡牌/技能池
- 平衡性测试量指数增长（N 个角色 × M 张卡牌）
- 美术/动画/音效资产

制作成本线性增长，但玩家体验的边际收益递减。

### 困境三：过度理由效应风险

> 参考：`09-过度理由效应.md`

```
玩家原本："我想玩，因为构筑很有趣"（内在驱动力）
加入解锁后："我得玩，因为想解锁下一个角色"（外在驱动力挤出内在驱动力）
解锁完毕："……还玩吗？"（内在驱动力已被稀释）
```

内容解锁本质上是**外在驱动力**——它以"新内容"为诱饵驱动玩家重复游玩，但这个过程会逐步削弱玩家对核心玩法本身的内在兴趣。

---

## 二、解法：规则变异（Rule Mutation）

### 核心定义

**规则变异**：不增加新内容（新卡牌/新角色/新地图），而是改变**已有内容之间的交互规则**，让同一套内容产生不同的体验。

### 与内容解锁的核心区别

```
内容解锁：增加新的"砖块"（新卡牌/新角色/新地图）
规则变异：改变"砖块之间的连接规则"（伤害公式/经济逻辑/成长曲线/敌人AI）
```

### 变异因子示例

| 变异类型 | 例子 | 改变的维度 |
|:---------|:-----|:-----------|
| **难度变异** | 敌人双倍血量，但掉落三倍金币 | 战斗节奏 + 经济逻辑 |
| **资源变异** | 商店每回合刷新，但售价+50% | 资源分配策略 |
| **构筑变异** | 每次升级可保留一张牌到下一局 | 跨局继承规则 |
| **信息变异** | 可提前看到下一层全部节点 | 决策条件 |
| **限制变异** | 禁用某类卡牌，获得额外起始资源 | 构筑约束 + 补偿 |
| **融合变异** | 两张相同卡牌可合成高级卡牌 | 卡牌成长规则 |
| **倒置变异** | 血量越低，攻击力越高 | 风险-收益关系 |
| **镜像变异** | 每场战斗会复制玩家的上一张出牌 | 对手行为规则 |

### 两种变异的深层分类

| 类型 | 本质 | 例子 | 对体验的影响 |
|:----|:-----|:-----|:-----------|
| 🎲 **规则型变异** | 改变玩法逻辑 | "击败敌人后可招募为已用" | 改变决策类型 |
| 🔢 **数值型变异** | 改变数值参数 | "所有卡牌费用-1，但起始手牌-1" | 改变效率曲线 |

> **设计原则**：优先设计规则型变异，数值型变异应有明确的代价或限制。每局至少选择 1 个规则型变异，数值型变异不超过 1 个。

## 三、规则变异的优势分析

### 对比维度

| 维度 | ❌ 内容解锁 | ✅ 规则变异 |
|:----|:-----------|:-----------|
| **体验上限** | 有穷——内容耗尽即终结 | **近乎无穷**——N 个因子 × N 个取值 × 自由组合 |
| **制作成本** | 高——每项新内容需完整制作 | **低**——同一套系统改参数/规则 |
| **体验多样性来源** | 新内容的线性叠加 | **组合爆炸**——变异因子的交叉作用 |
| **对内在驱动力的影响** | 外驱为主——"解锁了就爽一下" | **内驱为主**——"我想试试这个组合会怎样" |
| **与核心循环的关系** | 平行扩展，容易游离 | **垂直深化**——让已有系统产生新体验 |
| **消退速度** | 全解锁后迅速消退 | 更慢——仍有未尝试的组合 |
| **社区生命力** | 速通/攻略→全解锁→讨论减少 | 变异组合的发现和分享→持续话题 |

### 内在驱动力机制

规则变异利用的是人类的**好奇心**和**探索欲**——这是比"收集/解锁"更持久的内驱力来源：

```
规则变异 → "如果我把A和B组合会怎样？" → 尝试 → 发现结果
         → 要么是惊喜（正反馈）→ 再试其他组合
         → 要么是失败（学习）→ 调整策略后再试
```

这与斯金纳强化程式中的**可变比率强化**类似——玩家不知道下一次组合会带来什么效果，这种不确定性本身维持了期待感。

> 参考：`37-信号与强化体系.md`（可变比率），`21-驱动力模型.md`（内在驱动力）

## 四、设计原则与风险

### 设计原则

1. **变异改变规则而非堆砌内容**——变异因子的本质是修改已有系统的交互参数，而不是引入新系统
2. **变异应有可感知的差异**——玩家必须能清晰感受到"这局和上局不同"，否则变异失去意义
3. **变异之间应有协同/冲突**——不同变异因子的组合应产生非线性的效果（涌现），而非单纯的累加
4. **允许玩家自主选择变异组合**——这是权力下放的一部分，让玩家对"这局的体验"有决定权
5. **变异因子数量控制在认知舒适区**——建议 15-25 个因子，每局选 2-3 个

### 风险与缓解

| 风险 | 说明 | 缓解方案 |
|:----|:-----|:---------|
| **天花板依然存在** | 试过大部分有趣组合后，新鲜感仍会消退 | 接受"天花板存在"的事实，但比内容解锁高得多；定期增加少量新变异因子 |
| **最优解收敛** | 社区快速找出最强组合→路径依赖 | 引入随机轮换池（每局可用变异从更大的池子中随机抽取）|
| **与快车道冲突** | 快车道跳过战斗→变异因子无法生效 | 变异奖励仅在非跳过战斗中生效，跳关获得标准奖励而非变异奖励→制造取舍 |
| **隐性数值化** | 玩家的选择都收敛到纯数值变异 | 强制规则型变异比例（每局至少1个规则型）|
| **认知负荷** | 玩家需要理解变异因子的效果才能做有意义的选择 | 变异因子效果应一句话说清；新因子首次出现时高亮展示效果预览 |

## 五、适用场景

规则变异最适合以下类型的游戏：

| 场景 | 原因 |
|:----|:------|
| **数值深度较高**的构筑类游戏 | 数值系统越丰富，变异因子的可操作空间越大 |
| **单局时长适中**的游戏（10-40min） | 太短则变异来不及发挥作用，太长则试错成本高 |
| **已有核心循环成熟**的游戏 | 变异是对核心循环的深化，不能替代核心循环的设计 |
| **玩家群体偏策略/探索型** | 喜欢"试试看"的玩家更能从变异中获得乐趣 |
