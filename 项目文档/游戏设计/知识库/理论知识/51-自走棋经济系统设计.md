## 一、经济系统的核心功能

自走棋的经济系统承担以下功能：

| 功能         | 说明                               |
| :----------- | :--------------------------------- |
| **节奏控制** | 金资源获取速度决定阵容成型节奏     |
| **决策创造** | "花钱 vs 存钱"是贯穿整局的核心决策 |
| **容错机制** | 经济管理好 → 有资源转型/止损       |
| **分化玩家** | 不同经济策略 → 不同阵容路径        |

## 二、基础经济模型

### 2.1 金币来源

| 来源         |           量           | 条件                   |
| :----------- | :--------------------: | :--------------------- |
| **基础收入** |       5金币/回合       | 自动获得               |
| **利息收入** | +0~5金币（每存10金+1） | 回合结束时根据存金计算 |
| **连胜奖励** |  +1~3金币（累计连胜）  | 2连胜起算              |
| **连败奖励** |  +1~3金币（累计连败）  | 2连败起算              |
| **单位出售** | 单位购买价格的30%~100% | 随时                   |
| **特殊机制** |          不定          | 视版本/英雄而定        |

### 2.2 金币支出

| 支出项   |      费用      | 频率       |
| :------- | :------------: | :--------- |
| 购买单位 |   1~5金币/个   | 每次购买   |
| 刷新商店 |    2金币/次    | 随时       |
| 升级人口 | 4经验（5金币） | 随时       |
| 特殊操作 |      不定      | 视版本而定 |

## 三、利息系统（核心机制）

### 3.1 利息模型

利息是自走棋经济系统的**核心创新**：

```
存款 0-9   → 利息 0
存款 10-19 → 利息 +1
存款 20-29 → 利息 +2
存款 30-39 → 利息 +3
存款 40-49 → 利息 +4
存款 50+   → 利息 +5（上限）
```

**关键规则**：利息按回合结束时持有金币计算，上限50金 → 吃满+5利息。

### 3.2 利息的设计意义

| 存款区间 | 策略倾向                 | 风险                 |
| :------- | :----------------------- | :------------------- |
| 0-10金   | 即时战力型——有多少花多少 | 后期经济跟不上       |
| 10-30金  | 平衡型——留一部分吃利息   | 需要判断何时开始花钱 |
| 30-50金  | 经济型——最大化利息收入   | 前期血量可能不够     |
| 50金+    | 完美经济——吃完利息再花   | 需要保证血量撑得住   |

**核心决策**：**"是现在花钱提升战力，还是存钱吃利息以后更强？"**

这个决策创造了持续的**时间权衡**——每次花钱都是在"当前强度"和"未来收益"之间做选择。

### 3.3 利息上限的设计意图

为什么设50金上限？

1. **防止滚雪球**：如果没有上限，经济领先者会越来越领先
2. **创造消耗动机**：存到50后"多出来的钱不花就浪费了" → 推动玩家主动消费
3. **定义"满经济"**：玩家心理锚点——"我经济满了" → 开始花钱提升
4. **公平竞争**：所有人经济上限一致

## 四、连胜/连败奖励

### 4.1 奖励曲线

|   连胜次数   |   TFT奖励   | 设计意图              |
| :----------: | :---------: | :-------------------- |
|      2       |     +1      | 小幅奖励              |
|      3       |     +1      |                       |
|      4       |     +2      | 开始拉开差距          |
|      5+      |     +3      | 上限锁定              |
| **连败次数** | **TFT奖励** | **设计意图**          |
|      2       |     +1      | 安慰                  |
|      3       |     +1      |                       |
|      4       |     +2      | 连败也能发育          |
|      5+      |     +3      | 极限连败→后期翻盘可能 |

### 4.2 连胜 vs 连败的设计对比

| 维度           | 连胜                   | 连败                   |
| :------------- | :--------------------- | :--------------------- |
| **经济**       | 额外金币+保血量        | 额外金币+但掉血        |
| **选秀优先权** | 低（先选装备 vs 后选） | 高（后选→拿劣势装备）  |
| **心理**       | 正向——"我强"           | 负向——"我在憋"         |
| **策略**       | 保持最强阵容，压制对手 | 故意送分，追求完美经济 |
| **风险**       | 连胜中断→节奏断档      | 收不住→血量过低暴毙    |

**设计启示**：连败机制的引入让"输"不再是纯粹的惩罚——它给了落后玩家**双重追赶机制**（经济+选秀优先）。这是`19-浪费规避设计.md`中"失败但不白费"原则的典型实践。

## 五、经济管理策略

### 5.1 经典经济节奏

```
1-3回合：  全花 → 构建前期阵容
4-6回合：  开始存 → 到10金吃利息
7-12回合： 逐渐提升人口 → 到7级
13-20回合： 到50金 → 开始D牌/升级
21+回合：  维持50金以上 → 多余的钱全花
```

### 5.2 经济策略谱系

| 策略       | 存金节奏            | 升级时机       | 适用阵容     |
| :--------- | :------------------ | :------------- | :----------- |
| **速升**   | 少存，尽快升级      | 前期就升       | 高费核心阵容 |
| **经济流** | 存到50金再花        | 靠利息自动升级 | 中期发力阵容 |
| **D牌流**  | 不存钱，持续花钱D牌 | 不急于升级     | 低费3星阵容  |
| **灵活流** | 看情况，动态调整    | 根据发牌员     | 全能型       |

## 六、经济系统设计原则

### 6.1 核心原则

1. **每回合都有经济决策** —— 不是"有资源就花"，而是"花多少、怎么花"
2. **经济与战力对立** —— 钱花在人口上就不能D牌，D牌就不能存利息
3. **经济有上限** —— 避免无限滚雪球
4. **失败也有收益** —— 连败/利息让落后玩家保留翻盘可能性
5. **经济可视化** —— 玩家时刻知道自己的经济状况

### 6.2 常见设计陷阱

| 陷阱             | 表现                         | 对策                         |
| :--------------- | :--------------------------- | :--------------------------- |
| **利息惩罚过低** | 全员存钱到50→节奏过慢        | 提高前期压力，鼓励花钱       |
| **利息惩罚过高** | 存钱不如花钱→经济系统无效    | 降低升级成本，提升利息吸引力 |
| **连败收益过高** | 故意连败成最优策略           | 连胜连败收益对称             |
| **经济上限过低** | 很快就满50，后期经济系统失效 | 后期加入大额消费渠道         |
