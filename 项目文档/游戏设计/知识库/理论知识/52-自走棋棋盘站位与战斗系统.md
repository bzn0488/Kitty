## 一、棋盘布局设计

### 1.1 标准棋盘结构

自走棋的棋盘通常采用**对称网格布局**：

```
敌方区域
┌──┬──┬──┬──┬──┬──┬──┬──┐
│  │  │  │  │  │  │  │  │  ← 敌方后排
├──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │  ← 敌方前排
├──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │  ← 中线
├──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │  ← 己方前排
├──┼──┼──┼──┼──┼──┼──┼──┤
│  │  │  │  │  │  │  │  │  ← 己方后排
└──┴──┴──┴──┴──┴──┴──┴──┘
己方区域
```

**典型参数**：
- TFT：8列×4行（双方各2排，中间空地）
- Underlords：8列×4行
- 酒馆战棋：7格宽（简化版）

### 1.2 棋盘格的设计意义

| 维度       | 设计意义                                    |
| :--------- | :------------------------------------------ |
| **宽度**   | 影响单位密度和AOE效果价值                   |
| **深度**   | 影响前排/后排的区分和刺客/切入机制          |
| **格子数** | 人口上限 = 可用的格子数，稀缺性决定决策价值 |
| **对称性** | 影响站位博弈——镜像站位 vs 非对称站位        |

## 二、站位策略体系

### 2.1 基础站位原则

```
前排（肉盾）    ├── 坦克型单位
                 └── 控制型单位（顶在最前吃伤害放技能）

中排（输出）    ├── 核心输出（需要保护）
                 └── 功能辅助（Buff/治疗）

后排（保护）    ├── 脆皮输出
                 └── 关键单位（刺客跳后排时需保护）
```

### 2.2 常见站位阵型

| 阵型         | 布局               | 特点                 | 克制/被克制     |
| :----------- | :----------------- | :------------------- | :-------------- |
| **一字长蛇** | 全部放最后排       | 集中输出，但脆皮暴露 | 被AOE克制       |
| **铁桶阵**   | 前排全放，后排集中 | 保护核心，站位密集   | 被刺客克制      |
| **分散阵**   | 各单位分散放       | 防AOE，但站位稀释    | 被单体爆发克制  |
| **角落阵**   | 全部挤在角落       | 极端的保护/集火      | 被大范围AOE克制 |
| **钩子阵**   | 前排留空，诱导钩子 | 反刺客/反钩子        | 需要预判对手    |

### 2.3 站位决策层次

```
第一层：前排放谁？（坦克/控制/炮灰）
第二层：核心放哪？（保护还是放前排抢能量）
第三层：如何针对对手？（镜像站位 vs 调整站位）
第四层：如何防刺客？（包围核心 vs 分散）
```

## 三、自动战斗AI设计

### 3.1 目标选择逻辑

自走棋的战斗AI通常遵循以下目标选择规则：

| 优先级 | 目标选择       | 典型行为                        |
| :----: | :------------- | :------------------------------ |
|   1    | **最近敌人**   | 近战单位直接攻击最近的敌人      |
|   2    | **最低血量**   | 部分远程/刺客优先攻击低血量     |
|   3    | **最高攻击力** | 部分技能以最高攻为目标          |
|   4    | **随机**       | 平A在同等优先级中随机选择       |
|   5    | **特殊**       | 嘲讽/标记等技能效果强制改变目标 |

### 3.2 战斗AI的设计原则

| 原则         | 说明                 | 示例               |
| :----------- | :------------------- | :----------------- |
| **可预测性** | 玩家能预判单位的行为 | 近战总打最近敌人   |
| **一致性**   | 相同条件下相同行为   | 技能释放条件固定   |
| **透明度**   | 行为逻辑对玩家可见   | 攻击范围指示器     |
| **策略空间** | AI行为可被玩家利用   | 利用站位诱导AI集火 |

### 3.3 战斗节奏控制

```
战斗开始
    ↓
双方单位前进/接触（约1-2秒）
    ↓
集中火力期——前排互打（约3-5秒）
    ↓
减员期——开始出现死亡（约3-8秒）
    ↓
收割期——一方优势明显（约2-3秒）
    ↓
战斗结束
```

**节奏设计要点**：
- 战斗总时长控制在15-30秒（不拖沓）
- 初期接触要快（不浪费时间在走路上）
- 减员后节奏加速（优势方快速收割）
- 残局战斗应有悬念（残血翻盘的可能）

## 四、战斗结果呈现

### 4.1 观战体验设计

| 元素         | 设计要点                    |
| :----------- | :-------------------------- |
| **战斗速度** | 可快进（2x/3x），但不能跳过 |
| **血条动画** | 清晰展示伤害量，数值浮动    |
| **技能特效** | 关键技能要有足够视觉反馈    |
| **击杀反馈** | 单位死亡要有明确表现        |
| **战斗总结** | 伤害统计/DPS排行            |

### 4.2 信息呈现

```
战斗前：    血量对比、阵容预览（可选）
战斗中：    当前血量变化、击杀数
战斗后：    伤害排名、剩余单位数、对敌人造成的伤害
```

## 五、战斗平衡性

### 5.1 单位定位设计

| 定位         | 特征             | 站位      | 装备倾向    |
| :----------- | :--------------- | :-------- | :---------- |
| **坦克**     | 高血量/高护甲    | 前排      | 防御装      |
| **近战输出** | 中血量/高攻击    | 前排/中排 | 攻击装/吸血 |
| **远程输出** | 低血量/极高攻击  | 后排      | 攻击装/攻速 |
| **刺客**     | 低血量/高暴击    | 后排/侧翼 | 暴击装/攻击 |
| **控制**     | 中血量/控制技能  | 中排      | 回蓝装/防御 |
| **辅助**     | 中血量/Buff/治疗 | 中排/后排 | 回蓝装      |

### 5.2 战斗中的随机性

| 随机来源     | 影响程度 | 设计意图           |
| :----------- | :------: | :----------------- |
| 暴击         |    中    | 创造不确定性       |
| 技能目标选择 |   中高   | 防止计算最优解     |
| 攻击miss     |    低    | 少量随机性增加变化 |
| 特殊效果触发 |    中    | 间歇强化           |

## 六、背包管理类自走棋的站位变体（补充）

> 与 `理论知识/70-背包乱斗单位设计.md`、`理论知识/71-背包英雄单位设计.md` 配合阅读

背包管理类自走棋（背包乱斗、背包英雄）采用了一种与传统棋盘站位**截然不同**的站位方式——**网格背包索引站位**。

### 6.1 核心差异

| 维度         | 棋盘站位（TFT）    | 背包站位（背包乱斗/背包英雄）     |
| :----------- | :----------------- | :-------------------------------- |
| **空间维度** | 2D棋盘（行×列）    | 网格背包（行×列，但物品尺寸可变） |
| **位置意义** | 距离决定目标选择   | 索引顺序决定触发顺序              |
| **前后排**   | 近战前排/远程后排  | 无严格前排概念                    |
| **移动**     | 单位会移动接近目标 | 物品固定不移动，通过索引顺序触发  |
| **策略重点** | 保护核心/分散防AOE | 优化触发链/最大化格子利用率       |

### 6.2 背包站位模型

```python
# 背包触发顺序：从左到右，从上到下
背包网格:
┌────┬────┬────┬────┐
│ 1  │ 2  │ 3  │ 4  │  ← 第一行先触发
├────┼────┼────┼────┤
│ 5  │ 6  │ 7  │ 8  │  ← 第二行后触发
├────┼────┼────┼────┤
│ 9  │ 10 │ 11 │ 12 │
└────┴────┴────┴────┘
触发顺序: 1→2→3→4→5→6→7→8→...
```

### 6.3 背包站位 vs 棋盘站位的策略对应

| 棋盘策略           | 背包等价物                 | 设计转化                     |
| :----------------- | :------------------------- | :--------------------------- |
| 前排坦克吸收伤害   | 护甲类物品放在触发链前端   | 防御物品先触发——先套盾后输出 |
| 核心输出放后排保护 | 核心物品放触发链中段       | 先上Buff再核心输出           |
| 分散站位防AOE      | 关键物品不放在相邻格       | 避免连锁失效                 |
| 刺客跳后排         | 物品放在特定列攻击特定目标 | 位置决定目标选择优先级       |

### 6.4 背包站位的设计启示

背包站位对传统棋盘站位理论的扩展价值：

1. **空间约束替代数值约束**——不是因为"前后排属性差异"而站位，而是因为"格子不够"而必须取舍
2. **触发链替代目标选择**——单位不再通过AI选择目标，而是通过固定的索引顺序决定行为次序
3. **布局即Build**——物品位置的放置本身就是构筑策略的一部分，与卡片/单位的选择同等重要

## 七、设计自检清单

- [ ] 棋盘尺寸是否创造有意义的站位决策？ 
- [ ] 棋盘是否考虑了背包管理类的触发链变体？
- [ ] 战斗AI是否可预测且透明？
- [ ] 战斗时长是否在合理范围内（15-30秒）？
- [ ] 战斗结果呈现是否清晰（玩家知道为什么输/赢）？
- [ ] 站位策略是否有层次（从基础到进阶）？
- [ ] 是否有防单一最优站位的机制？
- [ ] 如果采用背包站位，物品索引顺序是否合理？
- [ ] 物品尺寸变化是否与站位策略关联？
