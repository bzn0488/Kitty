# 手牌重置与构筑规划的张力模型

> 来源：荒野与曙光 GDD 知识分析——手牌管理系统（每轮丢光重抽3张+唯一牌堆）与卡牌构筑深度的结构冲突
> 关联概念：`16-卡牌设计规范.md`, `25-杀戮尖塔卡牌设计分析.md`, `协同效应设计通用规则.md`, `无效进程与三层乐趣体系.md`

---

## 一、定义

手牌重置机制（每轮丢光手牌并随机抽取固定数量）与卡牌构筑规划（玩家设计卡组以实现特定连招）之间存在一种设计张力：

```
手牌重置 ← → 构筑规划
  ↑              ↑
随机性         确定性
即时应对      长远规划
低学习成本    高精通上限
```

这种张力本身不是"好"或"坏"的——它取决于设计者希望玩家获得什么样的体验。

## 二、张力的四个维度

### 2.1 战术规划性

| 维度         | 手牌重置倾向           | 构筑规划倾向              |
| ------------ | ---------------------- | ------------------------- |
| 连招可行性   | ❌ 难以设计多回合连招   | ✅ 可规划3-5回合的战术序列 |
| 关键牌保留   | ❌ 无法保留到"最佳时机" | ✅ 可等待理想时机          |
| 手牌管理深度 | 🟡 极浅——无需管理手牌   | ✅ 深度——需决定留/弃/用    |

> **设计启示**：如果允许保留1-2张手牌过轮，可以在手牌重置和构筑规划之间取得平衡。蓄力型卡牌跨轮存在是另一种替代方案。

### 2.2 随机性感知

| 维度             | 手牌重置倾向                 | 构筑规划倾向             |
| ---------------- | ---------------------------- | ------------------------ |
| "抽到什么打什么" | ✅ 高——每轮都是新的随机挑战   | ❌ 低——玩家希望减少随机性 |
| 玩家控制感       | ❌ 低——无法控制抽哪3张        | ✅ 高——牌库决定了概率分布 |
| "卡手"体验       | ✅ 极低——永不卡手（唯一牌堆） | 🟡 存在——key牌可能沉底    |

> **设计启示**：手牌重置+唯一牌堆的组合几乎完全消除了"卡手"体验，代价是牺牲了"蓄牌"的策略深度。

### 2.3 认知负荷

| 维度       | 手牌重置倾向          | 构筑规划倾向           |
| ---------- | --------------------- | ---------------------- |
| 入门门槛   | ✅ 低——不需要记牌/算牌 | ❌ 高——需要理解牌库交互 |
| 每轮决策量 | ✅ 3张中选择，简化决策 | ❌ 多张手牌需全面评估   |
| 精通上限   | ❌ 低——选项有限        | ✅ 高——优化空间大       |

> **设计启示**：手牌重置模型天然适合作为"母系设计"的入口——先让玩家轻松上手，再逐步深入构筑。

### 2.4 构筑驱动力

| 维度           | 手牌重置倾向             | 构筑规划倾向                     |
| -------------- | ------------------------ | -------------------------------- |
| "我要这个卡组" | 🟡 削弱——打不出来也没关系 | ✅ 强化——特定卡组产生特定体验     |
| 单张卡的价值   | ❌ 低——反正留不住         | ✅ 高——每张卡都可能在关键回合出现 |
| 构筑实验意愿   | ✅ 高——反正轮轮都是全新的 | 🟡 中——投入多但体验不到的风险     |

## 三、张力谱系（分类应用）

手牌重置与构筑规划并不是非此即彼的二元选择。以下是在这个谱系上的游戏定位：

```
纯随机                   手牌重置                构筑规划              纯构筑
  │                        │                        │                    │
  │   荒野与曙光            │   杀戮尖塔              │   万智牌            │
  │   (机制蓄牌补偿)        │   (每轮抽满手牌)        │   (起手调度)        │
  │                         │                        │                    │
  低 ←─────────────────── 构筑规划性 ───────────────────→ 高
```

> 注：荒野与曙光虽然在谱系上位于"强重置"端，但通过"机制蓄牌"（打出后卡牌自动驻留）补偿了构筑规划性。其实际构筑规划性略高于纯随机重置，约在"手牌重置"区间的中间偏左位置。

## 四、蓄牌的两种形式：主动蓄牌 vs 机制蓄牌

> ⚠️ **重要补充**：荒野与曙光的手牌重置与构筑规划之间的张力，实际上被一种特殊的"机制蓄牌"设计化解了。以下区分两种蓄牌形式。

### 4.1 主动蓄牌（玩家选择）

玩家手动选择保留某张手牌不打出，等待最佳时机。

| 维度                 | 说明                                    |
| -------------------- | --------------------------------------- |
| **谁决定蓄**         | 玩家主动选择"我留着这张牌等时机"        |
| **决策负担**         | 高——玩家需要判断"啥时候是最佳时机"      |
| **挫败风险**         | 留了但没用上→挫败感，"早知道刚才就打了" |
| **与丢光重抽的关系** | ❌ 冲突——手牌每轮重置，无法主动保留      |

### 4.2 机制蓄牌（卡牌自动驻留）

卡牌设计本身决定了"打出后保持在手卡直到效果触发"，玩家不需要额外决策。

| 维度                 | 说明                                             |
| -------------------- | ------------------------------------------------ |
| **谁决定蓄**         | 卡牌机制自动处理——打出去就自动保持在手           |
| **决策负担**         | 低——玩家不需要思考"留不留"                       |
| **挫败风险**         | 无——机制保证一定会触发                           |
| **与丢光重抽的关系** | ✅ 兼容——丢光清除的是未打出的牌，已驻留牌不受影响 |

### 4.3 核心区别

```
主动蓄牌：玩家说"我留着这张牌" → 需要判断 → 可能后悔 → 高决策深度 + 高挫败风险
机制蓄牌：卡牌说"我留在你手上" → 不需要判断 → 保证触发 → 低决策负担 + 零挫败风险
```

### 4.4 何时选择哪种蓄牌形式

| 设计目标       | 推荐形式 | 说明                                   |
| -------------- | -------- | -------------------------------------- |
| 降低认知负荷   | 机制蓄牌 | 玩家不需要额外做"留不留"的判断         |
| 提高策略深度   | 主动蓄牌 | 留牌 vs 打牌的取舍是经典策略乐趣       |
| 保持快节奏战斗 | 机制蓄牌 | 主动蓄牌需要玩家花时间评估"是否该留着" |
| 支持跨轮连招   | 两者结合 | 基础连招用机制蓄牌，高阶连招用主动蓄牌 |
| 新手友好       | 机制蓄牌 | 主动蓄牌需要更深的游戏理解才能用好     |

### 4.5 荒野与曙光的实际设计

荒野与曙光采用**手牌重置（强重置）+ 机制蓄牌**的组合：

- **未打出的牌**：每轮丢光重抽——保持"每轮都是新开始"的节奏
- **已打出的蓄力牌**：保持在手卡中等待效果触发——不受丢光重抽影响
- **效果**：既保留了手牌重置带来的低认知负荷，又通过机制蓄牌解决了"大剑蓄力"等跨轮机制的需求

> 这一设计的关键在于：**蓄牌的决策权从玩家手中转移到了卡牌设计者手中**。卡牌设计者决定"这张牌什么时候该留着、什么时候该释放"，玩家只需要打出去就行，不需要自己判断"要不要留"。

## 五、设计决策指南

当设计卡牌游戏的手牌机制时，以下问题可以帮助定位：

### 5.1 适用手牌重置（强重置）的场景

- 战斗节奏快，每轮都需要快速决策
- 希望降低入门门槛，减少"卡手"挫败感
- 时间轴/行动队列是核心机制，需要每个回合都"干净重启"
- **典型例子**：荒野与曙光（设计采用机制蓄牌作为补偿）

### 5.2 适用弱重置（允许保留）的场景

- 有"蓄力""准备"等跨回合机制
- 连招/组合技是核心乐趣
- 卡牌价值高度依赖于特定组合
- **典型例子**：杀戮尖塔（每回合保留残血手牌，抽满上限）

### 5.3 适用无重置（传统构筑）的场景

- 牌库管理是核心策略
- 玩家有多个回合来执行一个战术
- 弃牌/检索/牌库操作是机制的一部分
- **典型例子**：万智牌、游戏王

## 六、缓解张力的设计手法

| 手法                 | 说明                                  | 示例                           |
| -------------------- | ------------------------------------- | ------------------------------ |
| **机制蓄牌（驻留）** | 卡牌打出后保持在手卡直到效果触发      | 荒野与曙光——大剑蓄力牌跨轮存在 |
| **部分保留**         | 允许每轮保留1-2张，其余丢弃           | 保留手牌上限的0-2张            |
| **蓄力槽**           | 独立于手牌的"蓄力"资源槽              | 大剑蓄力——卡牌跨轮增加伤害     |
| **恒定区**           | 某些卡牌不进入抽牌池，每轮固定在手    | 角色自带的固有卡牌             |
| **延时注册**         | 打出的卡牌不是立即回牌堆，而是延迟N轮 | 行动队列的延迟效果             |
| **费用折扣**         | 保留的卡牌下轮费用-1                  | 鼓励战略保留而非全场清空       |
