# 多角色队伍的时间轴战斗设计

> 调研目标：针对荒野与曙光的"多角色队伍×时间轴战斗"复合设计，
> 分析多角色在时间轴上的协作模式、手牌/牌库管理、角色切换与分工、队伍成长与替换等核心设计问题。

---

## 一、多角色时间轴的四种模型

### 1.1 共享时间轴 vs 独立时间轴

| 模型           | 描述                           | 代表游戏                     | 对荒野的适用性       |
| -------------- | ------------------------------ | ---------------------------- | -------------------- |
| **共享时间轴** | 所有角色在同一时间轴上行动     | FF ATB（全员共用一个时间轴） | ✅ 最简单，容易管理   |
| **独立时间轴** | 每个角色有独立的行动进度条     | 八方旅人                     | ⚠️ 复杂度高，信息量大 |
| **混合时间轴** | 队伍共享时间轴但角色有独立速度 | 崩坏：星穹铁道               | ⚠️ 介于两者之间       |
| **回合分组**   | 按速度分组，同组同时行动       | 火焰纹章                     | ❌ 不适合卡牌战斗     |

**荒野的实际设计**（根据项目文档）：多角色共享一套手牌和费用系统？还是每个角色独立？
- 如果所有角色共享手牌 → 共享时间轴更合理
- 如果每个角色有独立牌库 → 独立或混合时间轴

**建议**：初始采用**共享时间轴**（降低复杂度），后续可通过角色特殊能力解锁独立时间轴机制。

### 1.2 共享时间轴下的角色分工

```
共享时间轴示例（荒野的推测模型）：

时间轴：──┬──┬──┬──┬──┬──┬──┬──┬──→
        角色A 怪物 角色B 角色A 怪物 角色B 角色A
        行动  行动  行动  行动  行动  行动  行动

每个时间节点，只有"轮到"的角色可以行动。
玩家在角色A的回合操作角色A的牌，角色B的回合操作角色B的牌。
```

**关键设计问题**：
1. 轮到某个角色时，其他角色能做什么？（等待/自动防御/支援）
2. 如何防止"板凳角色"没有参与感？
3. 角色之间的行动顺序如何决定？（速度属性？固定轮换？）

### 1.3 角色切换的五种模式

| 模式         | 操作                            | 策略深度 | 复杂度 | 适用性             |
| ------------ | ------------------------------- | -------- | ------ | ------------------ |
| **固定轮换** | 角色按固定顺序行动（A→B→C→A）   | 低       | 低     | 新手友好           |
| **速度排序** | 按速度属性决定行动顺序（CK3型） | 中       | 中     | 需要速度属性系统   |
| **玩家选择** | 每回合玩家选择让哪个角色行动    | 高       | 中     | **推荐**——策略最深 |
| **事件触发** | 特定条件触发角色切换            | 中高     | 高     | 适合特殊机制       |
| **实时切换** | 随时可以切换当前控制角色        | 高       | 高     | 需要暂停/ATB系统   |

**推荐**：**玩家选择型**——每回合玩家可以选一个角色行动，被选角色消耗费用推进时间。未被选的角色可以做一些辅助动作（自动防御/小幅度回复）。

## 二、多角色的手牌与牌库管理

### 2.1 手牌管理的四种模式

| 模式                | 描述                               | 策略特征                     | 代表游戏                       |
| ------------------- | ---------------------------------- | ---------------------------- | ------------------------------ |
| **共享手牌**        | 所有角色共用一套手牌               | 资源统一分配，角色间竞争手牌 | 杀戮尖塔（单人）               |
| **独立手牌**        | 每个角色有自己的手牌               | 每个角色是独立的"小卡组"     | 欺诈之地（协商 vs 战斗双牌库） |
| **主副手牌**        | 主角色有完整手牌，副角色有少量技能 | 主从关系明确                 | 暗黑地牢（技能位而非手牌）     |
| **公用池+个人手牌** | 公用卡池但每人手牌私有             | 协作+独立并存                | 怪物火车（契约+英雄）          |

**荒野的实际设计**：每个角色有**独立牌库和抽牌堆**，这意味着：

```
每人独立牌库的优势：
  - 角色差异化（战士有攻击牌，法师有法术牌）
  - 构筑多样化（每条构筑线对应一个角色）
  - 替换角色时不影响其他角色的构筑

每人独立牌库的挑战：
  - 管理复杂度翻倍（玩家需要管理多套卡组）
  - 战斗中的操作密度（轮到谁就操作谁的牌）
  - 构筑资源分配（商店奖励给谁？）
```

### 2.2 独立牌库的费用与时间协同

在多角色独立牌库模式下，费用的设计变成：

```
总费用预算 = 单个角色的费用（不共享）

例如：
  角色A（战士）：使用角色A的牌，费用从角色A的时间轴支付
  角色B（法师）：使用角色B的牌，费用从角色B的时间轴支付
  但时间轴是共享的 → 谁推进时间，怪物就行动
```

**关键问题**：如果角色A打了一张3费卡（推进3回合），角色B会受到这3回合的影响吗？

**两种设计选择**：

| 选择             | 规则                               | 影响                                         |
| ---------------- | ---------------------------------- | -------------------------------------------- |
| **费用全队共享** | 任何角色的费用推进都影响全队时间轴 | 一个角色打高费卡→全队等待→惩罚全队           |
| **费用个人承担** | 每个角色有独立的"行动窗口"         | 角色A打高费卡→角色A等待，但角色B可以立刻行动 |

**建议**：采用**费用全队共享**（更简单的模型）。这意味着：
- 玩家的核心策略是多角色之间的费用分配——"这回合让谁行动？"
- 高费卡的风险由全队承担→玩家会更谨慎使用高费卡
- 角色间需要协作而非各自为战

### 2.3 角色切换时的构筑资源分配

商店/事件奖励中，玩家需要决定将奖励分配给哪个角色：

```
奖励分配决策：
  🔴 稀有卡牌 → 给主C还是给辅助？
  🟡 通用升级 → 提升主C还是补强辅助？
  🟢 金币/资源 → 谁更需要？

设计建议：
  - 部分奖励是角色绑定的（"战士专属卡"）
  - 部分奖励是队伍共享的（"全队生命上限+"）
  - 稀有奖励应让玩家有"纠结"的决策空间
```

## 三、角色差异化设计

### 3.1 角色定位的五种原型

| 定位       | 核心功能           | 费用偏好 | 手牌特征         | 风险                 |
| ---------- | ------------------ | -------- | ---------------- | -------------------- |
| **输出型** | 主要伤害来源       | 1-2费    | 高伤害卡，连击卡 | 被集火时阵容崩溃     |
| **防御型** | 吸收伤害，保护队友 | 1费      | 护甲/嘲讽/减伤卡 | 输出不足时战斗拖长   |
| **辅助型** | 治疗/buff/过牌     | 0-1费    | 增益/回复/抽牌卡 | 自身脆皮             |
| **控制型** | 减速/打断/debuff   | 1-2费    | 控制卡，时间操纵 | 控制效果被抵抗时无用 |
| **混合型** | 多功能但专精度低   | 各费用   | 多样化但单卡弱   | 样样通样样松         |

### 3.2 角色协同的三种层次

```
第一层：简单配合（容易实现）
  角色A给怪物上debuff（+30%受伤）
  角色B对同一怪物输出（享受增伤）
  → 简单的"辅助→输出"链

第二层：条件触发（中等难度）
  角色A打出特定卡牌时"标记"怪物
  角色B攻击被标记的怪物时触发额外效果
  → 需要玩家规划行动顺序

第三层：组合技（高难度，高回报）
  角色A使用"蓄力"，角色B使用"协同攻击"
  如果两角色在同一回合先后使用，触发组合技
  → 需要精确的时间轴配合
```

### 3.3 角色退休与替换

荒野有角色退休机制（退休后需指定新队长）。这对多角色队伍提出特殊设计需求：

```
退休的触发条件：
  - 自愿退休（玩家选择）
  - 强制退休（年龄/伤病）
  - 剧情退休（事件触发）

退休的影响：
  - 该角色的所有牌库/构筑离队
  - 新手牌需要从零构筑（或继承部分）
  - 队伍需要时间重新磨合

设计要点：
  - 退休不应是纯粹负面——退休角色留下"遗产"（传承技能/资源）
  - 新人角色应有快速融入机制（初始送一套基础牌）
  - 关键角色退休应有剧情处理（而非突然消失）
```

## 四、时间轴上的多角色操作

### 4.1 操作流程设计

```
每回合的标准操作流程：

1. 查看当前时间轴状态（谁的位置到了）
2. 如果是角色A的回合：
   a. 查看角色A的手牌（独立手牌）
   b. 选择一张牌打出（消耗费用，推进时间）
   c. 结算卡牌效果（拼刀/伤害/治疗等）
   d. 如果还有操作机会→继续操作角色A或等待
3. 如果是怪物的回合：
   a. 查看怪物意图
   b. 无法操作，等待结算
4. 轮到下一个行动者
```

**操作密度问题**：如果队伍有3个角色，每个角色5张手牌，玩家每回合需要处理15张牌的信息。这可能导致**决策疲劳**。

### 4.2 减少决策疲劳的设计手段

| 手段         | 原理                                       | 副作用           |
| ------------ | ------------------------------------------ | ---------------- |
| **自动出牌** | 设置规则后角色自动出牌（如"最低费卡优先"） | 玩家失去控制感   |
| **简化手牌** | 副角色手牌限制（如最多3张）                | 角色差异化降低   |
| **行动建议** | UI高亮推荐出牌                             | 削弱策略自主性   |
| **操作队列** | 预先编排多回合操作                         | 上手门槛高       |
| **角色专注** | 每回合只有1个角色能操作，其他角色自动行动  | **推荐**——最平衡 |

**推荐方案：角色专注模式**

```
每回合操作规则：
  1. 玩家选择"本回合专注哪个角色"
  2. 该角色获得完整操作权（出牌/使用技能）
  3. 其他角色自动执行预设行为（自动攻击/防御/待机）
  4. 专注角色的行动推进时间轴
  5. 下回合玩家可以切换专注角色

优点：
  - 每回合只处理1套手牌（减少决策疲劳）
  - 角色切换创造策略节奏
  - 非专注角色也有贡献（自动行动）
```

### 4.3 多角色拼刀的特殊问题

荒野的拼刀系统在面对多角色时出现新的设计维度：

```
单角色拼刀：简单直接——1名玩家 vs 怪物多个部位
多角色拼刀：
  - 多个角色可以拼刀不同部位
  - 角色间可以协防（角色A拼刀失败→角色B补位）
  - 角色分工：输出角色专注攻击部位，防御角色专注格挡

设计问题：
  - 所有角色都能拼刀还是只有当前行动角色能拼刀？
  - 拼刀失败后其他角色能否补救？
  - 怪物的多个部位是否可以同时被不同角色拼刀？
```

**建议**：
- 只有当前行动角色能主动拼刀（简化规则）
- 非行动角色可以自动防御（减伤而非完全抵消）
- 特殊技能允许"协防"（消耗额外资源让副角色参与拼刀）

## 五、队伍成长与平衡

### 5.1 多角色的成长曲线同步

多角色队伍面临的核心挑战：**所有角色同步成长 vs 重点培养个别角色**。

```
方案A：齐头并进（低风险低效率）
  资源均分给所有角色
  → 所有角色"刚刚够用"，没有明显短板
  → 但也没有超强角色

方案B：主C+辅助（高效率高风险）
  80%资源给主C，20%给辅助
  → 主C超强，辅助脆皮
  → 辅助被秒→主C孤立无援

方案C：轮换培养（动态平衡）
  前中期主C，后期练新角色替代
  → 平滑过渡
  → 需要有效的"带新人"机制
```

**建议**：方案B+C混合——玩家可以自由决定资源分配，但提供"新人快速成长"机制（退休角色留下的遗产）。

### 5.2 队伍规模与战斗平衡

| 队伍规模 | 优势                       | 劣势               | 战斗节奏     |
| -------- | -------------------------- | ------------------ | ------------ |
| **1人**  | 操作简单，构筑专注         | 容错率低           | 快，但容错低 |
| **2人**  | 可分工，容错适中           | 覆盖面有限         | 适中         |
| **3人**  | 完整分工（输出/防御/辅助） | 资源分散           | 适中偏慢     |
| **4人+** | 战术丰富                   | 操作复杂，成长缓慢 | 慢，决策疲劳 |

**荒野的实际情况**：初始2人，可通过事件扩充。建议**队伍上限设为3-4人**，战斗中最多同时上场3人（板凳角色在战斗外待命）。

### 5.3 板凳角色的处理

多角色队伍必然产生"板凳角色"（不上场的角色）。需要让板凳角色有存在感：

```
板凳角色的参与方式：
  1. 战斗中可切换（消耗资源换人）
  2. 板凳角色提供被动光环（"全队攻击+5%"）
  3. 板凳角色在战斗中自动恢复（HP/卡牌）
  4. 板凳角色触发特殊事件（营救/对话）

避免的问题：
  - 板凳角色完全不参与会失去存在价值
  - 板凳角色提供太多增益会让主力角色差异被稀释
  - 频繁换人破坏战斗节奏
```

## 七、针对荒野的具体建议

### 7.1 推荐的核心方案

```
时间轴模型：共享时间轴 + 玩家选择型角色切换
手牌模式：每人独立牌库，每人独立手牌
操作模式：角色专注（每回合操作1个角色）
队伍规模：初始2人，上限3人上场 + 1-2人板凳
拼刀规则：当前行动角色主动拼刀，其他角色自动防御
退休机制：退休角色留下"传承牌"给继承者
```

### 7.2 设计自检清单

- [ ] 每个角色的定位是否清晰（输出/防御/辅助/控制/混合）？
- [ ] 角色间是否有有意义的协同？（不仅是数值叠加）
- [ ] 角色切换是否有策略成本？（无成本切换 → 每次选最优）
- [ ] 板凳角色是否有参与感？（不完全是看客）
- [ ] 角色退休/替换是否平滑？（新角色能快速融入）
- [ ] 多角色时信息量是否可控？（每回合最多处理几个决策？）
- [ ] 构筑资源的分配是否让玩家有"纠结"的决策？

### 7.3 需要进一步研究的问题

1. **角色永久死亡的可能**：Roguelike中角色死亡是常见的，但配合多角色培养体系，死亡成本如何设计？
2. **队伍化学效应**：特定角色组合产生额外效果（类似火焰纹章的支援等级）
3. **战斗外角色管理**：休息/治疗/升级等管理操作的界面和信息架构
4. **异步角色成长**：不同角色获取经验速度不同（后排角色成长慢的问题）
