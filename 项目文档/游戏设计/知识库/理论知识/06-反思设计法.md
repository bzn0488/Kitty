# 反思设计法——从0开始的设计流程

> 来源：千水《游戏创作心灵鸡汤》#16，从异星工厂组装机案例到完整六步设计方法。

## 完整六步流程

### 第一步：找痛点

找到一个游戏机制中让你觉得**不舒服的点**。不是大毛病，而是一个可以改进的小缺口。

> 案例：异星工厂的组装机——从游戏开始贯穿到最后，无论组装芯片还是传送带，玩家的体验没有变化。

### 第二步：衍生玩法

针对痛点，提出多种改进方案。不要只做一个方案，至少写 3~6 种。

> 案例：受我的世界合成界面启发→把组装机变成合成平台，需要按特定配置摆放原料。

### 第三步：展开设计核心玩法

选定最佳方案后，围绕它设计丰富的互动方式。

> 案例：给组装机配套三种零件——
> - **加长机械臂**：抓取距离×2，但每次只能旋转90°（与普通机械臂逻辑不同）
> - **装载器**：将传送带上的零件按顺序推入组装平台（引入顺序编程概念）
> - **机器人臂**：按地面路径规划放置（引入路径编程）

### 第四步：确定设计重心

母系思维 vs 父系思维的选择：
- 不给玩家设限 → 提供丰富工具让玩家自己设立目标
- 或者给玩家明确的挑战路径

### 第五步：验证与测试

原型 → 游玩 → 验证核心体验是否成立 → 回到第一步迭代

### 第六步：组织内容

将不同工具/机制划分为"技巧主题"（Skill Theme），对应游戏的前中后期：

> 异星工厂案例中：机械臂系列=前期、装载器=中期、机器人臂=后期——三种不同玩法，而非仅仅是数值差异。

## 与其他设计方法的关系

| 方法           | 与反思设计法的关系                                        |
| :------------- | :-------------------------------------------------------- |
| **自然创作观** | 反思设计法是自然创作观的具体落地方法                      |
| **MDA框架**    | 反思设计法更偏向 Mechanics 层面的设计迭代                 |
| **CCST**       | CCST 是关卡层面的组织方法，反思设计法是系统层面的设计方法 |
