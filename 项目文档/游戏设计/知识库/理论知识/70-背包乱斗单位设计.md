# 背包乱斗（Backpack Battle）单位设计

> 标签：`auto-battler`, `game-design`, `inventory-management`, `case-study`
> 关联概念：`50-自走棋商店与刷新机制设计.md`, `52-自走棋棋盘站位与战斗系统.md`, `53-自走棋协同机制设计.md`, `49-Auto-Battler品类定义与设计历史.md`

---

## 一、品类定位

背包乱斗是**背包管理×自走棋**的交叉产物：

| 自走棋特征            | 背包管理特征             |
| :-------------------- | :----------------------- |
| PvP八人对战，镜像匹配 | 物品放置在网格背包中     |
| 每回合自动战斗        | 背包布局决定触发顺序     |
| 商店刷新购买单位      | 物品即单位，无传统"棋子" |
| 协同系统驱动构筑      | 背包空间管理 = 核心策略  |

---

## 二、单位定义——物品即单位

### 2.1 核心概念

背包乱斗中最核心的设计决策：**去除了"棋子"的概念，物品本身就是单位。**

| 传统自走棋                   | 背包乱斗                   |
| :--------------------------- | :------------------------- |
| 英雄棋子有生命值/攻击力/技能 | 物品有攻击力/效果/触发条件 |
| 棋子放在棋盘格上             | 物品放在背包格中           |
| 棋子有主动技能（法力驱动）   | 物品自动触发（条件驱动）   |
| 棋子站位影响AI目标选择       | 物品位置影响触发链的顺序   |

### 2.2 物品的属性结构

```
物品 = {
  类型: 武器/护甲/饰品/消耗品/宝石
  稀有度: Common/Rare/Epic/Legendary
  基础属性: 攻击力/防御力/速度
  效果: 攻击时/回合开始时/被攻击时 触发
  职业Tag: 游侠/法师/战士/刺客/狂战士...
  合成配方: 可合成的高阶物品
}
```

### 2.3 物品的"自动战斗"逻辑

与英雄棋子不同，背包乱斗的物品没有主动技能和法力条。它们的行为逻辑是：

| 触发时机       | 行为              | 示例                     |
| :------------- | :---------------- | :----------------------- |
| **每回合开始** | 自动施放Buff/召唤 | 召唤物、初始护盾         |
| **攻击时**     | 触发攻击特效      | 附加伤害、吸血、连击     |
| **被攻击时**   | 触发防御/反击特效 | 反伤、护甲、闪避         |
| **回合结束时** | 触发结算效果      | 治疗、再生、毒伤         |
| **条件达成**   | 触发特殊效果      | 低血量时爆发、击杀后重置 |

## 三、背包布局即站位

### 3.1 网格位置的意义

背包乱斗的站位系统与棋盘自走棋**完全不同**——没有前后排概念，而是**物品在背包中的索引顺序**决定触发链。

```
背包网格（4×N）:
┌────┬────┬────┬────┐
│  A  │  B  │  C  │  D  │  ← 第一行（从左到右触发）
├────┼────┼────┼────┤
│  E  │  F  │  G  │  H  │  ← 第二行
├────┼────┼────┼────┤
│ ... │     │     │     │
└────┴────┴────┴────┘
```

| 位置特性         | 策略含义                           |
| :--------------- | :--------------------------------- |
| **从左到右触发** | 左侧物品先行动，决定了战斗的"先手" |
| **同行交互**     | 部分物品效果受同行物品影响         |
| **相邻触发**     | 相邻格子的物品可能触发连锁效果     |
| **列位置**       | 不同列可能对应不同目标选择逻辑     |

### 3.2 位置设计决策模型

```
第一层：物品主轴（先手/后手）
    ↓  强先手物品（匕首/迅捷武器）放左侧
第二层：物品间距（相邻放置触发协同）
    ↓  需要配合的物品放相邻
第三层：背包形状（如何最大化格子利用率）
    ↓  攻击物品→菱形放中间，防御物品→矩形填边角
第四层：反制对手（预判对手背包布局）
    ↓  针对当前对手的主流布局调整物品顺序
```

## 四、职业/Tag体系

背包乱斗的职业Tag比经典自走棋更简单——**单维度Tag，无种族叠加**：

| Tag        | 风格         | 核心机制                     |
| :--------- | :----------- | :--------------------------- |
| **游侠**   | 远程、攻速   | 攻击速度叠加，多重射击       |
| **法师**   | 爆发、AOE    | 法术伤害放大，冷却缩减       |
| **战士**   | 均衡、持续   | 攻击与防御兼备，战斗越久越强 |
| **刺客**   | 暴击、秒杀   | 背后攻击、暴击倍率、击杀重置 |
| **狂战士** | 高风险高回报 | 低血量时爆发，自伤加伤       |
| **召唤师** | 召唤物、铺场 | 召唤宠物占据格子，人海战术   |

### Tag触发的特殊性

背包乱斗的Tag触发**不要求"场上同时有N个"**，而是通过**背包中的物品构成**来决定职业倾向。这意味着：
- 混合职业可行（多Tag共存）
- 没有"凑不够人数"的问题
- 职业判定更灵活，但也更难捕捉"核心Build"

## 五、合成体系——升星的等价物

背包乱斗没有传统的"3合1升星"，而是用**物品合成**替代：

| 方式         | 规则                      | 效果                |
| :----------- | :------------------------ | :------------------ |
| **同级合成** | 2个同名+同稀有度 → 高一级 | 属性提升 + 效果强化 |
| **异级合成** | 低级+高级 → 特定配方      | 产生固定高阶装备    |
| **宝石镶嵌** | 宝石+装备槽 → 附加效果    | 自定义装备特效      |

### 合成 vs 升星的设计对比

| 维度           | 传统升星（TFT）      | 合成（背包乱斗）             |
| :------------- | :------------------- | :--------------------------- |
| **所需数量**   | 3个1星→2星，9个→3星  | 2个同物品→更高品质           |
| **资源消耗**   | 同卡合并，不消耗金币 | 消耗金币购买+合成            |
| **效果变化**   | 数值提升+技能增强    | 效果质变（可能改变触发逻辑） |
| **决策复杂度** | 追还是不追？         | 合成A还是保留两个各自触发？  |

## 六、与经典自走棋的差异总结

| 维度         | 经典自走棋（TFT）             | 背包乱斗               |
| :----------- | :---------------------------- | :--------------------- |
| **单位定义** | 英雄棋子（技能+属性）         | 物品（自动触发效果）   |
| **站位**     | 六边形棋盘，前后排            | 网格背包，索引顺序     |
| **协同**     | 双维度（职业+种族），阶梯触发 | 单维度Tag，自动匹配    |
| **升星**     | 3合1                          | 2合1合成，品质提升     |
| **装备**     | 3组件合成成装，装备槽         | 物品即单位，部分可镶嵌 |
| **经济**     | 利息+连胜连败                 | 固定收入，无利息       |
| **战斗节奏** | 15-30秒自动战斗               | 较短（约10-20秒）      |

## 七、设计目的反推——从知识理论看背包乱斗的规则意图

### 7.1 "物品即单位"的减法设计

> 关联理论：`12-减法设计.md`、`33-无效进程与三层乐趣体系.md`、`18-快乐总量.md`

背包乱斗选择**去除了"棋子"概念，物品本身就是单位**，这一设计的核心目的是：

| 减法了什么            | 设计目的                                                        | 理论依据                                                        |
| :-------------------- | :-------------------------------------------------------------- | :-------------------------------------------------------------- |
| 移除了英雄属性面板    | 降低认知负荷——玩家只需要关注物品效果，不需要同时管理血/蓝/攻/防 | `12-减法设计.md` — "聚焦而非删除"，保留核心乐趣                 |
| 移除了主动技能/法力条 | 消除"等能量"的无效进程——物品条件触发让每回合都有事发生          | `33-无效进程与三层乐趣体系.md` — 时间不能空转                   |
| 移除了前后排站位      | 将站位简化为"索引顺序"→减少"怎么放"的决策时间                   | `18-快乐总量.md` — 单位进程快乐总量：压缩决策时间，提升决策密度 |

**减法设计揭示的意图**：背包乱斗的设计者刻意去除了所有"棋子模拟"的复杂性，因为背包管理本身已经提供了足够的策略深度。**物品即单位 = 背包管理即构筑，不再需要额外的棋子维度。**

### 7.2 "背包索引即站位"的创造性表达

> 关联理论：`03-创造性表达.md`、`22-权力下放设计.md`、`41-涌现式设计.md`

背包乱斗的站位不是"前排抗伤后排输出"的棋盘逻辑，而是**物品索引顺序 = 触发链**。这一设计的目的是：

| 设计意图                    | 实现方式                                                        | 理论解释                                                            |
| :-------------------------- | :-------------------------------------------------------------- | :------------------------------------------------------------------ |
| **让站位变成Build的一部分** | 物品位置影响触发顺序 → 同一个物品放左边和放右边是完全不同的策略 | `03-创造性表达.md` — 规则内自由选择，表达结果可见                   |
| **权力下放：背包布局权**    | 玩家自主决定物品排列，没有"设计师预设的最佳站位"                | `22-权力下放设计.md` — 游戏节奏权+体验选择权的双重让渡              |
| **涌现式触发链**            | 物品在背包中的相邻关系可能产生意外连锁效果                      | `41-涌现式设计.md` — 一个操作有多种用处，系统交互产生预料之外的效果 |

**为什么不用棋盘站位？** 棋盘（前后排+AOE+刺客跳后）需要额外的目标选择AI，而背包索引使触发链完全透明——玩家一看背包就知道"这把会怎么打"，降低了随机性的不可预测感。

### 7.3 "单维度Tag"的快乐总量优化

> 关联理论：`18-快乐总量.md`、`53-自走棋协同机制设计.md`

背包乱斗选择单维度Tag（无职业+种族叠加），这看似简化，实则有明确设计意图：

| 对比项   | 双维度（TFT）     | 单维度（背包乱斗） | 设计意图                     |
| :------- | :---------------- | :----------------- | :--------------------------- |
| 学习成本 | 高——需要记2套Tag  | 低——只看职业       | 降低进入门槛                 |
| 构筑深度 | 深——双维度交叉    | 浅——单轴发展       | 背包布局提供另一维深度       |
| 失败体验 | 凑不够Tag＝阵容崩 | 灵活混合＝不死追   | 浪费规避——没有"凑不够"的挫败 |
| Tag数量  | 约60个/赛季       | 约6-8个            | 聚焦核心差异                 |

**真正意图**：单维度Tag让"凑Build"的认知负担降低，把决策精力留给**背包布局**这一更独特的策略维度。快乐总量 = 构筑乐趣 + 布局乐趣，而非"凑Tag的乐趣×2"。

### 7.4 "合成体系"的投篮效应与浪费规避

> 关联理论：`30-投篮效应.md`、`19-浪费规避设计.md`、`46-正反馈与负反馈循环.md`

背包乱斗用"2合1合成"替代传统"3合1升星"，其设计目的：

| 规则                   | 设计目的                                                    | 理论依据                                              |
| :--------------------- | :---------------------------------------------------------- | :---------------------------------------------------- |
| 2合1而非3合1           | 降低升星门槛——每买到2个就能合成，减少"差一个追不到"的负反馈 | `19-浪费规避设计.md` — 转化途径存在就不算浪费         |
| 合成产生质变而非纯数值 | 每次合成都是"新的期待"——物品可能改变触发逻辑                | `30-投篮效应.md` — 低成本(买2个)+小概率质变=持续期待  |
| 保留2个不合成也是策略  | 玩家可以选择"保留两个各自触发"还是"合成一个更强"            | `46-正反馈与负反馈循环.md` — 多行为串联形成正反馈循环 |

**背后的核心洞察**：升星本质上是一种**损失厌恶触发器**——追星的时候差一个是痛苦的。背包乱斗用"2合1"把痛苦减半，同时用"留两个不合成"创造了新的决策空间。

### 7.5 "无利息经济"的快车道设计

> 关联理论：`17-快车道设计.md`、`43-游戏节奏设计.md`

背包乱斗移除了经典自走棋的利息系统（固定收入无利息），目的是：

| 规则           | 意图                                                             | 理论                                        |
| :------------- | :--------------------------------------------------------------- | :------------------------------------------ |
| 每回合固定收入 | 消除"存钱vs花钱"的经济决策——玩家只需关注"买什么"而非"什么时候买" | `17-快车道设计.md` — 去掉低正反馈的"存钱期" |
| 无利息         | 战斗节奏更快——不需要等经济成型，每回合都在加强阵容               | `43-游戏节奏设计.md` — 节奏紧凑，无"空窗期" |
| 节奏紧凑       | 单局10-15分钟完成                                                | 适应移动端/碎片化时间                       |

**真正意图**：利息系统是经典自走棋的"节奏拖延器"——你要存钱，你就得等。背包乱斗移除了利息，等于移除了**经济维度的无效进程**，让快乐密度最大化。
