# 无效进程与三层乐趣体系

> 标签：`invalid-process`, `happiness-total`
> 关联概念：`18-快乐总量.md`, `17-快车道设计.md`, `43-游戏节奏设计.md`

## 无效进程

**定义**：玩家无法从中获得显著正向体验（或正反馈极低/为零）、且不包含内容表达的环节。

**示例**：
- 多层菜单才能施放动作
- 开放世界中反复跑图（路径已体验过）
- 无意义QTE

## 三层乐趣体系

| 层级 | 定义 | 进程需求 | 示例 |
|------|------|----------|------|
| **Hard Fun** | 核心乐趣，游戏结构 | 最长 | 核心战斗、关卡挑战 |
| **Easy Fun** | 低成本乐趣，额外系统 | 中等 | 滑翔翼、载具、小游戏 |
| **无意识快乐** | 本能驱动，无需动机 | 最短 | 点击反馈、收集音效 |

## 填充原则（装瓶比喻）

```
瓶子的容量 = 玩家时间
优先放石头（Hard Fun）→ 再放石子（Easy Fun）→ 最后放沙粒（无意识快乐）
```

## 设计流程

1. 优先构建 Hard Fun（**必须存在**）
2. 评估是否存在无效/低体验密度进程
3. 能缩短/简化则缩短
4. 无法缩短 → 加入 Easy Fun 填充
5. 在不影响前两者的前提下考虑无意识快乐
