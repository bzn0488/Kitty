## 一、商店刷新模型

### 1.1 基础结构

自走棋商店的核心是一个**随机生成+选择购买**的系统：

```
每回合开始 → 商店生成 N 个单位 → 玩家决策（买/不买/刷新）
                   ↑                              │
                   └── 花费金币刷新 ───────────────┘
```

**关键参数**：

| 参数           | TFT 标准值 | 设计意义                 |
| :------------- | :--------: | :----------------------- |
| 每次刷新单位数 |     5      | 提供足够选择但不Overload |
| 初始可用格数   |     1      | 人口限制驱动购买决策     |
| 商店等级       |    1-10    | 控制高费单位的出现时机   |
| 每次刷新费用   |   2金币    | 刷新是一种经济决策       |

### 1.2 刷新概率体系

自走棋的刷新概率是**随等级动态变化**的：

```
等级 1-2:  1费 100%       → 前期快速凑2星
等级 3-4:  1费 70-75%     → 过渡期
等级 5-6:  2费 35-40%     → 主力阵容成型期
等级 7-8:  3费 30-35%     → 寻找核心3费单位
等级 9:    4费 25-30%     → 大后期找5费神卡
等级 10:   5费 30%        → 决战期
```

**设计意图**：
- 低等级高概率出低费 → 前期节奏友好
- 等级提升使高费概率上升 → 玩家有动力升级人口
- 不存在"某费率为0"（除等级1时5费为0）→ 即使低等级也有极小概率出高费（惊喜感）

## 二、卡池系统设计

### 2.1 共享卡池 vs 独立卡池

| 模式         | 机制                           |     代表     | 设计影响                                |
| :----------- | :----------------------------- | :----------: | :-------------------------------------- |
| **共享卡池** | 所有玩家共用有限数量的同名单位 |     TFT      | 抢卡成为策略——热门Build会被其他玩家稀释 |
| **独立卡池** | 每个玩家有自己独立的卡池       | 炉石酒馆战棋 | 不存在"互卡"，决策更独立                |
| **混合卡池** | 部分共享+部分独立              |   部分变体   | 折中方案                                |

### 2.2 共享卡池的设计意义

共享卡池是自走棋的核心创新之一：

| 行为            | 对其他玩家的影响         | 策略意义                        |
| :-------------- | :----------------------- | :------------------------------ |
| 你买了某单位    | 该单位在公共池中减少一个 | 观察其他玩家阵容，预判竞争程度  |
| 别人买了某单位  | 你能刷到的概率降低       | 需要备选方案，不能死追单一Build |
| 多家追同一个3星 | 谁都难3星                | 创造了"转阵容"的决策空间        |

**设计启示**：共享卡池的本质是**玩家之间的间接交互**——即使不在同一局对战，你的购买行为也在影响其他人的决策。

### 2.3 卡池数量设计

| 费用 | 每张卡的数量 | 单位种类数 |
| :--- | :----------: | :--------: |
| 1费  |     30张     |   约13种   |
| 2费  |     20张     |   约13种   |
| 3费  |     15张     |   约13种   |
| 4费  |     10张     |   约11种   |
| 5费  |     5张      |   约8种    |

**设计逻辑**：
- 低费卡数量多 → 容易凑3星（新手友好）
- 高费卡数量少 → 3星极为稀有（成就感和内容更新点）
- 全局卡池有限 → 总卡数可控，平衡性可调节

## 三、人口与等级系统

### 3.1 人口的作用

人口（等级）是自走棋的**单位上场限制**：

```
等级 1 → 可上场 1 个单位
等级 2 → 可上场 2 个单位
...
等级 10 → 可上场 10 个单位
```

| 等级  | 升级所需经验 | 累计经验 | 解读         |
| :---: | :----------: | :------: | :----------- |
|  2→3  |      4       |    4     | 前期快速升级 |
|  3→4  |      8       |    12    |              |
|  4→5  |      12      |    24    | 中期关键节点 |
|  5→6  |      20      |    44    |              |
|  6→7  |      32      |    76    | 后期         |
|  7→8  |      50      |   126    |              |
|  8→9  |      70      |   196    |              |
| 9→10  |     100      |   296    | 大后期       |

### 3.2 经验获取

| 来源         |    经验量     |    频率    |
| :----------- | :-----------: | :--------: |
| 自动获得     |    2/回合     |   每回合   |
| 花费金币购买 | 4/次（5金币） |    随时    |
| 特殊机制     |     不等      | 视版本而定 |

**核心决策**：
```
"升级人口" vs "刷新单位" vs "存钱吃利息"
        三者只能选其一二
```

### 3.3 设计意义

人口系统的本质是**节奏控制器**：
- 前期人口低 → 无法上高质量阵容 → 自然节奏慢
- 花钱升级人口 → 牺牲经济换即时战力
- 等级上限固定 → 后期所有人最终都有相同人口 → 公平竞争

## 四、D牌决策模型

"D牌"（刷新商店寻找关键单位）是自走棋中最核心的决策之一：

### 4.1 决策维度

| 维度     | 问题              | 策略意义                           |
| :------- | :---------------- | :--------------------------------- |
| **时机** | 什么时候开始D牌？ | 过早→经济崩；过晚→血量崩           |
| **深度** | 花多少钱D牌？     | 花光→没容错；留钱→概率低           |
| **目标** | 追什么单位？      | 追热门→竞争大；追冷门→可能不兼容   |
| **止损** | D不到怎么办？     | 转阵容→损失沉没成本；硬追→可能暴毙 |

### 4.2 常见的D牌策略

| 策略           | 做法                   | 适用场景               |
| :------------- | :--------------------- | :--------------------- |
| **现实现用**   | 刷到当前阵容的缺位就买 | 保血量、中期过渡       |
| **All-in D牌** | 一次性花光所有金币     | 血量告急、就差几个3星  |
| **稳步升级**   | 先升级人口再D牌        | 经济健康、追求后期上限 |
| **存钱吃息**   | 攒50金币吃满利息再花   | 经济最大化的保守策略   |

## 五、设计自检清单

- [ ] 商店刷新是否有明确的概率曲线？
- [ ] 人口升级的节奏是否合理（早升级有优势但牺牲经济）？
- [ ] 卡池是否支持多种Build方向，而非单一最优解？
- [ ] 共享卡池（如使用）的竞争是否创造了有意义的决策？
- [ ] D牌决策是否有「早 vs 晚」「深 vs 浅」「追 vs 转」的权衡空间？
