# 输入随机与输出随机（Input vs Output Randomness）

> 标签：`randomness`, `design-framework`, `player-psychology`
> 来源：Mark Brown（GMTK）、Keith Burgun、游戏设计社区
> 关联概念：`30-投篮效应.md`, `37-信号与强化体系.md`, `69-强制循环.md`, `49-Auto-Battler品类定义与设计历史.md`

## 核心分类框架

随机性在游戏设计中最基本的分类是**作用于决策前还是决策后**：

```
输入随机（Input Randomness）          输出随机（Output Randomness）
    │                                       │
    ▼                                       ▼
  决策前的信息不确定性                     决策后的结果不确定性
  "我现在面对什么情况？"                   "我的选择结果怎样？"
```

| 维度         | 输入随机           | 输出随机             |
| :----------- | :----------------- | :------------------- |
| **作用时机** | 玩家做决策之前     | 玩家做决策之后       |
| **玩家感知** | "面对未知做选择"   | "选择的结果不确定"   |
| **控制感**   | 高（决定权在玩家） | 低（结果由系统决定） |
| **适合场景** | 策略/构筑/探索     | 战斗/掉落/惊喜事件   |

## 详细分析

### 输入随机（Input Randomness）

玩家在**信息不完全**的情况下做决策。随机性创造了多样化的情境，但玩家仍可通过决策质量影响结果。

**示例**：
- 《杀戮尖塔》的地图布局 — 你不知道接下来会碰到什么敌人/事件，但路线选择影响全局
- 自走棋的商店刷新 — 每回合随机出5个单位，玩家决定买什么、刷不刷
- 牌类游戏的抽牌 — 你拿到什么牌是随机的，但怎么打出是玩家的选择
- Roguelike 的地图生成 — 每次开局地形不同，但路径选择由玩家决定

**设计优势**：
- 保留玩家**决策的意义**（好的选择仍然带来优势）
- 制造**多样化情境**而牺牲控制权
- 鼓励**适应能力**（玩家根据情况调整策略）
- 重玩性的健康来源（每次面对不同的决策问题）

### 输出随机（Output Randomness）

玩家做出决策后，结果由随机性决定。随机性直接作用于行为的产出。

**示例**：
- 暴击判定 — 你攻击了，但伤害是否暴击是随机的
- 随机掉落 — 你击败了Boss，但掉什么装备是随机的
- 命中率 — 你瞄准了，但子弹是否命中是随机的
- 关键Miss — 你做出了正确的战术选择，但随机失败

**设计优势**：
- 制造**惊喜峰值**（投篮效应）
- 创造**悬念和紧张感**（每次攻击都有期待）
- 增加**不可预测性**（避免公式化体验）

**设计风险**：
- 玩家**归因困难**（赢了是运气好还是技术好？输了呢？）
- **控制感降低**（好决策也可能带来坏结果）
- **过度使用**会导致"运气游戏"感

## 设计原则

### 黄金法则：优先使用输入随机

> **让玩家在不完整信息下做决策，而不是让决策的结果随机化。**

```diff
+ ✅ 输入随机：敌人未知 → 玩家选择装备和战术
- ❌ 输出随机：玩家进攻 → 随机判定是否命中
```

### 何时可以使用输出随机

| 场景                 | 原因                         | 注意                         |
| :------------------- | :--------------------------- | :--------------------------- |
| 单次影响有限         | 每次判定的影响足够小         | 避免一次随机决定胜负         |
| 可重复行为           | 玩家可以多次尝试（投篮效应） | 每次尝试成本需低             |
| 惊喜体验             | 作为正反馈的调味剂           | 不要依赖随机性作为核心正反馈 |
| 无法预测时更鼓励探索 | 探索行为本身有价值           | 不要惩罚探索行为             |

### 混合使用策略

许多优秀游戏同时使用两种随机：

| 游戏           | 输入随机                 | 输出随机                   |
| :------------- | :----------------------- | :------------------------- |
| 《杀戮尖塔》   | 地图生成、卡牌奖励三选一 | 战斗中的随机判定（少数）   |
| 自走棋         | 商店刷新、对手匹配       | 战斗中的攻击目标选择、暴击 |
| 《以撒的结合》 | 房间布局、道具掉落       | 弹道扩散、射速随机         |
| XCOM           | 地图生成、任务类型       | **命中率（争议点）**       |

### 设计检查清单

| 问题                       | 说明                                       |
| :------------------------- | :----------------------------------------- |
| 这个随机是输入还是输出？   | 明确知道自己在用哪一类                     |
| 玩家的决策有意义吗？       | 输入随机保留决策意义，输出随机削弱         |
| 随机的结果能被对冲吗？     | 是否有策略手段缓解坏运气                   |
| 玩家会觉得"不公平"吗？     | 输出随机更容易引发不公平感                 |
| 随机性在制造还是消耗体验？ | 好的随机创造决策空间，坏的随机消耗玩家时间 |

## 与随机性的其他分类关系

```
随机性
├── 按时机：输入随机 vs 输出随机（本文件）
├── 按范围：全局随机（种子生成） vs 局部随机（每帧判定）
├── 按可控性：纯随机 vs 伪随机（PRD）
├── 按感知：透明随机 vs 隐藏随机
└── 按功能：表现型随机（氛围） vs 机制型随机（核心玩法）
```
