# 单次检定最优概率（80%法则）

> 标签：`player-psychology`, `probability`, `game-mechanics`, `dice-roll`, `game-balance`
> 关联概念：`72-输入随机与输出随机.md`, `73-伪随机分布PRD.md`, `74-确定性vs随机性掉落设计.md`, `01-Roguelike设计规范.md`, `44-游戏平衡性设计.md`
> 来源：桌游设计经验总结——多款经典桌游的单次检定成功率集中在 80% 附近。

---

## 一、核心观察

多款经典桌游中，单次检定的成功概率设计集中在 **70%~82%** 区间：

| 游戏         | 检定方式             | 成功概率 | 备注                 |
| :----------- | :------------------- | :------: | :------------------- |
| **三角机构** | 6个4面骰投出至少1个3 |   ≈82%   | 成功条件较低，容错高 |
| **忍神**     | 2D6 ≥ 5              |   ≈81%   | 基础难度设定         |
| **D&D 5e**   | DC 10，加值+2~+4     | 70%~80%  | 标准难度检定         |
| **D&D 5e**   | DC 15，加值+4~+6     | 50%~65%  | 中高难度，明显有压力 |
| **D&D 5e**   | DC 20，加值+5        | 25%~30%  | 高难度，大概率失败   |

> **核心结论**：**80% 是单次检定最合适的概率**——高到让玩家感到"我应该能成功"，又低到失败时不至于觉得系统在欺诈。

---

## 二、为什么是 80%？

### 2.1 玩家心理锚点

当成功概率在 80% 左右时：
- **成功时**：玩家归因为自身能力/策略（"我本来就应该成功"）→ 维持胜任感
- **失败时**：玩家归因为运气（"20% 被我撞上了"）→ 不损害胜任感

当概率高于 90%：
- 失败时玩家感到被系统背叛（"99% 都能失败？"）→ 信任崩塌
- 成功时无成就感（"本来就该成功"）

当概率低于 60%：
- 失败时玩家感到挫败（"又没中"）→ 挫伤积极性
- 成功时虽然更有成就感，但整体体验负面化

### 2.2 风险感知的激活

80% 概率恰好激活玩家的**风险感知**而不激活**恐惧感知**：
- 风险感知 → 玩家感到"这件事有变数" → 专注度提升 → 心率微升
- 恐惧感知 → 玩家感到"我大概率要输" → 焦虑 → 逃避或消极应对

这类似于**耶克斯-多德森定律**（`游戏设计理论/04-游戏节奏与心流设计.md` §1.1）中的最佳唤醒状态——80% 的成功概率处于"适度风险"的理想区间。

### 2.3 期望值与实际体验的 Gap

```
期望：80% → "我觉得我能行"
实际：成功 → "果然如此"（胜任感维持）
实际：失败 → "运气不好"（不伤及自我）
```

这个 Gap 的设计精妙之处在于：**玩家的预期锚定在"我能成功"，但系统保留了"惊喜/意外"的空间**。

## 三、游戏设计中的应用

### 3.1 何时使用 80% 概率

| 场景         | 推荐概率 | 说明                             |
| :----------- | :------: | :------------------------------- |
| 基础攻击命中 | 70%~80%  | 保证战斗流畅性                   |
| 核心技能发动 | 75%~85%  | 玩家依赖的核心手段，不能频繁失效 |
| 稀有掉落     |  5%~20%  | 低概率结合 PRD 保底              |
| 高风险奖励   | 30%~50%  | 高回报高风险，激活决策           |
| 负面效果抵抗 | 75%~85%  | 失败→增益是惊喜而非沮丧          |

### 3.2 结合 PRD（伪随机分布）

当使用 80% 作为基准概率时，**配合 PRD（`73-伪随机分布PRD.md`）效果更佳**：

```
名义概率 80% → PRD 实现中前几次概率递增 → 实际体验更平滑
```

- 第一次：~60%（触发失败 → 玩家感到运气不好但不崩溃）
- 第二次：~75%
- 第三次：~90%
- 第四次：必然成功

这样即使触发 20% 的失败，也不会出现"连续三次失败"的极端情况。

### 3.3 输入随机 vs 输出随机

根据 `72-输入随机与输出随机.md` 的分类：

| 随机类型 | 80% 的应用                     | 适合的玩家群体               |
| :------- | :----------------------------- | :--------------------------- |
| 输出随机 | 攻击命中率 80%                 | 成就者、探索者（可策略应对） |
| 输入随机 | 资源分配中 80% 优质 / 20% 劣质 | 杀戮者（不打断操作节奏）     |

## 四、在不同游戏类型中的实践

### 4.1 战棋 / SRPG

```
攻击命中率 80% → 玩家会主动寻找"背刺/高打低/属性克制"等加成 → 
将概率提升至 95%+ → 归因为策略成功
```

设计要点：让玩家通过策略操作将 80% 提升到更高，而非一开始就给 95%。

### 4.2 Roguelike DBG

```
基础卡牌命中率 80% → 可通过升级/遗物提升至 90%+
```

要在"随机性的惊喜"和"策略的确定性"之间取得平衡——80% 恰好站在中间。

### 4.3 桌游 / TRPG

```
DM 设计 DC 时，默认标准难度 = 70%~80% 成功率
高于此 → 简单（适合让玩家感到"变强了"）
低于此 → 困难（适合 boss 战或高压场景）
```

---

## 五、例外情况

80% 法则并非放之四海皆准：

| 例外情况     | 原因                               |          建议概率          |
| :----------- | :--------------------------------- | :------------------------: |
| 连续多次检定 | 每次 80%，三次连续的期望成功仅 51% | 适当提高单次概率或加入保底 |
| 高惩罚失败   | 失败代价过大时 80% 仍不够安全      | 提高到 90%+ 或使用"安全网" |
| 低惩罚失败   | 失败无实质损失，只是"换个方式继续" |  可降至 60%~70% 增加变数   |
| 喜剧/娱乐向  | 频繁失败本身就是笑点               | 可低至 50%，失败也带来欢乐 |
