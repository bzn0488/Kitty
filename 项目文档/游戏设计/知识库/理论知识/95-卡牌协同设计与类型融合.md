# 卡牌协同设计与类型融合

> 调研目标：在现有卡牌设计规范的基础上，深入卡牌协同的设计模式、
> 卡牌与不同游戏类型的融合规律、卡牌的叙事潜力与玩家心理模型。

---

## 一、卡牌协同的设计模式目录

### 1.1 七种基础协同模式

| #     | 模式          | 原理                                | 示例                                      | 适用场景             |
| ----- | ------------- | ----------------------------------- | ----------------------------------------- | -------------------- |
| **1** | **标签聚合**  | 多张同标签卡牌互相增强              | "每有1张'火'标签卡，伤害+1"               | 元素/种族主题的构筑  |
| **2** | **条件触发**  | 卡牌A创造了条件，卡牌B消费条件      | "上流血"→"对有流血的敌人伤害翻倍"         | 状态/异常体系        |
| **3** | **资源转换**  | 一种资源转化为另一种更高效的形态    | "护甲→伤害"转换卡                         | 需要跨资源流通的场景 |
| **4** | **费用联动**  | 使用某张卡降低或触发其他卡          | "本回合每打X张卡，下一张卡费用-1"         | 连击/快攻体系        |
| **5** | **位置协同**  | 卡牌效果与卡牌在手中/场上的位置相关 | "最左侧的卡效果翻倍"                      | 需要空间管理的设计   |
| **6** | **链式反应**  | 卡牌A触发卡牌B，B触发C              | 卡牌被消耗时触发相邻卡牌效果              | 组合技/连击体系      |
| **7** | **积累/阈值** | 累积资源/标记达到阈值后触发强力效果 | "每打出3张攻击卡，下一张攻击卡不消耗费用" | 蓄力/爆发体系        |

### 1.2 协同强度的设计原则

```
协同强度曲线：

弱协同（标签聚合）：
  强度：线性增长
  问题：可能变成"无脑堆Tag"
  控制：设上限（"最多叠加3层"）

中协同（条件触发/资源转换）：
  强度：非线性但可控
  问题：需要玩家规划
  控制：条件的稀缺性

强协同（链式反应/累积阈值）：
  强度：指数级增长
  问题：容易失控
  控制：设触发上限/消耗品/一次性
```

**核心原则**：协同的强度越高，触发条件就应该越苛刻。一个简单触发的强力协同会摧毁构筑多样性。

### 1.3 协同的"说明书式" vs "涌现式"

| 维度           | 说明书式协同       | 涌现式协同                     |
| -------------- | ------------------ | ------------------------------ |
| **认知成本**   | 低（看了就知道）   | 高（需要玩家发现）             |
| **可预测性**   | 高（结果确定）     | 低（取决于多种因素）           |
| **重复可玩性** | 低（看一次就够）   | 高（每次都不一样）             |
| **典型案例**   | "与'火'标签卡配合" | "利用地形+击退+持续伤害的连锁" |
| **适合场景**   | DBG构筑游戏        | 系统交互丰富的游戏             |

**建议**：在同一个游戏中混合使用两种模式——说明书式协同引导新手构筑，涌现式协同提供深度。

## 二、卡牌与游戏类型的融合设计

### 2.1 卡牌在模拟经营游戏中的角色

| 角色           | 说明                           | 对设计的要求           | 家族纪元的应用             |
| -------------- | ------------------------------ | ---------------------- | -------------------------- |
| **微调工具**   | 调整经营参数（人口/金钱/效率） | 效果可量化，反馈即时   | ✅ 现有设计                 |
| **危机应对**   | 应对突发事件                   | 在危机时保持可用性     | ✅ 可加强                   |
| **长期投资**   | 持续多回合的效果               | 边际价值递减的预防     | ⚠️ 需注意持续效果的时间成本 |
| **叙事触发器** | 卡牌附带剧情事件               | 效果服务于故事而非数值 | 🆕 **可新增**               |

### 2.2 卡牌在Roguelike战斗中的角色

| 角色         | 说明                   | 对设计的要求         | 荒野的应用   |
| ------------ | ---------------------- | -------------------- | ------------ |
| **战术决策** | 每回合选择打什么牌     | 费用与效果的价值匹配 | ✅ 核心设计   |
| **构筑方向** | 卡牌引导玩家的构筑路线 | 有明确的Build信号    | ✅ 已有原则   |
| **资源管理** | 手牌上限、费用、消耗品 | 多重约束的平衡       | ✅ 已覆盖     |
| **风险回报** | 高费高风险高回报       | 补偿机制的必要性     | ✅ 93号已调研 |

### 2.3 关键洞察：卡牌类型与游戏类型的匹配矩阵

```
               战斗型游戏         模拟经营型        混合型（荒野/家族纪元）
                ─────────         ─────────        ─────────────────────
0费卡          战术价值           运营价值         需要双重价值审查
1费卡          主力战斗           日常操作         最安全的费用段
2费卡          核心战术           重大决策         需要额外补偿
3费卡          终结技             —                仅限特殊场景

持续效果卡      buff/debuff        长期投资         需要独立评估（持续vs即时）
条件触发卡      组合技             事件连锁         适合两边的桥梁
消耗品卡        一次性道具         突发事件应对     天然的"稀有感"
```

## 三、卡牌的叙事与主题融合

### 3.1 卡牌作为叙事载体

卡牌不只是"数值工具"，也可以是**叙事载体**。这对家族纪元这类历史主题游戏尤为重要：

| 叙事层次     | 实现方式                     | 案例                                                     |
| ------------ | ---------------------------- | -------------------------------------------------------- |
| **卡名叙事** | 卡名本身包含故事             | "焚书坑儒"——玩家看到卡名就联想到历史事件                 |
| **卡面叙事** | 卡牌描述/文案补充背景        | "始皇下令焚毁百家之言，文化传播效率-50%，统治稳定度+30%" |
| **效果叙事** | 效果机制反映事件逻辑         | 焚书的机制是"减少文化产出但增加稳定度"——机制即叙事       |
| **连锁叙事** | 相关卡牌先后使用触发额外叙事 | 先打"修筑长城"再打"匈奴入侵"→长城自动防御匈奴            |

### 3.2 机制即叙事的设计原则

好的卡牌设计应该让玩家**通过机制感受到叙事**，而不需要读长篇文案：

```
原则1：效果反映原因
  ❌ "焚书坑儒：获得30文化"
  ✅ "焚书坑儒：文化产出-50%，统治稳定度+30%，持续10回合"
  → 效果机制本身讲述了"压制思想换取稳定"的故事

原则2：连锁产生剧情
  家族纪元中，先打"独尊儒术"再打"科举制度"
  → 触发事件"儒家学者在科举中占据优势"
  → 效果：获得一位"儒家学者"角色

原则3：卡牌的"代价"本身就是叙事
  "背水一战"：本回合所有属性×2，但下回合所有属性归零
  → 机制本身就讲述了"破釜沉舟"的故事
```

### 3.3 历史主题卡牌的三层设计

对家族纪元的历史主题，卡牌设计可以分为三个层次：

| 层次       | 描述                    | 例子                               | 设计难度                 |
| ---------- | ----------------------- | ---------------------------------- | ------------------------ |
| **事件卡** | 历史事件作为一次性效果  | "大运河开通"→贸易路线效率+50%      | 低——直接映射             |
| **人物卡** | 历史人物作为可获取资源  | "拥有'商鞅'时，改革类卡牌效果+50%" | 中——需人物与机制匹配     |
| **思潮卡** | 思想/制度作为持续性影响 | "法家思想"→严刑峻法+效率但-民心    | 高——需抽象概念转化为机制 |

## 四、卡牌的玩家心理学

### 4.1 玩家评估卡牌的五个维度

当玩家看到一张卡牌时，他们在潜意识中评估：

```
1. 强度：这张牌比我已有的牌强吗？
2. 协同：这张牌和我的构筑路线匹配吗？
3. 时机：我现在需要这种效果吗？
4. 稀缺：以后还能再见到这张牌吗？
5. 趣味：这张牌打起来好玩吗？
```

**设计影响**：
- 如果一张牌在"时机"维度上扣分（当前用不上），但在"稀缺"维度上高分（错过不再有）→ 玩家会"为了未来投资"而拿牌
- 如果"趣味"维度太低 → 玩家即使拿了也不爱用

### 4.2 "感觉有用" vs "实际有用"

卡牌设计中一个重要的心理现象：**玩家选择卡牌的依据是"感觉"而非"计算"**。

```
感觉有用 > 实际有用 的情况：
  - 高数值但简单的卡（如"造成20点伤害"）→ 玩家高估其价值
  - 低数值但功能性卡（如"抽1张牌"）→ 玩家低估其价值
  - 有条件的强力卡（如"如果护甲>10则造成30点伤害"）→ 玩家高估其触发概率

设计指导：
  - 功能性卡牌需要更显眼的效果呈现（动画/数值显示）
  - 条件触发卡的触发条件需要清晰可见
  - 不要依赖玩家精确计算——他们不会
```

### 4.3 卡牌的"爽点"设计

卡牌游戏中，玩家记住的不是"平衡"，而是"爽点"：

| 爽点类型     | 触发条件                   | 典型案例             | 设计方法              |
| ------------ | -------------------------- | -------------------- | --------------------- |
| **一锤定音** | 用一张卡终结艰难的战斗     | 高费终结技           | 视觉效果+伤害数字放大 |
| **连环组合** | 多张卡打出预想中的连锁     | 组合技触发时         | 连锁动画+额外奖励     |
| **绝境翻盘** | 在濒死时一张卡扭转局势     | 低血量触发的强力效果 | "背水一战"类卡牌      |
| **白嫖快感** | 用0费卡打出意想不到的价值  | 0费过牌找到关键卡    | 0费卡的效果触发期待感 |
| **构筑成型** | 拿到构筑路线的最后一块拼图 | 核心卡出现           | Build信号的确认感     |

## 五、卡牌设计的常见陷阱

### 5.1 十大陷阱

| #   | 陷阱           | 表现                         | 解法                         |
| --- | -------------- | ---------------------------- | ---------------------------- |
| 1   | **数值膨胀**   | 新卡越来越强，旧卡报废       | 设计时设数值上限             |
| 2   | **功能重叠**   | 多张卡做同一件事             | 每张卡应有独特定位           |
| 3   | **必选卡**     | 某张卡在任何构筑中都是最优选 | 削弱或增加限制条件           |
| 4   | **死卡**       | 在大多数情况下无法发挥作用   | 增加备用效果或条件放宽       |
| 5   | **配合太窄**   | 只与特定一张卡配合           | 协同应基于标签而非指名       |
| 6   | **效果模糊**   | 玩家看不懂这张牌有什么用     | 明确效果目标，增加Build信号  |
| 7   | **节奏断层**   | 某费用段没有可用卡           | 检查费用曲线分布             |
| 8   | **稀有度错配** | 普通卡太强或传说卡太弱       | 稀有度≠强度，稀有度=复杂度   |
| 9   | **叙事脱节**   | 效果与卡名/背景不匹配        | 机制即叙事                   |
| 10  | **记忆负担**   | 玩家记不住卡牌效果           | 效果不超过一行字，使用关键词 |

### 5.2 卡牌设计的快速自检

- [ ] 这张卡有明确的Build信号吗？
- [ ] 这张卡是自闭环吗？（移除所有其他卡后仍能工作？）
- [ ] 这张卡在三选一里出现时，玩家会选吗？（诅咒卡检验）
- [ ] 它的效果是否对应它的费用？（时间成本检验）
- [ ] 它的效果是否对应它的稀有度？（复杂度检验）
- [ ] 它的效果是否对应它的卡名？（叙事一致性检验）
- [ ] 存在其他卡能更好地完成同样的功能吗？（功能重叠检验）

## 六、针对两个项目的具体建议

### 6.1 对家族纪元的卡牌设计

| 建议                                       | 理由                             |
| ------------------------------------------ | -------------------------------- |
| 优先使用**叙事融合**设计（机制即叙事）     | 历史主题需要卡牌讲述故事         |
| 适度使用**持续效果卡**（多回合效果）       | 模拟经营的节奏需要长期投资感     |
| 引入**思潮/政策卡**作为跨回合的全局影响    | 区别于一次性事件卡，提供策略深度 |
| 利用**连锁事件**（特定卡组合触发隐藏事件） | 鼓励玩家探索历史故事线           |

### 6.2 对荒野与曙光的卡牌设计

| 建议                                              | 理由                                  |
| ------------------------------------------------- | ------------------------------------- |
| 严格控制0费卡的数量和伤害值                       | 避免0费霸权（93号调研已分析）         |
| 高费卡必须附带**时间补偿**                        | 让高费构筑可行                        |
| 使用**累积阈值协同**（如累积3次攻击触发额外效果） | 在时间轴战斗中创造"蓄力→爆发"的节奏感 |
| 角色专属卡 vs 通用卡的比例控制在6:4               | 保持角色差异化的同时允许构筑灵活性    |
