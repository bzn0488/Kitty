# 游戏平衡性设计

> 综合千水《游戏创作心灵鸡汤》#2、#18 及行业实践经验。

## 一、平衡性的核心目的

**平衡 ≠ 公平。** 平衡的目的是让选择变得有价值，避免变成霍布森选择。

街头小子（最早期格斗游戏）展示了一个教训：
- 绝对平衡 = 只有一个角色 → 不好玩
- 平衡的目的是让尽可能多的**选择有意义**，而不是让所有选项完全相等

### 平衡性调整的两个方向

| 调整对象   | 含义                   | 何时使用                           |
| :--------- | :--------------------- | :--------------------------------- |
| **削数值** | 降低伤害/血量/CD等数据 | 优先使用。保留英雄特色             |
| **削机制** | 移除或重做技能效果     | 仅当该机制过于强大或毫无乐趣时使用 |

> 机制=特色，砍机制=砍特色。数值可以调，机制应慎动。

## 二、BP（Ban/Pick）作为难度平衡机制

BP 系统不仅是玩家博弈，更是**游戏设计层面的难度平衡工具**：

| 环节       | 设计功能                                 |
| :--------- | :--------------------------------------- |
| **Ban**    | 让玩家有权移除过于强势或自己不喜欢的选项 |
| **先Pick** | 先选方获得优势，后选方获得Counter机会    |
| **后Pick** | 通过反选机制平衡先选优势                 |

BP 使得：即使版本存在强势选项，玩家间也能通过博弈达到动态平衡。

## 三、胜率 ≠ 平衡

**50% 胜率不代表平衡。** 数据会骗人——多种因素影响胜率：

| 干扰因素     | 说明                                           |
| :----------- | :--------------------------------------------- |
| **人气差异** | 冷门英雄的少量数据样本会产生偏差               |
| **掌握程度** | the shy 的阿卡丽 vs 普通玩家的阿卡丽——天壤之别 |
| **上手难度** | 简单英雄低分段胜率天然更高                     |
| **段位差异** | 同一个英雄不同段位表现不同                     |

> 正常的表现是：一个英雄在不同段位 50% 上下浮动。**不存在精确的"平衡线"。**

---

## 四、归因理论与玩家抱怨

海德（1958）归因理论在游戏中的体现：

| 结果     | 玩家归因             | 心理机制           |
| :------- | :------------------- | :----------------- |
| **赢了** | "我厉害"             | 成功归因于自身能力 |
| **输了** | "版本不平衡""队友菜" | 失败归因于外部因素 |

这是正常的心理防御机制。设计师应理解而非对抗。

---

## 五、平衡性的终极困境

即使游戏真的达到绝对平衡，也**不能停止更新**——因为：
- 公平不是游戏的第一追求
- **持续更新带来的新鲜感**才是
- 版本更替本身也是游戏体验的组成部分

> **结论**：平衡不是终点，而是持续的过程。平衡性调整应该让选择更多、更有意义，而不是让所有选项趋于同质。
