# 渐进式生成内容设计

> 来源：千水《游戏创作心灵鸡汤》#29 星空发售前分析——对贝赛思达开放世界设计理念的预测与推演。

## 一、核心理念

不同于传统的 "手工铺地图 + 在地图上标记事件"，渐进式生成内容是一种**根据玩家行为动态生成内容**的方式。

### 传统方式 vs 渐进式生成

| 维度       | 传统开放世界       | 渐进式生成                       |
| :--------- | :----------------- | :------------------------------- |
| 地图       | 手工铺设，固定位置 | 结构固定，内容动态分配           |
| 事件       | 固定位置触发       | 根据玩家状态从库中挑选           |
| 内容利用率 | 玩家错过就无法体验 | 未触发的事件可重复在其他地点出现 |
| 承载力     | 受限于手工工作量   | 可通过扩充库来提升               |

---

## 二、与传统方式的对比

### 传统开放世界（2D地图模型）

```
地图=一个大正方形
├── 任务A（固定位置）
├── 宝箱B（固定位置）
├── 野怪营地C（固定位置）
└── ...
```

问题：所有内容固定→玩家不走回头路就错过了→内容浪费

### 渐进式生成（星际地图模型）

```
地图=两层结构
├── 星球层：1000+个星球（结构固定）
│   └── 每个星球有标签（气候温和/有大气层/敌对势力等）
└── 星系层：连接星球的太空
    └── 跃迁→进入新星系→根据玩家状态生成内容
```

---

## 三、渐进式生成的三个优势

### 1. 解决支线悖论

玩家进行主线时，主线相关事件**直接出现在玩家身边**（而非固定在地图上某一点）：

> 例：飞船燃料泵被盗窃 → 需要夺回 → 不然不能离开当前星球

利用"无法离开"的设计，将主线融入玩家的日常探索中。

### 2. 内容承载力

手工制作的内容是有限的。渐进式生成允许：
- 从内容池中按规则挑选
- 未使用的内容可以重复分配
- Mod 作者可以轻松添加新内容到池中

### 3. 不浪费内容

传统开放世界中，大部分内容玩家都看不到。渐进式生成确保：
- 未触发的事件可以在其他星球/场景重复出现
- 同一批佣兵可以在不同地点遇到（符合严肃性："他们有飞船"）

---

## 四、设计挑战

| 挑战         | 解法                                       |
| :----------- | :----------------------------------------- |
| 避免重复感   | 需要有足够大的内容库和丰富的组合规则       |
| 保持叙事连贯 | 事件之间需要有关联链条（而非独立的碎片）   |
| 控制体验密度 | 根据玩家最近的体验类型来调整生成内容的类型 |
| 动态难度     | 生成的难度需匹配当前玩家强度               |
