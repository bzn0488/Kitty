## 二、核心玩法架构

### 2.1 游戏模式：异步PvP + Roguelite构筑

The Bazaar 最大的创新在于将**异步PvP**与**Roguelike构筑**结合——玩家在不同时间构建自己的阵容，完成后与其他玩家的"镜像"自动对战：

```
选英雄 → 进入大巴扎（商店/事件）→ 构筑10格棋盘配装 → PvP对战（异步）
    ↑                                                         │
    └──────────────────── 继续推进/失败结算 ───────────────────┘
```

**与杀戮尖塔的关键差异**：
- 不是打预设AI怪物，而是与其他真实玩家的构筑成果对战
- 无法"背板"——每次PvP都面对不同的构筑
- 构筑深度要求显著更高

### 2.2 物品系统（核心创新）

The Bazaar 使用**物品（Item）**而非传统卡牌，这是与杀戮尖塔式DBG的核心差异：

| 维度     | 传统卡牌（杀戮尖塔） | The Bazaar 物品                 |
| :------- | :------------------- | :------------------------------ |
| 表现形式 | 手牌，消耗性使用     | 装备在棋盘的永久物品，持续生效  |
| 交互方式 | 回合制打出手牌       | 自动触发（类似自走棋）          |
| 构筑维度 | 卡牌加入牌库         | 物品放入10格棋盘                |
| 成长方式 | 获得新牌、删旧牌     | 物品升级（青铜→白银→黄金→钻石） |
| 协同方式 | 牌序+效果联动        | 物品之间的持续效果联动          |

### 2.3 节奏：异步对战 → 无实时压力

- 构筑决策**不受时间压迫**
- 玩家可以充分思考每步决策
- 体现了 `22-权力下放设计.md` 中"游戏节奏权让渡"的原则

## 三、六层核心系统分析

### 3.1 声望系统（Prestige）——外层决策引擎

| 机制     | 说明                   |
| :------- | :--------------------- |
| 初始声望 | 若干点（决定"容错率"） |
| PvP失败  | 损失声望               |
| 胜场目标 | 累计10胜               |
| 主动退出 | 随时提现当前进度       |
| 结算条件 | 10胜 或 声望归零       |

**设计价值**：声望系统实质上是将**风险决策**引入了构筑游戏的外层循环：

```
失败 → 扣声望 → 玩家决策：现在提现（确定收益）vs 继续赌（更大收益+可能全输）
```

这个决策本身创造了持续的策略乐趣密度，类似于扑克中的筹码管理。

> 与 `07-分段奖励设计.md` 关联——分段的另一种形式：由玩家主动决定提现时机。

### 3.2 物品品质与升级系统

物品分为四个品质等级：

| 等级                | 效果                      |
| :------------------ | :------------------------ |
| **青铜（Bronze）**  | 基础效果                  |
| **白银（Silver）**  | 数值提升                  |
| **黄金（Gold）**    | 数值提升 + 额外效果       |
| **钻石（Diamond）** | 峰值强度 + 多重释放等质变 |

升级不仅提升数值，还可能解锁额外效果、减少冷却、增加多重释放等。这是`04-叠层设计与延迟满足.md`中"叠层+质变"的经典实践。

> 与 `项目经验/07-增量自走棋-升星机制设计.md` 的"三档升星效果"理念一致——升级应该改变决策逻辑，而非纯数值。

### 3.3 10格棋盘——空间资源管理

玩家拥有10格棋盘，需要在此有限空间中配置物品：

| 管理维度     | 说明                                       |
| :----------- | :----------------------------------------- |
| **物品类型** | 武器/工具/药水/技能/恐龙等                 |
| **物品冷却** | 每个物品有独立冷却，时序排列影响输出节奏   |
| **协同触发** | 物品之间通过效果联动，位置关系影响触发顺序 |

10格是**有意义的稀缺资源**——放什么、放哪、升级哪个，都是持续的取舍决策。

### 3.4 物品类型体系

| 类型                    | 功能             | 设计要点             |
| :---------------------- | :--------------- | :------------------- |
| **武器（Weapon）**      | 攻击性物品       | 冷却+伤害的基准模型  |
| **工具（Tool）**        | 功能性物品       | 减CD、增益、辅助     |
| **药水（Potion）**      | 一次性/消耗品    | 高回报、一次性使用   |
| **技能（Skill）**       | 全局被动效果     | 不占棋盘格，永久生效 |
| **附魔（Enchantment）** | 附加到物品的增强 | 增加物品的额外维度   |
| **恐龙（Dinosaur）**    | 特殊生物类       | 高成长性、大后期     |

### 3.5 英雄系统——差异化构筑池

| 英雄                           | 风格倾向         | 获取方式   |
| :----------------------------- | :--------------- | :--------- |
| **Vanessa（初始）**            | 平衡型，新手友好 | 免费       |
| **Pygmalian**                  | 商人/经济型      | 免费       |
| **Dooley**                     | 机械/工具型      | 免费       |
| **Karnok（卡诺克）**           | 怒火/猎人型      | DLC（¥76） |
| **Celebrity Chef Warrior Mom** | —                | DLC（¥76） |
| **Ingenious Engineer**         | —                | DLC（¥76） |
| **Ancient Alchemist**          | —                | DLC（¥76） |

每个英雄拥有 **100+ 独特物品**，物品池独立 → **不同英雄玩的是不同游戏**。这与 `25-杀戮尖塔卡牌设计分析.md` 中四角色差异化的设计理念一致。

### 3.6 商人/事件系统——构筑决策的"节奏点"

The Bazaar 的核心交互是**逛市场**——一个不断变化的选择空间：

| 交互类型 | 功能                   | 设计价值             |
| :------- | :--------------------- | :------------------- |
| **商人** | 出售随机物品，各有所长 | 创造"买不买"的决策   |
| **事件** | 全配音奇遇，分支选择   | 增加叙事层，丰富体验 |
| **升级** | 提升物品品质           | 长期目标驱动         |
| **附魔** | 为物品附加额外效果     | 增加构筑的垂直深度   |

## 四、"对轴"理论——涌现式构筑的Bazaar版

B站攻略中提出的"对轴"（冷却同步）概念，与 `11-机制轴设计方法.md` 高度吻合：

### 4.1 五大轴

| 轴              | 依赖因素       | 设计要点                    |
| :-------------- | :------------- | :-------------------------- |
| **时间轴**      | 物品冷却同步   | 冷却相近的物品同时触发→爆发 |
| **击杀轴**      | 击杀→减CD/触发 | 高攻速+击杀联动             |
| **资源轴**      | 金币→商人刷新  | 经济管理→更优构筑           |
| **冻结/缓速轴** | 打乱敌方时间轴 | PvP独有的对抗维度           |
| **附魔轴**      | 物品×附魔组合  | 垂直深化的协同维度          |

### 4.2 PvP独有的"轴对抗"

与单机Roguelike的不同——The Bazaar 中玩家不仅要优化自己的"轴"，还要**预判和打乱对手的轴**：

- 冻结、缓速、减速等机制直接作用于对手的时间轴
- 构筑时需要考虑"我的轴被干扰了怎么办"
- 这是 PvP 模式为构筑游戏带来的额外策略深度

## 五、"三步劝退"——新手体验的设计隐患

根据B站蜥蜴君50.7万播放的深度分析，The Bazaar 存在以下新手体验问题：

| 问题       | 表现                 | 设计根源                      | 对应知识库                          |
| :--------- | :------------------- | :---------------------------- | :---------------------------------- |
| **第一步** | 新玩家不知"该拿什么" | 物品池太大，Build信号不清晰   | `16-卡牌设计规范.md` — Build信号自检   |
| **第二步** | 构筑成型后仍输       | 异步PvP对手更强，缺乏难度曲线 | `20-难度调节方案.md` — 单机vsPvP的差异 |
| **第三步** | 不知道输在哪         | 无法看到对手构筑过程          | 信息透明的边界问题                  |

### 设计启示

异步PvP的**"归因困难"**是 The Bazaar 面临的核心设计挑战：
- 输给AI → "我操作/构筑不够好"（内部归因，可改进 → 胜力感）
- 输给异步对手 → 无法看到过程 → "对方运气好/匹配不公平"（外部归因 → 负反馈）
- 这解释了 Steam 评测从"多半好评"到近期"褒贬不一"的变化趋势
