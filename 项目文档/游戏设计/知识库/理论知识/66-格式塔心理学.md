# 格式塔心理学（Gestalt Psychology）

> 标签：`player-psychology`, `ui-design`, `perception`, `cognitive-bias`
> 关联概念：`36-心流通道.md`, `34-系统交互与体验调节.md`
> 来源：Wertheimer, Koffka, Köhler（20世纪初）
> 完整展开见：`游戏设计理论/02-玩家心理学.md §14.2`

## 核心命题

> **整体不等于部分之和**——人的感知天然倾向于将碎片信息组织为有意义的整体。

## 关键法则

| 法则          | 含义                       | UI应用                   | 关卡/系统应用               |
| :------------ | :------------------------- | :----------------------- | :-------------------------- |
| **接近律**    | 距离近的元素被视为一组     | 相关功能按钮在视觉上靠近 | 同一区域的敌人被视为"一组"  |
| **相似律**    | 相似的元素被视为一组       | 同类物品用统一视觉语言   | 相同等级/类型的怪物统一配色 |
| **闭合律**    | 未闭合的形被认为整体       | 残缺图标激发"补全"欲望   | 地图上未探索区域的暗示      |
| **连续律**    | 延续方向上的元素被视为一组 | 流程引导中的箭头/路径线  | 关卡路径上的视觉线索连贯性  |
| **图形-背景** | 焦点与背景的自动区分       | 重要目标从环境中"跳出来" | Boss从背景中突出显示        |

## 顿悟式学习（Insight Learning）

Köhler的黑猩猩实验：解决"悬挂香蕉"问题不是渐进的（如斯金纳箱），而是突然的"灵光一现"。

### 游戏设计中的应用

- 对应游戏中"我懂了！"的时刻（Build之间的协同发现、隐藏机制的领悟）
- 与玩具系统设计相关——当玩家自主发现系统间的关联时产生顿悟式正反馈
- 顿悟不可被强制引导，只能通过**提供足够的系统交互线索**来创造机会

## 功能固着（Functional Fixedness）

人很难把锤子当作锤子以外的东西——对应认知的"思维定势"。

### 游戏设计中的应用

- 打破功能固着 = 创造惊喜（如：《塞尔达》中磁铁能力不仅可以吸金属门，也可以吸敌人的盔甲）
- 中间层设计（物理属性总线）天然具有打破功能固着的能力
- 需要注意：过度打破功能固着可能导致玩家认知负担

## 审查要点（设计完成后自检）

| 问题                                           | 说明                             |
| :--------------------------------------------- | :------------------------------- |
| UI布局是否遵循接近律？相关功能是否视觉上靠近？ | 减少玩家的视觉搜索成本           |
| 同类元素是否用了统一的视觉语言？               | 降低认知分类负担                 |
| 是否有未闭合的形激发玩家"补全"欲望？           | 可用于任务提示/区域解锁          |
| 重要目标是否从环境中自然突出？                 | 避免玩家错过关键交互点           |
| 是否有系统在无意中触发了功能固着？             | 检查是否有被玩家忽视的交互可能性 |
| 是否存在虚假可供性（看起来能做但实际不能）？   | 这是"严肃性崩溃"的根源之一       |
