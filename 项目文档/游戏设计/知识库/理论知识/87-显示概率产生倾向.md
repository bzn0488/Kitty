# 显示概率产生倾向——概率信息对玩家心理的影响

> 标签：`player-psychology`, `probability`, `game-mechanics`, `decision-design`, `feedback`
> 关联概念：`84-单次检定最优概率80%.md`, `72-输入随机与输出随机.md`, `73-伪随机分布PRD.md`, `74-确定性vs随机性掉落设计.md`, `46-正反馈与负反馈循环.md`
> 来源：心理学实验（斯金纳箱延伸）+ 玩家行为观察——揭示了"概率信息本身如何影响玩家决策"

---

## 一、核心命题

**是否向玩家公示概率，决定了玩家的关注焦点和情感体验。**

### 实验现象

| 情况  | 信息呈现                                  | 玩家的关注点                 |
| :---- | :---------------------------------------- | :--------------------------- |
| **A** | "该卡牌背后只可能出现 1 和 5"             | 关注"出现什么"和"不出现什么" |
| **B** | "该卡牌有 80% 概率出现 1，20% 概率出现 5" | 关注**概率本身**             |

### 核心机制

一旦公示概率，玩家就会产生**倾向**（preference）：

```
显示概率 → 产生倾向 → 有倾向就有正反馈和负反馈
                ↓
        80% → 玩家倾向"出现 1"
        20% → 玩家不希望"出现 5"
```

---

## 二、心理学原理

### 2.1 锚定效应

玩家会以公示的概率作为**心理锚点**：
- 80% → "我应该抽到 1"（锚点建立）
- 抽到 1 → 符合预期（无情绪或轻微正反馈）
- 抽到 5 → 偏离预期（负反馈）

### 2.2 损失厌恶

当玩家将 80% 视为"应得"时，20% 的失败被感知为**损失**：
```
损失厌恶系数 ≈ 2×
"80% 概率失败"带来的沮丧 ≈ "20% 概率成功"带来的喜悦 × 2
```

### 2.3 归因转移

| 概率信息     | 成功时归因           | 失败时归因            |
| :----------- | :------------------- | :-------------------- |
| 不显示概率   | "我选对了"           | "我选错了"            |
| 显示概率(高) | "概率高所以我成功了" | "20%撞上了，运气不好" |

注意：显示概率在**高概率场景**保护玩家归因（失败归因于运气），但在**低概率场景**可能让玩家感到"被系统坑了"。

---

## 三、设计决策：显示 vs 不显示

### 3.1 显示概率的优缺点

| 优点                          | 缺点                             |
| :---------------------------- | :------------------------------- |
| 玩家感到信息透明 → 信任度提升 | 玩家产生概率倾向 → 负反馈更强烈  |
| 玩家可以做概率计算 → 策略决策 | 低概率时玩家可能放弃尝试         |
| 失败时归因于运气（保护自尊）  | 连续失败时信任崩塌（"伪随机？"） |
| 激活期待感 → 多巴胺分泌       | 过度关注数字 → 出戏              |

### 3.2 何时显示概率

| 场景              | 建议             | 理由                       |
| :---------------- | :--------------- | :------------------------- |
| 高概率(>70%)      | ✅ 显示           | 保护归因，失败归运气       |
| 中等概率(30%~70%) | ⚠️ 谨慎显示       | 玩家可能误判概率或感到不安 |
| 低概率(<30%)      | ❌ 隐藏或模糊提示 | 显示会让玩家感到"这不公平" |
| 策略决策需要      | ✅ 必须显示       | 如 Build 构筑中的暴击率    |
| 氛围/沉浸优先     | ❌ 隐藏           | 如《黑暗之魂》的隐藏命中率 |

### 3.3 概率呈现方式

| 方式         | 例子                           | 效果                     |
| :----------- | :----------------------------- | :----------------------- |
| **精确数字** | "80% 概率"                     | 最透明，但也最理性化     |
| **区间描述** | "高概率" / "极高概率"          | 保留模糊感，减少数字焦虑 |
| **视觉提示** | 进度条/星数/颜色               | 直觉化，无需计算         |
| **行为暗示** | 敌人的"疲惫"状态暗示成功率提高 | 融入情境，保持沉浸       |
| **不提示**   | 隐藏概率                       | 维持神秘感，减少理性分析 |

## 四、应用案例

### 4.1 显示概率 → 创造正负反馈

在**三选一优化**（`16-卡牌设计规范.md`）中，显示概率可以：

```
选项 A：80% 获得稀有物品，20% 获得普通物品
选项 B：50% 获得史诗物品，50% 获得普通物品
选项 C：必定获得稀有物品
```

- 玩家在 A 和 B 之间产生倾向
- 抽到想要的结果 → 正反馈（"我赌对了"）
- 抽到不想要的结果 → 负反馈（但不伤害自尊）

### 4.2 隐藏概率 → 维持神秘感

在**爬行者体系**（`82-爬行者体系.md`）中：

```
击杀怪物 → 掉落物品（概率隐藏）
```

- 玩家不会因"90% 掉落率却未掉落"而产生负反馈
- 掉落的惊喜感更强（多巴胺在"获得"时而非"期待"时释放）
- 但对追求效率的核心玩家来说，隐藏概率可能造成烦躁

### 4.3 混合使用

**预期管理 + 结果惊喜**的混合策略：

```
显示："该宝箱有较高概率开出好物品"（模糊高概率）
实际：80% 好物品，20% 普通物品
结果：好物品 → "果然"（维持预期）
      普通物品 → "运气差了点"（归因运气）
```

## 五、设计原则总结

|   #   | 原则                   | 说明                                                   |
| :---: | :--------------------- | :----------------------------------------------------- |
|   1   | **概率即倾向**         | 显示概率一定会让玩家产生倾向和情感反应，不是中立信息   |
|   2   | **高概率显、低概率隐** | >70% 显示保护归因，<30% 隐藏避免负反馈                 |
|   3   | **策略必要则显**       | 需要玩家做概率计算的场景必须显示                       |
|   4   | **概率+PRD=平滑体验**  | 公示概率 + 伪随机分布减少极端情况                      |
|   5   | **文字>数字**          | 非策略场景用模糊描述代替精确数字，维持沉浸             |
|   6   | **失败时的归因保护**   | 无论是否显示概率，都要确保玩家失败时不归因于"系统不公" |
