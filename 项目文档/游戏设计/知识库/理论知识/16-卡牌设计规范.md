# 卡牌设计规范

> 综合自：千水《电竞教父》项目经验、知识库已有项目经验、MTG 等经典卡牌游戏设计原则。

## 一、卡牌经济系统——费用曲线

卡牌的费用（Cost）是卡牌设计中最核心的平衡维度。合理的费用曲线决定了游戏的节奏健康度。

### 费用曲线原则

| 费用            | 效果预期                       | 设计要点                                           |
| :-------------- | :----------------------------- | :------------------------------------------------- |
| **低费（1-2）** | 节奏铺垫、过牌、小幅度场面影响 | 不能直接决定胜负，但不影响战局                     |
| **中费（3-5）** | 核心战术执行、关键场面转折     | 决定对局走向的区间，需要合理分配                   |
| **高费（6+）**  | 终结比赛、大场面翻盘           | 必须足够强大到配得上其费用，但不应让低费卡完全无用 |

### 费用模型的三种类型

| 类型         | 代表               | 特点                   | 设计注意                       |
| :----------- | :----------------- | :--------------------- | :----------------------------- |
| **线性增长** | 每费固定增长属性   | 简单直白，易于平衡     | 容易导致高费碾压低费           |
| **指数增长** | 高费卡效果远超低费 | 后期卡牌感觉强大       | 需要防止低费卡完全失去存在意义 |
| **功能型**   | 费用不直接对应强度 | 效果独特，无法简单量化 | 需要大量测试来评估平衡性       |

### 费用曲线与游戏节奏

```
费用曲线图（理想分布）
       ↑
       │    ╱╲
       │   ╱  ╲
       │  ╱    ╲
       │ ╱      ╲
       │╱        ╲
       └────────────────→ 费用
         1 2 3 4 5 6 7
```

理想分布：中费卡占主体（约 50%），低费和高费递减（各约 25%）。这确保游戏中后期都有可用的选择。

## 二、三选一优化原理

> 整合自 `03-游戏机制设计技法.md` §1.12。

在卡牌和随机 Build 游戏中，三选一是一种经过验证的核心交互模式：

### 三选一的三个设计效果

| 效果           | 说明                                      |
| :------------- | :---------------------------------------- |
| **限制信息量** | 每次只呈现 3 个选项，玩家不会面临信息过载 |
| **简化评估**   | 3 选 1 比从大量牌中挑选更易于比较优劣     |
| **增加期待感** | 每次选择都是一次"小开奖"——可能出想要的牌  |

### 三选一的变体

| 变体             | 说明                       | 例                     |
| :--------------- | :------------------------- | :--------------------- |
| **标准三选一**   | 3 张牌中选 1 张            | 杀戮尖塔               |
| **二选一**       | 更快的决策，适合快节奏     | 部分手游               |
| **多选一（4+）** | 选项更多，但增加了认知负担 | 需配合选项数量限制机制 |

## 三、DBG（牌库构筑游戏）的核心设计循环

DBG 区别于传统卡牌游戏（如炉石、MTG）的关键：**战斗中获得新牌，而非赛前提前组好卡组。**

### DBG 的独特设计挑战

| 挑战         | 原因                                             | 解法               |
| :----------- | :----------------------------------------------- | :----------------- |
| **牌库膨胀** | 每次拿牌都加入卡组，后期牌库膨胀导致关键牌抽不到 | 删牌机制、烧牌机制 |
| **路径依赖** | 构筑成型后不愿意换组件（正反馈→强度固化）        | 设软上限、递增成本 |
| **认知负荷** | 牌越多，每次决策的选项就越多                     | 三选一、删牌奖励   |
| **随机方差** | 牌序影响远大于传统卡牌                           | 过滤机制、检索机制 |

### DBG 与经济卡的互动

经济卡在 DBG 中的定位：

| 经济卡类型    | 作用             | 设计注意                                   |
| :------------ | :--------------- | :----------------------------------------- |
| **增加手牌**  | 扩充选择范围     | 不能直接增加强度，否则会挤压战斗卡空间     |
| **减少费用**  | 加速高费卡出场   | 需要检查是否会导致某些卡牌过早登场破坏平衡 |
| **复制/回收** | 重复利用强力卡牌 | 最容易导致平衡失控的类型                   |

## 四、卡牌类型学

### 按功能分类

| 类型       | 目的               | 设计规范                         |
| :--------- | :----------------- | :------------------------------- |
| **核心**   | 构成主力Build方向  | 每局至少见到 2-3 张同类型卡      |
| **支援**   | 辅助核心卡发挥作用 | 不能独立存在，必须与核心卡配合   |
| **经济**   | 管理资源/抽牌/减费 | 与战斗卡分路设计，不兼得         |
| **消耗品** | 一次性效果         | 价格与效果强度正相关，稀有度递增 |

### 按稀有度分类

稀有度体系的基本原则：

| 稀有度            | 出现率 | 强度预期            | 复杂度 |
| :---------------- | :----- | :------------------ | :----- |
| 普通（Common）    | 高     | 基础强度            | 低     |
| 稀有（Rare）      | 中     | 中等强度+独特效果   | 中     |
| 史诗（Epic）      | 低     | 高强度+独特机制     | 高     |
| 传说（Legendary） | 极低   | Build核心或终结手段 | 最高   |

> **稀有度 ≠ 最强**：稀有度更多代表机制独特性和复杂度，而非绝对强度。普通卡也有其不可替代的作用（基础曲线填充、兜底功能）。

## 五、卡牌的自检标准

### 自闭环检测（来自项目经验）

移除场上所有其他单位后，该卡是否仍能满额运作？
- 能 → ❌ 自闭环（删除或重做）
- 不能 → ✅ 需要队友参与

### 诅咒卡检验

> "如果三选一里只有这一张，我会选它吗？"

- 不会 → ❌ 这张卡是"诅咒卡"，根本不会有人选
- 会（即使很弱）→ ✅ 及格

### 协同效应检验

> 源自 `项目经验/协同设计经验.md`，补充自 `35-协同效应.md` §涌现式协同

- 协同是否基于共享中间层（标签/属性类型/玩家行为），而非指名卡牌联动？
- 协同是"说明书式"（堆Tag）还是"涌现式"（找行为）？（参考 `35-协同效应.md`）
- 玩家是否有决策空间？
- 是否存在通解/最优解？
- 协同强度的上限是否可控？

### Build信号自检

> 源自 `项目经验/07-增量自走棋-升星机制设计.md`

设计完成时自问：**"在三选一里看到这张卡时，玩家脑中会浮现什么构筑方向？"**

- 有明确方向 → ✅ 通过
- 没有方向 → ❌ 需要重写

衡量标准：
- ✅ **好信号**："周围每有一个同Tag友军就变强" → 指向"多找同Tag单位"
- ❌ **坏信号**："相邻有友军时减伤" → 不指向任何特定搭配
- ❌ **坏信号**："运气好很猛，运气差也不亏" → 运气不改变Build方向
