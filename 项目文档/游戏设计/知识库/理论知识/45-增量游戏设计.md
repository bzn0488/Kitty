# 增量游戏（Incremental Game）设计

> 基于维基百科资料及增量自走棋项目经验整理。

## 一、什么是增量游戏

增量游戏（Incremental Game，又称 Idle Game / 放置游戏 / 挂机游戏）是一种以**持续自动产出资源**为核心机制的游戏类型。玩家投入少量操作启动资源生产，然后游戏会自动运行积累资源，玩家再消耗资源解锁升级，形成循环。

### 核心特征

| 特征                 | 说明                                                   |
| :------------------- | :----------------------------------------------------- |
| **自动资源生成**     | 游戏在玩家不在线时也持续产出                           |
| **点击/操作加速**    | 主动操作可临时提高产出速度（如 Cookie Clicker 的点击） |
| **升级解锁层**       | 消耗资源解锁升级，提高效率或解锁新系统                 |
| **Prestige（重置）** | 重置游戏，保留部分关键进度以获得永久加成               |
| **指数增长**         | 产出和成本呈指数级增长，后期数字快速增长               |

---

## 二、核心循环

增量游戏的核心循环是典型的**正反馈循环**：

```
资源产生 → 资源积累 → 消耗升级 → 效率提升 → 资源产生加速
     ↑                                          |
     └──────────────────────────────────────────┘
```

### 循环设计要点

| 环节         | 设计要点                                         |
| :----------- | :----------------------------------------------- |
| **资源产生** | 基础产出要持续可见（数字滚动），给玩家稳定反馈   |
| **资源积累** | 展示累积量、每秒产出（per second）是核心数据     |
| **消耗升级** | 性价比梯度——前期便宜提升快，后期昂贵但解锁新机制 |
| **效率提升** | 乘法叠加效果最好（×2、×3），加法容易快速疲劳     |
| **新层解锁** | 每隔一段时间解锁新的资源/系统，维持新鲜感        |

---

## 三、Prestige（重置）机制

Prestige 是增量游戏的核心深度来源——玩家主动放弃当前进度以换取永久加成。

### 设计原则

| 原则             | 说明                                     |
| :--------------- | :--------------------------------------- |
| **保留关键资产** | 重置后保留部分解锁项（如特殊升级、成就） |
| **加速后续循环** | 重置后基础速度更快，能更快到达上次的进度 |
| **解锁新系统**   | 重置到一定次数后解锁新的玩法层           |
| **自愿原则**     | 让玩家自己决定何时重置，而不是强制       |
| **重置收益递增** | 每次重置的收益应比上次更高（但不是线性） |

### 常见问题

- **重置后失去动力** — 如果重置后玩家感觉"一切又从头开始"，会失去动力。需要确保重置后前期的节奏明显快于初次游玩
- **最佳重置时机** — 玩家需要能判断何时重置最优（通常边际收益递减时），需要通过 UI 和数据引导

## 四、成长曲线设计

### 常见曲线类型

| 曲线         | 特点             | 适用场景      |
| :----------- | :--------------- | :------------ |
| **线性增长** | 每级增加固定数值 | 基础属性      |
| **指数增长** | 每级乘以固定倍数 | 核心产出      |
| **对数增长** | 越往后收益越低   | 边际收益递减  |
| **S型曲线**  | 慢→快→慢         | 软上限/天花板 |

### 软上限（Soft Cap）

增量游戏的关键设计技巧：**不是阻止玩家变强，而是让变强的速度逐渐放缓。** 例如：

```
等级 1-10: 每级 +100% 产出（快 → 爽）
等级 10-20: 每级 +50% 产出（中 → 持续）
等级 20-50: 每级 +10% 产出（慢 → 推动 Prestige）
```

### 里程碑与峰谷

增量游戏容易变得单调，需要设计**峰谷体验**：

| 手段           | 说明                                           |
| :------------- | :--------------------------------------------- |
| **里程碑事件** | 达到一定资源量时触发特殊事件/解锁              |
| **阶段性成就** | 达成目标给予一次性大额奖励                     |
| **惊喜事件**   | 低概率触发的高收益事件                         |
| **内容解锁**   | 每进入新阶段解锁全新系统（如从农场→工厂→太空） |

## 五、与知识库已有理论的关联

### 快乐密度理论

增量自走棋的"分母最小化"策略，本质上是增量游戏设计的核心理念的体现——**去除所有无效时间，让每一秒都在积累价值**。

| 增量游戏                    | 快乐密度理论                             |
| :-------------------------- | :--------------------------------------- |
| 自动战斗 ≈ 砍掉执行层       | 分母最小化——不让玩家花时间在无趣的操作上 |
| 离线收益 ≈ 时间利用率最大化 | 即使不玩也在积累快乐                     |
| Prestige ≈ 快乐总量重新分配 | 重置后更快达到之前的快乐峰值             |

### 与 Roguelike 的对比

| 维度           | Roguelike              | 增量游戏               |
| :------------- | :--------------------- | :--------------------- |
| **核心驱动力** | 每次不同的新体验       | 持续增长的满足感       |
| **永久死亡**   | 失去一切重新开始       | Prestige 保留部分进度  |
| **节奏**       | 回合制，玩家控制节奏   | 自动运行，玩家干预节奏 |
| **失败体验**   | 失败是学习过程的一部分 | 没有失败，只有快/慢    |
