# 去目标化指引设计

> 来源：千水知乎《如何做到去目标化的指引》
> 关联：`理论知识/22-权力下放设计.md` `理论知识/36-心流通道.md` `理论知识/58-去显性选择设计方法.md`

---

## 一、传统目标化指引的问题

传统游戏依赖**显性目标**来驱动玩家：

```
任务：杀死10只狼
奖励：100金币
→ 玩家知道该做什么，但缺乏情感动机
```

**问题**：
- 玩家变成"任务完成机器"，机械地执行指令
- 目标的达成感替代了体验本身
- 目标之间的情感是断裂的——玩家从一个目标跳到下一个，没有连贯的体验
- 公式化的目标设计是千水所厌恶的"公式化游戏"的核心问题

---

## 二、去目标化的核心思想

去目标化不是"没有目标"，而是**目标从玩家的自发动机中产生，而非外部强加**。

| 维度         | 目标化指引          | 去目标化指引 |
| :----------- | :------------------ | :----------- |
| **目标来源** | 系统/任务列表       | 玩家自发产生 |
| **驱动力**   | 外部奖励            | 内在好奇心   |
| **情感**     | 短暂满足→下一个目标 | 持续的探索欲 |
| **玩家状态** | 被动执行            | 主动探索     |
| **经典案例** | 传统RPG任务链       | 旷野之息     |

---

## 三、去目标化的实现方法

### 3.1 引力式设计（旷野之息）

用视觉/物理/好奇心的引力替代任务列表：

| 引力类型       | 表现形式           | 效果               |
| :------------- | :----------------- | :----------------- |
| **视觉引力**   | 远处的塔/神庙/光柱 | "那是什么？去看看" |
| **物理引力**   | 高山/河流/山谷     | "怎么过去？"       |
| **好奇心引力** | 可疑的地形/NPC行为 | "为什么会这样？"   |

### 3.2 环境叙事

让世界本身讲述故事，而非任务文本：

- 废墟中的日记碎片 → 玩家自己拼凑故事
- 战场的遗留物 → 玩家推测发生了什么
- NPC的日常行为 → 世界有"生机"

### 3.3 玩家驱动的目标生成

玩家基于自己的观察和兴趣自行制定目标：

```
玩家：看到一座高山 → 目标：爬上去看看有什么
玩家：发现一个洞穴 → 目标：探索里面有什么
玩家：遇到一个NPC → 目标：了解这个人的故事
```

## 四、去目标化的风险与对策

| 风险           | 表现                 | 对策                           |
| :------------- | :------------------- | :----------------------------- |
| **玩家迷茫**   | 不知道做什么         | 用引力引导，而非任务强制       |
| **缺乏方向感** | 失去推进的动力       | 设置宏观目标（拯救/探索/收集） |
| **体验碎片化** | 探索缺乏连贯性       | 区域设计有主题和层次           |
| **难度失控**   | 玩家过早进入高难区域 | 软性阻拦（高级怪物/恶劣环境）  |

---

## 五、设计自检

- [ ] 玩家能否通过观察环境自行得出目标？
- [ ] 是否有明确的视觉/物理/好奇心引力引导玩家？
- [ ] 玩家是否知道"接下来去哪里"而不需要看任务列表？
- [ ] 探索是否有有效的回报（非纯数值）？
- [ ] 玩家能否在1分钟内说出"我想做什么"（而非"任务要我做什么"）？
