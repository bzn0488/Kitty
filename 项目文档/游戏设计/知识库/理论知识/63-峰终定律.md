# 峰终定律（Peak-End Rule）

> 标签：`player-psychology`, `happiness-total`, `experience-design`
> 关联概念：`18-快乐总量.md`, `33-无效进程与三层乐趣体系.md`, `43-游戏节奏设计.md`
> 来源：Daniel Kahneman & Barbara Fredrickson（1993）
> 完整展开见：`游戏设计理论/02-玩家心理学.md §十六`

## 核心定义

人们对一段体验的整体评价，**不取决于体验的总和或平均值**，而主要取决于两个时刻：

1. **峰值（Peak）**：体验中情绪强度最高的瞬间
2. **终值（End）**：体验结束时的感受

即：**记忆 ≈ 峰值 + 终值的平均**，而非体验的总时长或总快乐量。

## 实验证据

### 冰水实验（Kahneman et al., 1993）

| 条件 | 过程                                   | 总痛苦时间 | 回顾评价 |
| :--- | :------------------------------------- | :--------- | :------- |
| A组  | 14°C冰水 60秒                          | 60秒       | 更差     |
| B组  | 14°C冰水 60秒 → 缓慢升温至15°C再浸30秒 | **90秒**   | **更好** |

结论：**更长但结尾改善的体验，被评价为更好**——时长被忽略（duration neglect），峰终值主导记忆。

### 结肠镜检查实验（Redelmeier et al., 2003）

结肠镜多留3分钟（不移动、不疼但不适）的患者，对体验的评价反而更好，且更愿意回访。

## 时长忽略（Duration Neglect）

峰终定律的一个重要推论：体验的**持续时间**对事后评价影响极小。

### 设计含义

- **不一定要缩短糟糕的体验**——如果能优化峰值和结尾，更长的体验反而可能被记住更好
- **不要依赖"总时长"来累积快乐**——玩家记忆的是高光时刻，不是平均时长

## 游戏设计应用

### 关卡/章节结尾设计

| 策略         | 做法                           | 示例                          |
| :----------- | :----------------------------- | :---------------------------- |
| **结尾高潮** | 在关卡末尾放置最具冲击力的内容 | Boss战放在关卡最后            |
| **结尾安抚** | 艰苦战斗后给予平静的"收尾时刻" | 胜利后的篝火对话/营地场景     |
| **避免烂尾** | 不要在高负反馈后立即结束       | 失败后先给复活/重试选项再结算 |

### 日常体验设计

| 场景          | 峰值设计              | 终值设计                          |
| :------------ | :-------------------- | :-------------------------------- |
| **每日签到**  | 累计奖励的高光时刻    | 签到完成后的确认动画+明日预告     |
| **一局游戏**  | 关键决策/精彩操作时刻 | 结算界面的正反馈展示（亮点回放）  |
| **章节/任务** | 剧情高潮/重大奖励     | 任务完成时的阶段性总结+下一步预告 |

### 与海明威效应的配合

海明威效应（积极离场）本质上是利用峰终定律：在**峰值之后、感觉良好时**主动结束——让玩家记住的是巅峰时刻，而非疲劳时刻。

## 审查要点（设计完成后自检）

| 问题                                           | 目的                 |
| :--------------------------------------------- | :------------------- |
| 当前关卡/任务的**峰值**在哪里？是否足够强？    | 确保有明确的高光时刻 |
| 结束时刻玩家感受如何？是满足还是疲惫？         | 优化终值体验         |
| 是否有不应该出现的负向峰值？是否可避免？       | 移除意外的糟糕体验   |
| 时长是否被过度填充导致负向峰值稀释了正向峰值？ | 检查无效进程         |
