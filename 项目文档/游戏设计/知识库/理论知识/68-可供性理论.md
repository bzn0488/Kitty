# 可供性理论（Affordance Theory）

> 标签：`ui-design`, `player-psychology`, `perception`, `interaction-design`
> 关联概念：`66-格式塔心理学.md`, `34-系统交互与体验调节.md`, `41-涌现式设计.md`
> 来源：James J. Gibson（1966/1979）；Donald Norman（1988）《设计心理学》
> 完整展开见：`游戏设计理论/02-玩家心理学.md §14.3`

## 定义

**可供性（Affordance）**：环境"提供"给行为体的行为可能性——它依赖于行为体的能力和意图，是**关系性**的而非绝对的。

> 一把椅子"可供"坐——但只有对能坐的人来说如此；对婴儿来说它不"可供"坐。

## Norman的四种可供性分类

| 类型           | 含义                    | 游戏中的例子           | 设计建议                               |
| :------------- | :---------------------- | :--------------------- | :------------------------------------- |
| **真实可供性** | 物理/逻辑上真的能做的事 | 一扇可以打开的门       | 确保视觉反馈与实际行为一致             |
| **感知可供性** | 玩家**认为**能做的事    | 门的把手暗示可以拉     | 利用设计线索引导玩家行为               |
| **虚假可供性** | 看起来能做但实际不能    | 装饰性门但实际上打不开 | **⚠️ 避免**——这是"严肃性崩溃"的主要来源 |
| **隐藏可供性** | 能做但玩家不知道        | 鞋底可以开酒瓶         | 涌现式玩法的生长土壤                   |

## 游戏设计中的关键应用

### 虚假可供性的危害

虚假可供性是**严肃性崩溃**（Immersion Break）的核心原因之一：

- 玩家看到岩石 → 尝试用锤子砸 → 无效 → **不信任感**产生
- 玩家看到门 → 尝试打开 → 提示"无法交互" → 下次看到门不再尝试
- 这与你知识库中01章"岩石和锤子"的例子直接对应

**设计原则**：要么让交互可用，要么让不可交互的对象在视觉上明确区别于可交互对象。

### 隐藏可供性与涌现式玩法

隐藏可供性是涌现式设计的土壤：
- 玩家偶然发现系统间的意外交互 → 产生"我发现了"的正反馈
- 通过**中间层设计**（物理属性总线）创造隐藏可供性的空间
- 注意：隐藏可供性的发现过程应该符合**合理预期**——玩家发现后觉得"原来如此"而非"这怎么可能"

### 感知可供性的引导作用

通过设计线索引导玩家"无意识"地做出正确行为：
- 发光的可收集物品引导探索路径
- 敌人的朝向暗示攻击方向
- 场景中的颜色引导注意力焦点

## 审查要点（设计完成后自检）

| 问题                                       | 说明                                 |
| :----------------------------------------- | :----------------------------------- |
| 是否有虚假可供性？是否被玩家多次尝试？     | 装饰性门/岩石是最常见的问题          |
| 可交互对象是否有明确的可供性暗示？         | 发光/突出/符合直觉的交互提示         |
| 是否利用了隐藏可供性创造涌现玩法？         | 检查中间层设计中是否存在意外交互可能 |
| UI元素是否遵循了可供性设计原则？           | 按钮看起来可点击、滑块看起来可拖动   |
| 玩家第一次遇到新交互时，是否能凭直觉理解？ | 不应依赖说明书/教程来解释基础交互    |
