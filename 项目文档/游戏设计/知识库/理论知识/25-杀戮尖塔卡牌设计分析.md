# 杀戮尖塔（Slay the Spire）卡牌设计分析

> 对《杀戮尖塔》卡牌游戏设计体系的系统性分析，作为卡牌设计规范的具体案例参考。
> 
> 关联：`理论知识/16-卡牌设计规范.md` `理论知识/01-Roguelike设计规范.md` `理论知识/35-协同效应.md`

---

## 一、核心资源系统：能量（Energy/费用）设计

杀戮尖塔使用了极简的 0-3 费曲线体系，与常规卡牌游戏的费用曲线有本质差异：

### 费用体系特征

| 费用     | 占比         | 效果定位               | 典型设计要点                              |
| :------- | :----------- | :--------------------- | :---------------------------------------- |
| **0 费** | 较低         | 白嫖节奏、触发类效果   | 往往附带条件（如"消耗""固有"）或数值较低  |
| **1 费** | 主体（~40%） | 常规强度，构成主力卡组 | 数值基准：单体 6-9 伤害 / 格挡 5-8 点     |
| **2 费** | 主体（~35%） | 强效卡，核心战术组件   | 数值基准：单体 12-20 伤害 / 格挡 10-15 点 |
| **3 费** | 较低（~15%） | 终结技 / 强力效果      | 必须有足以改变战局的效果                  |

### 能量设计的减法思考

杀戮尖塔的能量系统是**减法设计的典范**：

- **固定上限**：每回合 3 费，玩家无需像 MTG/炉石那样计算增长曲线
- **聚焦决策**：3 费意味着每一张卡的选择都重要——不是"我能打什么"，而是"我该打什么"
- **Boss遗物抉择**：+1 能量的 Boss 遗物（如宝石核心、能量石）意味着牺牲其他可能性，是重要的取舍点

### 与卡牌设计规范的关系

对比 `16-卡牌设计规范.md` §一的费用曲线，杀戮尖塔属于**功能型费用模型**——费用不直接等比对应强度，而是通过卡牌效果的特殊性来体现价值。0 费卡的设计尤其体现了"费用是约束条件而非强度标尺"的理念。

## 二、四角色差异化设计体系

杀戮尖塔四个角色的卡牌设计展现了同一框架下截然不同的设计方向，是"**框架约束下的差异化**"的极好案例：

### 角色设计对比

| 角色                     | 核心机制         | 数值特征           | 决策重点                | 设计创新点                                                             |
| :----------------------- | :--------------- | :----------------- | :---------------------- | :--------------------------------------------------------------------- |
| **铁甲战士**（Ironclad） | 力量（力量倍率） | 高爆发、高格挡     | 爆发时机 / 血量资源化   | 首次建立"血量也是一种资源"的卡牌消耗逻辑                               |
| **静默猎手**（Silent）   | 敏捷 + 毒        | 过牌多、攻击频次高 | 牌序优化 / 毒层管理     | 毒叠层机制是"延迟满足"的典型（叠层→质变）                              |
| **故障机器人**（Defect） | 能量球 + 集中    | 球位管理、法术联动 | 球位排布 / 抓取时机     | 能量球是卡牌之外的**第二资源维**，突破传统卡牌交互模式                 |
| **观者**（Watcher）      | 姿态切换         | 爆发极强、攻守转换 | 姿态切换节奏 / 费用计算 | 姿态设计本质是**角色自身的状态机**——一张卡同时影响"你是谁"和"你做什么" |

### 设计启示

四个角色的设计验证了**协同效应**（`理论知识/35-协同效应.md`）的不同实现路径：

| 角色       | 协同模式                           | 示例                                     |
| :--------- | :--------------------------------- | :--------------------------------------- |
| 铁甲战士   | **数值乘区**：力量 × 多段攻击      | 突破极限 → 连续打击                      |
| 静默猎手   | **叠层累计**：层数 → 质变          | 毒雾 → 催化毒爆                          |
| 故障机器人 | **资源联动**：球体 × 集中值 × 释放 | 冰球护甲 → 闪电输出 → 充能球爆发         |
| 观者       | **状态切换**：姿态 × 对应卡牌      | 平静（攒费）→ 愤怒（爆发）→ 神格（终结） |

## 三、爬塔地图系统——自由的路径决策

爬塔地图是杀戮尖塔最具辨识度的系统，本质是**决策密度的外层结构**：

### 节点类型与决策

| 节点                    | 功能               | 决策信号                         |
| :---------------------- | :----------------- | :------------------------------- |
| **普通战斗（Monster）** | 金币 + 卡牌选择    | 升级打磨还是探索新牌             |
| **精英战斗（Elite）**   | 高难度 + 遗物掉落  | 风险 vs 收益——当前卡组是否够强   |
| **篝火（Rest Site）**   | 回血 或 升级卡牌   | 短期生存 vs 长期强化             |
| **商店（Shop）**        | 购买卡牌/遗物/删牌 | 决策点：删掉什么牌？买什么遗物？ |
| **神秘（Mystery）**     | 随机事件           | 不确定性带来的期待感             |
| **首领（Boss）**        | 稀有遗物 + 稀有卡  | 构筑成色检验                     |

### 地图设计的三个核心作用

1. **控制信息量**：每次只展示当前层 → 下一层，玩家不会被全图信息压垮
2. **制造资源权衡**：路线选择 = 优先级的具象化——爬血量还是拿遗物？打精英还是保血量？
3. **重玩性**：每次看到的地图不同，路径不同，每局的节奏感不同

## 四、意图系统——信息透明的策略博弈

意图系统是杀戮尖塔对卡牌战斗的**核心创新**，将传统卡牌游戏的"信息不完全博弈"转化为"信息透明的策略规划"。

### 意图系统的设计价值

| 传统卡牌问题                | 杀戮尖塔解法             | 设计效果                         |
| :-------------------------- | :----------------------- | :------------------------------- |
| 对手下一步不可知 → 被迫防守 | 显示怪物下一回合行动     | 玩家可以做**有信息支撑的决策**   |
| 运气成分过高                | 意图告知后，决策是确定的 | 失败更像"决策失误"而非"运气不好" |
| 难以制定策略                | 根据意图制定多回合计划   | 强化了**智力优越感**的正反馈     |

### 意图类型与响应策略

| 意图类型                   | 含义           | 玩家应对方针              |
| :------------------------- | :------------- | :------------------------ |
| **攻击意图**（红色箭头）   | 下回合攻击     | 格挡 / 减伤 / 控制        |
| **Buff意图**（绿色螺旋）   | 给自己加Buff   | 优先爆发输出 / 准备解Buff |
| **Debuff意图**（紫色箭头） | 给玩家加Debuff | 优先闪避 / 净化           |
| **召唤意图**               | 召唤小兵       | AOE清理 / 集火本体        |

### 意图系统的"父系"特征

意图系统是典型的**父系设计**（`08-父系与母系设计哲学.md`）：

- **信息先行**：怪物告诉你它要做什么 → 你选择应对 → 执行
- **过程清晰**：黑白分明，执行结果可预期
- **智力优越感**：正确的预判和应对带来强烈的正反馈

## 五、卡牌构筑的节奏控制

作为 DBG（牌库构筑游戏），杀戮尖塔在构筑节奏上有精妙的设计：

### 构筑阶段划分

| 阶段                | 楼层范围   | 构筑任务     | 设计重点                      |
| :------------------ | :--------- | :----------- | :---------------------------- |
| **前期（1-3层）**   | Act 1      | 确立基础方向 | 费用曲线搭建、核心卡筛选      |
| **中期（4-6层）**   | Act 1 后半 | 打磨协同方向 | 删掉无用牌、补充关键词联动    |
| **后期（Act 2起）** | Act 2-3    | 应对Boss检验 | 单一Build无法通吃，需要多功能 |

### 删牌机制的战略意义

杀戮尖塔的删牌设计是 DBG 中**解决牌库膨胀**（`16-卡牌设计规范.md` §三）的典范：

| 删牌方式         | 成本                        | 设计意图                          |
| :--------------- | :-------------------------- | :-------------------------------- |
| **商店删牌**     | 金币（递增：75→100→125...） | 经济约束——删掉打击/防御是重要投资 |
| **事件删牌**     | 事件条件                    | 偶发机会，不可控但有期待          |
| **转化删牌**     | 新牌替代旧牌                | 不损失卡牌数量但提升质量          |
| **Boss遗物删牌** | 牺牲其他遗物                | 高成本高回报                      |

**关键洞察**：删牌的经济学是杀戮尖塔最精妙的平衡点之一——一张"打击"（Strike）的基础卡只有 6 伤，删掉它意味着牌库更纯净、更强的卡更易上手。但金币有限，是删牌还是买遗物/买药水？这一永恒的权衡创造了持续的决策密度。

## 六、遗物系统——全局征效的Build定义器

遗物是卡牌之外的**第二构筑维度**：

### 遗物分层

| 稀有度       | 获取方式         | 设计定位                                 |
| :----------- | :--------------- | :--------------------------------------- |
| **普通遗物** | 普通战斗/商店    | 功能型辅助，小幅提升                     |
| **稀有遗物** | 精英/商店/Boss池 | Build定义者，改变玩法方向                |
| **Boss遗物** | 击败Boss         | 收益巨大但必有取舍（如+1费 vs 不能喝药） |

### 遗物的设计原则

1. **永续被动**：遗物一旦获得就永久生效，无需操作——降低了认知负荷
2. **协同强化**：遗物与卡牌构筑形成互补协同（`35-协同效应.md`）
   - 例：手里剑（每打 3 张攻击牌获得 +1 力量）→ 多段攻击卡的价值提升
3. **机会成本**：Boss遗物三选一 → 每次都是艰难抉择

## 七、篝火抉择——经典困境设计

篝火（Rest Site）是杀戮尖塔中最简单但最有效的**困境设计**之一：

### 回血 vs 升级

| 选项              | 短期收益       | 长期代价               | 心理模型               |
| :---------------- | :------------- | :--------------------- | :--------------------- |
| **休息（Rest）**  | 回复 30% 血量  | 少一次卡牌强化         | 安全感（能活到下一战） |
| **升级（Smith）** | 永久强化一张卡 | 当前和后续战斗风险更高 | 成长感（长远更强）     |

### 设计效果

- **制造紧张感**：血量不足时，"升级"的诱惑 vs "回血"的需求构成内心冲突
- **叙事投影**（`31-玩家投影与游戏肌理.md`）：玩家在篝火前的纠结是"我"与"我的卡组"之间的张力——我是相信我的卡组能过关，还是保命要紧？
- **差异化**：不同角色/构筑对篝火的需求不同——力量型战士可能更需要升级核心卡，毒猎手可能更需要回血稳节奏

## 八、进阶系统（Ascension）——难度作为奖励

| 层级   | 效果         | 设计意图                       |
| :----- | :----------- | :----------------------------- |
| A1     | 精英更多     | 更多遗物机会 + 更高风险        |
| A2-A20 | 逐层增加约束 | 难度逐步递增，每次只加一个变量 |
| A20    | 终局挑战     | 双Boss + 全部约束              |

进阶系统是**难度作为奖励解锁**（`游戏设计理论/04-游戏节奏与心流设计.md` §4.4）的经典实践——不是玩家变强了，而是玩家在清晰的约束下不断优化策略。

## 九、稀有度与卡牌轮换控制

### 每局卡牌池控制

杀戮尖塔并非一次性暴露全部卡牌池，而是通过分层获取控制信息量：

| 机制             | 效果                                            |
| :--------------- | :---------------------------------------------- |
| **每战三选一**   | 每次战斗后只看到 3 张卡 → 决策不 overload       |
| **跳过选项**     | 三选都不满意 → 跳过，不拿牌 → 牌库不膨胀        |
| **角色专属牌池** | 四个角色各有独立牌库 → 每个角色都是不同的游戏   |
| **稀有度轮换**   | 普通/稀有/史诗按比例出现 → 稀有卡是惊喜而非常态 |

### 可重玩性设计

杀戮尖塔的可重玩性来源于**三个维度的组合变异**：

```
卡牌池 (各角色专属) × 遗物随出 × 地图路径随机
         ↓
   每局不同的协同可能性
```
