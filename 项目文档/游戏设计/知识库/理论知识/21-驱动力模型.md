# 驱动力模型

> 标签：`player-psychology`
> 关联概念：`09-过度理由效应.md`, `48-自我决定理论SDT.md`, `37-信号与强化体系.md`, `13-渐进式生成.md`

## 行为选择模型

玩家的每个可选行为都有对应的驱动力结构：

```
驱动力总值 = 内在驱动力 + 外在驱动力 - 排斥力
```

- 驱动力超过**阈值** → 玩家选择该行为
- 多个行为超过阈值 → 选择第一个到达的
- 无一超过阈值 → 选择驱动力最高的

## 驱动力构成

| 力                     | 定义                 | 示例                                 |
| ---------------------- | -------------------- | ------------------------------------ |
| **内在驱动力（内驱）** | 玩家自发的行为意愿   | 喜欢过程体验、对结果期待感、自发目标 |
| **外在驱动力（外驱）** | 外部因素             | 已知奖励、任务目标、强制行为         |
| **排斥力（负数）**     | 让玩家不想体验的因素 | 难度太高、体验差、时间成本高、奖励差 |

## 关键原则

### 激发内驱 > 给予外驱
- 外驱元素可被"包装"成内驱（如不揭示奖励全貌）
- 任务奖励只显示类型/稀有度，不显示具体内容

### 避免内驱流失
- 驱动力不要超过行为阈值太多
- 过度理由效应：玩家总是先倒外驱→再倒内驱→溢出的部分（内驱）流失

## 增加驱动力技法

| 技法               | 原理                     | 示例                                          |
| ------------------ | ------------------------ | --------------------------------------------- |
| **目标梯度效应**   | 越接近目标，追求力度越强 | 跑商奖励中产出占领进度→提升对劫掠行为的驱动力 |
| **目标可见性调整** | 提前揭示未来内容         | 展示A区域后B区域的详情                        |

## Fogg 行为模型（补充）

> 来源：BJ Fogg，斯坦福大学
> 与驱动力模型的互补关系见：`游戏设计理论/02-玩家心理学.md §3.1`

Fogg 模型描述行为发生的三要素：

> **行为 = 动机 × 能力 × 提示**（Behavior = Motivation × Ability × Prompt）

### 三要素

| 要素                   | 含义                                                 | 游戏设计中的应用                |
| :--------------------- | :--------------------------------------------------- | :------------------------------ |
| **动机（Motivation）** | 愉悦/痛苦、希望/恐惧、社会认同/排斥                  | 即时爽感、预期奖励、排行榜/公会 |
| **能力（Ability）**    | 时间、金钱、体力、脑力、社会偏差、非例行             | 操作是否便捷、认知负荷是否合理  |
| **提示（Prompt）**     | 火花（低动机时）、引导者（低能力时）、信号（皆高时） | UI红点、任务更新通知、教程提示  |

### 与水桶理论的关系与分工

| 模型                       | 用途                                | 适用场景                             |
| :------------------------- | :---------------------------------- | :----------------------------------- |
| **Fogg 模型（B=MAP）**     | **验证**—排查玩家为何没有进行某行为 | 定位问题：从三要素逐一检查缺口       |
| **水桶理论（驱动力模型）** | **设计**—驱动玩家行为               | 具体设计：围绕内驱/外驱/排斥力的加减 |

**互补关系**：水桶理论负责"怎么让玩家做"，Fogg模型负责"为什么玩家没做"。

### 提示（Prompt）的设计落地

| 途径         | 说明                         | 示例                                   |
| :----------- | :--------------------------- | :------------------------------------- |
| **做进 UI**  | 视觉提示元素直接嵌入游戏界面 | 任务追踪箭头、红点提醒、按钮高亮       |
| **做进设计** | 通过关卡/系统设计自然触发    | 走到某位置触发事件、场景中灯光引导视线 |

两种途径可以结合使用。做进设计的提示比做进 UI 的提示更"自然"，但 UI 提示在动机/能力都足够时更高效。

## 如何改变玩家体验

当玩家长时间处于同一体验下：
- **增加**其他行为驱动力
- **减少**当前行为驱动力（⚠️不破坏内驱，路径依赖常源于内驱）
- 调整排斥力（距离、难度、奖励等）
