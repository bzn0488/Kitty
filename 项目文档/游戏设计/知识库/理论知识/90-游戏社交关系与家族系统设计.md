# 游戏社交关系与家族系统设计

> 调研目标：系统梳理游戏中的社交关系设计模式，特别聚焦于家族/血缘关系系统，
> 为家族纪元等项目的家族成员互动、世代传承、亲属关系网络提供设计参考。

---

## 一、社交关系系统的核心设计维度

### 1.1 关系量化模型

游戏中最常见的三种关系模型：

| 模型         | 描述                                | 代表游戏                 | 优势         | 劣势                 |
| ------------ | ----------------------------------- | ------------------------ | ------------ | -------------------- |
| **单轴线性** | 一个数值（好感度/友谊）从负到正     | 星露谷物语、动物森友会   | 简单直观     | 无法表达复杂关系     |
| **多轴独立** | 多个独立维度（友谊/浪漫/尊重/恐惧） | The Sims、Crusader Kings | 关系更真实   | 复杂度高，UI 负担大  |
| **多轴依赖** | 维度之间有相互约束                  | 火焰纹章：风花雪月       | 防止关系跳跃 | 需要精心设计约束规则 |

**设计原则**：如果游戏需要表达"两人互相尊重但阵营对立"这种复杂关系，必须用多轴模型。如果只需要"是否友好"，单轴足够。

### 1.2 对称性 vs 非对称性

- **对称关系**：A对B的态度 = B对A的态度（大多数游戏采用，易于实现）
- **非对称关系**：A喜欢B但B讨厌A（更真实，但需要额外跟踪和UI处理）

**家族纪元的启示**：家族内部宜采用**非对称+多轴**模型——父母对子女的感情不同于子女对父母，婚姻中夫妻各有独立关系值。

### 1.3 关系可见度

| 可见度       | 特征                                | 适用场景           |
| ------------ | ----------------------------------- | ------------------ |
| **完全透明** | 玩家看到所有数值                    | 星露谷送礼反馈     |
| **模糊提示** | 仅给出定性描述（"他看起来很满意"）  | 巫师3对话选项      |
| **隐藏**     | 完全不可见，通过行为后果间接感知    | 质量效应关键决策   |
| **混合**     | 基础数值可见，隐藏阈值/事件触发条件 | 女神异闻录Coop系统 |

## 二、经典游戏关系系统分析

### 2.1 火焰纹章 Support 系统（系列标杆）

**核心机制**：
- 角色间通过并肩作战积累 Support 点数
- 达到阈值后触发 Support 对话（C → B → A → S）
- 高 Support 提供战斗加成（命中、回避、必杀）
- S 级仅限一对角色，触发结婚/结局

**设计要点**：
- **正向反馈循环**：一起战斗 → 关系提升 → 战斗更强 → 更愿意一起战斗（链接到 `正反馈循环` 知识）
- **叙事收益分离**：即使不看 Support 对话，纯战斗玩家也能获得数值收益
- **稀缺性创造选择**：每角色只能一个 S 级，迫使玩家做配对决策
- **世代传承**（三房子/觉醒）：结婚后下一代继承技能和属性

**对家族纪元的适用性**：
- Support 的"条件触发对话"模式适合家族成员间的特殊事件
- 世代传承机制直接可参考——后代继承父母部分特质
- 稀缺性配对（只选一个配偶）可应用于婚姻系统

### 2.2 《Crusader Kings》家族与外交系统（深度参考）

**核心机制**：
- **多轴关系**：Opinion（-100 ~ +100）+ 好感修正列表（每项修正独立）
- **家族树**：血缘关系自动生成，影响继承权和婚姻策略
- **婚姻联盟**：婚姻 = 外交工具，产生同盟和宣称权
- **压力系统**：角色行为违背性格特质时积累压力，影响心理健康
- **秘密系统**：私生子、阴谋、通奸——关系网络中的隐藏信息层

**设计要点**：
- **关系修正列表**比单一数值更透明：玩家知道"为什么他讨厌我"
- **血缘距离**影响继承优先级、婚姻允许度、近亲惩罚
- **角色性格特质**（Traits）驱动行为倾向，而非纯数值计算
- **压力与 coping 机制**：性格与行为的冲突产生叙事

**对家族纪元的直接借鉴**：
- 关系修正列表（"因嫁妆争议 -20"）的透明设计
- 血缘距离的计算和继承权影响
- 角色性格特质系统——让家族成员有差异化行为

### 2.3 《The Sims》多维关系系统

**核心机制**：
- **关系双轴**：友谊（Friendship）+ 浪漫（Romance），每轴 -100 ~ +100
- **长期关系**：即使长期不互动，关系缓慢衰退
- **记忆系统**：重要事件（结婚、背叛、死亡）永久影响关系
- **自主行为**：NPC 之间自主发展关系，不依赖玩家驱动
- **世代玩法**：家庭树、遗传特征、年龄增长

**设计要点**：
- **自主关系发展**是关键亮点——NPC 之间可以独立结婚生子，世界自己转动
- **记忆事件**比数值更有叙事价值："他救了我一命"比 "+30 好感" 更有意义
- **长期衰退**让关系维护成为游戏循环的一部分

### 2.4 其他重要参考

| 游戏            | 核心设计                                | 可借鉴点                                 |
| --------------- | --------------------------------------- | ---------------------------------------- |
| **女神异闻录5** | Coop 系统：时间锁 + 事件解锁 + 能力奖励 | 关系升级解锁具体游戏能力（"家族技能树"） |
| **质量效应**    | 全局决策影响多角色关系网络              | 分支因果链：一个决策波及其他角色         |
| **星露谷物语**  | 送礼 + 生日 + 事件触发                  | 简单但有效的日常关系经营循环             |
| **文明系列**    | 议程 (Agenda) 驱动 AI 外交行为          | 每个角色有固定行为倾向+随机变化          |
| **RimWorld**    | 社交状态 + 关系事件生成                 | 随机关系事件（出轨/和解/宿怨）创造叙事   |

## 三、家族/血缘系统专项设计

### 3.1 血缘关系建模

```
血缘距离 = 0   → 本人
血缘距离 = 1   → 父母/子女/同胞
血缘距离 = 2   → 祖父母/孙辈/叔伯/甥侄
血缘距离 = 3   → 曾祖/堂表亲
血缘距离 = 4+  → 远亲/同族
```

**应用**：
- 继承权排序：距离越近优先级越高
- 婚姻限制：近亲婚姻产生负面效果
- 家族声望影响：近亲行为对家族声望影响更大
- 遗传特质传递概率：与血缘距离正相关

### 3.2 世代传承设计模式

| 模式         | 描述                       | 游戏案例                     |
| ------------ | -------------------------- | ---------------------------- |
| **直接继承** | 玩家角色死亡后控制继承人   | Crusader Kings               |
| **多代经营** | 同时控制整个家族，世代更替 | The Sims、太阁立志传         |
| **血脉积累** | 后代继承前代的属性和成就   | Rougelike 传承、Rogue Legacy |
| **基因混合** | 后代从父母双方获取特质组合 | The Sims 遗传系统            |

**家族纪元的适用性**：建议采用**直接继承 + 血脉积累**混合模式——玩家主要控制族长/主角，但家族其他成员有独立行动逻辑；后代继承部分前代积累的资源/声望/特质。

### 3.3 家族内部关系张力

好的家族系统需要内置**冲突来源**：

- **继承权竞争**：多个子嗣争夺继承权 → 内战/阴谋/谋杀
- **世代价值观冲突**：父辈保守 vs 子辈改革
- **联姻 vs 爱情**：为了家族利益安排的婚姻 vs 个人感情
- **资源分配不公**：有限的家族资源在多子之间的分配产生怨恨
- **声誉连带**：一个家族成员的丑闻影响全族

**设计原则**：冲突不是负面，而是**叙事发动机**。没有内部张力的家族系统是乏味的。

### 3.4 家族声望与外部关系

```
家族声望 = 累积值（-1000 ~ +1000）
  ├── 先祖声望（去世成员的贡献，缓慢衰减）
  ├── 当代声望（在世成员的行为贡献）
  ├── 声望事件（大的成就/丑闻产生跳跃）
  └── 公众认知（外界对家族的标签化印象）
```

家族声望影响：
- 外交信誉（他国愿意结盟/通婚的程度）
- 联姻筹码（高声望家族能娶到高地位配偶）
- 人才吸引力（能吸引/招募到的人才质量）
- 贸易/经济优惠
- 敌对势力对家族的态度

## 四、与现有知识库的连接

### 4.1 与自我决定理论（SDT）的关系

社交关系系统直接满足 SDT 的**归属感（Relatedness）**需求：
- 与家族成员建立关系 → 归属感满足
- 关系丧失（亲人去世/背叛）→ 归属感受损 → 叙事驱动力
- 家族外社交 → 归属感扩展 → 开放新游戏维度

**设计提示**：关系系统应产生真实的"失去感"，而非仅仅是数值衰减。失去一个有关系的角色比失去一件装备更有情感冲击力。（链接到 `峰终定律`、`快乐总量`）

### 4.2 与正反馈循环的关系

关系系统的正反馈循环设计：
```
正向：互动 → 关系提升 → 更多互动选项 → 更高效关系增长
负向：忽视 → 关系衰退 → 互动效果减弱 → 更快速衰退
```

**问题**：正反馈会导致关系两极分化（永远最好或永远最差）。解决方案：
- **Crusader Kings 模式**：性格特质冲突制造"不得不恶化"的关系
- **Sims 模式**：长期衰退需要持续维护
- **Persona 模式**：时间锁（不能快速连续升级，需要等待）

### 4.3 与玩家投影的关系

当玩家控制一个家族而非单一个体时：
- 玩家在"族长"和"整个家族"之间建立双重投影（链接到 `玩家投影`）
- 让玩家为家族利益做出牺牲个人情感的决策 → 更强的叙事张力
- 世代更替时：玩家需要与"老角色"告别，接受"新角色"

## 五、针对家族纪元的实施建议

### 5.1 推荐模型：多轴非对称 + 关系修正列表

```
每个角色关系包含：
  血缘距离: int          // 自动计算
  友好度: -100 ~ 100     // 基础关系轴
  尊重度: 0 ~ 100        // 影响命令接受度
  恐惧度: 0 ~ 100        // 影响背叛概率
  修正列表: [
    {原因: "嫁妆纠纷", 友好修正: -20, 持续回合: 10},
    {原因: "战场救命", 友好修正: +40, 持续回合: 永久}
  ]
```

### 5.2 关键设计决策

1. **关系是否双向独立？** 建议是——父亲对儿子的感情 ≠ 儿子对父亲
2. **NPC之间关系是否自主发展？** 建议有限自主——家族成员间有一定自主发展，但关键关系（婚姻/结盟）由玩家决策
3. **关系可见度？** 建议混合——对主角的关系透明可见（友好度数值），NPC之间的仅给出定性描述
4. **关系如何驱动叙事？** 建议事件触发——关系达到特定阈值或状态时触发对应事件

### 5.3 最小可行版本（MVP）

```
第一步：单轴友好度（-100~100） + 血缘关系自动生成
第二步：增加关系事件触发（结婚/生子/背叛/去世）
第三步：增加独立 NPC-NPC 关系（有限自主）
第四步：增加多轴（尊重/恐惧） + 修正列表
```
