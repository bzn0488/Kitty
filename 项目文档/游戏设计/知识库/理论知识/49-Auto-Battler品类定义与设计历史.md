## 一、品类定义

Auto-Battler（自走棋）是一种**半自动策略游戏**子类型，核心特征包括：

| 特征           | 说明                                               |
| :------------- | :------------------------------------------------- |
| **自动战斗**   | 玩家不直接控制单位行动，战斗由AI自动执行           |
| **构筑先行**   | 玩家在战前通过商店购买/合成/站位进行构筑决策       |
| **多人回合制** | 多名玩家同时在商店阶段决策，战斗阶段异步或同步匹配 |
| **棋盘空间**   | 有限棋盘格上的站位策略是核心维度之一               |
| **协同系统**   | 单位之间通过Tag/职业/种族等产生协同效应            |

### 与传统策略游戏的分界

| 维度         | 传统策略游戏（如星际争霸） | 自走棋                  |
| :----------- | :------------------------- | :---------------------- |
| **微操**     | 核心——操作决定胜负         | 无——战斗全自动          |
| **决策时间** | 实时压力                   | 无时间压迫              |
| **构筑**     | 单局内临时决策             | 核心乐趣——持续优化阵容  |
| **随机性**   | 有限（地图/开局）          | 高（商店刷新/战斗随机） |

## 二、品类起源与发展历史

### 2.1 起源：Dota Auto Chess（2019）

2019年1月，在《Dota 2》自定义地图中诞生了 **Dota Auto Chess**，由开发者 **drodo**（中文名：巨鸟多多）创作。

**核心创新**：
- 将Dota的英雄单位放在棋盘上自动战斗
- 引入商店刷新系统（每回合随机刷出5个英雄）
- 同名英雄合成升星（3合1）
- 种族/职业协同系统

**爆发式增长**：
- 上线不到3个月，同时在线玩家峰值超30万
- 成为2019年最热门的Dota 2自定义地图
- 催生了整个Auto-Battler品类

### 2.2 三大商业作品的崛起（2019年中）

| 作品                          | 开发商     |         发布时间          | 核心差异                                           |
| :---------------------------- | :--------- | :-----------------------: | :------------------------------------------------- |
| **Teamfight Tactics (TFT)**   | Riot Games |         2019年6月         | 引入"选秀"（Carousel）机制、装备系统、英雄专属羁绊 |
| **Dota Underlords**           | Valve      | 2019年6月（Early Access） | 引入"霸主"下棋者单位、回合制战斗、联盟系统         |
| **Hearthstone Battlegrounds** | Blizzard   |        2019年11月         | 简化版自走棋、基数小（7名对手）、旅店升级代替人口  |

### 2.3 品类分化（2019-2020）

自走棋品类迅速分化出不同方向：

| 方向            | 代表             | 设计要点                        |
| :-------------- | :--------------- | :------------------------------ |
| **经典自走棋**  | TFT / Underlords | 8人对战、共享卡池、多人淘汰     |
| **轻量自走棋**  | 炉石酒馆战棋     | 7人对战、独立卡池、简化经济系统 |
| **卡牌+自走棋** | The Bazaar       | 异步PvP、物品/卡牌构筑+自动战斗 |
| **RPG+自走棋**  | 增量自走棋       | 增量游戏理念+自走棋战斗         |
| **PVE自走棋**   | 各种单机变体     | 对战AI、Roguelike式单局构筑     |

### 2.4 现代演进（2020-2026）

- **TFT**：持续赛季更新，每年3-4个大型赛季，引入城邦机制、英雄强化等新系统
- **Battlegrounds**：加入伙伴机制、双打模式、法术等
- **The Bazaar**（2025）：将自走棋概念与卡牌构筑/DBG深度融合
- **移动端自走棋**：《金铲铲之战》（TFT移动版）、《多多自走棋》等

## 三、品类核心设计维度

### 3.1 自走棋的"三重决策空间"

```
商店决策层 ──→ 买什么？刷新吗？存钱还是花钱？
     │
     ▼
构筑决策层 ──→ 上什么单位？什么星级？触发什么协同？
     │
     ▼
站位决策层 ──→ 前排/后排？分散/集中？保护关键单位？
```

### 3.2 与DBG/Roguelike的交叉

| 自走棋                  | DBG（杀戮尖塔） | Roguelike |
| :---------------------- | :-------------- | :-------- |
| 商店购买单位 ≈ 卡牌获得 | ✓               | —         |
| 升星合成 ≈ 卡牌升级     | ✓               | —         |
| 随机刷新 ≈ 三选一       | ✓               | ✓         |
| 每局不同 ≈ 程序化生成   | —               | ✓         |
| 失败归因困难            | —               | —         |

## 四、设计启示

### 自走棋解决的核心问题

1. **降低操作门槛**：自动战斗消除微操障碍，让策略决策成为唯一胜负因子
2. **提供持续期待**：每次商店刷新 ≈ 老虎机式的间歇强化
3. **创造协同满足**：阵容成型时的"智力优越感"

### 自走棋的设计陷阱

| 陷阱           | 表现           | 对策                       |
| :------------- | :------------- | :------------------------- |
| **随机性过大** | "运气游戏"感   | 提供经济管理手段来对冲随机 |
| **协同固化**   | 最优Build单一  | 赛季轮换、平衡性调整       |
| **归因困难**   | 输了不知道原因 | 战斗回放、伤害统计         |
| **长尾无聊**   | 后期公式化     | 引入后期质变单位/机制      |
