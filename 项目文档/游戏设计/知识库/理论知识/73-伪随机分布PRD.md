# 伪随机分布（Pseudo-Random Distribution / PRD）

> 标签：`randomness`, `probability`, `system-design`, `mechanics`
> 来源：Warcraft 3 引擎实现、Dota 2 继承
> 关联概念：`30-投篮效应.md`, `36-心流通道.md`, `46-正反馈与负反馈循环.md`
> 大文档对应：`游戏设计理论/04-游戏节奏与心流设计.md §4.6 幸运原则`

## 定义

PRD（Pseudo-Random Distribution）是一种**概率递增机制**——事件每次未触发时，下次触发的概率按固定增量提高，直到事件触发后重置为初始概率。

**目的**：让实际触发频率趋近期望值，避免极端RNG情况（连续触发或连续不触发）。

> 注意：这里的"伪随机"不是指计算机生成随机数的算法(PRNG)，而是指通过概率递增来**修正概率分布**的设计模式。

## 核心原理

### C值机制

PRD 使用一个称为 **C值（常数）** 的增量参数：

```
第1次尝试：概率 = C
第2次尝试：概率 = 2C
第3次尝试：概率 = 3C
...
第N次尝试：概率 = N × C
触发后 → 重置为 C
```

**C值决定了最终的实际触发概率**。C值越大，触发越频繁，但连续不触发的最大次数也越小。

## PRD vs 纯随机

| 对比         | 纯随机（独立概率） | PRD                    |
| :----------- | :----------------- | :--------------------- |
| **每次概率** | 固定（如20%）      | 递增（C → 2C → 3C...） |
| **极端情况** | 可能连续10次不触发 | 有限最大不触发次数     |
| **玩家感知** | "运气太差了！"     | "也该触发了..."        |
| **数学分布** | 几何分布           | 修正伯努利分布         |
| **可控性**   | 不可控             | 可控下界               |

### 连续不触发概率对比（以20%为例）

| 连续不触发次数 | 纯随机 |   PRD   |
| :------------: | :----: | :-----: |
|      5次       | 32.8%  |  26.9%  |
|      10次      | 10.7%  |  5.1%   |
|      15次      |  3.5%  |  0.5%   |
|      20次      |  1.2%  |  0.02%  |
|      25次      |  0.4%  | <0.001% |

> PRD 大幅降低了连续不触发的极端尾部概率，同时保持了长期的期望值。

## 设计应用

### 典型应用场景

| 场景          | 使用PRD                      | 不使用PRD             |
| :------------ | :--------------------------- | :-------------------- |
| **暴击系统**  | 每次未暴击→下次暴击率提高    | 可能连续10刀不暴击    |
| **技能触发**  | 被动技能稳定触发频率         | 关键战不触发=体验灾难 |
| **稀有掉落**  | 保底式掉率（但不是固定保底） | 非酋刷不到=弃坑       |
| **闪避/格挡** | 不可预测但期望可控           | 连续闪避=破坏平衡     |
| **随机事件**  | 渐进式生成的事件间隔         | 长时间无事=无聊       |

### PRD的变体

#### 1. 传统PRD（Warcraft 3）
- 每次事件未触发时概率增加固定C值
- 触发后重置为C
- 适用于暴击/重击/闪避

#### 2. 递增保底（Gacha pity）
- 固定概率直到N次未触发 → 第N次必然触发
- 常见于抽卡游戏（如原神的90抽保底）
- 是PRD的离散化变体

#### 3. 温和PRD（Soft PRD）
- 未触发时概率非线性增长（加速/减速）
- 适用于更平滑的概率曲线

#### 4. 隐藏计数器（Hidden Counter）
- 系统记录玩家未触发次数
- 在后台悄悄修正概率
- 玩家感知不到但体验平滑

## 设计注意事项

### 1. C值校准
C值需要通过数值模拟或查表获得。直接猜测C值会使实际触发率偏离目标。

### 2. 阈值效应
玩家可能会"感觉到"概率在增加——尤其是当系统明确显示"下次概率提高"时。设计上可以选择：
- **透明**：显示概率递增（如"第N次未暴击，下次暴击率+5%"）
- **隐藏**：不告知玩家，平滑体验

### 3. PRD的局限
- **不适用于一次性事件**（只能触发一次的事件不能用PRD）
- **玩家博弈**（如果玩家知道PRD机制，可能会故意等概率堆高后再行动）
- **多事件交叉**（多个PRD同时运行时，需要独立维护各自的计数器）

### 4. 玩家心理
- PRD改善了"连续差运气"的体验
- 但玩家仍然会记住"坏运气"场景（负面记忆偏差）
- 结合「幸运原则」——不仅要平滑分布下限，还要整体提升下限感知
