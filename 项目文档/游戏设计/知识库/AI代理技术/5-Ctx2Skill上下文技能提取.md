# Ctx2Skill：从上下文中提取可复用技能

> 学术论文 · 已在知识库调研阶段引用

## 核心思想

Agent 在与环境/用户交互的过程中积累了丰富的上下文（对话历史、工具调用记录、错误修复过程），其中隐含着**可复用的技能模式**。Ctx2Skill 的目标是从这些上下文中**自动提炼**出结构化的技能知识。

## 与技能文件的关系

我们系统中的技能文件（`_共享资源/技能/`）本质上是 Ctx2Skill 的手动版本：

| Ctx2Skill 概念 | 我们系统的对应              |
| -------------- | --------------------------- |
| 上下文 → 技能  | 使用经验 → 写入 `.skill.md` |
| 技能触发条件   | `triggers` 字段             |
| 技能内容       | `.skill.md` 中的知识        |
| 技能注册       | `_registry.json`            |
| 经验沉淀       | `.memory.md` 文件           |

## 改进方向

1. **自动化检测**：当某个 prompt/workflow 被反复修改时，自动提示「是否从中提取技能？」
2. **交叉关联**：多个 `.memory.md` 中出现相似教训时，自动合并为一个技能
3. **版本追踪**：技能随上下文演变（v1 → v2 → v3）

## 应用场景

- 从多次对弈自检的改进清单中提取通用设计原则
- 从知识调研记录中提取搜索方法论
- 从工作流执行失败中提取调试模式
