# Self-Refine：自生成反馈与自改进

> Madaan et al., 2023 · arXiv:2303.17651

## 核心思想

同一个 LLM 同时扮演「生成者」和「改进者」：先生成输出，然后自己给自己提反馈意见，再根据反馈修正输出。循环直到满足条件。

```
生成 → 自我反馈 → 改进 → 再反馈 → 再改进 → ...
```

## 三步循环

| 步骤                   | 动作               | 示例                         |
| ---------------------- | ------------------ | ---------------------------- |
| ① **生成（Generate）** | 产出初始回答       | 写一段报告                   |
| ② **反馈（Feedback）** | 对自己的产出提批评 | "这里太啰嗦了，缺少具体数据" |
| ③ **改进（Refine）**   | 根据反馈修改       | 精简语言，加入数字支撑       |

## 关键特征

- **同一模型**：生成者和改进者是同一个 LLM，不需要额外训练
- **零样本**：不需要示例或微调
- **可迭代**：可以循环多轮直到满意

## 与我们系统的关联

Self-Refine 模式在我们系统中多处出现：

| 场景                     | 生成                  | 反馈                | 改进                 |
| ------------------------ | --------------------- | ------------------- | -------------------- |
| **对弈自检**             | 答题方作答            | 裁判判定            | 改进步骤修复         |
| **日记写作**             | 写初稿（原则1趣味性） | 自检（原则3具体性） | 重写（加入更多细节） |
| **workflow prompt 迭代** | 写 prompt             | 跑工作流发现问题    | 修改 prompt          |

## 与 Reflexion 的区别

```
Self-Refine: 单次任务内 生成→反馈→改进 循环（横向）
Reflexion:   跨任务   失败→反思→记忆→下次参考（纵向）
```

两者互补。Self-Refine 让单次输出更好，Reflexion 让跨任务表现持续提升。
