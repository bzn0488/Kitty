## 体系说明

本知识库采用 **三套体系并行** 的知识组织策略：

| 体系             | 路径            | 定位                             | 阅读方式                   |     状态      |
| :--------------- | :-------------- | :------------------------------- | :------------------------- | :-----------: |
| **大文档体系**   | `游戏设计理论/` | 教材级系统知识，一个主题一个文档 | 按 01→02→03 顺序通读       |  多数 draft   |
| **UI 设计理论**  | `UI设计理论/`   | 游戏 UI/UX 专题理论，独立组织    | 通过 `_索引.md` 按标签查找 |     待建      |
| **原子文档体系** | `理论知识/`     | 卡片级概念知识，一个概念一个文档 | 通过 `_索引.md` 按标签查找 | 全部 verified |
| **桥梁层**       | `归纳文档/`     | 中间层，已梳理验证               | 两者皆可                   |   verified    |

> 📖 **想系统学习？** → 从 `游戏设计理论/00-总览与阅读导引.md` 开始顺序阅读
> 🔍 **想快速查找？** → 读 `理论知识/_索引.md` 或 `UI设计理论/_索引.md` 通过标签定位概念
> 📋 **差异对照** → 读 `_差异清单.md` 了解两套体系各自的独有内容

## 规范

### 添加新知识（使用知识入库工作流）
触发 **知识入库工作流**（`工作流/知识入库.json`），自动完成：
1. 分类定位 → 2. 生成 YAML frontmatter → 3. 写入文件 → 4. 更新索引 → 5. 校验

也可手动添加：
1. 在合适的子目录下创建 `.md` 文件
2. 文件头必须包含 YAML frontmatter：
   ```yaml
   ---
   title: 知识标题
   tags: [标签1, 标签2]
   created: 2024-01-01
   status: draft | verified
   ---
   ```
3. 运行 `python 脚本/generate_index.py` 更新索引

### 标签系统
- 用标签组织知识，而不是目录结构
- 一个知识可以有多个标签
- 标签命名：小写 + 下划线（如 `roguelike`, `card_design`）

### 维护
- 定期运行 `python 脚本/validate.py` 检查知识引用一致性
- 删除知识文件后，记得更新索引
- 新增独有内容后，同步更新 `_差异清单.md`
