# 快乐总量与正反馈技法

> 整理自：`_原始文档归档/快乐总量.txt`

## 一、快乐总量

### 1.1 定义
在游戏中获得的正向体验和正反馈（正值）与负面体验和负反馈（负值）的总和。

- **讨论范围**：默认指所有玩家的快乐总量
- **核心推论**：快乐总量越高，所有玩家的总体验越好

### 1.2 单位进程快乐总量

```
单位进程快乐总量 = 快乐总量 / 进程
```

- 增加快乐总量 **且** 减少进程（非缩减体量，而是减少无效时间）

## 二、提供正反馈的技法

| 技法                | 核心机制                                 | 关键点                           |
| ------------------- | ---------------------------------------- | -------------------------------- |
| **协同效应**        | 1+1>2，组合产生远大于单个的效果          | 智力优越感、构筑类游戏适用       |
| **叠层**            | 较小即时奖励累积到阈值触发大奖励         | 延迟满足、对阈值的期待           |
| **投篮效应**        | 低成本行为×大概率基础奖励+小概率高额奖励 | 反复尝试→期待感→惊喜峰值         |
| **精准操作**        | 克服操作/策略挑战完成明确目标            | 目标明确、权力下放               |
| **浪费规避**        | 减少资源/体验浪费，提供补救途径          | 挫败感→可控微小正反馈            |
| **他者确认**        | 多人游戏中提供交互入口                   | 避免任何他人给予负反馈的可能性   |
| **额外挑战&及格线** | 分层目标：及格+额外挑战                  | 克服越多正反馈越强               |
| **创造性表达**      | 在有规则范围内自由选择和塑造             | 源于自我认可的正反馈（母系游戏） |
| **双轨收益**        | 同一行为多套奖励机制（固定+随机）        | 固定保底 + 可变比率强化          |

### 2.1 协同效应
让两个维度获得提升时总强度获得更大提升，需要和玩家决策相关。

**例**：增加暴击率和暴击伤害、每次攻击造成额外固定伤害+多次攻击次数。

### 2.2 叠层
**例1**：技能每次施放永久提升攻击力→强化该行为
**例2**：攻击3次后下一次释放强力攻击→从攻击行为获得更大正反馈

### 2.3 投篮效应
行为低成本（时间/资源），可多次进行。
- 通常"未命中"→挫败感低
- "命中"→惊喜峰值高
- 因为可多次进行，负反馈持续时间短

**例**：《英雄联盟》非指向性技能——命中/未命中都能提高整体快乐总量

### 2.4 精准操作
目标明确是重点——明确的预期奖励提供强大动机和目标感。
可结合权力下放让玩家自主选择操作难度。

### 2.5 浪费规避
人天然有规避损失的倾向。无法避免时，设计补救途径让浪费转化为微小正反馈。

### 2.6 他者确认（多人游戏）
- 提供交互入口满足被认可需求
- ★ 必须避免任何他人给予负反馈（负反馈影响更深更长）

### 2.7 额外挑战 & 及格线
- **及格**：能继续进程
- **挑战**：额外目标，更多正反馈

**例**：《胡闹厨房》1星及格→3星挑战

### 2.8 双轨收益
固定奖励（保底）+ 随机奖励（间歇性强化）

★ 固定奖励保证基础正反馈，随机奖励基于可变比率强化（最有效维持行为的方法）

## 三、正反馈与负反馈的使用策略

### 3.1 多使用正反馈加强正确行为，谨慎使用负反馈
**做法1**：正确→大正反馈，错误→负反馈
- ✅ 快速建立正确认知
- ❌ 触发损失厌恶，易陷入负面情绪

**做法2（推荐）**：正确→巨大正反馈，错误→小正反馈
- ✅ 基础满足感 + 相对剥夺感→调整行为
- ❌ 可能模糊正确/错误界限，需辅助设计（随机性诱使选择新选项）

### 3.2 需要调整行为时，使用小的负反馈而非大的
损失厌恶：对损失的敏感度远高于同等收益。小的负反馈最小化损失厌恶，让玩家更关注修正行为而非陷入负面情绪。

## 四、减少无效进程

### 4.1 无效进程的定义
玩家无法获得显著正向体验、且不包含内容表达的环节。

**例**：
- 进入多个层级菜单才能施放动作
- 开放世界中反复前往目的地（无体验的移动过程）

### 4.2 三层乐趣体系

| 层级           | 定义                 | 进程需求 | 举例                         |
| -------------- | -------------------- | -------- | ---------------------------- |
| **Hard Fun**   | 核心乐趣，游戏结构   | 最长     | 核心战斗、关卡挑战           |
| **Easy Fun**   | 低成本乐趣，额外系统 | 中等     | 滑翔翼、一次性车辆等移动手段 |
| **无意识快乐** | 本能驱动，不需要动机 | 最短     | 点击反馈、收集               |

**填充原则**：瓶子优先放石头（Hard Fun），再放石子（Easy Fun），最后放沙粒（无意识快乐）。

**设计流程**：
1. 优先构建Hard Fun（必须存在）
2. 评估是否存在无效/低体验密度进程
3. 能缩短/简化则缩短→无法缩短则加入Easy Fun

### 4.3 快车道
玩家快速跳过正反馈较低的进程，同时获得更多奖励。

**例**：跳关 + 下次奖励×3倍（连续跳关累乘）→ 强度高时跳过低难度关卡 + 获得更多奖励

→ 同时做到**增加正反馈量**和**减少进程**。

## 五、按奖励类型的完整分类

### 5.1 四种基本奖励类型（Wang & Sun 2011）

| 类型         | 定义         | 正反馈原理            | 示例                 |
| ------------ | ------------ | --------------------- | -------------------- |
| **进度奖励** | 推进游戏进程 | 胜任感 + 目标梯度效应 | 经验值、升级、新区域 |
| **资源奖励** | 可积累的资源 | 连续强化 + 期待感     | 金币、材料、体力     |
| **能力奖励** | 增强玩家能力 | 权力感 + 掌控感       | 新技能、属性提升     |
| **装饰奖励** | 改变外观     | 自我表达 + 社会认同   | 皮肤、称号、宠物     |

### 5.2 按确定性分类

| 类型         | 原理                   | 示例             |
| ------------ | ---------------------- | ---------------- |
| **固定奖励** | 连续强化（学习最快）   | 每次击杀给经验   |
| **递增奖励** | 目标梯度效应           | 连续登录奖励递增 |
| **随机奖励** | 可变比率强化（最持久） | 稀有装备掉落     |
| **条件奖励** | 塑造（Shaping）        | 首次达成条件解锁 |
| **惊喜奖励** | 多巴胺峰值             | 隐藏成就、彩蛋   |
| **社交奖励** | 他者确认               | 点赞、排行榜     |

## 六、按时间维度分类

| 时间窗口                    | 技法                     | 作用                       |
| --------------------------- | ------------------------ | -------------------------- |
| **即时反馈**（0-3秒）       | 屏幕震动、音效、伤害数字 | 持续强化当前行为，维持心流 |
| **短期反馈**（3秒-5分钟）   | 小怪掉落、任务完成提示   | 建立行为-奖励因果链        |
| **中期反馈**（5分钟-1小时） | 关卡完成、Boss击杀       | 阶段性满足感，维持会话     |
| **长期反馈**（1小时-数天）  | 成就解锁、剧情推进       | 回归驱动力，防流失         |
